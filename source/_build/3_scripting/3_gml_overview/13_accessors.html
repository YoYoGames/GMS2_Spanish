Accesorios


El GameMaker Language (GML) también permite acceder a ciertas estructuras de datos y los tampones a través de la utilización de expresiones lógicas llamadas descriptores de acceso. Esto se hace de forma similar a como lo haría para trabajar con matrices, solo usamos un símbolo identificador antes del primer argumento para decirle a GameMaker Studio 2 que está trabajando en una estructura de datos (previamente creada).

Estructuras de datos

Puede agregar, reemplazar y obtener valores y claves para los siguientes tipos de estructura de datos: ds_maps, ds_lists y ds_grids. A cada uno se le ha dado su propio símbolo y método para acceder, cambiar o establecer valores, como se muestra a continuación.

La sintaxis para las listas es:

list_index[| index]


Entonces cuando has usado ds_list_create() para crear su lista, debe usar el índice de lista (que ha almacenado en una variable) para hacer referencia a ella, con el valor de "índice" como la posición en la lista para establecer o agregar. Por ejemplo, el siguiente código crea una lista y luego agrega 10 entradas, estableciendo cada entrada en un número aleatorio de 0 a 9:

ds = ds_list_create();
var index = 0;
repeat(10)
   {
   ds[| index++] = irandom(9);
   }


Una vez que haya creado su estructura de lista y la haya llenado con datos, para obtener valores de la lista, tendría algo como:

value = ds[| 5];


Lo anterior obtendrá el valor de la posición 5 (el sexto índice, ya que las listas comienzan en 0) y lo almacenará en una variable. Tenga en cuenta que si está usando una expresión para agregar una referencia a un índice que ya tiene un valor, el valor anterior será reemplazado en lugar de tener un índice adicional agregado a la lista. Para agregar más entradas, necesitarás saber el tamaño de ds_list y agregarlas al final. También vale la pena señalar que puede establecer un índice de lista que sea mayor que el tamaño de la lista a la que se hace referencia, y esto establecerá ese valor, ampliando la lista al mismo tiempo, e inicializando todas las posiciones en la lista. al índice dado como 0.

La sintaxis para los mapas es:

map_index[? key]


Después de crear su mapa con ds_map_create(), usaría el índice del mapa que ha almacenado en una variable para hacer referencia a él, con el valor "clave" como la clave del mapa para establecer o obtener. Por ejemplo, el siguiente código crea un mapa y luego agrega algunas entradas usando esta sintaxis:

ds = ds_map_create();
ds[? "Name"] = "Hamish";
ds[? "Company"] = "MacSeweeny Games";
ds[? "Game"] = "Catch The Haggis";


Una vez que haya creado su estructura de mapa y la haya llenado con datos, para obtener valores de una clave de mapa específica, tendrá algo como:

value = ds[? "Name"];


Lo anterior obtendrá el valor de la clave "Nombre" y lo almacenará en una variable. Tenga en cuenta que si el mapa ya contiene el mismo valor clave que está intentando agregar, no creará una clave duplicada con el nuevo valor, sino que reemplazará el valor anterior.

La sintaxis para grids es:

grid_index[# xpos, ypos]


Después de crear su grilla con el ds_grid_create() función, usaría el índice de cuadrícula que ha almacenado en una variable para referenciarlo, con "xpos" e "ypos" es la posición dentro de la cuadrícula para obtener o establecer un valor. Por ejemplo, el siguiente código crea una grilla, la borra a 0, y luego agrega algunas entradas a ella:

ds = ds_grid_create();
ds_grid_clear(ds, 0);
var gw = ds_grid_width(ds) - 1;
var gh = ds_grid_height(ds) - 1;
repeat(10)
   {
   var xx = irandom(gw);
   var yy = irandom(gh);
   if ds[# xx, yy] == 0 ds[# xx, yy] = 1;
   }


Una vez que haya creado su estructura de cuadrícula y la haya llenado con datos, para obtener valores de una posición de cuadrícula específica, tendrá algo como:

value = ds[# mouse_x div 16, mouse_y div 16];


Lo anterior obtendrá el valor de la ds_grid dada en función de la posición del mouse (dividido por el ancho de "celda" en la sala para obtener la ubicación correcta).


Arrays

Las matrices también tienen sus propios descriptores de acceso que funcionan de forma similar a los enumerados anteriormente para las estructuras de datos. Sin embargo, los accesores de matriz tienen una propiedad interesante y es permitirle modificar una matriz a partir de una secuencia de comandos sin tener que copiarla. Cuando pasa una matriz a una secuencia de comandos, se pasa por referencia, lo que significa que la matriz no se está copiando en la secuencia de comandos, sino que simplemente se está haciendo referencia a ella para obtener los datos. Normalmente, si necesita cambiar la matriz, se copiará en la secuencia de comandos y luego tendrá que volver a pasar la matriz copiada para la matriz original que se actualizará. Esto puede tener costosos gastos indirectos de procesamiento, y puede usar el acceso en su lugar, ya que eso cambiará el arreglo original directamente sin la necesidad de copiarlo. Puedes ver cómo funciona esto en los ejemplos a continuación.

La sintaxis para las matrices es:

array[@ xpos]
array[@ xpos, ypos]

Después de haber creado su matriz en una instancia, puede pasarla a una secuencia de comandos por referencia y usar el accesorio @ para cambiarlo directamente Por ejemplo, crearía la matriz y llamaría la secuencia de comandos de esta manera:

array[99] = 0;
scr_Array_Populate(array);

El script en sí tendría algo como esto:

var a = argument0; var i = 0; repeat(25)
   {
   i = irandom(99);
   while (a[i] != 0)
      {
      i = irandom(99);
      }
   a[@ i] = 100;
   }

Todo lo que esta secuencia de comandos está haciendo es seleccionar 25 posiciones aleatorias en la matriz y establecer el valor de la posición de la matriz elegida en 100.
