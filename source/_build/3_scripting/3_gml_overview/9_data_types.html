Tipos de datos


En secciones anteriores, hemos cubierto las variables y sus reglas de alcance, pero se ha dicho muy poco acerca de los diferentes tipos de datos que una variable puede almacenar. Por lo tanto, este capítulo explica los diferentes tipos y para qué se pueden usar.

Antes de continuar, expliquemos brevemente qué queremos decir con "tipos de datos". Cuando crea una variable, puede usarse para contener información, y cuando llama a una función GML, también puede devolver información. Sin embargo, esta información puede venir en varios "sabores", como puede (por ejemplo) ser un número real o puede ser una cadena. Estos diferentes tipos de valores que se utilizan se denominan tipos de datos y, al usar GML, pueden ser cualquiera de los siguientes:

Una cadena es simplemente cualquier texto que se ha colocado entre comillas (ya sea '...' o "..."). Puede realizar ciertas operaciones en cadenas, como agregar dos cadenas juntas para crear una cadena más larga (concatenación) y también puede cambiar las propiedades de la cadena e incluso extraer números reales de ella. Para obtener más información sobre las funciones de cadena, consulte: Referencia del lenguaje de GameMaker - Cadenas.


Los números reales son cualquier valor que no sea una cadena, ni ninguno de los otros tipos de datos posibles. Entonces, 124, 45639.566546456, 0, -45.5 etc... son todos ejemplos de números reales. Todos los números reales se almacenan como valores de punto flotante de 32 bits (incluso valores enteros), por lo que puede experimentar ligeros errores de redondeo al tratar estos. Para obtener más información sobre esta y otras funciones relacionadas con números, consulte: Referencia GML - Números reales.

Tenga en cuenta que las variables creadas en GameMaker Studio 2 se almacenan como números de punto flotante de precisión doble, pero puede usar otros formatos cuando se trata de extensiones. Estos se pueden pasar a GameMaker Studio 2 desde una extensión y luego se pueden verificar usando el is_ función, enumerada en la parte inferior de esta página.


Una matriz es un tipo de datos especial que se puede usar para contener múltiples valores. Usted asigna la matriz a una variable y luego "completa" diferentes índices de la matriz con valores. Esta matriz se puede pasar a los scripts y funciones en una base de paso por punto, sin embargo, si modifica la matriz pasada, se convertirá en una copia de la matriz original y, por lo tanto, deberá devolverse a la variable original. para la matriz que se actualizará. Para obtener más información sobre matrices, consulte: Descripción general de GML: matrices.


Un booleano es simplemente un valor que puede ser true o false. Tenga en cuenta que actualmente GameMaker Studio 2 interpretará un número real igual o inferior a 0.5 como una false valor, y cualquier número real mayor que 0.5 como true. Sin embargo, esto no significa que debe verificar 1 y 0 (o cualquier otro número real) para verdadero y falso, ya que también se le proporcionan las constantes. true y false que siempre debe usarse en su código para evitar problemas en caso de que se agreguen tipos de datos booleanos reales en una actualización futura.

Puede convertir cualquier número real en un valor booleano implícito usando la siguiente función:

  • bool

Un puntero es un tipo de datos que "apunta" a una ubicación de memoria. No puede hacer operaciones en un puntero y se usa solo para algunas funciones muy específicas, como obtener una textura o dirección de memoria intermedia para otra función. Para ver ejemplos de funciones que devuelven un puntero, puede ver buffer_get_address o sprite_get_texture.

También hay una función para verificar si un valor es un puntero (consulte " Verificación de los tipos de datos " a continuación) y una función para convertir un valor en un puntero:

  • ptr

También puede usar (y obtener devoluciones) las siguientes constantes integradas al usar punteros:

Constante Descripción
pointer_null Esta constante indica que el puntero no apunta a nada significativo (lo mismo que NULL en C ++ o nulo en C #)
pointer_invalid Esta constante simplemente significa que el valor no es un puntero válido

Una enumeración es un "enumerador", y esencialmente le permite crear su propio tipo de datos limitados con una lista de valores constantes. Los mensajes son variables de alcance global (esto es implícito y no requieren " global "prefijo" y tienen la siguiente estructura:

enum <variable>{<constant> [= <value>]}

En el siguiente ejemplo, creamos una enumeración para los colores del arco iris y le asignamos varias constantes y valores predeterminados:

enum rainbowcolors {
   red,
   orange,
   yellow,
   green,
   blue,
   indigo,
   violet
   }

Las entradas enum solo pueden ser números reales o expresiones con enumeraciones previas, y de manera predeterminada están numeradas de 0 hacia arriba, por lo que nuestro ejemplo dado anteriormente sería el predeterminado para red = 0, orange = 1, yellow = 2, etc... También puede asignar valores a las variables enumeradas en el momento de la creación:

enum rainbowcolors {
   red = 5,
   orange = 5 * 2,
   yellow = 15,
   green = 20,
   blue = 25,
   indigo = 30,
   violet = 35 * enum_test.entry
   }

Observe que en el ejemplo anterior usamos otra enumeración para crear una expresión para "violeta". Esto solo funciona si la enumeración a la que se hace referencia se creó antes de la enumeración que la está utilizando en una expresión, pero no funcionará para las variables o funciones, ya que el valor enum debe poder evaluarse como una constante en la compilación. También tenga en cuenta que todos los valores enum se evalúan a valores enteros, y cuando crea los propios, debe tener en cuenta que solo los valores enteros están permitidos para que funcionen las enumeraciones.

Para acceder más tarde al valor dentro de un tipo de enumeración dado, puede usar el punto "." método, como este:

var value = rainbowcolours.green;

Tenga en cuenta que no puede modificar los valores de ninguna constante enum después de que se haya creado.


Un valor indefinido (también conocido como valor "nulo") es aquel en el que una expresión no tiene un valor correcto, aunque es sintácticamente correcta, por lo que debe devolver algo. Por ejemplo, supongamos que tiene una ds_map y usa la función ds_map_find_value(). Ahora, ¿qué sucede cuando el mapa no tiene el valor que se busca? Bueno, dado que la función está formateada correctamente, y el problema es que no existe tal valor, entonces devolvería la constante undefined, y puedes verificar esta constante como verificarías true o cualquier otro valor.


Comprobando los tipos de datos

GameMaker Studio 2 permite verificar el tipo de datos de una variable dada usando las siguientes funciones:

  • is_string
  • es real
  • is_bool
  • is_array
  • is_ptr
  • is_int32
  • is_int64
  • is_vec3
  • is_vec4
  • is_matrix
  • es indefinido
  • tipo de
