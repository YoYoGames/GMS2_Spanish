Tipos de datos


En las secciones anteriores hemos cubierto las variables y sus reglas de alcance, pero poco se ha dicho sobre los diferentes tipos de datos que una variable puede almacenar. Por lo tanto, este capítulo explica los diferentes tipos y para qué se pueden usar.

Antes de continuar, expliquemos brevemente qué entendemos por "tipos de datos". Cuando creas una variable, se puede usar para guardar información, y cuando llamas a una función GML, también puede devolver información. Sin embargo, esta información puede venir en varios "sabores", como puede ser (por ejemplo) un número real o puede ser una cadena. Estos diferentes tipos de valores que se utilizan se denominan tipos de datos y cuando se usa GML pueden ser cualquiera de los siguientes:

Una cadena es simplemente cualquier texto que se ha colocado entre comillas "...". Puede realizar ciertas operaciones en cadenas, como agregar dos cadenas juntas para crear una cadena más larga (concatenación) y también puede cambiar las propiedades de la cadena e incluso extraer números reales de ella. Para obtener más información sobre las cadenas y las funciones de las cadenas, consulte: Referencia del lenguaje de GameMaker - Cadenas.


Los números reales son cualquier valor que no sea una cadena, ni ninguno de los otros tipos de datos posibles. Entonces, 124, 45639.566546456, 0, -45.5 etc... son todos ejemplos de números reales. Todos los números reales se almacenan como valores de punto flotante de doble precisión (incluso valores aparentemente enteros), por lo que puede experimentar pequeños errores de redondeo al tratar con estos. Para obtener más información sobre esta y otras funciones relacionadas con números, consulte: Referencia de GML - Números reales.

Tenga en cuenta que si bien las variables creadas en GameMaker Studio 2 se almacenan como números de punto flotante de doble precisión, aún puede usar otros formatos cuando se trata de extensiones. Estos pueden pasarse a GameMaker Studio 2 desde una extensión y luego verificarse usando el is_ función, que figuran en la parte inferior de esta página.


Una matriz es un tipo de datos especial que se puede usar para mantener varios valores. Usted asigna la matriz a una variable y luego "completa" diferentes índices de la matriz con valores. Esta matriz se puede pasar a las secuencias de comandos y funciones paso por referencia, sin embargo, si modifica la matriz pasada, se convertirá en una copia de la matriz original y, por lo tanto, deberá volver a la variable original. para que la matriz sea actualizada. Para obtener más información sobre matrices, consulte: Descripción general de GML - Matrices.


GameMaker Studio 2 también aceptará literales hexadecimales como valores legítimos. Los valores hexadecimales son especialmente comunes cuando se trabaja con colores, pero se pueden usar en cualquier lugar donde se requiera un valor entero positivo. Tenga en cuenta que los valores hexadecimales se pueden formatear de una de las dos formas siguientes:

$abcd
0xabcd

Por ejemplo, los siguientes valores se pueden expresar como hexadecimales como se muestra:

11406 = $2c8e = 0x2c8e
16777215 = $ffffff = 0xffffff


Un booleano es simplemente un valor que puede ser true o false. Tenga en cuenta que actualmente GameMaker Studio 2 interpretará un número real igual o inferior a 0,5 como un false valor, y cualquier número real mayor que 0.5 como siendo true. Esto no significa, sin embargo, que debe estar marcando 1 y 0 (o cualquier otro número real) para ver si es verdadero o falso, ya que también se le proporcionan las constantes true y false que siempre debe usarse en su código para evitar cualquier problema en caso de que se agreguen tipos de datos booleanos reales en una actualización futura.

Puede convertir cualquier número real en un valor booleano implícitamente usando la siguiente función:

  • bool

Un puntero es un tipo de datos que "apunta" a una ubicación de memoria. No puede realizar operaciones en un puntero y se usa solo para algunas funciones muy específicas, como obtener una textura o una dirección de búfer de la memoria para otra función. Para ejemplos de funciones que devuelven un puntero puedes ver buffer_get_address o sprite_get_texture.

También hay una función para verificar si un valor es un puntero (consulte " Comprobación de los tipos de datos ", a continuación) y una función para convertir un valor en un puntero:

  • ptr

También puede usar (y obtener) las siguientes constantes integradas al usar punteros:

Constante Descripción
pointer_null Esta constante indica que el puntero no apunta a nada significativo (lo mismo que NULL en C ++ o nulo en C #)
pointer_invalid Esta constante simplemente significa que el valor no es un puntero válido

Un enum es un "enumerador", y esencialmente le permite crear su propio tipo de datos limitado con una lista de valores constantes. Las enumeraciones son variables de alcance global (esto es implícito, y no requieren " global "prefijo) y tienen la siguiente estructura:

enum <variable>{<constant> [= <value>]}

En el siguiente ejemplo, creamos una enumeración para los colores del arco iris y le asignamos varias constantes y valores predeterminados:

enum rainbowcolors {
   red,
   orange,
   yellow,
   green,
   blue,
   indigo,
   violet
   }

Las entradas de enumeración solo pueden ser números enteros o expresiones con enumeraciones previas que se evalúan como un número entero, y de forma predeterminada están numeradas desde 0 en adelante, por lo que nuestro ejemplo dado de manera predeterminada sería red = 0, orange = 1, yellow = 2, etc...

También puede asignar valores a las variables de enumeración en el momento de la creación:

enum rainbowcolors {
   red = 5,
   orange = 5 * 2,
   yellow = 15,
   green = 20,
   blue = 25,
   indigo = 30,
   violet = 35 * enum_test.entry
   }

Note que en el ejemplo anterior usamos otra enumeración para crear una expresión para "violeta". Esto solo funciona si la enumeración a la que se hace referencia se creó antes que la enumeración que la está utilizando en una expresión, pero no funcionará para variables o funciones, ya que el valor de enumeración debe poder evaluarse como una constante en la compilación. También tenga en cuenta que todos los valores de enumeración se evalúan como valores enteros, y cuando cree los suyos propios, debe tener en cuenta que solo se permiten valores enteros para que funcionen las enumeraciones. Este valor puede ser cualquier número entero que pueda representar un número de precisión doble de punto flotante.

Para acceder posteriormente al valor dentro de un tipo de enumeración dado, puede usar el punto "." método, como este:

var value = rainbowcolours.green;

Tenga en cuenta que no puede modificar los valores de ninguna constante de enumeración después de que se haya creado.


Un valor no definido (también conocido como "valor nulo") es uno donde una expresión no tiene un valor correcto, aunque es sintácticamente correcto, y por lo tanto debe devolver algo. Por ejemplo, digamos que tiene un ds_map y usar la función ds_map_find_value(). Ahora, ¿qué sucede cuando el mapa no tiene el valor que se está buscando? Bueno, ya que la función está formateada correctamente y el problema es que no existe tal valor, entonces devolvería la constante undefined, y puede verificar esta constante como comprobaría true o cualquier otro valor.


Comprobación de tipos de datos

GameMaker Studio 2 permite verificar el tipo de datos de una variable dada usando las siguientes funciones:

  • is_string
  • es real
  • is_bool
  • is_array
  • is_ptr
  • is_int32
  • is_int64
  • is_vec3
  • is_vec4
  • is_matrix
  • es indefinido
  • tipo de
