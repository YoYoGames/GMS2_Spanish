Características del lenguaje


El lenguaje GameMaker (GML) tiene una serie de características que forman parte de la estructura esencial de todo el código escrito con el idioma. Cada uno de estos se utilizará en cualquier código que escriba varias veces, por lo que debe saber qué son y cómo funcionan antes de comenzar a programar un proyecto.

Un simple "if" La declaración toma esta forma:

if (<expression>) <statement>


o puede tener un poco más complejo "if... else..." formar:

if (<expression>) <statement> else <statement>


En este caso, la expresión se evaluará y si el valor (redondeado) es <= 0 ( false ) la sentencia after else se ejecuta, de lo contrario ( true ) se ejecuta la otra sentencia. Es un buen hábito colocar siempre corchetes alrededor de las declaraciones en el if ", y tomar una nueva línea en el bloque para cada declaración, por lo que el código final tendrá este formato:

if (<expression>)
   {
   <statement>
   ...
   <statement>
   }
else
   {
   <statement>
   }


Como un pequeño ejemplo, considere el siguiente código que moverá la instancia hacia la posición x = 200 en la sala:

if (x < 200)
   {
   x += 4;
   }
else
   {
   x -= 4;
   }


Tenga en cuenta que también puede hacer verificaciones compuestas en un if, es decir: verificar varios valores o expresiones en la misma declaración. Al hacer esto, GameMaker Studio 2 evaluará cada uno de ellos uno por uno, y si alguno de ellos se evalúa como falso, el resto se omitirá. Por ejemplo:

if (keyboard_check_pressed(vk_enter) && !instance_exists(obj_Player))
   {
   go = false
   alarm[0] = room_speed
   }


También puede realizar operaciones condicionales (también conocidas como operaciones ternarias ), que es esencialmente una forma "abreviada" de realizar un "if" básico. Tiene la siguiente sintaxis:

variable = condition ? <expression1 (if true)> : <expression2 (if false)>


El operador condicional devolverá uno de los dos valores dados dependiendo de si la condición se evalúa como true o false, por ejemplo:

var temp_x = x < (room_width / 2) ? 32: room_width - 32;


El código anterior verificará el valor de "x" con el valor de la mitad del ancho de la sala y luego, si es menor, establecerá "temp_x" en 32, de lo contrario "temp_x" será room_width - 32. Aquí hay algunos ejemplos más de utilizar:

draw_text(x, y, "The fee is " + (global.Member ? "$2.00" : "$10.00"));
path_start((global.level > 10 ? path_hard : path_easy;), 2, path_action_reverse, true);
--hp <= 0 ? instance_destroy() : score += 10;


Vale la pena señalar que puede anidar operaciones condicionales, pero que si lo hace, entonces cada operación deberá incluirse entre paréntesis, por ejemplo:

var c = a ? "foo" : (b ? "bar" : "wii"); // Correct
var c = a ? "foo" : b ? "bar" : "wii";   // Will cause an error


UNA "repeat" declaración tiene la forma

repeat (<expression>) <statement>


La instrucción se repite el número de veces indicado por el valor redondeado de la expresión. Por ejemplo, el siguiente programa crea cinco bolas en posiciones aleatorias.

{
repeat (5) instance_create_layer(random(400), random(400), "Instances", obj_ball);
}


Esto puede ser muy útil para evitar escribir el mismo código varias veces, o para usar arrays, o para contar a través de una serie de operaciones, etc. Por ejemplo:

{
var i, total;
i = 0;
total = 0;
repeat (10)
   {
   total += array[i];
   i += 1
   }
draw_text(32, 32, total);
}


UNA "while" declaración tiene la forma

while (<expression>) <statement>


Siempre que la expresión sea verdadera, se ejecuta la instrucción (que también puede ser un bloque de código). ¡Ten cuidado con tus bucles while! Puedes hacer bucles infinitos fácilmente, en cuyo caso tu juego se bloqueará y no reaccionará a ninguna entrada del usuario. A continuación puede encontrar un ejemplo de una forma típica de usar "while":

{
while (!place_free(x, y))
   {
   x = random(room_width);
   y = random(room_height);
   }
}


El programa anterior intenta colocar el objeto actual en una posición libre (esto es casi lo mismo que la acción para mover un objeto a una posición aleatoria).

Un "hacer" es realmente el "do... until" Declaración como no puedes tener una sin la otra. Tiene esta forma:

do <statement> until (<expression>)


La instrucción (que también puede ser un bloque de código) se ejecuta hasta que la expresión se encuentra verdadera, y la instrucción inicial siempre se ejecuta al menos una vez. Tenga cuidado con sus bucles de hacer, ya que puede hacerlos con facilidad para siempre, en cuyo caso su juego se bloqueará y no reaccionará a ninguna entrada del usuario. A continuación puede encontrar un ejemplo de una forma típica de uso. "do... until":

{
do
   {
   x = random(room_width);
   y = random(room_height);
   }
until (place_free(x, y)); }


El programa anterior intenta colocar el objeto actual en una posición libre (esto es casi lo mismo que la acción para mover un objeto a una posición aleatoria).

UNA "for" La declaración tiene esta forma:

for (<assigment1> ; <expression> ;<statement1>) <statement2>


Esto funciona de la siguiente manera: primero se ejecuta assigment1, luego se evalúa la expresión y, si es verdadera, se ejecuta la sentencia2. Luego se realiza la sentencia1 y luego la expresión se evalúa nuevamente. Este bucle continuará hasta que la expresión sea falsa.

Ahora, esto puede sonar complicado cuando se escribe así, pero debes interpretarlo como:

  • La primera instrucción inicializa el for-loop.
  • La expresión comprueba si el bucle debe finalizar.
  • Statement2 es la declaración de "paso" que va a la siguiente evaluación de bucle.
Esto es extremadamente útil para realizar tareas repetitivas que involucrarían múltiples líneas de código de cualquier otra manera, y se usa comúnmente como un contador para evaluar matrices o dibujar cosas. El siguiente ejemplo de código ilustra un uso típico para este tipo de declaración:

{
for (var i = 0; i < 10; i += 1)
   {
   draw_text(32, 32 + (i * 32), string(i) + ". "+ string(scr[i]));
   }
}


El código anterior inicializa un bucle for, comenzando en 0 y contando hasta 9, y luego usa el valor del bucle "i" para dibujar los valores almacenados en una matriz en la pantalla. Observe cómo "for" la variable de bucle "i" se usa no solo para recorrer la matriz, sino también para dibujar un número, así como para indicar a GameMaker Studio 2 dónde dibujar los valores en la sala. Esta flexibilidad es una de las razones principales por las que "for" Los bucles son tan importantes en la programación.

En una serie de situaciones, desea que sus instancias completen una acción en función de un valor particular. Puedes hacerlo usando una cantidad de " if "declaraciones pero cuando las opciones posibles superan los dos o tres, generalmente es más fácil usar el "switch" declaración. UNA switch La declaración tiene la siguiente forma:

switch (<expression>)
{
case <expression1>: <statement1>; ... ; break;
case <expression2>: <statement2>; ... ; break;
...
default: <statement>;
}


Esto funciona de la siguiente manera:

  • Primero se ejecuta la expresión.
  • A continuación, se compara con los resultados de las diferentes expresiones después de cada una de las declaraciones de casos.
  • La ejecución continúa después de la primera declaración de caso con el valor correcto, hasta que se encuentra una declaración de interrupción.
  • Si ninguna declaración de caso tiene el valor correcto, la ejecución continúa después de la declaración predeterminada (no se requiere que tenga una declaración predeterminada, en cuyo caso no se tomará ninguna medida).
Tenga en cuenta que se pueden colocar varias declaraciones de caso para la misma declaración. Además, no se requiere la interrupción, y si no hay una declaración de interrupción, la ejecución simplemente continúa con el código para la siguiente declaración de caso. Esto significa que puede crear un "interruptor" de jerarquía en el que se ejecutan diferentes secciones de código dependiendo del valor de entrada. Aquí hay un ejemplo de un típico "switch" de un juego:

{
switch (keyboard_key)
   {
   case vk_left:
   case ord("A"):
      x -= 4;
      break;
   case vk_right:
   case ord("D"):
      x += 4;
      break;
   case vk_up:
   case ord("W"):
      y -= 4;
      break;
   case vk_down:
   case ord("S"):
      y += 4;
      break;
   }
}


El código anterior utiliza "switch" para verificar un evento de teclado y luego lo compara con los casos enumerados. Si cumple alguno de los valores requeridos, se ejecuta el código correspondiente. Note cómo en el código hemos usado la forma en que "switch" puede verificar varios casos y continuar si no se encuentra una interrupción para permitir que se utilicen varias claves para obtener el mismo resultado. Esta es solo una de las formas en que puede permitir múltiples configuraciones de movimiento en sus juegos.

los "break" declaración se utiliza para terminar prematuramente una for, repeat, while, do... until bucle de algún tipo, o para decirle a un switch declaración para terminar en ese punto, o para terminar prematuramente un with llamada. A continuación, puede ver algunos ejemplos de cómo se puede usar, y su sintaxis es simplemente:

break;


"romper" en un for lazo:

{
var i;
for (i = 0; i < 10; i += 1)
   {
   if array[i] = 234 break;
   }
num = i;
}


"romper" en un repeat lazo:

{
var i, temp;
i = 0;
temp = 0;
repeat (10)
   {
   temp += array[i];
   if temp > max_total break else i += 1;
   }
}


"romper" en un while lazo:

{
var i;
i = 0;
while (!place_free(x, y))
   {
   x = random(room_width);
   y = random(room_height);
   if i > 50 break else i+=1;
   }
}


"romper" cuando se usa with:

{
var count = 0;
with (obj_Enemy)
   {
   count++;
   if count > 10 break;
   hp = 100;
   }
}


los "continue" La declaración tiene la forma:

continue


Si se usa dentro de una declaración que forma un bucle ( repeat, while, do... until o for ), saltará inmediatamente al comienzo del bucle, como si el bucle se hubiera ejecutado y regresado. También hará lo mismo al usar el with Función, donde hará que el código salte a la siguiente instancia y se ejecute de nuevo, pero si se usa fuera de contexto, dará un error.

A continuación se muestra un ejemplo de su uso en un bucle "for":

{
var i;
for (i = 0; i < 10; i += 1)
   {
   if array[i] = "" continue;
   array[i] = "";
   }
}


El código anterior volverá al principio del bucle si el valor de la matriz [i] ya es una cadena vacía.

los "exit" La declaración tiene la forma:

exit;


"Salir" simplemente finaliza la ejecución del script o evento actual. Tenga en cuenta que hay una ligera diferencia en el uso aquí, dependiendo del alcance: si usa exit en una secuencia de comandos simplemente saldrá de la secuencia de comandos y volverá al código que llamó a la secuencia de comandos; sin embargo, si usa este evento en un bloque de código dentro de un objeto, saldrá de todo el evento incluso si hay varios bloques de código separados después de la secuencia de comandos. La función ha sido llamada. Normalmente se usa para evitar que una instancia ejecute un bloque de código específico, por ejemplo, un evento de colisión. El siguiente código da un ejemplo simple de esto:

{
if !visible exit;
while (place_meeting(x, y))
   {
   x -= lengthdir_x(1, direction - 180);
   y -= lengthdir_y(1, direction - 180);
   }
}


El código anterior verifica una variable y si se resuelve en verdadero, entonces existe el bloque de código, de lo contrario, continúa y ejecuta el resto del código.
Nota: No finaliza la ejecución del juego. Para eso necesitas usar la función. game_end.

Como se indica en la sección Cómo abordar las variables en otras instancias, es posible leer y cambiar el valor de las variables en otras instancias. Pero en algunos casos desea hacer mucho más que cambiar una sola variable con esas otras instancias. Por ejemplo, imagina que quieres mover todos los objetos de bolas en tu juego de 8 píxeles. Puede pensar que esto se logra simplemente mediante el siguiente código:

obj_ball.y = obj_ball.y + 8;


Pero esto no es correcto, ya que el lado derecho de la asignación obtiene el valor de la coordenada y de la primera bola y le agrega 8. A continuación, este nuevo valor se establece como la coordenada y de todas las bolas, por lo que el resultado es que todas las bolas obtienen la misma coordenada y, e incluso si usa lo siguiente:

obj_ball.y += 8;


Tendrá exactamente el mismo efecto porque es simplemente una abreviatura de la primera declaración. Entonces, ¿cómo lograr esto? Para ello existe el with Declaración en GML. Su forma global es:

with (<expression>) <statement>


<Expression> indica una o más instancias, y para esto puede usar un id de instancia, el nombre de un objeto (que indica que todas las instancias de este objeto son para ejecutar el bloque de código) o una de las palabras clave especiales ( all, self, other ). <Estado de cuenta> ahora se ejecuta para cada una de las instancias indicadas, como si esa instancia fuera la instancia actual (propia). Por lo tanto, para mover todas las instancias del objeto de bola 8 píxeles hacia abajo, puede escribir:

with (obj_ball) y += 8;


Si desea ejecutar varias instrucciones, coloque rizos entre ellas, de la misma forma que lo haría con cualquier otro programa. Por ejemplo, para mover todas las bolas a una posición aleatoria, puedes usar:

with (obj_ball)
   {
   x = random(room_width);
   y = random(room_height);
   }


Tenga en cuenta que, dentro de la (s) declaración (es), la instancia indicada se ha convertido en la instancia de destino (auto) que ejecuta el bloque de código, lo que significa que las declaraciones de la instancia original (que contiene el "with" y el bloque de código) se ha convertido en el other ejemplo. Por ejemplo, para mover todas las bolas a la posición de la instancia actual, puede escribir esto:

with (obj_ball) { x = other.x; y = other.y; }


La declaración with es una herramienta extremadamente poderosa y es útil en muchas, muchas circunstancias, por lo que es importante que entiendas completamente cómo se puede usar. Para ayudar hay algunos ejemplos más de uso a continuación:

with (instance_create_layer(x, y, "Instances", obj_Ball))
   {
   speed = other.speed;
   direction = other.direction;
   }


El código anterior creará una instancia de obj_Ball y le asignará la velocidad y la dirección de la instancia que ejecuta todo el bloque de código.

with (instance_nearest(x, y, obj_Ball))
   {
   instance_destroy();
   }


El código anterior destruirá la instancia de obj_Ball más cercana a la instancia que ejecuta el código.

var inst;
inst = noone;
with (obj_ball)
   {
   if str > other.str inst = id;
   }
if inst != noone target = inst;


El código anterior es ligeramente más complejo que los anteriores debido a que utiliza una variable local. Esta variable es local para el script y no para la instancia y, por lo tanto, todas las instancias a las que se hace referencia dentro del bloque de código pueden usar y acceder a ellas. Así, en el código lo hemos establecido a la palabra clave especial noone tt> y luego use la construcción "con" para que cada instancia de obj_Ball verifique su variable "str" contra la de la instancia que ejecuta el bloque de código. Si el valor de la variable es mayor, entonces almacenan su ID único en la variable local "inst", lo que significa que al final del código, solo la instancia con un valor mayor que la instancia que llama (o la palabra clave) noone si ninguno es más grande) se almacenará en la variable local inst. Para obtener más información sobre las variables locales, consulte la sección Variables y alcance de la variable.

los "return" La declaración tiene la forma:

return (<expression>)


Solo usas el return en los scripts, y se usa para devolver un valor del script que se usará en otras llamadas de código o script. Se debe tener en cuenta que la ejecución del script finaliza en la declaración de devolución, lo que significa que no se ejecutará ningún código que venga después de que se haya llamado a la devolución. Aquí hay un breve ejemplo de script llamado " scr_sqr "que calcula el cuadrado del valor que se le pase, e incluye la captura de errores en caso de que el argumento de que se pasa no sea un número real:

{
if !is_real(argument0)
   {
   return 0;
   }
else
   {
   return (argument0 * argument0);
   }
}


Para llamar a un script desde un fragmento de código, actúe de la misma manera que cuando llama a las funciones, es decir, escriba el nombre del script con los valores de argumento entre paréntesis. Entonces, el script anterior se llamaría así:

if keyboard_check_pressed(vk_enter)
   {
   val = scr_sqr(amount);
   }
