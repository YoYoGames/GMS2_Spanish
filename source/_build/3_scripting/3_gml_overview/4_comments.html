Comentarios en el código


Cuando se trabaja en colaboración con otras personas, cuando se trata de un proyecto grande, o incluso para su propia referencia y depuración, es muy importante dejar comentarios en sus códigos y scripts. En GameMaker Studio 2 tienes varios mecanismos para ayudarte a dejar notas y comentarios sobre secciones de código e incluso bloques completos de código que se pueden ver en la lista de eventos del objeto que estás editando actualmente, y de esta manera puedes dejar notas para usted y sus colegas, o para explicar una pieza de código particularmente engañosa, o simplemente dejar un recordatorio sobre lo que hace algo.

Lo primero que puedes hacer es dejar un breve comentario usando // antes del texto Por ejemplo:

//initialize variables
sr = 10;
hp = 100;

También puede dejar comentarios de varias líneas, para dar crédito, omitir una sección completa de código para la depuración, o incluso para explicar los argumentos de un guión. Para eso puedes usar /* ... */ Me gusta esto:

/*
usage:
diff = angle_difference(angle1,angle2);

angle1 first direction in degrees, real
angle2 second direction in degrees, real

returns: difference of the given angles in degrees, -180 to 180

GMLscripts.com
*/
{
return ((((argument0 - argument1) mod 360) + 540) mod 360) - 180;
}

Finalmente, si está escribiendo scripts personalizados y quiere tener algún control sobre cómo son tratados por el IDE (es decir, tiene autocompletar o mostrar los argumentos del script) entonces debe escribir sus comentarios usando la notación JSDoc como se explica aquí.
