Funciones


La definición general de una función en GML es algo como esto:

Una función tiene una entrada y una salida, y la salida está relacionada de alguna manera con la entrada.

En GameMaker Studio 2 usa funciones como parte de la estructura del código para que suceda lo mismo y hay una gran cantidad de funciones GML disponibles para usted, todas explicadas en la sección de Referencia del lenguaje del manual.

En GML, una función tiene la forma de un nombre de función, seguido por los argumentos de entrada entre paréntesis y separados por comas (si la función no tiene argumentos de entrada, entonces solo se usan corchetes). Aquí hay un resumen de la estructura de una función:

<function>(<arg0>, <arg1> ,... <arg15>);

En general, hay dos tipos de funciones: primero, existe la enorme colección de funciones integradas que se utilizan para controlar todos los aspectos de su juego y, en segundo lugar, cualquier secuencia de comandos que defina en su juego también se puede usar como una función ( pero no siempre).

Algunas funciones devuelven valores y se pueden usar en expresiones, mientras que otras simplemente ejecutan comandos, como se ilustra en los dos ejemplos siguientes:

instance_destroy();                              // Destroy the calling instance - this requires no arguments and returns nothing
dist = point_distance(x, y, mouse_x, mouse_y;    // Get the distance from the current instance position to the mouse position - takes four arguments and returns a real value

Debe tener en cuenta que es imposible usar una función por sí mismo como el lado izquierdo de una tarea. Por ejemplo, el siguiente código le daría un error:

instance_nearest(x, y, obj).speed = 0;

El valor de retorno para la expresión en ese ejemplo de código es un número real (el valor único de ID para la instancia más cercana) y debe estar entre corchetes para ser utilizado de esta manera y abordar adecuadamente la instancia requerida (ver Direccionamiento de variables en otro Instancias para más información). El código anterior se escribiría correctamente como:

(instance_nearest(x, y, obj)).speed = 0;
//or
var inst = instance_nearest(x, y, obj);
inst.speed = 0;
