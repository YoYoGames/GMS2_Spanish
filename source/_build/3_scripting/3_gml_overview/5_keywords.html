Palabras clave


Para facilitar ciertas cosas en GameMaker Studio 2, puede usar una de varias palabras clave en sus secuencias de comandos y acciones. Estas palabras clave se usan principalmente para identificar instancias y cada una se explica en el texto a continuación. Tenga en cuenta que todas las palabras clave están representadas por un valor entero negativo internamente, por lo que debe tener cuidado al asignar valores a las variables, ya que puede obtener resultados inesperados más adelante ya que GameMaker Studio 2 interpreta el valor que ha utilizado como otra cosa. También debe tener en cuenta que el uso de los valores en lugar de las palabras clave en su código no es en absoluto recomendado y podría causar problemas más adelante.

Palabra clave Descripción valor
self La instancia que está ejecutando el bloque actual de código. -1

self se puede utilizar para identificar la instancia de llamada del bloque de código actual, ya que siempre es la ID única para la instancia que se encuentra actualmente en el alcance. Por ejemplo:

var val = 100;
with (instance_create_layer(x, y, "Instances", obj_Fire))
  {
  self.val = val;
  }

En este ejemplo, puede ver que tenemos una variable local llamada val y queremos que establezca la variable de instancia con el mismo nombre en la instancia de objeto recién creada. Para identificar la variable de instancia correctamente y decirle a GameMaker Studio 2 que la configure en la instancia que llama al bloque de código, usamos el self palabra clave. En la mayoría de los casos, también puede usar id palabra clave en lugar de self, pero self ofrece ciertos beneficios. Para empezar, es más rápido para el compilador recuperar el valor de ID de instancia usando self más bien que id, como el id el valor pasa por la tabla de búsqueda de instancia. En segundo lugar, para las personas que realizan extensiones, es muy útil garantizar el alcance correcto de las variables, ya que es posible que un proyecto que utiliza una extensión tenga una variable de ámbito global con el mismo nombre que una variable en la extensión.


Palabra clave Descripción valor
other La otra instancia involucrada en un evento de colisión, o la otra instancia de un with función. -2

La palabra clave especial other tiene dos maneras diferentes en que puede usarse para hacer referencia a una instancia específica: cuando se usa en una with función (explicada aquí ) o cuando se utiliza en un evento de colisión, que es lo que esta sección va a explicar.

Un evento de colisión solo puede ocurrir entre dos instancias. Puede tener varias colisiones entre varias instancias, pero todos ellos son resueltos por GameMaker Studio 2 sobre una base de 1-a-1, con la instancia que tiene el evento de colisión y la "otra" instancia que está implicado. Imagine que tiene un objeto jugador, múltiples objetos enemigos y múltiples objetos de bala que el enemigo puede disparar contra usted. Puedes asignar a cada enemigo una única instancia de bala pero con una variable de daño diferente asignada al azar cuando se crea, por ejemplo:

var nnn;
nnn = instance_create_layer(x, y, "Bullets", obj_Bullet);
nnn.damage = 5 + irandom(5);
nnn.speed = 8;
nnn.direction = point_direction(x, y, obj_Player.x, obj_Player.y);

Vea cómo establecemos sus variables usando el método de puntos descrito en la sección sobre Variables de direccionamiento ? Esto le dará a los objetos de bala un valor de daño diferente. Pero, ¿y el objeto jugador? ¿Cómo detectará el daño que tiene que tomar? Mediante el uso other en el evento de colisión:

hp -= other.damage;
if hp <= 0 instance_destroy();

El código anterior deducirá la cantidad almacenada en la otra instancia en la variable "daños" de colisiones de la variable "hp" del jugador, luego verificará si el "hp" es menor o igual a 0. Si es entonces destruirá el objeto jugador. Tenga en cuenta que otros utilizados de esta manera solo funcionan en el evento de colisión y que la otra instancia debe tener la variable que se verifica o de lo contrario se producirá un error. Sin embargo, puede asignar valores a las variables, o incluso crear nuevas, usando otras en el evento de colisión también, como esta:

other.mana += 10; //add ten to the other instance "mana" variable
other.hit = true; //set the other instance variable "hit" to true, creating it if the variable doesn't already exist


Palabra clave Descripción valor
all Todas las instancias actualmente activas en la sala. -3

Esta palabra clave se usa para indicar a GameMaker Studio 2 que se debe aplicar una función, o para verificar, todas las instancias activas dentro de una habitación (las instancias desactivadas no se verificarán ni se accederá a ellas). No puedes usar all para acceder o establecer variables en otras instancias usando el método de punto (ver aquí ), pero puedes usarlo cuando llamas with(), por ejemplo:

with (all)
   {
   speed = 0;
   }

El código anterior establecerá la velocidad de todas las instancias en la sala a 0. También puede usar all dentro de las funciones para apuntar o verificar todas las instancias en la sala, por ejemplo:

inst = instance_position(mouse_x, mouse_y, all);               //Check a point for any active instance in the room

if collision_line(x, y, mouse_x, mouse_y, all, false, true) {} //Check all instances for a collision along a line

mp_grid_add_instances(grid, all, false);                       //Add all instances in the room into a motion planning grid

all es una palabra clave muy útil y se puede utilizar en numerosas situaciones dentro de su código y acciones, a menudo reduciendo la cantidad de código que necesita escribir para lograr el efecto deseado.


Palabra clave Descripción valor
noone Ninguna instancia en absoluto. -4

Puede parecer extraño, pero muchas veces mientras programa sus juegos, encontrará la necesidad de verificar si no hay instancias encontradas en una ubicación, colisión, etc. En esos casos, usaría esta palabra clave para no verificar nada, algo como esto:

if instance_nearest(x, y, obj_enemy) != noone
   {
   //do something as there is an enemy instance near
   }

En este ejemplo, la función instance_nearest() Regresará ya sea noone o la identificación única de la instancia encontrada más cercana. Básicamente, cada vez que necesite verificar una instancia, puede esperar obtener noone o una ID de instancia única devuelta.

Vale la pena señalar que la palabra clave struct es una palabra clave reservada para posibles adiciones futuras a GML y, como tal, no se puede usar como una variable o nombre de script, y tampoco tendrá código de color en el IDE.
