Expresiones


Una expresión es una frase matemática que puede contener números ordinarios, variables, cadenas o funciones GML, así como uno o más operadores (como sumar, restar, multiplicar, etc.). Los valores utilizados en una expresión pueden ser números reales (por ejemplo, 3.4 * 6), números hexadecimales que comienzan con un signo $ (ej. $00FFAA | $88FFAA ), cadenas entre comillas dobles (ej. "Hola" + "mundo") o más complicados expresiones que usan múltiples operadores y valores.

Para expresiones, existen los siguientes operadores (en orden de prioridad):

= - Se usa para asignar un valor a una variable. Tenga en cuenta que esto también se puede usar para comparar variables en GameMaker Studio 2 y puede ver esto en ejemplos y otros códigos de personas. Sin embargo, este es un legado de versiones anteriores de GameMaker y debe usar los operadores == para comparar y = para asignar, como se muestra en estos ejemplos:

a = 12;
speed = 5;
val = (old_val + 5);


&&, ||, ^^ (y, oy xor) - Combina valores booleanos para dar verdadero o falso. Si alguno de los siguientes ejemplos resuelve true entonces el código se ejecutará:

if (a == b && c == d) {do something...}// and
if (a == b || c ==d) {do something...}// or
if (a == b ^^ c == d) {do something...} // xor


<, <=, ==,! =,>,> = - Estas son comparaciones y solo pueden dar una true o false resultado (donde verdadero también se puede interpretar como 1 y falso como 0). Ejemplos de uso:

if (a < b) {do something...}
if (a != b) {do something...}


|, &, ^, <<, >>: puede realizar operaciones bit a bit con estos, donde | = bitwise o, & = bitwise y, ^ = bitwise xor, << = shift left, >> = shift right. Ejemplos de uso:

x = (x & $ffffffe0) + 32;
if (y ^ $1f) > 0 {do something...};


Puede encontrar información adicional sobre cómo usar los operadores bit a bit y qué hacen desde la sección: Operadores bit a bit.

+, -, *, / - Sumar, restar, multiplicar y dividir. Ejemplos de uso:

c = a * b;
str = a + "world";


++, - - Agregar o restar uno de un valor. Vale la pena señalar que colocar esto antes o después del valor para agregar o restar tendrá resultados ligeramente diferentes. Por ejemplo:

  • ++ a incrementará la variable y devolverá el valor incrementado.
  • a ++ incrementará la variable pero devolverá el valor antes de que se incremente.
Por lo tanto, si tienes algo como esto:

var a = 1;
show_debug_message(string(a++));
show_debug_message(string(++a));


La salida de depuración sería 1 y 3. Aquí hay algunos ejemplos de uso:

for (var i = 0; i < 10; i++;) {do something...};
if hit == true score--;


NOTA: En las plataformas de objetivos del Compilador YoYo (aquellas marcadas (YYC)), estas expresiones se evalúan de izquierda a derecha, mientras que en todas las demás plataformas de destino se evalúan de derecha a izquierda, lo que significa que:

val = max(num, ++num, num++);


dará diferentes resultados dependiendo de la plataforma.

div, mod (%) - División y módulo, donde div te da la cantidad en que se puede dividir un valor produciendo solo un cociente entero, mientras que mod te da solo el resto de una división. Ejemplos de uso:

secs = time mod 60;
time_str = string(time div 60);


Además, existen los siguientes operadores unarios:
  • !: booleano "no", entonces !true == false
  • -: niega el siguiente valor real o entero (no válido para cadenas o booleanos)
  • ~: niega el siguiente valor en modo bit

Como valores en todas las expresiones, puede usar números, variables o funciones que devuelven un valor, y las subexpresiones también pueden colocarse entre corchetes. Todos los operadores trabajan para valores reales, pero las comparaciones también funcionan para cadenas y "+" concatena cadenas.


Orden de operaciones

Cuando se realizan operaciones múltiples en una sola expresión, es muy importante que utilice corchetes () para separar el orden de la operación, ya que las diferentes plataformas pueden realizarlos de manera diferente si no se establece explícitamente de esta manera. Por ejemplo, considere el siguiente código:

a = b == c || d;

Los diferentes compiladores de destino realizarán las operaciones en diferentes órdenes, ya que no se muestran explícitamente, lo que da lugar a resultados "extraños" que no puede esperar al jugar su juego. para evitar esto, use () para separar las partes, como esta:

a = (b == c || d); //better
a = ((b == c) || d); //best

Aquí hay algunos ejemplos finales de las diferentes expresiones diferentes:

{
x = 23;
colour = $FFAA00;
str = "hello world";
y += 5;
x *= y;
x = y << 2;
x = 23 * ((2 + 4) / sin(y));
str = "hello" + "world";
b = (x < 5) && !(x==2 || x==4);
}
