collision_line

Sintaxis:

collision_line( x1, y1, x2, y2, obj, prec, notme );


Argumento Descripción
x1 La coordenada x del inicio de la línea.
y1 La coordenada y del inicio de la línea.
x2 La coordenada x del final de la línea.
y2 La coordenada y del final de la línea.
obj El objeto para verificar, por ejemplo, las colisiones.
prec Si la verificación se basa en colisiones de píxeles perfectos (verdadero = lento) o su cuadro delimitador en general (falso = rápido).
notme Si la instancia de llamada, si corresponde, debe excluirse (verdadero) o no (falso).


Devoluciones: id de instancia o noone


Descripción

La línea de colisión comprueba a lo largo de una línea desde el punto x1, y1 hasta el punto x2, y2 para una colisión con cualquier instancia del objeto especificado por el argumento "obj". esta comprobación puede ser precisa o no, pero para que las colisiones precisas estén habilitadas, el objeto o instancia que está comprobando también debe tener colisiones precisas habilitadas para su sprite. De lo contrario, la verificación predeterminada se basa en recuadros delimitadores. La siguiente imagen ilustra cómo funciona esto:

Recuerde, para que se consideren las colisiones precisas, tanto el objeto sprite como la función de colisión deben marcarse con precisión como on. También se debe tener en cuenta que el valor de retorno de la función puede ser el id de cualquiera de las instancias consideradas en colisión a lo largo de la línea, por lo que si tres instancias se superponen a la línea definida, cualquiera de sus identificadores podría ser el valor de retorno de la función.


Ejemplo:

with (obj_Enemy)
   {
   if collision_line(100, 400, 100, 600, id, false, false) instance_destroy();
   }

Este código obtiene todas las instancias de "obj_Enemy" para verificar a lo largo de una línea de 100x, 400y a 100x, 600y para una colisión con ellos mismos, y si hay uno, se destruyen.
