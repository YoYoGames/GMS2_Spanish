colision_circle


Descripción

El círculo de colisión crea una zona circular del radio dada por el usuario alrededor de los puntos x1, y1 y luego verifica esa zona para detectar una colisión con cualquier instancia del objeto especificado por el argumento "obj". Esta comprobación puede ser precisa o no, pero para habilitar las colisiones precisas, el objeto o la instancia que está verificando también debe tener colisiones precisas habilitadas para su sprite. De lo contrario, la verificación predeterminada se basa en cuadros delimitadores. La siguiente imagen ilustra cómo funciona esto:

Aquí, la instancia en el medio está usando un círculo de colisión para verificar si hay objetos de pelota. Ahora, los azules no tienen un cuadro delimitador preciso y, como puede ver, incluso si el sprite no toca realmente el círculo, la colisión aún puede ocurrir (incluso si establece la opción precisa en la función como verdadera) el cuadro delimitador de ese sprite sobrepasa el área circular definida por collision_circle. Por otro lado, las bolas verdes solo se considerarán en colisión si el sprite real se sobrepasa con el círculo definido. Recuerde, para que las colisiones precisas se consideren, tanto el objeto sprite como la función de colisión deben estar marcadas con precisión como activadas. También se debe tener en cuenta que el valor de retorno de la función puede ser el ID de cualquiera de las instancias consideradas en colisión.


Sintaxis:

collision_circle(x1, y1, rad, obj, prec, notme);

Argumento Descripción
x1 La coordenada x del centro del círculo a comprobar.
y1 La coordenada y del centro del círculo a verificar.
rad El radio (distancia en píxeles desde su centro hasta su borde).
obj El objeto para comprobar las colisiones de instancia.
prec Si la comprobación se basa en colisiones de píxeles perfectos (verdadero = lento) o en su cuadro delimitador en general (falso = rápido).
notme Si la instancia que llama, si es relevante, debe ser excluida ( true ) o no ( false ).


Devoluciones:

Instance id or noone


Ejemplo:

if collision_circle(x, y, 20, obj_Cursor, false, true)
   {
   image_index = 1;
   }
else image_index = 0;

El código anterior verificará que las circulares estén en un radio de 20 píxeles para una colisión con "obj_Cursor" y si hay una, establecerá el índice de imagen del objeto que ejecuta el código en 1, pero si no existe, establecerá el índice de imagen del objeto a 0.
