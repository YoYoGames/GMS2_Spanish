colision_rectangle


Descripción

Collision_rectangle usa los primeros cuatro argumentos (x1, y1, x2, y2) para definir un área dentro de la sala actual y luego verifica si algún objeto definido por el argumento "obj" está en colisión con esa área. Esta colisión se puede verificar como precisa o no, y también puede optar por verificar si la instancia ejecuta el código o no. Considera esta imagen:

Aquí, la instancia en el medio está usando un rectángulo de colisión para verificar si hay objetos de pelota. Ahora, los azules no tienen un cuadro delimitador preciso y, como puede ver, incluso si el sprite no toca realmente el rectángulo, la colisión seguirá ocurriendo (incluso si establece la opción precisa en la función en verdadero) como la cuadro delimitador de ese sprite sobrepasa el collision_rectangle. Por otro lado, las bolas verdes solo se considerarán en colisión si el sprite real sobrepasa el rectángulo. Recuerde, para que las colisiones precisas se consideren, tanto el objeto sprite como la función de colisión deben estar marcadas con precisión como activadas. También se debe tener en cuenta que el valor de retorno de la función puede ser el ID de cualquiera de las instancias consideradas en colisión.


Sintaxis:

collision_rectangle(x1, y1, x2, y2, obj, prec, notme);

Argumento Descripción
x1 La coordenada x del lado izquierdo del rectángulo a verificar.
y1 La coordenada y del lado superior del rectángulo a verificar.
x2 La coordenada x del lado derecho del rectángulo a verificar.
y2 La coordenada y del lado inferior del rectángulo a verificar.
obj El objeto para comprobar las colisiones de instancia.
prec Si la verificación se basa en colisiones precisas ( true, que es más lento) o su caja delimitadora en general ( false, Más rápido).
notme Si la instancia que llama, si es relevante, debe ser excluida ( true ) o no ( false ).


Devoluciones:

Instance id or noone


Ejemplo:

var inst;
inst = collision_rectangle(50, 50, 200, 100, obj_Ball, false, true);
if inst != noone
   {
   with (inst) instance_destroy();
   }

Este código abreviado utiliza collision_rectangle para verificar un área en la habitación desde 50x, 50y (parte superior izquierda del rectángulo) hasta 200x, 200y (parte inferior derecha del rectángulo) para una instancia de un objeto llamado "obj_ball". Almacena el valor de retorno en una variable temporal que luego se verifica para ver si ese valor es un id de instancia o la palabra clave nadie. Si no hay nadie entonces se utiliza el ID de instancia almacenada para destruir el objeto.
