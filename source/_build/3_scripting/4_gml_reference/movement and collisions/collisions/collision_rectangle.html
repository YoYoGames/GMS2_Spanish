collision_rectangle

Sintaxis:

collision_rectangle( x1, y1, x2, y2, obj, prec, notme );


Argumento Descripción
x1 La coordenada x del lado izquierdo del rectángulo para verificar.
y1 La coordenada y del lado superior del rectángulo para verificar.
x2 La coordenada x del lado derecho del rectángulo para verificar.
y2 La coordenada y del lado inferior del rectángulo para verificar.
obj El objeto para verificar, por ejemplo, las colisiones.
prec Si la verificación se basa en colisiones de píxeles perfectos (verdadero = lento) o su cuadro delimitador en general (falso = rápido).
notme Si la instancia de llamada, si corresponde, debe excluirse (verdadero) o no (falso).


Devoluciones: id de instancia o noone


Descripción

Collision_rectangle usa los primeros cuatro argumentos (x1, y1, x2, y2) para definir un área dentro de la habitación actual y luego verifica si algún objeto que está definido por el argumento "obj" está en colisión con esa área. Esta colisión puede verificarse como precisa o no, y también puede optar por verificar si la instancia ejecuta el código o no. Considera esta imagen:

Aquí, la instancia en el medio está usando un rectángulo de colisión para buscar objetos de bola. Ahora, los azules no tienen un cuadro delimitador preciso y como puede ver, incluso si el sprite no está tocando realmente el rectángulo, la colisión aún ocurrirá (incluso si establece la opción precisa en la función en verdadero) como el la caja de ese sprite sobrepasa el collision_rectangle. Por otro lado, las bolas verdes solo se considerarán en colisión si el sprite real sobrepasa el rectángulo. Recuerde, para que se consideren las colisiones precisas, tanto el objeto sprite como la función de colisión deben marcarse con precisión como on. También se debe tener en cuenta que el valor de retorno de la función puede ser el ID de cualquiera de las instancias que se considera que están en colisión.


Ejemplo:

var inst;
inst = collision_rectangle(50, 50, 200, 100, obj_Ball, false, true);
if inst != noone
   {
   with (inst) instance_destroy();
   }

Este código abreviado usa collision_rectangle para verificar un área en la habitación desde 50x, 50y (esquina superior izquierda del rectángulo) hasta 200x, 200y (esquina inferior derecha del rectángulo) para una instancia de un objeto llamado "obj_ball". Almacena el valor de retorno en una variable temporal que luego se verifica para ver si ese valor es una ID de instancia, o la palabra clave noone. Si no hay nadie entonces se utiliza el ID de instancia almacenada para destruir el objeto.
