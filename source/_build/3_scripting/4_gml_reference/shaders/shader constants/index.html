Shader Constantes

Esta sección enumera las constantes específicas de GameMaker Studio 2 utilizadas para crear sombreadores.

Además de las funciones y constantes de sombreado definidas en las páginas de referencia del lenguaje de sombreado de OpenGL ES (GLSL ES), también hay varias constantes de sombreado disponibles para usted que son exclusivas de GameMaker Studio 2.

Matrices

Las siguientes constantes de matriz de visualización se pueden usar como índices de matriz cuando se utiliza la constante de matriz de sombreado gm_Matrices:

Constante Descripción
MATRIX_VIEW Esta constante de índice de matriz contiene el índice de la matriz de vista actual. El índice devuelto se usaría como un valor de matriz al llamar al gm_Matrices constante dentro del código de sombreado.
MATRIX_PROJECTION Esta constante de índice de matriz contiene el índice de la matriz de proyección actual. El índice devuelto se usaría como un valor de matriz al llamar al gm_Matrices constante dentro del código de sombreado.
MATRIX_WORLD Esta constante de índice de matriz contiene el índice de la matriz mundial actual. Esto se puede usar para cosas como la iluminación si tienes información liviana en el espacio-mundo. El índice devuelto se usaría como un valor de matriz al llamar al gm_Matrices constante dentro del código de sombreado.
MATRIX_WORLD_VIEW Esta constante de índice de matriz mantiene el índice del resultado del mundo y las matrices de vista se multiplican juntas. Esto se usa a menudo para cosas como la niebla. El índice devuelto se usaría como un valor de matriz al llamar al gm_Matrices constante dentro del código de sombreado.
MATRIX_WORLD_VIEW_PROJECTION Esta constante de índice de matriz mantiene el índice del resultado del mundo, las matrices de proyección y de proyección se multiplican juntas. Esta es la matriz de transformación normal utilizada para las posiciones de los vértices. El índice devuelto se usaría como un valor de matriz al llamar al gm_Matrices constante dentro del código de sombreado.
MATRIX_MAX Esto no es un índice de matriz, sino que devuelve el tamaño de la matriz matriz en el sombreador de vértices.
MAX_VS_LIGHTS Esto no es un índice de matriz, sino que devuelve el número de luces en el sombreado de vértices.

Otras constantes

Los siguientes uniformes y constantes de matriz predefinidos se pueden usar en su sombreador para acceder a los valores específicos de GameMaker Studio 2:

Constante Descripción
gm_Matrices[matrix] Esta constante de matriz devuelve una matriz de transformación y es uno de los uniformes predefinidos disponibles que GameMaker Studio 2 crea para su uso dentro del editor de código de sombreado. El índice de matriz se elige de una de las constantes enumeradas anteriormente.
gm_BaseTexture Esta es una constante de muestreador 2D que devuelve la textura del objeto actual, tal como lo establece GameMaker Studio 2. Por lo tanto, sería el sprite, la superficie o la textura actuales que se utilizarían normalmente al dibujar sin que se llame al sombreador.
gm_LightingEnabled Esto se puede usar para obtener o configurar la iluminación de GameMaker Studio 2 cuando se usa 3D.
gm_FogStart Esto se puede usar para obtener la distancia donde los polígonos comienzan a mezclarse con el color de la niebla.
gm_RcpFogRange Esto se puede usar para obtener la distancia a la que la niebla es máxima y ya no se puede ver nada.
gm_PS_FogEnabled Esto regresará true o false si la GPU tiene habilitada la niebla de píxeles o no.
gm_FogColour Esto se puede usar para obtener el color de niebla utilizado por GameMaker Studio 2.
gm_VS_FogEnabled Esto regresará true o false si la GPU tiene niebla de vértices habilitada o no.
gm_AlphaTestEnabled Esto se puede usar para obtener pruebas alfa en el sombreador. Ver gpu_set_alphatestenable para obtener más información sobre las pruebas alfa.
gm_AlphaRefValue Esto se puede usar para obtener el valor de referencia de prueba alfa actual. Ver gpu_set_alphatestref para más información sobre la referencia de prueba alfa.
