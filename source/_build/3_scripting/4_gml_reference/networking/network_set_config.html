network_set_config

Sintaxis:

network_set_config(config_value, setting);


Argumento Descripción
config_value La constante de valor de configuración a establecer (enumerada a continuación).
setting La configuración del valor de configuración.


Devoluciones: Cadena


Descripción

Con esta función puede configurar diferentes configuraciones de red. Estas configuraciones se dan como constantes y la configuración dependerá de la constante que haya elegido. La siguiente tabla enumera las constantes disponibles y sus configuraciones:

Constante Descripción Ajuste
network_config_connect_timeout Establecer un valor de tiempo de espera de conexión A value in milliseconds
network_config_use_non_blocking_socket Dígale a GameMaker Studio 2 no bloquee en la conexión. 0 = block, 1 = don't block
network_config_enable_reliable_udp Habilita el protocolo "UDP confiable" para un socket UDP existente A value in milliseconds
network_config_disable_reliable_udp Desactiva el protocolo "UDP confiable" para un UDP existente socked. 0 = block, 1 = don't block



La primera opción ( network_config_connect_timeout ) simplemente establece el tiempo de espera para conectarse a un servidor, pero no cambia el comportamiento de conexión aparte del tiempo que tiene que esperar. Se recomienda que no establezca este valor demasiado bajo y que se mantenga en aproximadamente 1000 para un juego solo de LAN o 4000 o más para internet, o el juego puede fallar al conectarse al azar. Si desea establecer un valor de tiempo de espera para enviar / recibir paquetes, utilice la función network_set_timeout().

La segunda opción ( network_config_use_non_blocking_socket ) significa que todas las funciones de conexión de red devolverán un socket_id al instante, pero no puede enviar o recibir en ese socket hasta que haya recibido un evento de red asíncrono. El evento desencadenado tendrá la tecla "tipo" establecida en network_type_non_blocking_connect (Puede encontrar más detalles en la página sobre Evento asíncrono de red ). Esta es una configuración global ya que GameMaker Studio 2 no admite una combinación de bloqueo y no bloqueo en una aplicación.

La tercera y cuarta opciones ( network_config_enable_reliable_udp, network_config_disable_reliable_udp ) son para habilitar o deshabilitar el protocolo UDP confiable de GameMaker Studio 2 y, como tal, solo deben usarse con conexiones UDP. Lo que "confiable" en este caso significa que, cuando está habilitado en ambos extremos de una conexión, GameMaker Studio 2 intentará verificar que sus paquetes hayan llegado correctamente y reenviarse los que no lleguen (tenga en cuenta que no garantizamos en sus paquetes llegarán en orden de transmisión). Cuando se usa un tipo de socket confiable, se agregará un encabezado de 12 bytes a todos los paquetes udp que contiene la información que necesita GameMaker Studio 2 para verificar que el paquete no presente errores y reenviar paquetes faltantes.


Ejemplo:

network_set_config(network_config_connect_timeout, 1000);
network_set_config(network_config_use_non_blocking_socket, 1);

El código anterior establecerá el tiempo de espera para la conexión de red a 1000 ms y le dirá a GameMaker Studio 2 no se bloquee en la conexión.
