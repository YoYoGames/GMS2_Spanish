Redes

Las funciones de red de GameMaker Studio 2 se basan en sockets, que es un estándar de la industria en todas las plataformas. Con sockets, tiene la capacidad de crear clientes y servidores usando GML simple que le permite crear incluso un juego de un solo jugador usando el modelo cliente / servidor (algo que permitirá que el multijugador se agregue fácilmente más adelante).

Básicamente, un socket es un objeto que puede enviar, recibir, conectar y escuchar puertos en la red. Estas funciones se refieren solo a las conexiones TCP / IP, que es en lo que se basa Internet (IPv4 para ser más preciso). En esencia, esto le permite conectar un socket usando una dirección IP (digamos 192.168.1.10) a otro socket en otra dirección IP (digamos 192.168.1.11). Una vez conectados, estos dos enchufes pueden enviar datos hacia adelante y hacia atrás usando el network_send_* funciones (como network_send_packet ) y el evento asíncrono dedicado que se desencadena por ellos. Este evento asíncrono generará un ds_map que contiene los datos recibidos, así como otros detalles, y usted puede encontrar los detalles completos en la sección sobre el Evento Asíncrono de la Red.

En lugar de que los programas tengan que leer y lidiar con cada bit de tráfico de red que llega a una máquina, las direcciones IP también tratan con los puertos, que están numerados de 0 a 65535. Esto permite que cada programa obtenga solo los paquetes de datos que le interesan, y no todo lo que ha sido enviado por cada programa. Esto también ahorra un tiempo de CPU considerable, ya que reduce seriamente el procesamiento de datos, por lo que con GameMaker Studio 2 también puede "escuchar" los puertos. Entonces, un servidor creará un socket y luego lo vinculará a un puerto específico para escuchar. Luego recibirá notificaciones de conexiones, desconexiones y transferencias de datos desde ese socket.

Los "paquetes" de datos en sí mismos están formados por datos binarios tomados de un búfer. Este buffer se puede crear y manipular usando las funciones de Buffer de GameMaker Studio 2.

NOTA: Estas funciones no están actualmente disponibles para el módulo de destino HTML5.

Las siguientes funciones existen para que las use para redes:

  • network_create_server
  • network_create_server_raw
  • network_create_socket
  • network_create_socket_ext
  • network_connect
  • network_connect_raw
  • network_resolve
  • network_set_config
  • network_set_timeout
  • network_send_broadcast
  • network_send_packet
  • network_send_raw
  • network_send_udp
  • network_send_udp_raw
  • red_destroy
