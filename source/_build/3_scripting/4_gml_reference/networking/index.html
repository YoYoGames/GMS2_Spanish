Redes

Las funciones de red de GameMaker Studio 2 se basan en sockets, que es un estándar de la industria en todas las plataformas. Con los sockets, tiene la capacidad de crear clientes y servidores utilizando GML simple, lo que le permite crear incluso un juego para un solo jugador utilizando el modelo cliente / servidor (algo que permitirá agregar el multijugador fácilmente más adelante).

Básicamente, un socket es un objeto que puede enviar, recibir, conectarse y escuchar puertos en la red. Estas funciones solo tratan con conexiones TCP / IP, que es en lo que se basa Internet (IPv4 para ser más precisos). En esencia, esto le permite conectar un socket utilizando una dirección IP (digamos 192.168.1.10) a otro socket en otra dirección IP (digamos 192.168.1.11). Una vez conectados, estos dos sockets pueden enviar datos de un lado a otro utilizando el network_send_* funciones (como network_send_packet ) y el evento asíncrono dedicado que es activado por ellos. Este evento asíncrono generará un ds_map que contiene los datos recibidos, así como otros detalles, y puede encontrar los detalles completos en la sección sobre el evento asíncrono de la red.

En lugar de que los programas tengan que leer y lidiar con cada bit del tráfico de red que ingresa a la máquina, las direcciones IP también se ocupan de los puertos, que están numerados de 0 a 65535. Esto permite que cada programa obtenga solo los paquetes de datos que le interesan. en, y no todo lo que ha sido enviado por cada programa. Esto también ahorra un tiempo considerable de CPU, ya que reduce seriamente el procesamiento de datos, por lo que con GameMaker Studio 2 también puede "escuchar" en los puertos. Por lo tanto, un servidor creará un socket y luego lo vinculará a un puerto específico para escuchar. Luego recibirá notificaciones de conexiones, desconexiones y transferencias de datos desde ese socket.

Los "paquetes" de datos se componen de datos binarios tomados de un búfer. Este búfer se puede crear y manipular usando las funciones de la memoria intermedia de GameMaker Studio 2.

NOTA: estas funciones no están disponibles actualmente para el módulo de destino HTML5.

Las siguientes funciones existen para que las uses en redes:

  • network_create_server
  • network_create_server_raw
  • network_create_socket
  • network_create_socket_ext
  • network_connect
  • network_connect_raw
  • red_resolver
  • network_set_config
  • network_set_timeout
  • network_send_broadcast
  • network_send_packet
  • network_send_raw
  • network_send_udp
  • network_send_udp_raw
  • network_destroy
