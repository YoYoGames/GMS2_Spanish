clickable_add


Sintaxis:

clickable_add(x, y, tpe, url, target, params);


Argumento Descripción
x La posición x dentro de la ventana.
y La posición y dentro de la ventana.
tpe La entrada de la página de textura para el sprite que se utilizará (uso sprite_get_tpe )
url La URL (dirección del sitio web) a la que se debe vincular o la secuencia de comandos que se ejecutará.
target Esta es el área objetivo para abrir la URL en (ver descripción).
params Los diversos parámetros para el icono (ver descripción).


Devoluciones: Real


Descripción

Con esta función, puede crear un icono personalizado, que se puede hacer clic, que actúa como un "botón" fuera del lienzo del juego, pero dentro de la ventana del navegador. La función devolverá un valor de identificación que debe almacenarse en una variable para ser utilizado en todas las funciones de botón adicionales que hacen referencia al nuevo icono. Esta función es muy útil para evitar que el navegador cree una ventana emergente al hacer clic en el botón, ya que todo se hace a través de GameMaker: Studio en sí.

Primero debes establecer la posición, que es relativa a la (0,0) de la ventana del navegador (esquina superior izquierda) y luego dar la entrada de la página de textura para el sprite que deseas que tenga el icono del botón (ver: sprite_get_tpe ) A continuación, puede especificar una URL para abrir o una secuencia de comandos (desde el árbol de recursos de GameMaker: Studio ) para abrir o ejecutar. El parámetro "objetivo" que especifique es el mismo que el valor "nombre" de JavaScript estándar cuando usa el open() método (tenga en cuenta que todos menos '_self' pueden provocar el bloqueo del navegador, o preguntar al usuario si desean permitirlo) y el argumento "params" es el mismo que el parámetro "especificaciones" de JavaScript para controlar qué propiedades tiene el nuevo se debe mostrar la ventana / pestaña (no todos los navegadores pueden admitir todas las características). Si está utilizando el icono para ejecutar un script interno, estos dos últimos argumentos pueden dejarse como cadenas vacías "".

Los objetivos válidos son:

Objetivo Descripción
_blanco Abre el documento vinculado en una nueva ventana o pestaña.
_yo Abre el documento vinculado en el mismo cuadro en el que se hizo clic (es el valor predeterminado).
_padre Abre el documento vinculado en el marco principal.
_parte superior Abre el documento vinculado en todo el cuerpo de la ventana.

Los parámetros válidos son:

Parámetros Descripción
'height = [ px ]' La altura de la ventana, con un valor mínimo de 100.
'width = [ px ]' El ancho de la ventana, con un valor mínimo de 100.
'left = [ px ]' La posición izquierda de la ventana.
'top = [ px ]' La posición superior de la ventana (solo IE).
'location = [ boolean ]' Si se muestra o no el campo de dirección (el valor predeterminado es 1).
'menubar = [ boolean ]' Si se muestra o no la barra de menú (el valor predeterminado es 1).
'resizable = [ boolean ]' Si la ventana es redimensionable o no (por defecto es 1).
'scrollbars = [ boolean ]' Si se muestran o no las barras de desplazamiento (el valor predeterminado es 1).
'status = [ boolean ]' Si se agrega o no una barra de estado (el valor predeterminado es 1).
'titlebar = [ boolean ]' Si mostrar o no la barra de título. Esto se ignora a menos que la aplicación de llamada sea una aplicación HTML o un cuadro de diálogo de confianza (el valor predeterminado es 1);
'toolbar = [ boolean ]' Si se muestra o no la barra de herramientas del navegador (el valor predeterminado es sí).

También vale la pena señalar que si el argumento de URL que ingresa es el nombre de una secuencia de comandos (es decir, no comienza con "http: //") Y se llama la secuencia de comandos gmcallback_* luego puede obtener feeds directos en un evento html5. GameMaker: los nombres de las funciones de Studio no se ofuscan si comienzan con gmcallback_.


Ejemplo:

home_but = clickable_add(32, 32, sprite_get_tpe(spr_MS_Home, 0), "http://macsweeney_games.com", "_blank", "width=700, height=500, menubar=0, toolbar=0, scrollbars=0");

El código anterior crea un icono de DOM seleccionable en la posición (32, 32) de la página en la que se ejecuta el lienzo del juego. El ícono utiliza el sprite al que se hace referencia desde la página de texturas como "spr_MS_Home" y cuando se presiona el ícono abrirá una nueva ventana para la URL especificada y con las propiedades definidas para esa ventana.
