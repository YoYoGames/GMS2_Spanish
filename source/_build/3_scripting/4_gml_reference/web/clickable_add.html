clickable_add


Sintaxis:

clickable_add(x, y, tpe, url, target, params);


Argumento Descripción
x La posición x dentro de la ventana.
y La posición y dentro de la ventana.
tpe La entrada de la página de textura para el sprite que se utilizará (use sprite_get_tpe ).
url La URL (dirección del sitio web) para vincular o la secuencia de comandos que se ejecutará.
target Esta es el área de destino para abrir la URL en (ver descripción).
params Los distintos parámetros para el icono (ver descripción).


Devoluciones: Real


Descripción

Con esta función, puede crear un ícono personalizado, seleccionable, que actúa como un "botón" fuera del lienzo del juego, pero dentro de la ventana del navegador. La función devolverá un valor de identificación que debe almacenarse en una variable para ser utilizada en todas las funciones de botones posteriores que hagan referencia al nuevo icono. Esta función es muy útil para evitar que el navegador cree una ventana emergente al hacer clic en el botón, ya que todo se hace a través de GameMaker Studio 2.

Primero debe establecer la posición, que es relativa al (0,0) de la ventana del navegador (esquina superior izquierda) y luego dar la entrada de la página de textura para el sprite que desea que tenga el icono del botón (ver: sprite_get_tpe ). Luego puede especificar una URL para abrir o una secuencia de comandos (del árbol de recursos de GameMaker Studio 2 ) para abrir o ejecutar. El parámetro "objetivo" que especifique es el mismo que el valor estándar de "nombre" de JavaScript cuando usa open() Método (tenga en cuenta que todos excepto '_self' pueden provocar el bloqueo del navegador o preguntar al usuario si desean permitirlo) y el argumento "params" es el mismo que el parámetro "especificaciones" de JavaScript para controlar las propiedades del nuevo La ventana / pestaña debe aparecer (no todos los navegadores pueden admitir todas las funciones). Si está utilizando el icono para ejecutar un script interno, estos dos últimos argumentos se pueden dejar como cadenas vacías "".

Los objetivos válidos son:

Objetivo Descripción
_blanco Abre el documento vinculado en una nueva ventana o pestaña.
_yo Abre el documento vinculado en el mismo marco en el que se hizo clic (este es el valor predeterminado).
_padre Abre el documento vinculado en el marco padre.
_parte superior Abre el documento vinculado en el cuerpo completo de la ventana.

Los parámetros válidos son:

Parámetros Descripción
'altura = [ px ]' La altura de la ventana, con el valor mínimo de 100.
'ancho = [ px ]' El ancho de la ventana, con el valor mínimo de 100.
'izquierda = [ px ]' La posición izquierda de la ventana.
'top = [ px ]' La posición superior de la ventana (solo IE).
'location = [ boolean ]' Si se muestra o no el campo de dirección (el valor predeterminado es 1).
'barra de menú = [ booleano ]' Si se muestra o no la barra de menús (el valor predeterminado es 1).
'resizable = [ booleano ]' Si la ventana es o no redimensionable (el valor predeterminado es 1).
'scrollbars = [ boolean ]' Si se muestran o no las barras de desplazamiento (el valor predeterminado es 1).
'status = [ booleano ]' Si agregar o no una barra de estado (el valor predeterminado es 1).
'titlebar = [ booleano ]' Si se muestra o no la barra de título. Esto se ignora a menos que la aplicación de llamada sea una aplicación HTML o un cuadro de diálogo confiable (el valor predeterminado es 1);
'toolbar = [ boolean ]' Si se muestra o no la barra de herramientas del navegador (el valor predeterminado es sí).

También vale la pena señalar que si el argumento de la URL que pasa es el nombre de un script (es decir, no comienza con "http: //") Y el script se llama gmcallback_* entonces puede obtener feeds directos en un evento html5. GameMaker Studio 2 nombres de funciones de GameMaker Studio 2 no están ofuscados si comienzan con gmcallback_.


Ejemplo:

home_but = clickable_add(32, 32, sprite_get_tpe(spr_MS_Home, 0), "http://macsweeney_games.com", "_blank", "width=700, height=500, menubar=0, toolbar=0, scrollbars=0");

El código anterior crea un icono DOM en el que se puede hacer clic en la posición (32, 32) de la página en la que se está ejecutando el lienzo del juego. El icono utiliza el elemento sprite al que se hace referencia en la página de textura como "spr_MS_Home" y cuando se hace clic en él, se abrirá una nueva ventana para la URL especificada y con las propiedades definidas para esa ventana.
