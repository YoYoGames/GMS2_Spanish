clickable_add_ext

Sintaxis:

clickable_add_ext(x, y, tpe, url, target, params, scale, alpha);


Argumento Descripción
x La posición x dentro de la ventana.
y La posición y dentro de la ventana.
tpe La entrada de la página de textura para el sprite que se utilizará (uso sprite_get_tpe )
url La URL (dirección del sitio web) a la que se debe vincular o la secuencia de comandos que se ejecutará.
target Esta es el área objetivo para abrir la URL en (ver descripción).
params Los diversos parámetros para el icono (ver descripción).
scale La escala del ícono (por defecto 1).
alpha La imagen alfa del icono (por defecto 1).


Devoluciones: Real


Descripción

Con esta función, puede crear un icono personalizado, que se puede hacer clic, que actúa como un "botón" fuera del lienzo del juego, pero dentro de la ventana del navegador. La función devolverá un valor de identificación que debe almacenarse en una variable para ser utilizado en todas las funciones de botón adicionales que hacen referencia al nuevo icono. Esta función es muy útil para evitar que el navegador cree una ventana emergente al hacer clic en el botón, ya que todo se hace a través de GameMaker: Studio en sí.

Esta función funciona exactamente igual que clickable_add función, donde debe establecer la posición relativa al (0,0) de la ventana del navegador, ingrese la entrada de la página de textura para el sprite que desea que tenga el icono del botón (ver: sprite_get_tpe ), especifique una URL para abrir o una secuencia de comandos (desde el árbol de recursos de GameMaker: Studio ) para abrir o ejecutar, luego configure el objetivo y los parámetros a usar. Sin embargo, esta función también tiene dos opciones adicionales que le permiten establecer el ícono alfa entre 0 (totalmente transparente) y 1 (totalmente opaco) y la escala del ícono, que es un valor multiplicador aplicado al sprite utilizado. Entonces, un valor de 1 es la escala predeterminada, mientras que (por ejemplo) 0.5 sería la mitad de la escala y 2 sería el doble.

También vale la pena señalar que si el argumento de URL que ingresa es el nombre de una secuencia de comandos (es decir, no comienza con "http: //") Y se llama la secuencia de comandos gmcallback_* luego puede obtener feeds directos en un evento html5. GameMaker: los nombres de las funciones de Studio no se ofuscan si comienzan con gmcallback_.


Ejemplo:

home_but = clickable_add_ext(32, 32, sprite_get_tpe(spr_MS_Home, 0), "http://macsweeney_games.com", "_blank", "width=700, height=500, menubar=0, toolbar=0, scrollbars=0", 2, 1);

El código anterior crea un icono de DOM seleccionable en la posición (32, 32) de la página en la que se ejecuta el lienzo del juego. El ícono utiliza el sprite al que se hace referencia desde la página de texturas como "spr_MS_Home" y cuando se presiona el ícono abrirá una nueva ventana para la URL especificada y con las propiedades definidas para esa ventana. el botón también se escalará al doble del tamaño del sprite original y a la mitad de una imagen alfa de 1.
