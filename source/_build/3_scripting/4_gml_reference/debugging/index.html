Depuración


Mientras crea un juego, a menudo es muy importante que conozca ciertos detalles de la mecánica del juego en momentos específicos, o que pueda ingresar ciertos valores clave para algunas funciones para probar diferentes métodos e ideas para su juego. A tal fin, GameMaker Studio 2 proporciona una serie de funciones de depuración que mostrarán mensajes y solicitarán entrada mientras se encuentra en modo de depuración (o en un destino de Windows), pero que se ignorarán cuando se compilen para otros módulos de destino debido a que estos Las funciones ponen el juego en un bucle cerrado hasta que se resuelve la entrada.

NOTA: Aunque pueden funcionar en ciertas plataformas, no deben utilizarse para nada más que para la depuración.
  • modo de depuración
  • evento de depuración
  • debug_get_callstack
  • get_integer
  • get_string
  • show_error
  • Mostrar mensaje
  • show_question
  • show_debug_message
  • show_debug_overlay
  • code_is_compiled
  • fps
  • fps_real

Además de esas funciones especializadas, para ayudarlo a crear su juego, GameMaker Studio 2 también tiene una larga lista de mensajes de error que se pueden mostrar al compilar su juego o cuando el juego se está ejecutando. Estos mensajes apuntan al error específico en cuestión y se pueden usar para identificar un problema dentro de un objeto o script específico, lo que ayuda enormemente con el proceso de depuración del juego. Sin embargo, puede que no siempre sea obvio qué significa exactamente el mensaje de error, por lo que la siguiente sección enumera todos los mensajes posibles y explica lo que significan para hacerte la vida más fácil:

  1. Errores de compilación y sintaxis
