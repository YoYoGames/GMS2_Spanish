keyboard_set_map


Descripción

A veces, al hacer un juego, es posible que desee que una tecla haga lo mismo que otra. Por ejemplo, muchas personas usan las teclas WASD para el movimiento, pero muchas personas también usan las teclas de flecha. ¿Entonces lo que hay que hacer? Bueno, podría codificar el sistema de movimiento dos veces, pero eso es un poco complicado y afortunadamente redundante, ya que esta función le permite "asignar" una tecla a otra y, por lo tanto, cualquier entrada de cualquiera de las teclas se interpretará como la misma. Para ello elige una clave que desea asignar (clave2 - esta será la clave que se escribe el código para) y una clave que desea que se asigna a (key1). Después de eso, las pulsaciones de teclas en cualquiera de las teclas serán interpretadas por GameMaker Studio 2 como provenientes de key2. También puede usar esta función para diseñar un sistema en el que el usuario pueda definir sus propias teclas para jugar simplemente asignando la tecla de entrada del usuario a la tecla que usted ha codificado en el juego.


Sintaxis:

keyboard_set_map(key1, key2);

Argumento Descripción
key1 Esta es la clave a la que se asignará key1
key2 Esta es la clave que debe ser mapeada


Devoluciones:

Boolean


Ejemplo:

keyboard_set_map(ord("A"), vk_left);

El código de ejemplo anterior asignará la tecla "A" a la tecla de flecha izquierda. Esto significa que el jugador puede usar la tecla de flecha "A" o izquierda, y que todo el código escrito para la flecha izquierda también responderá a la tecla "A" que se está utilizando.
