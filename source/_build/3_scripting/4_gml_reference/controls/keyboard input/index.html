Entrada de teclado


Al tratar con el teclado en GameMaker Studio 2, tiene una variedad de funciones que se pueden utilizar para reconocer diferentes estados del teclado, como presionar o soltar. También hay algunos que almacenan todas las pulsaciones de teclas como una cadena o que pueden indicarle cuál fue la última tecla presionada, así como otras que le permiten borrar completamente el estado del teclado.

NOTA: Estas funciones están diseñadas solo para plataformas de escritorio Windows / Mac / Ubuntu. Es posible que algunas de las variables y constantes integradas no sean válidas en otras plataformas y que muchas de las funciones no funcionen en los dispositivos móviles.

Ahora, cada tecla normalmente está definida por un número, llamado código ascii, y puede ingresar directamente este número en estas funciones y funcionarán bien... Pero, como es un poco difícil recordar tantos números y la relación que tienen con su teclado, GameMaker Studio 2 tiene una serie de constantes para las teclas especiales del teclado más usadas y una función especial ord() para devolver el número de caracteres tipificados normales (letras o números).

El siguiente es un pequeño ejemplo de cómo usar ord():

if keyboard_check(ord("A"))
   {
   hspeed = -5;
   }

Por lo tanto, lo anterior marcará la tecla "A" y, si se está presionando, establecerá la velocidad horizontal del objeto en -5. Tenga en cuenta que la "A" es una "A" mayúscula y que cuando se usa ord() la tecla del teclado para verificar siempre debe escribirse entre comillas y como mayúscula. Ahora, ¿qué pasa si quieres usar las teclas de flecha? ¿O si quiere modificar una acción usando la tecla "shift"? Bueno, para eso GameMaker Studio 2 tiene una serie de vk_ constantes ( vk_ significa teclado virtual ) que puede usar en lugar de ord o del código ascii.

Aquí hay una lista completa de vk_ constantes:

Constante Descripción
vk_nokey código clave que representa que no se presiona ninguna tecla
vk_anykey código clave que representa que se presiona cualquier tecla
vk_left código clave para la tecla de flecha izquierda
vk_right código clave para la tecla de flecha derecha
vk_up código clave para la tecla de flecha hacia arriba
vk_down código clave para la tecla de flecha hacia abajo
vk_enter introducir clave
vk_escape tecla de escape
vk_space Llave del espacio
vk_shift cualquiera de las teclas de cambio
vk_control cualquiera de las teclas de control
vk_alt tecla Alt
vk_backspace tecla de retroceso
vk_tab tecla de tabulación
vk_home llave de la casa
vk_end tecla de finalización
vk_delete borrar clave
vk_insert insertar clave
vk_pageup tecla de búsqueda
vk_pagedown tecla de búsqueda hacia abajo
vk_pause pause / break key
vk_printscreen tecla printscreen / sysrq
vk_f1... vk_f12 código clave para las teclas de función F1 a F12
vk_numpad0... vk_numpad9 teclas numéricas en el teclado numérico
vk_multiply tecla multiplicar en el teclado numérico
vk_divide dividir clave en el teclado numérico
vk_add tecla en el teclado numérico
vk_subtract restar la tecla en el teclado numérico
vk_decimal teclas de puntos decimales en el teclado numérico

Las siguientes constantes solo se pueden usar con keyboard_check_direct():

Constante Descripción
vk_lshift tecla Mayús izquierda
vk_lcontrol tecla de control izquierda
vk_lalt tecla alt izquierda
vk_rshift tecla Mayús derecha
vk_rcontrol tecla de control derecha
vk_ralt tecla alt derecha



El siguiente es un pequeño ejemplo de cómo usar el vk_ constantes:

if keyboard_check_pressed(vk_tab)
   {
   instance_create_layer(x, y, "Controllers", obj_Menu);
   }

El código anterior detectará si se presiona la tecla "Tab" y crea una instancia del objeto "obj_Menu" si es así.
