Entrada de teclado


Cuando se trata del teclado en GameMaker Studio 2, tiene una variedad de funciones que se pueden usar para reconocer diferentes estados del teclado, como presionar o soltar. También hay algunos que almacenan todas las pulsaciones de teclas como una cadena o que pueden decirle cuál fue la última tecla presionada, así como otras que le permiten borrar completamente el estado del teclado.

NOTA: Estas funciones están diseñadas solo para plataformas de escritorio Windows / Mac / Ubuntu. Es posible que algunas de las variables y constantes integradas no sean válidas en otras plataformas y muchas de las funciones no funcionen en los móviles.


Cada carácter de entrada de una clave (o varias claves) se define por su código UTF8, que es un valor numérico. Este valor puede ser recuperado para cualquier personaje usando el ord() función, pero GameMaker Studio 2 también tiene una serie de constants Para las teclas especiales más usadas del teclado y funciones especiales. Normalmente utilizarías una combinación de ord() con el keyboard_check*() Funciones, algo como esto:

if keyboard_check(ord("A"))
    {
    hspeed = -5;
    }


Por lo tanto, lo anterior verificará la tecla "A" y, si se presiona, establecerá la velocidad horizontal del objeto en -5. Tenga en cuenta, que utilizando ord() de esta manera solo funcionará correctamente si la cadena de entrada tiene una longitud de solo un carácter y es un número del 0 al 9 o un carácter romano en mayúscula de la A a la Z. La función ord() devolverá un valor UTF8 completo, pero el keyboard_check*() Las funciones solo detectarán A - Z y 0 - 9. ¿Pero qué pasa si quieres usar las teclas de flecha? ¿O si quieres modificar una acción usando la tecla "shift"? Bueno, para eso GameMaker Studio 2 tiene una serie de vk_ constantes vk_ significa teclado virtual ) que puede usar en lugar de ord():

Constante Descripción
vk_nokey código clave que representa que no se presiona ninguna tecla
vk_anykey código clave que representa que se presiona cualquier tecla
vk_left código clave para la tecla de flecha izquierda
vk_right código clave para la tecla de flecha derecha
vk_up código de tecla para la tecla de flecha hacia arriba
vk_down código clave para la tecla de flecha hacia abajo
vk_enter introducir clave
vk_escape tecla de escape
vk_space Llave del espacio
vk_shift cualquiera de las teclas de cambio
vk_control cualquiera de las teclas de control
vk_alt tecla Alt
vk_backspace tecla de retroceso
vk_tab tecla de tabulación
vk_home llave de la casa
vk_end tecla de finalización
vk_delete eliminar clave
vk_insert insertar llave
vk_pageup clave de página
vk_pagedown clave de paginación
vk_pause tecla pausa / pausa
vk_printscreen Imprime la tecla / sysrq
vk_f1... vk_f12 Keycode para las teclas de función F1 a F12.
vk_numpad0... vk_numpad9 teclas numéricas en el teclado numérico
vk_multiply tecla multiplicar en el teclado numérico
vk_divide tecla de división en el teclado numérico
vk_add añadir tecla en el teclado numérico

Las siguientes constantes solo pueden ser usadas con keyboard_check_direct():

Constante Descripción
vk_lshift tecla de cambio izquierda
vk_lcontrol tecla de control izquierda
vk_lalt tecla alt izquierda
vk_rshift tecla de cambio a la derecha
vk_rcontrol tecla de control derecha
vk_ralt tecla derecha derecha
El siguiente es un pequeño ejemplo de cómo usar el vk_ constantes

if keyboard_check_pressed(vk_tab)
    {
    instance_create_layer(x, y, "Controllers", obj_Menu);
    }


El código anterior detectará si se presiona la tecla "Tabulador" y creará una instancia del objeto "obj_Menu" si lo está.

Si necesita buscar un carácter clave que no sea 0 - 9, A - Z o una de las constantes VK, entonces debería estar marcando uno de los keyboard_* variables, como keyboard_lastchar por ejemplo:
