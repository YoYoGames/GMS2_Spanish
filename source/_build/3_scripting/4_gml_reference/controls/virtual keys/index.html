Teclas virtuales y teclados virtuales


Al desarrollar juegos para dispositivos de pantalla táctil, a menudo es importante obtener información del teclado del usuario. Cuando se utiliza GameMaker Studio 2 esto se puede hacer de una de las siguientes dos maneras:

  • Crea tus propias teclas virtuales en pantalla para obtener entrada
  • Abre el teclado virtual del sistema operativo y recibe información de ese

Lo que use dependerá en gran medida del juego y lo que necesite, pero en general, usaría teclas virtuales cuando quiera tener botones en pantalla para el control del jugador o para los menús, mientras que usaría el teclado virtual cuando necesite la entrada del jugador., como obtener un nombre o establecer un valor. Las secciones a continuación describen los usos y funciones asociadas con cada una de estas opciones de entrada.

Para que sus juegos funcionen correctamente en dispositivos móviles con una pantalla táctil, es posible que deba asignar áreas de la pantalla para responder a ciertas cosas cuando son tocadas por el usuario. Esto se hace asignando una "clave virtual" a una tecla de teclado estándar asignada internamente y luego utilizando los eventos de teclado que generan para controlar su aplicación. Vale la pena señalar que estas "teclas virtuales" se asignan a la posición de la pantalla en lugar de la posición de la sala, por lo que los valores de x / y son absolutos, según el tamaño de la pantalla. Esto significa que no necesita preocuparse por el uso de las vistas o las coordenadas relativas de la habitación, y simplemente puede dibujar sus sprites de clave y definir sus claves virtuales en el Evento de GUI de un objeto de Draw.

NOTA: estas funciones no funcionarán en Windows, Ubuntu (Linux) ni en MacOS, plataformas de destino que utilizan una pantalla táctil, aunque recibirán clics del mouse como "toques".

Las funciones clave virtuales disponibles son:

  • virtual_key_add
  • virtual_key_show
  • virtual_key_hide
  • virtual_key_delete

Si necesita entrada de texto o numérica en su juego en un dispositivo con pantalla táctil, puede usar las funciones del teclado virtual proporcionadas para usar el teclado del SO del dispositivo. Estas funciones activarán u ocultarán el teclado del sistema operativo y aceptarán sus entradas, sin embargo, no activarán los eventos regulares del teclado, sino que actualizarán keyboard_string variable con la entrada del usuario (esto se debe al hecho de que el teclado virtual puede ingresar palabras completas y no solo caracteres individuales).

Abrir o cerrar el teclado virtual generará eventos asíncronos del sistema, en los que el async_load El mapa DS se llenará con los siguientes pares clave / valor:

  • " event_type ": el tipo de evento del sistema recibido, que será el " estado del teclado virtual " para los teclados virtuales.

  • " screen_height ": la altura del teclado virtual (en píxeles). Esto será 0 si el teclado es invisible.

  • " keyboard_status ": el estado actual del teclado, devuelto como una de las siguientes cadenas:

    • "ocultación"
    • "oculto"
    • "demostración"
    • "visible"

Las funciones de teclado virtual disponibles son:

  1. keyboard_virtual_show
  2. keyboard_virtual_hide
  3. keyboard_virtual_status
  4. keyboard_virtual_height
NOTA: estas funciones solo son válidas para las plataformas de destino de Android (incluidas Amazon Fire, FireTV y Android TV), iOS y UWP.
