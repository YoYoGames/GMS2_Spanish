gamepad_test_mapping


Descripción

Esta función se puede usar para establecer la asignación de gamepad en aquellos objetivos que lo permitan. Usted suministra el índice de "ranura" para que se establezca el gamepad, y luego la cadena del mapa, que debería haberse creado utilizando el formato SDL con los siguientes campos:

<guid>,<description>,platform:<platform-name>,<bindings>

Los detalles que deben incluirse en cada uno de estos campos son:

  • guid - una cadena de dígitos y letras que identifica de forma única un tipo de dispositivo (esto se puede recuperar utilizando el gamepad_get_guid() función)

  • description - una descripción de Ascii del dispositivo (esto se puede recuperar usando el gamepad_get_description() función)

  • platform-name - será "platform: android", "platform: windows", "linux" o "mac" (tenga en cuenta que esto se agregará automáticamente a la cadena cuando llame a esta función, por lo que no necesita suministrarlo) al construir sus propias cadenas de mapeo)

  • bindings - un conjunto de entradas separadas por ',' para vincular la entrada real a constantes GML específicas

La lista de entradas separadas por comas sigue un formato específico para cada entrada:

<abstract-gp-name>:<value-binding>

aquí <value-binding> (es decir, la entrada directa del gamepad) puede ser cualquiera de los siguientes:

  • a<N> - Esto representa un eje del gamepad, donde <N> es uno de los ejes de 0 a gamepad_axis_count() -1
  • b<N> - Esto representa un botón del gamepad, donde <N> es uno de los botones de 0 a gamepad_button_count() -1
  • h<N>.<mask> - Esto representa un sombrero del gamepad, donde <N> representa un sombrero de 0 a gamepad_hat_count() -1, y el <mask> será 1, 2, 4 u 8.

Cada valor de entrada debe estar vinculado a un nombre SDL, que se expresa como <>abstract-gp-name>, que a su vez está vinculado a una constante GML. La siguiente tabla muestra la equivalencia entre el nombre de SDL y la constante de GameMaker Studio 2:

b gp_face2
x gp_face3
y gp_face4
leftshoulder gp_shoulderl
lefttrigger gp_shoulderlb
rightshoulder gp_shoulderr
righttrigger gp_shoulderrb
guide gp_select
start gp_start
leftstick gp_stickl
rightstick gp_stickr
dpup gp_padu
dpdown gp_padl
dpleft gp_padr
dpright gp_padd
leftx gp_axislh
lefty gp_axislv

Cuando haya construido su cadena de mapeo debería verse algo como esto:

"050000005e040000fd020000ffff3f00, Xbox Wireless Controller, a: b0, b: b1, inicio: b4, dpdown: h0.4, dpleft: h0.8, dpright: h0.2, dpup: h0.1, guía: b6, leftshoulder: b9, leftstick: b7, lefttrigger: a4, leftx: a0, zurdo: a1, rightshoulder: b10, rightstick: b8, righttrigger: a5, rightx: a2, righty: a3, x: b2, y: b3, platform: android "


Sintaxis:

gamepad_test_mapping(index, mapping_string);

Argumento Descripción
index Que gamepad indexa "slot" para configurar.
colour La cadena del mapa a usar (ver la descripción para más información).

Ejemplo:

var mapping = gamepad_get_guid(global.padIndex) + "," + gamepad_get_description(global.padIndex);
var len = array_length_1d(global.PadInstances);
for( i = 0; i < len; i += 2)
    {
    var left = global.PadInstances[i];
    var right = global.PadInstances[i+1];
    mapping += "," + left.sdlLabel + ":" + right.binding;
    }
gamepad_test_mapping(global.padIndex, mapping);

El código anterior pasará por una serie de instancias y utilizará los valores de las diferentes variables que contienen para crear una cadena de mapeo, que luego se establece para su uso en el gamepad en el índice de intervalo dado.
