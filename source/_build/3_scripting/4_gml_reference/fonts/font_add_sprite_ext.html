font_add_sprite_ext


Descripción

Con esta función puedes usar una "tira de sprites" (el sprite debe ser un recurso de sprite del árbol de recursos, o un sprite que hayas agregado a los recursos del juego usando sprite_add ) para crear un nuevo recurso de fuente, donde cada sub-imagen sería un símbolo o letra individual. A diferencia de lo normal font_add_sprite que tiene un orden específico para las subimágenes del sprite, esta función mapeará las subimágenes de sprites basadas en el argumento "string_map" de la función. Este argumento es una cadena que puede usar para indicar a GameMaker Studio 2 orden en que se encuentran las subimágenes de la fuente de sprite y las asignará automáticamente al escribir el texto. Entonces, por ejemplo, si tiene un mapa de cadena de "AaBbCcDdEeFfGgHh", su fuente de sprite debe tener las subimágenes ordenadas de esta manera. Puede definir "espacio" como cualquier carácter que desee, por ejemplo, una sola línea del tamaño que desea que sea el espacio, y todos los espacios del texto se mostrarán en ese ancho (pero la imagen elegida nunca se representará), o si no proporciona un sprite para el espacio, entonces se usará el ancho del carácter más ancho en la fuente del sprite.

También puede establecer el espaciado para que la fuente sea proporcional ( true ) o no ( false ), donde una fuente proporcional está espaciada en función del ancho real de las letras (por lo que la letra "i" ocupa menos espacio que la letra "w", por ejemplo), mientras que una fuente no proporcional se espaciará según el subtítulo ancho de imagen, como una fuente monoespaciado. Finalmente, puede definir el espacio que debe dejarse entre cada letra al escribir, y este puede ser cualquier valor entero, donde 0 es sin espacio (las letras "tocarán" si son proporcionales). La función devuelve un valor de índice que debe almacenarse en una variable, ya que será necesario en todos los demás códigos que hagan referencia a esta fuente.

Vale la pena señalar que para que funcionen las funciones de alineación de fuentes (como draw_set_halign() ) entonces el sprite de la fuente debería tener su origen en la esquina superior izquierda. Si usa otros valores, tendrá que tener en cuenta la compensación de origen al dibujar texto con la fuente.

NOTA: Cuando cargue una fuente en GameMaker Studio 2, debe recordar eliminarla nuevamente (con font_delete ) cuando ya no sea necesario, de lo contrario existe el riesgo de una pérdida de memoria que disminuirá la velocidad y, finalmente, se bloqueará el juego.
¡IMPORTANTE! Esta función no está disponible con la licencia de prueba del producto.


Sintaxis:

font_add_sprite_ext(spr, string_map, prop, sep);


Argumento Descripción
spr El sprite para agregar una fuente basada en
string_map Cadena desde la que se toma el orden de subimagen de sprite.
prop Establecer como fuente proporcional o no.
sep El espacio para irse entre cada letra.


Devoluciones:

Real


Ejemplo:

global.Font = font_add_sprite_ext(spr_CalcFont, "0123456789+-*/=", true, 2);

El código anterior creará un nuevo activo de fuente del sprite indexado en la variable "spr_CalcFont" y almacenará el índice de la nueva fuente en la variable "global.Font". Las subimágenes de la fuente de sprite se asignarán a la cadena especificada.
