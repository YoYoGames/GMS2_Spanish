iap_acquire

Esto solicita la adquisición del producto dado.

Sintaxis:

iap_acquire(purchase_id, payload);


Argumento Descripción
purchase_id La cadena de identificación del producto para la compra.
payload Esta es una cadena que se puede usar para la verificación de compras en Google Play (use una cadena vacía, "", en todas las demás plataformas).


Devoluciones: Real


Descripción

Esta función es la que utiliza para adquirir una compra a través de la tienda objetivo para el dispositivo que ejecuta su juego. Esta cadena de identificación de producto es la que usted especifica al activar el IAP usando iap_activate y debe ser el mismo que usaste para identificar el producto en la tienda objetivo. Esta función devolverá un valor ya que es un "índice" para que pueda identificarse cuando se reciban los datos de compra y llamar a la función activará un evento IAP del "tipo". iap_ev_purchase, que contendrá la siguiente clave adicional:

  • " índice ": la identificación de compra del producto activado.

Este ID de compra se puede usar para verificar qué compra ha desencadenado el evento (comparándola con el valor del índice devuelto que almacenó), para que pueda manejarlo en consecuencia. Si está desarrollando para Google Play (en el destino de Android ), también puede configurar la cadena de "Carga útil". Esto es utilizado por Google para verificar el producto y se envía junto con la solicitud de compra a Google Play. Cuando se completa con éxito una compra, el juego recibirá la carga utilizada como parte de la información de compra. Al generar una carga útil con cada compra, puede proporcionar una forma simple de verificación de compra.


Ejemplo:

if mouse_check_button_pressed(mb_left)
   {
   iap_acquire("noads", "")
   }

El código anterior enviará una solicitud de compra cuando se presiona el botón del mouse.
