iap_purchase_details

Agregue detalles de compra de un solo producto a un ds_map.

Sintaxis:

iap_purchase_details(product_id, ds_map);


Argumento Descripción
product_id La cadena de identificación del producto para la compra.
ds_map El ds_map que almacenará la información de compra.


Devoluciones: N / A


Descripción

Con esta función puedes poblar una previamente creada ds_map con una serie de pares clave / valor que describen los detalles de compra del producto dado. Los siguientes datos se devuelven dentro del mapa:

  • " producto " - La cadena de identificación del producto.

  • " orden ": el ID único de la orden, donde esté disponible (no todas las tiendas admiten esto).

  • " token ": la cadena de token de compra, donde esté disponible (no todas las tiendas admiten esto).

  • " carga útil ": la carga útil devuelta por la tienda objetivo, donde esté disponible (no todas las compras lo requieren, y no todas las tiendas lo admiten).

  • " recibo ": la cadena de recibos, cuando esté disponible (no todas las tiendas admiten esto).

  • " respuesta ": un valor de respuesta (como un número real), donde esté disponible, en función de los valores de respuesta de Google Play (consulte aquí para obtener más información).

El mapa también contendrá una clave adicional, " estado ", que se puede usar para verificar el estado actual de la compra. Esta clave tendrá una de las siguientes constantes según su valor:

Constante Descripción
iap_available El producto está disponible o el juego está esperando el resultado de una compra.
iap_failed El intento de compra falló.
iap_purchased El producto fue comprado con éxito.
iap_canceled El usuario canceló la compra.
iap_refunded La compra ha sido anulada.



Las siguientes notas están relacionadas con las teclas "token" y "receipt" del mapa devuelto en referencia a la tienda de destino que se utiliza:

  1. Para Amazon Store, las entradas de "token" y "recibo" de compra se tratan como una sola.

  2. Para Google Play no hay datos de "recibo" y se espera que verifique las compras a través de la "carga útil" devuelta (si está verificando mediante una carga útil, se recomienda que no utilice valores codificados). GameMaker Studio 2 aún verificará la firma de datos devuelta con una compra de Google Play contra la clave pública proporcionada en la pestaña IAP de las Opciones del juego.

  3. Para Google Play, el "token" provisto con una compra está destinado a consumir compras.

  4. Para las tiendas de destino iOS, MacOS, Windows 8 (N y JS) y Windows Phone, hay datos de "recibo" disponibles para las compras, pero el "token" asociado se genera internamente para fines de desarrollo.



NOTA: el ds_map used no es creado por la función, lo que significa que debe crearlo previamente y eliminarlo nuevamente de la memoria cuando no esté en uso con la función apropiada.


Ejemplo:

var p_map = ds_map_create();
var p_index = ds_map_find_value(iap_data, "index");
iap_purchase_details(p_index, p_map);
if ds_map_find_value(p_map, "status") == iap_purchased
   {
   global.Gold += 100000;
   }
ds_map_destroy(p_map);

El código anterior creará un ds_map y luego lo rellenará con los pares de clave / valor para la compra con el ID de producto almacenado en la variable "p_index", que obtenemos del especial iap_data ds_map que se crea exclusivamente en el evento IAP. Si la tecla "estado" devuelve la constante iap_purchased, luego establecerá una variable global.
