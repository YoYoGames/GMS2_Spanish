iap_activate

Habilita la compatibilidad para realizar compras en la aplicación.

Sintaxis:

iap_activate(product_index);


Argumento Descripción
product_index El índice de ds_list que contiene las compras disponibles.


Devoluciones: N / A


Descripción

Esta función permite la compatibilidad para realizar compras en la aplicación y prepara GameMaker Studio 2 mediante el suministro del producto ds_list eso tiene una serie de ds_maps con la información de cada compra disponible. Esto significa que tendrá que indicarle a GameMaker Studio 2 todas las opciones de compra disponibles configurándolas en forma individual. ds_maps (uno para cada compra disponible), que luego se almacenan dentro de un ds_list, que luego se "activa" con esta función.

Los mapas de compra individuales que deben almacenarse en el ds_list debe tener el siguiente formato de pares clave-valor:

  • " id ": la identificación del producto para la compra, por ejemplo: "LevelPack".

  • " título " - El nombre de la compra, por ejemplo: "Nivel Paquete 1".

  • " descripción " - La descripción de la compra, por ejemplo: "Level Pack 1 for Catch The Clown".

  • " precio ": el precio de la compra, por ejemplo: " $1.00".

  • " tipo " - ¡ Windows 8 y Windows Phone solamente! El tipo de compra, que debe ser "Consumible" o "Durable". "Durable" es el valor predeterminado (no consumible).

Vale la pena señalar que las únicas claves esenciales para cualquier configuración de tienda objetivo (por ID de producto) es la "id", excepto para Windows 8 destinos de Windows 8 y Windows Phone, en cuyo caso también necesita la clave "tipo". Tenga en cuenta que para estas plataformas de destino, si desea que la tienda informe datos significativos (correctos) cuando se encuentre en modo Sandbox, entonces el "título", la "descripción", etc. son necesarios.

La activación de compras también activará un evento IAP, que crea un especial iap_data ds_map del tipo de evento iap_ev_product. Este ds_map tendrá la siguiente clave adicional:

  • " índice ": la identificación del producto activado.

Si está activando múltiples productos, cada producto activará su propio evento IAP del tipo iap_ev_product donde puede obtener la identificación del producto. Vale la pena señalar que la tienda Google Play (para Android ) solo puede procesar los detalles de los productos 20 a la vez, lo que puede generar tiempos de carga bastante largos para las aplicaciones con un número significativo de productos.



NOTA: ¡ Todos los pares clave / valor que componen un mapa de compra son cadenas!


Ejemplo:

var purchaseList, purchase1;
purchaseList = ds_list_create();
purchase1 = ds_map_create();
ds_map_add(purchase1, "id", "LevelPack");
ds_map_add(purchase1, "title", "ExtraLevels1");
ds_map_add(purchase1, "description", "Level Pack 1 for Catch The Clown");
ds_map_add(purchase1, "price", "$1.00");
ds_list_add(purchaseList, purchase1);
iap_activate(purchaseList);
ds_map_destroy(purchase1);
ds_list_destroy(purchaseList);

El código anterior creará un ds_list y un ds_map, que luego se completa con la información para realizar una compra. Este mapa se agrega a la lista y luego se activa como una compra disponible para el juego. Finalmente, el ds_list y ds_map se eliminan para evitar fugas de memoria ya que ya no son necesarios.
