Archivos Ini

Los archivos INI son archivos pequeños y livianos que son compatibles con la mayoría de las plataformas. Son ideales para almacenar pequeñas cantidades de información, como preferencias de interfaz, puntajes locales altos, datos de nivel, etc. y, por lo tanto, son una forma excelente y fácil para que comiences a guardar información de tus juegos. Sin embargo, vale la pena señalar que tienen un límite en los datos que pueden almacenar, que es un máximo de 64 KB, por lo que si está guardando cosas como estructuras de datos, es posible que desee ver algunas de las otras funciones del archivo como el texto. Tenga en cuenta que para los juegos con localizaciones es muy importante que las letras acentuadas se puedan leer desde archivos externos. Esto significa que primero debe crear el archivo ini en formato UTF8 y luego agregarlo a GameMaker Studio 2 como un archivo incluido para que se exporte al ejecutar el juego y se use en lugar del archivo en formato ANSI predeterminado creado por el ini. funciones cuando ninguna ha sido suministrada previamente. De esta forma, puede leer y escribir correctamente con todos los acentos y manteniendo las letras no romanas.

Las siguientes funciones existen para archivos ini:

  • ini_open
  • ini_close
  • ini_write_real
  • ini_write_string
  • ini_read_real
  • ini_read_string
  • ini_key_exists
  • ini_section_exists
  • ini_key_delete
  • ini_section_delete
  • ini_open_from_string
¡ADVERTENCIA! ¡Es posible que estas funciones no funcionen como esperaba debido a que GameMaker Studio 2 está utilizando en la zona de pruebas! Consulte la sección sobre Límites del sistema de archivos para obtener más información.
