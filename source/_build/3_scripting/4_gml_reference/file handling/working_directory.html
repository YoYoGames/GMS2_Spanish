directorio de trabajo

Sintaxis:

working_directory


Devuelve: Cadena.


Descripción

Working_directory en realidad puede devolver dos valores diferentes en función de lo que lo está utilizando. Si está escribiendo un archivo en el disco, working_directory señala el área de almacenamiento local que está reservada para su juego en el dispositivo de destino (en Windows esto es "% LOCALAPPDATA% / gamedir /" donde el "gamedir" es el directorio con el nombre de su juego). Sin embargo, al leer desde el disco, working_directory pueden ser o bien el almacenamiento local o el área donde se almacenan los archivos incluidos.

NOTA: el working_directory la variable devolverá la ruta incluyendo la última barra invertida.

Por ejemplo, si tiene un valor predeterminado *.txt archivo incluido con su juego y leerlo en la memoria usando working_directory, entonces ese archivo se tomará del área donde se almacenan los archivos incluidos. Si luego elige escribir esa información en un archivo de texto usando working_directory, esto escribirá el archivo en el almacenamiento local de las plataformas de destino, donde se leerá la próxima vez, a menos que le indique a GameMaker Studio 2 que elimine el archivo, en cuyo caso working_directory indicará una vez más el área donde se almacenan los archivos incluidos.


¡ADVERTENCIA! ¡Es posible que esta función no funcione como espera debido a que GameMaker Studio 2 está siendo protegido! Consulte la sección sobre Límites del sistema de archivos para obtener más información.


Ejemplo:

ini_open(working_directory + "temp_ini.ini");

Esto abrirá un archivo ini del directorio de trabajo del juego (crearlo si aún no existe). Este podría ser el almacenamiento local, o el área donde los archivos incluidos dependen de si el archivo que se busca existe en cualquier lugar.
