get_open_filename_ext

Sintaxis:

get_open_filename_ext(filter, fname, directory, caption);


Argumento Descripción
filter El filtro de tipo de archivo y / o nombre.
fname El nombre de archivo sugerido para usar.
directory El directorio para iniciar la búsqueda desde.
caption El título de la ventana.


Devoluciones: String.


Descripción

Esta función abre un diálogo y le pide al jugador que abra un nombre de archivo con el filtro dado. El filtro tiene la forma " nombre1 | máscara1 | nombre2 | máscara2 |... ", donde la máscara puede contener las diferentes opciones con un punto y coma entre ellas y también puede usar un "*" para indicar cualquier cadena. Por ejemplo: " mapas de bits | *.bmp; *. Wmf ", le pediría al usuario que seleccione el nombre de un archivo de mapa de bits del sistema, pero solo aquellos que tengan las dos extensiones especificadas ( bmp y png ). Esta función no abre el archivo en sí! Solo devuelve una cadena con el nombre completo y la ruta del archivo. Si el usuario presiona " Cancelar ", se devuelve una cadena vacía "". También puede agregar un directorio para iniciar la búsqueda, y un título que se mostrará en la parte superior de la ventana en lugar del predeterminado.

Una cosa importante a tener en cuenta al usar esta función es que le otorga ciertos permisos para ese archivo, durante la duración de su juego. Entonces, una vez que obtenga la ruta del archivo del usuario, puede acceder a ella una y otra vez sin tener que preguntar. Sin embargo, como esta función es para leer un archivo, los permisos otorgados solo para leer y guardar no serán permitidos por el sistema operativo. Si necesita leer y escribir en el archivo, puede obtener permiso al usar la función get_save_filename.

NOTA: Esta función solo es válida en los destinos de Windows y macOS.


Ejemplo:

var file;
file = get_open_filename_ext("text file|*.txt", "", working_directory, "Open a saved level");
if file != ""
   {
   file_text_open_read(file);
   }

El código anterior le pedirá al usuario que seleccione un archivo de texto para abrir desde el working_directory y luego verifique el nombre y la ruta del archivo devuelto, y si existe abrirá el archivo para su lectura.
