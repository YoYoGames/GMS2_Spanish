Manejo de archivos

En algún momento mientras crea su juego, será necesario almacenar información externamente y poder recuperar esta información. Por ejemplo, puede guardar detalles de inicio de sesión, o puntajes altos, o quizás la progresión del jugador a través de sus niveles. Cualquiera que sea el motivo, deberá saber cómo usar las diferentes funciones de archivo que están disponibles para usted cuando trabaje con GameMaker Studio 2.

Es muy importante darse cuenta de que GameMaker Studio 2 es Sandboxed, lo que significa que hay un número de límites y complicaciones menores cuando se utilizan las funciones del sistema de archivos disponibles, y también vale la pena señalar que el sistema de archivos funcionará de forma ligeramente diferente dependiendo de la plataforma a la que te diriges Para obtener más información, consulte la siguiente sección:

  1. Límites del sistema de archivos

Si Steam te ha otorgado permiso para usar su API dedicada con tu juego, verás que también hay un nuevo conjunto de funciones de archivos disponibles para ti. Estos se explican en la sección Steam API de este manual.

¡IMPORTANTE! Todos los proyectos de GameMaker Studio 2 tienen un archivo llamado " opions.ini ", por lo que no puede nombrar ningún archivo con ese nombre (obtendrá un fallo leve cuando intente compilar su proyecto de juego).

Obviamente, habrá momentos en los que necesite hacer referencia al directorio en el que se guardan los archivos y GameMaker Studio 2 también tiene algunas funciones especiales relacionadas con los diferentes directorios posibles que pueda necesitar:

  • directory_exists
  • directory_create
  • directory_destroy
  • temp_directory
  • directorio de trabajo
  • program_directory

Cuando se trata de archivos externos, siempre existe la posibilidad de que el usuario final pueda abrir y cambiar la información que contienen, y así cambiar su juego. Esto puede dar como resultado elementos de juego rotos o puntajes fraudulentos en línea (por ejemplo), por lo que GameMaker Studio 2 proporciona algunas funciones básicas de codificación, así como funciones para realizar comprobaciones de md5 en cadenas y archivos para asegurarse de que tienen mantuvo su integridad antes de ser utilizado. También se proporcionan dos funciones para codificar y decodificar cadenas de formato JSON (JavaScript Object Notation).

  • base64_encode
  • base64_decode
  • json_encode
  • json_decode
  • md5_string_utf8
  • md5_string_unicode
  • md5_file
  • sha1_string_utf8
  • sha1_string_unicode
  • sha1_file
  • zip_unzip
  • load_csv
NOTA: ¡La codificación NO es cifrado! Una codificación base64 hace que el archivo sea ilegible a simple vista y requerirá un esfuerzo por parte del usuario para decodificar, pero no es seguro contra piratería. Se recomienda que combine esas funciones con su propio cifrado (hay muchas formas de cifrado y los scripts están disponibles en Internet).
