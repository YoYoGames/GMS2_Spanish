Steam API

Esta sección es para aquellos usuarios que tienen acceso a la API de Steam para publicar su juego en esa plataforma. Para poder utilizar estas funciones, debe haber sido aceptado previamente en Steam, ya sea a través de un editor o a través del sistema Greenlight, y haber configurado las Preferencias de Steam y las Opciones del Juego de Steam correctamente.

Hay una gran cantidad de funciones diferentes relacionadas con la API de Steam, por lo que las hemos dividido en las siguientes secciones para facilitar la navegación:

El siguiente conjunto de funciones sirve para verificar la disponibilidad de ciertos aspectos del cliente Steam o la API del servidor, y como tal retornan true o false dependiendo de factores como que el servidor esté ocupado o el usuario haya iniciado sesión, etc. Esto significa que estas funciones deben usarse antes de cualquier otra llamada de función para garantizar que la configuración del cliente / servidor sea correcta y se comunique con su juego:

  • steam _inicializado
  • steam _stats_ready
  • steam _is_overlay_enabled
  • steam _get_persona_name
  • steam _get_user_persona_name
  • steam _get_user_ steam _id
  • steam _is_user_logged_on

Las siguientes funciones se pueden utilizar para obtener la ID de la aplicación (juego) y la ID de la cuenta para el usuario actual, que luego se puede usar en otras funciones relacionadas con Steam y el Taller de Steam:

  • steam _get_app_id
  • steam _get_user_account_id
  • steam _current_game_language
  • steam _available_languages

Steam Overlay es la pantalla visual que se puede mostrar para mostrar la interfaz de Steam al usuario. Esto normalmente lo hace el usuario usando una combinación de teclas, pero también tiene la posibilidad de hacerlo desde dentro de su juego, de modo que puede asignar un botón o un evento para mostrar la superposición, usando las siguientes funciones:

  • steam _activate_overlay
  • steam _activate_overlay_browser
  • steam _activate_overlay_store
  • steam _activate_overlay_user
  • steam _is_overlay_activated

Steam API admite tablas de clasificación persistentes con entradas ordenadas automáticamente. Estas tablas de clasificación se pueden usar para mostrar tablas de clasificación globales y de amigos en tu juego y en la página web de la comunidad para tu juego. Cada juego puede tener hasta 10,000 marcadores, y cada tabla de líderes puede recuperarse inmediatamente después de que se haya insertado el puntaje de un jugador, pero tenga en cuenta que para cada tabla de líderes, un jugador puede tener solo una entrada, aunque no hay límite en el número de jugadores por tabla de líderes.

Cada entrada de tabla de clasificación contiene un nombre, una puntuación y un rango para la tabla de clasificación, y estos datos serán reemplazados cuando se cree una nueva entrada de tabla de clasificación para el usuario, y las siguientes funciones se pueden usar para agregar y recuperar esta información de las tablas de clasificación de tu juego:

  • steam _create_leaderboard
  • steam _upload_score
  • steam _upload_score_buffer
  • steam _download_scores
  • steam _download_scores_around_user
  • steam _download_friends_scores

La API Stats and Achievements de Steam proporciona una manera fácil para que tu juego proporcione seguimiento persistente de roaming y estadísticas para tus usuarios. Los datos del usuario están asociados a su cuenta de Steam, y los logros y las estadísticas de cada usuario se pueden formatear y mostrar en su perfil de comunidad Steam.

Además de proporcionar premios altamente valiosos a los jugadores de sus juegos, los logros son útiles para alentar y recompensar el trabajo en equipo y la interacción del jugador, proporcionando una mayor dimensionalidad a los objetivos del juego y recompensando a los usuarios por pasar más tiempo en el juego. se recomienda que tu juego tenga algunos. Se configuran fácilmente desde Steam Dashboard, pero requerirán que usted cree Iconos especiales para ellos.

Las estadísticas rastrean datos de gran precisión, como el tiempo de reproducción, el número de power-ups utilizados, etc. Puede optar por usarlos simplemente para rastrear los datos internos del juego, para que, por ejemplo, pueda otorgar un logro basado en múltiples estadísticas de juego de sesión recopiladas del usuario en varias computadoras. O bien, puede rastrear datos interesantes del juego para mostrarlos en la página de Steam Community del usuario, donde los usuarios pueden comparar sus propias estadísticas con sus amigos.

Las estadísticas vienen en tres tipos, enteros, flotantes y promedios, y GameMaker Studio 2 permite modificar cada uno de estos desde su juego, pero deberían haber sido inicializados previamente desde el panel de control de Steam Works para su juego.

NOTA: debes esperar hasta steam_stats_ready ha vuelto verdadero, antes de intentar leer o escribir estadísticas y logros.


Se pueden encontrar más detalles e instrucciones sobre cómo usar estadísticas y logros en las siguientes páginas que describen las funciones:

  • steam _set_achievement
  • steam _get_achievement
  • steam _clear_achievement
  • steam _set_stat_int
  • steam _set_stat_float
  • steam _set_stat_avg_rate
  • steam _get_stat_int
  • steam _get_stat_float
  • steam _get_stat_avg_rate
  • steam _reset_all_stats
  • steam _reset_all_stats_achievements

Si el usuario está en modo fuera de línea, Steam mantiene un caché local de las estadísticas y los datos de rendimiento para que las API se puedan utilizar de la forma habitual. Cualquier estadística que no se pueda confirmar se guardará para la próxima vez que el usuario esté en línea. En el caso de que haya habido modificaciones en más de una máquina, Steam fusionará automáticamente los logros y elegirá el conjunto de estadísticas que haya tenido más progreso. Debido a que Steam mantiene un caché local de datos de estadísticas, no es necesario que el juego también guarde un caché local de los datos en el disco, especialmente porque dichos cachés a menudo entran en conflicto y cuando lo hacen se ve a los usuarios como si su progreso hubiera ha sido revertido, lo cual es una experiencia frustrante.


Steam Cloud proporciona un sistema de almacenamiento de archivos remoto fácil y transparente para su juego. Todos los archivos escritos en el disco utilizando las funciones en la nube de GameMaker Studio 2 (enumeradas a continuación) se replicarán en los servidores de Steam vez que el juego haya finalizado. Si el usuario cambia las computadoras, los archivos se descargarán a la nueva computadora antes de que se inicie el juego, lo que significa que el juego puede acceder a los archivos leyéndolos con las funciones de Steam apropiadas. Steam Client se encarga de garantizar que los archivos se mantengan sincronizados en todas las computadoras a las que el usuario puede acceder.

NOTA: Por defecto, la nube no está habilitada para un juego en Steam. debe estar habilitado previamente desde la pestaña 'Cloud' del administrador del juego Steam Works, donde debe establecer la cuota de bytes y archivos. La próxima vez que publique la configuración de los juegos de Steam, el almacenamiento en la nube estará listo para usar.


Las siguientes funciones se pueden usar para acceder a Steam Cloud desde GameMaker Studio 2

  • steam _is_cloud_enabled_for_app
  • steam _is_cloud_enabled_for_account
  • steam _get_quota_total
  • steam _get_quota_free
  • steam _file_exists
  • steam _file_size
  • steam _file_persisted
  • steam _file_write
  • steam _file_write_file
  • steam _file_read
  • steam _file_share
  • steam _file_delete

Steam admite contenido descargable gratuito y pago (DLC), y en el cliente Steam, un juego con contenido descargable aparece como una aplicación única en la lista de juegos del usuario con el contenido descargable visible a través del cuadro de diálogo de propiedades de los juegos. Una vez que se posee, el contenido descargable se trata como una parte integral del juego y Steam automáticamente actualizará el contenido cuando haya un parche disponible e instalará el contenido cuando el usuario instale el juego.

Como todo esto lo manejan los servidores de Steam y la configuración de cualquier DLC se realiza a través del panel de control de Steam Works, solo hay algunas funciones necesarias en GameMaker Studio 2 para verificar este contenido adicional:

  • steam _user_owns_dlc
  • steam _user_installed_dlc


Contenido generado por el usuario

La integración de Steam API en GameMaker Studio 2 permite compartir ciertos contenidos generados por los usuarios. Puede usarlos para compartir archivos en Steam Workshop para su juego, así como para publicar capturas de pantalla en el perfil del usuario.

  • steam _is_screenshot_requested
  • steam _send_screenshot
