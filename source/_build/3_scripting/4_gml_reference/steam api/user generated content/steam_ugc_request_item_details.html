steam _ugc_request_item_details

Sintaxis:

steam_ugc_request_item_details(published_file_id, max_age_seconds);


Argumento Descripción
published_file_id La identificación de archivo único para el UGC a verificar.
max_age_seconds La "edad" de los datos a verificar (recomendado 30 - 60 segundos).


Devoluciones: identificación Async


Descripción

Esta función se puede usar para recuperar información sobre una identificación de archivo determinada. Usted proporciona la identificación del archivo y proporciona una edad máxima para la verificación (consulte los documentos de Steam para obtener más información), luego la función devuelve un valor único de identificación asíncrona que se puede usar para verificar los detalles correctamente en el evento Steam Async. Este evento tendrá el async_load Mapa DS que se completará con los siguientes pares clave / valor:

  1. " id ": la identificación asíncrona devuelta por la función que activó el evento

  3. " resultado " - El resultado de la operación (un valor real). Esta será la constante GML ugc_result_success o algún otro número real. Por lo tanto, debe verificar esta constante para asegurarse de que la llamada fue exitosa y, si no, algo no funcionó correctamente. El resto de los posibles valores devueltos se muestran como resultado del valor "EResult" de Steam y debería ver steamclientpublic.h en los encabezados del SDK para los 89 valores posibles.

  5. " event_type " - La cadena " ugc_item_details "

  7. " cached_data " - Será true si los detalles devueltos son de la memoria caché local o false si se toman del servidor

  9. " published_file_id" "-Realiza el único ID de archivo publicado para el elemento

  11. " file_type " - El tipo de archivo utilizado

  13. " creator_app_id ": la identificación de Steam del creador del elemento

  15. " consumer_app_id ": la identificación de Steam del consumidor del artículo

  17. " título " - El título del artículo

  19. " descripción " - La descripción del artículo

  21. " steam _id_owner " - La identificación de Steam del propietario del elemento

  23. " time_created ": la hora a la que se creó el elemento por primera vez

  25. " time_updated ": la última vez que se actualizó el elemento

  27. " time_added_to_user_list ": la hora a la que se suscribió el elemento

  29. " visibilidad " - La visibilidad del artículo (ver steam_ugc_set_item_visibility() para las constantes devueltas)

  31. " prohibido ": si el artículo ha sido prohibido ( true ) o no ( false )

  33. " accepted_for_use ": si el artículo ha sido aceptado para su uso ( true ) o no ( false )

  35. " tags_truncated " - Versión corta de las etiquetas como una matriz

  37. " etiquetas ": una matriz de etiquetas para el elemento

  39. " handle_file " - El manejador de archivo único para el elemento

  41. " handle_preview_file ": el identificador único para la vista previa de la imagen del elemento (se puede usar como argumento con steam_ugc_download() para descargar una imagen de vista previa)

  43. " nombre de archivo " - El nombre del archivo de artículo

  45. " file_size ": el tamaño del archivo de elemento

  47. " preview_file_size ": el tamaño de la imagen de vista previa

  49. " url ": la URL completa del elemento

  51. " votes_up ": la cantidad de votos recibidos

  53. " votes_down ": la cantidad de votos atrasados recibidos

  55. " puntaje " - El puntaje general del artículo

  57. " account_id_owner ": la identificación de la cuenta del propietario de Steam ID (esto se puede usar en la función steam_ugc_create_query_user_ex() )


Ejemplo extendido:

En este ejemplo enviamos una solicitud de detalles para un artículo y luego analizamos el resultado async_load DS mapa para establecer algunas variables. Primero enviamos la solicitud:

steam_details = steam_ugc_request_item_details(global.fileID, 60);

El código anterior solicitará detalles sobre el elemento con el ID de archivo almacenado en la variable global y activará un evento Steam Async con la información devuelta. En este caso, podemos analizar el mapa y almacenar algunos de los valores en variables que luego se pueden usar para mostrar la información al usuario:

var map_id = async_load[? "id"];
var result = async_load[? "result"];
if (map_id == steam_details) && (result == ugc_result_success)
   {
   mTitle = async_load[? "title"];
   mDesc = async_load[? "description"];
   mTags = async_load[? "tags"];
   m_hPreviewFile = async_load[? "handle_preview_file"];
   m_hOwnerSteamId = async_load[? "steam_id_owner"];
   mOwnerAccountId = async_load[? "account_id_owner"];
   mPubFileId = async_load[? "published_file_id"];
   mScore = async_load[? "score"];
   }
