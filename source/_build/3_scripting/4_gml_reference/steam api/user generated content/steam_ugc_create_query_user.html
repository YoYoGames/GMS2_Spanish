steam _ugc_create_query_user

Sintaxis:

steam_ugc_create_query_user(list_type, match_type, sort_order, page);


Argumento Descripción
list_type El tipo de lista de datos para crear (vea las constantes enumeradas a continuación).
match_type El tipo de elementos UGC a consultar (ver las constantes enumeradas a continuación).
sort_order La forma en que se deben ordenar los datos (ver las constantes enumeradas a continuación).
page El número de página para consultar.


Devoluciones: Manija de consulta


Descripción

Esta función se puede usar para consultar la base de datos de UGC. La función utiliza automáticamente la ID predeterminada para la aplicación y el usuario, y asume que la consulta la realiza el consumidor (en lugar del creador). La función requiere que utilice las siguientes constantes para el tipo de datos a consultar, el tipo de elemento a consultar y el orden en que se ordenarán los elementos devueltos, así como un número de página: tenga en cuenta que una consulta arrojará un máximo numero de 50 articulos

La función devuelve un valor de identificador de consulta único que debe almacenarse en una variable para usar en las otras funciones de consulta. Tenga en cuenta que esta función solo prepara la consulta pero en realidad no la envía, para eso debe llamar a la función steam_ugc_send_query - y puedes usar más steam_ugc_query_*() funciones para refinar la solicitud de búsqueda antes de que realmente se envíe.


Ejemplo:

query_handle = steam_ugc_create_query_user(ugc_list_Published, ugc_match_Items, ugc_sortorder_TitleAsc, 1);

El código anterior crea una solicitud de consulta y almacena su manejador en una variable para uso futuro.
