steam _ugc_create_item

Sintaxis:

steam_ugc_create_item(consumer_app_id, file_type);


Argumento Descripción
consumer_app_id La ID de aplicación única para tu juego en Steam.
file_type Una de las constantes de tipo de archivo disponibles (enumeradas a continuación).


Devoluciones: identificación Async


Descripción

Esta función se usa para preparar la API de taller y generar una identificación de archivo publicada para el artículo que se agregará. Debe invocarse la función antes de hacer cualquier otra cosa con el elemento que se va a cargar, ya que se le solicitará que use el valor único de ID publicado que devuelve en el Evento Async de Steam para su actualización. Para usar esta función, debe proporcionar el ID de la aplicación Steam para su juego, y el uso de una de las siguientes constantes para el file_type argumento:

Constante Descripción
ugc_filetype_community Esto se usa para crear archivos que serán cargados y puestos a disposición de cualquier persona en la comunidad.
ugc_filetype_microtrans Esto se usa para describir los archivos que se cargan pero que solo se tienen en cuenta para que el juego considere agregarlos como contenido oficial.



Al usar esta función, se devolverá un valor de identificación asíncrona que luego se puede analizar cuando el evento Steam Asynchronous se activa para informar la creación del elemento. El evento contendrá los siguientes valores de clave / mapa en async_load ds_map:

  1. " id ": la identificación asíncrona devuelta por la función de llamada

  3. " resultado " - El resultado de la operación (un valor real). Esta será la constante GML ugc_result_success o algún otro número real. Por lo tanto, debe verificar esta constante para asegurarse de que la llamada fue exitosa y, si no, algo no funcionó correctamente. El resto de los posibles valores devueltos se muestran como resultado del valor "EResult" de Steam y debería ver steamclientpublic.h en los encabezados del SDK para los 89 valores posibles.

  5. " event_type " - Esta tecla mantendrá el valor " ugc_create_item "

  7. " legal_agreement_required " - Será true o false (mira los documentos de Steam para más detalles)

  9. " published_file_id ": esta clave contiene el ID único publicado para el elemento




Ejemplo extendido:

En este ejemplo, primero llamamos a la función y almacenamos el valor de la identificación asíncrona en una variable:

var app_id = steam_get_app_id();
new_item = steam_ugc_create_item(app_id, ugc_filetype_community);

Esto enviaría una solicitud a la API de Steam para crear el nuevo elemento Worksop, generando un evento asíncrono que trataríamos de la siguiente manera:

var event_id = async_load[? "id"];
if event_id == new_item
   {
   var type = async_load[? "event_type"];
   if type == "ugc_create_item"
      {
      global.Publish_ID = async_load[? "published_file_id"];
      }
   }

El código anterior comprueba el tipo de evento y, si es "ugc_create_item", recupera el ID del archivo publicado y lo almacena en una variable global para referencia futura.
