Funciones de números


Los números reales en GameMaker Studio 2 se consideran números de coma flotante de precisión doble, es decir, tienen un punto decimal sin un número fijo de dígitos antes o después del punto. 2, por ejemplo, es un entero binario, pero 2.0 es un punto flotante real. Esta distinción entre enteros y flotadores es muy importante cuando se trata del desarrollo multiplataforma, ya que la precisión de los cálculos realizados en una PC con Windows no es la misma que la precisión de esos mismos cálculos cuando se realiza en un dispositivo móvil. Esto significa que debe prestar especial atención al hacer comparaciones, por ejemplo:

if image_index == 1 {do something...}

Ahora puede parecer que funciona, pero debido a la forma en que funcionan las matemáticas de coma flotante, puede significar que image_index en realidad es igual a 1.00000000001, en cuyo caso la comparación nunca resultará en un true Resultado y el código en ejecución. Estos tipos de errores pueden ser bastante difíciles de depurar y, por lo tanto, siempre es bueno conocerlos y planificar con anticipación y usar otros medios para verificar los valores o usar las funciones de piso o redondeo apropiadas (enumeradas a continuación) para convertir el número a comprobar en un número entero. Por ejemplo, el código anterior podría escribirse como:

if floor(image_index) == 1 {do something...}

También vale la pena señalar que cuando se utilizan los objetivos del compilador YoYo, todas las expresiones y funciones se evalúan de izquierda a derecha, mientras que en todas las demás plataformas de destino se evalúan de derecha a izquierda, lo que significa que esto, por ejemplo, dará diferentes resultados dependiendo de en la plataforma:

val = max(num, ++num, num++);

También puede usar una función especial disponible en GameMaker Studio 2 para establecer el valor épsilon para las matemáticas de coma flotante. Cuando un número real se redondea al número de punto flotante más cercano, el épsilon (también conocido como "épsilon de la máquina") forma un límite superior en el error relativo, y puede obtener y configurar el valor de épsilon usando las siguientes funciones:

  • math_set_epsilon
  • math_get_epsilon

Todas estas funciones tratan con el uso de números y valores aleatorios:

  • escoger

  • random_range
  • al azar
  • irandom_range
  • random_set_seed

NOTA: Al usar las funciones aleatorias, GameMaker Studio 2 mantiene la misma semilla aleatoria cada vez que inicias el juego. Esto hace que la depuración sea mucho más fácil ya que tiene la garantía de que las funciones aleatorias siempre devolverán el mismo valor. Sin embargo, si no desea que esto suceda, primero debe establecer una nueva semilla aleatoria al comienzo del juego, ya sea utilizando randomise o random_set_seed.
