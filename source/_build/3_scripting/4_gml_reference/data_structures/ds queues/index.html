Colas DS

Una ds_queue es una estructura de datos de primero en entrar, primero en salir.

Una ds_queue es algo similar a una ds_stack pero funciona en base a primero en entrar primero en salir (FIFO). El valor que se coloca primero en la cola también es el primero que se elimina de él. Puede pensar que es como una cola en una tienda para pagar, donde la persona que está primero en la cola paga primero y luego sale de la tienda. Las colas generalmente se utilizan para almacenar acciones o comportamientos (en forma de scripts, por ejemplo) que una instancia aún tiene que hacer, pero hay muchos otros usos.

NOTA: Al igual que con todos los recursos dinámicos, las estructuras de datos ocupan la memoria y, por lo tanto, siempre deben destruirse cuando ya no son necesarias para evitar fugas de memoria que disminuirán la velocidad y, finalmente, bloquearán su juego.

Existen las siguientes funciones que se ocupan de ds_queues:

  1. ds_queue_create
  2. ds_queue_destroy
  3. ds_queue_clear
  4. ds_queue_empty
  5. ds_queue_size
  6. ds_queue_dequeue
  7. ds_queue_enqueue
  8. ds_queue_head
  9. ds_queue_tail
  10. ds_queue_copy
  11. ds_queue_read
  12. ds_queue_write
NOTA: si necesita verificar si existe una estructura de datos, puede usar ds_exists() función.
