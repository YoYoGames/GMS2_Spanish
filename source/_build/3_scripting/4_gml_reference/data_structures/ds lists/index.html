Listas DS

A ds_list almacena una colección de valores en un orden particular.

Una ds_list es una estructura de datos que almacena la información secuencialmente a medida que se agrega (al igual que una matriz 1D ). Las listas de DS son estructuras de datos muy flexibles que le permiten agregar valores al final o insertarlos en algún lugar en el medio de la lista, así como también darle la posibilidad de mezclar el orden (para asignar los valores aleatoriamente) u ordenarlos en forma ascendente. o en orden descendente

Al acceder a las estructuras de datos de la lista DS, siempre debe tratar de usar valores enteros para la posición de la lista, y GameMaker Studio 2 todos los índices que no sean enteros si no lo hace. Si eso no es lo que necesita, tendrá que hacer el redondeo usted mismo antes de pasar el índice que desea verificar.

NOTA: Al igual que con todos los recursos dinámicos, las estructuras de datos ocupan la memoria y, por lo tanto, siempre deben destruirse cuando ya no son necesarias para evitar fugas de memoria que disminuirán la velocidad y, finalmente, bloquearán su juego.

Las siguientes funciones existen que tratan con listas de DS.

  1. ds_list_create
  2. ds_list_destroy
  3. ds_list_clear
  4. ds_list_empty
  5. ds_list_size
  6. ds_list_add
  7. ds_list_set
  8. ds_list_delete
  9. ds_list_find_index
  10. ds_list_find_value
  11. ds_list_insert
  12. ds_list_replace
  13. ds_list_shuffle
  14. ds_list_sort
  15. ds_list_copy
  16. ds_list_read
  17. ds_list_write

Las listas Ds se pueden usar también para mantener ds_maps y otras ds_lists, pero solo para usar junto con la json_encode función. Esto requiere que "marque" o "marque" la entrada dentro de la lista para que pueda codificarse correctamente utilizando las siguientes funciones especiales:

  1. ds_list_mark_as_list
  2. ds_list_mark_as_map

Además de estas funciones específicas, todos pueden usar una expresión (llamada un descriptor de acceso ) para agregar o modificar el contenido de su ds_list. Este acceso se parece a una matriz 1D con la siguiente sintaxis:

list_index[| index]

Puede encontrar más información, además de ejemplos, de la página de Descripción de GML en Accesorios.

NOTA: si necesita verificar si existe una estructura de datos, puede usar ds_exists() función.
