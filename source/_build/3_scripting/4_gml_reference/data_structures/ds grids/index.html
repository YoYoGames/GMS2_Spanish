DS Grids


Un ds_grid es básicamente un tipo de matriz bidimensional. Defina la cuadrícula configurando el ancho y la altura de su entero, que luego establece el número total de "celdas" dentro de la cuadrícula (el ancho y la altura se multiplican) y estas celdas se usan para contener diferentes valores de datos (números reales o cadenas).

Este tipo de estructura le permite establecer y recuperar el valor de las celdas en la cuadrícula al proporcionar los índices x e y (que comienza con 0 para los ejes x y y, y sube hasta el ancho - 1 y la altura -1). Pero también puede establecer el valor en regiones de cuadrícula específicas, agregar valores individuales y recuperar la suma, el valor máximo, el mínimo y el valor medio de una región determinada, lo que hace que esta estructura de datos sea la más versátil de todas las disponibles para usted.

Al acceder a las estructuras de datos de la cuadrícula DS, siempre debe intentar usar valores enteros para la posición de la celda, y GameMaker Studio 2 todos los índices no enteros si no lo hace. Si eso no es lo que necesita, deberá realizar el redondeo usted mismo antes de aprobar el índice que desea verificar.

NOTA: Al igual que con todos los recursos dinámicos, las estructuras de datos ocupan memoria y, por lo tanto, siempre deben ser destruidas cuando ya no sean necesarias para evitar pérdidas de memoria, lo que ralentizará y eventualmente bloqueará su juego.

Aparte de estas funciones específicas, todos pueden usar una expresión (llamada acceso ) para agregar o modificar el contenido de su ds_grid. Este elemento de acceso es similar a una matriz 2D con la siguiente sintaxis:

grid_index[# xpos, ypos]

Puede encontrar más información, más ejemplos, en la página de información general de GML en Accessors.

NOTA: si necesita comprobar si existe una estructura de datos, puede utilizar la ds_exists() función.
