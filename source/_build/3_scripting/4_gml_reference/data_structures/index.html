Estructuras de datos

Esta sección trata sobre las diferentes funciones especializadas que puede usar para almacenar diferentes tipos de datos.

En los juegos, a menudo necesita almacenar información de una manera precisa y ordenada. Por ejemplo, puede que necesite almacenar listas de artículos que una persona transporta o puede querer almacenar una grilla de lugares que aún necesitan ser visitados. Ahora, puede utilizar las matrices para esto, pero si usted quiere hacer las operaciones más complicadas, como la clasificación de los datos o la búsqueda de un artículo en particular, tiene que escribir grandes piezas de código GML que pueden ser lentos para ejecutar, difíciles de depurar y un verdadero dolor para escribir!

Para remediar esto, GameMaker Studio 2 tiene una serie de estructuras de datos integradas a las que se puede acceder a través de funciones especializadas. Hay seis tipos diferentes de estructura de datos disponibles, cada uno con sus propios beneficios según el tipo de información que desee almacenar y cómo desea manipularlo más tarde: pilas, colas, listas, mapas, colas de prioridad y cuadrículas.

Esencialmente, todas las estructuras de datos funcionan de la misma manera: usted crea una estructura de datos y almacena su valor de índice en una variable. A continuación, utiliza este índice para hacer referencia a la estructura de datos en todas las demás llamadas a funciones que pueden realizar operaciones en él. Finalmente, una vez que termina, destruye la estructura de datos nuevamente para eliminarla de la memoria. Puedes usar tantas estructuras al mismo tiempo como necesites, y todas las estructuras pueden almacenar tanto cadenas como valores reales.

NOTA: Al igual que con todos los recursos dinámicos, las estructuras de datos ocupan la memoria y, por lo tanto, siempre deben destruirse cuando ya no son necesarias para evitar fugas de memoria que disminuirán la velocidad y, finalmente, bloquearán su juego.

Antes de utilizar estructuras de datos en su juego, también debe tener en cuenta que hay ciertos momentos en que, debido a errores de redondeo, puede obtener un resultado que no es el esperado. Esto puede resolverse cambiando la precisión de DS usando la siguiente función:

  • ds_set_precision

También hay una función especial para verificar si existe una estructura de datos de cualquier tipo:

  • ds_exists
