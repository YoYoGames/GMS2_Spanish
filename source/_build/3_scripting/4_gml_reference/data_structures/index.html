Estructuras de datos


En los juegos, a menudo es necesario almacenar información de forma precisa y ordenada. Por ejemplo, es posible que necesite almacenar listas de elementos que una persona transporta o tal vez desee almacenar una cuadrícula de lugares que aún deben visitarse. Ahora, puede usar las matrices para esto, pero si desea realizar operaciones más complicadas, como ordenar los datos o buscar un elemento en particular, necesita escribir grandes partes de código GML que pueden ser lentos de ejecutar, difíciles de depurar ¡Y un verdadero dolor al escribir!

Para remediar esto, GameMaker Studio 2 tiene una serie de estructuras de datos integradas a las que se puede acceder a través de funciones especializadas. Hay seis tipos diferentes de estructuras de datos disponibles, cada una con sus propios beneficios según el tipo de información que desea almacenar y cómo desea manipularla posteriormente: pilas, colas, listas, mapas, colas de prioridad y cuadrículas.

Esencialmente, todas las estructuras de datos funcionan de la misma manera: crea una estructura de datos y almacena su valor de índice en una variable. A continuación, utiliza este índice para hacer referencia a la estructura de datos en todas las llamadas a funciones adicionales que pueden realizar operaciones en él. Finalmente, una vez que haya terminado, destruya la estructura de datos nuevamente para eliminarla de la memoria. Puede usar tantas estructuras al mismo tiempo como necesite, y todas las estructuras pueden almacenar tanto cadenas como valores reales.

NOTA: Al igual que con todos los recursos dinámicos, las estructuras de datos ocupan memoria y, por lo tanto, siempre deben ser destruidas cuando ya no sean necesarias para evitar pérdidas de memoria, lo que ralentizará y eventualmente bloqueará su juego.
¡IMPORTANTE! Cuando creas una estructura de datos, el valor del índice para identificarlo es un valor entero que comienza en 0. Esto significa que diferentes estructuras de datos pueden tener el mismo valor de índice, por lo que si tienes dudas debes usar el ds_exists Funciona antes de acceder a ellos. También tenga en cuenta que los índices se reutilizan, por lo que un valor nuevo creado de la estructura de datos destruidos puede ser utilizado posteriormente, y le recomendamos que establezca cualquier variable que contenga un índice DS en -1 después de haber destruido la estructura de datos.

Antes de usar las estructuras de datos en su juego, también debe tener en cuenta que hay ciertos momentos en que, debido a errores de redondeo, puede obtener un resultado que no es el que esperaba. Esto se puede resolver cambiando la precisión de DS usando la siguiente función:

  • ds_set_precision

También hay una función especial para verificar si existe una estructura de datos de cualquier tipo dado:

  • ds_exists
