Mapas de DS

Un ds_map es una estructura de datos que almacena datos en pares vinculados que constan de una clave y un valor correspondiente.

La estructura de datos ds_map es excepcionalmente útil ya que le permite almacenar claves y pares de valores. Por ejemplo, un personaje en su juego puede tener una cantidad de elementos diferentes ( claves ) y para cada elemento individual puede tener un número de ellos ( valores ), como en un juego de rol donde puede tener 10 pociones de salud, 5 pociones de maná y 100 de oro Los mapas mantienen todos estos pares juntos en un solo lugar y puede agregar pares al mapa y buscar el valor correspondiente a ciertas teclas también usando algunas funciones simples. Sin embargo, hay un par de cosas que debes saber sobre los mapas antes de usarlos. Los mapas no están ordenados de ninguna manera (reconocible), lo que significa que para encontrar una determinada clave, es posible que tengas que repetir todo el proceso (lo cual es muy lento). Tampoco hay forma de mantener dos claves iguales, ni puede asignarle a una tecla dos valores.

NOTA: Las funciones ds_map han cambiado desde las versiones anteriores de GameMaker por lo que los juegos importados pueden no funcionar o comportarse como usted espera.

NOTA: Al igual que con todos los recursos dinámicos, las estructuras de datos ocupan la memoria y, por lo tanto, siempre deben destruirse cuando ya no son necesarias para evitar fugas de memoria que disminuirán la velocidad y, finalmente, bloquearán su juego.

Existen las siguientes funciones que se ocupan de los mapas de DS.

  1. ds_map_exists
  2. ds_map_create
  3. ds_map_add
  4. ds_map_clear
  5. ds_map_copy
  6. ds_map_replace
  7. ds_map_delete
  8. ds_map_empty
  9. ds_map_size
  10. ds_map_find_first
  11. ds_map_find_last
  12. ds_map_find_next
  13. ds_map_find_previous
  14. ds_map_find_value
  15. ds_map_read
  16. ds_map_write
  17. ds_map_destroy

Hay otras cuatro funciones disponibles para guardar y cargar un ds_map. Estas funciones encriptarán el mapa y lo almacenarán en una ubicación segura en la plataforma objetivo, para que pueda usarlo para guardar In App Purchase detalles, contraseñas, información sensible del juego, etc.

  1. ds_map_secure_save
  2. ds_map_secure_save_buffer
  3. ds_map_secure_load
  4. ds_map_secure_load_buffer

Además de estas funciones específicas, todos pueden usar una expresión (llamada un descriptor de acceso ) para agregar o modificar el contenido de su ds_map. Este acceso se parece a una matriz 1D con la siguiente sintaxis:

map_index[? key]

Puede encontrar más información, además de ejemplos, de la página de Descripción de GML en Accesorios.

JSON y Ds_maps

Uno de los usos que tienen los ds_maps es cuando se trabaja con JSON, por lo que hay algunas funciones especiales que son específicas para eso:

  1. ds_map_add_list
  2. ds_map_add_map
  3. ds_map_replace_list
  4. ds_map_replace_map

También hay un par de funciones complementarias para ds_lists:

  1. ds_list_mark_as_list
  2. ds_list_mark_as_map

NOTA: Si bien estas funciones le permiten agregar listas y mapas dentro de un mapa, son inútiles para cualquier otra cosa que no sea JSON, y los mapas y listas anidados no se leerán correctamente si se escriben en el disco o se accede de otra manera.

NOTA: si necesita verificar si existe una estructura de datos, puede usar ds_exists() función.
