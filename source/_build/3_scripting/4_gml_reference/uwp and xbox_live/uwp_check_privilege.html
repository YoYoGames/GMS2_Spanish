uwp_check_privilege


Descripción

Con esta función puede verificar que un usuario determinado tenga un privilegio elegido. La función devolverá verdadero o falso en el evento asíncrono del sistema dependiendo de si el privilegio está habilitado o no, y si establece el argumento de intento de resolución en verdadero y el privilegio no está habilitado, también abrirá un diálogo del sistema (suspendiendo el juego ) para solicitar al usuario que actualice la cuenta o lo que sea necesario para obtener el privilegio. Si el usuario habilita la opción requerida, la función devolverá verdadero.

La función requiere que use una de las siguientes constantes para la solicitud de privilegio:

Constante Descripción
uwp_privilege_internet_browsing Compruebe si está permitida la navegación por internet
uwp_privilege_social_network_sharing Compruebe si está permitido compartir en redes sociales.
uwp_privilege_share_kinect_content Compruebe si está permitido compartir el controlador Kinect
uwp_privilege_video_communications Compruebe si la comunicación por video está permitida.
uwp_privilege_communications Compruebe si la comunicación de internet está permitida.
uwp_privilege_user_created_content Compruebe si el acceso al contenido creado por el usuario está permitido
uwp_privilege_multiplayer_sessions Compruebe si se permiten sesiones de multijugador en línea
uwp_privilege_sessions Compruebe si las sesiones en línea están permitidas
uwp_privilege_fitness_upload Compruebe si se permite la carga de datos de fitness


Una vez que se ha llamado a la función, se activará un evento del sistema asíncrono cuando se reciba el resultado de la devolución de llamada, que tendrá el mapa especial de DS async_load. Este mapa debe ser analizado para la siguiente clave:

  • " event_type ": debe contener el valor "check_privilege_result" si el evento fue activado por esta función

Si el tipo de evento se relaciona con esta función, también habrá las siguientes teclas adicionales:

  • " resultado ": será una o más de las siguientes constantes (valores combinados):
    Constante Valor Descripción
    uwp_privilege_result_no_issue 0 There are no privilege issues with the user
    uwp_privilege_result_purchase_required 1 The user must purchase something additional, usually a subscription, for access
    uwp_privilege_result_aborted 2 The check was aborted
    uwp_privilege_result_banned 4 The user is banned
    uwp_privilege_result_restricted 8 The user is restricted from access, usually through parental controls

  • " privilegio ": será una de las constantes enumeradas anteriormente para su uso en la función, por lo que puede verificar qué privilegio ha solicitado en el caso de verificaciones múltiples.


Sintaxis:

uwp_check_privilege(user_id, privilege, attempt_resolution);

Argumento Descripción
user_id El puntero de ID de usuario para comprobar.
privilege La constante de privilegios a verificar (ver descripción, arriba).
attempt_resolution Habilitar el intento de resolución si se establece en true, o ignorar si se establece en false.


Devoluciones:

Real


Ejemplo:

var user_one = xboxlive_get_user(0);
if (xboxlive_user_is_signed_in(user_one))
    {
    uwp_check_privilege(user_one, xboxlive_privilege_multiplayer_sessions, true);
    }

Lo anterior verifica si el usuario ha iniciado sesión y si lo está, verifica el permiso de las sesiones de varios jugadores.
