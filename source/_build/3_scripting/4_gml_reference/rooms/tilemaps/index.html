Funciones de capa de Tilemap

El Editor de habitaciones GameMaker Studio 2 permite agregar fichas en cualquier habitación usando capas. Como su nombre lo indica, una capa de mosaico de azulejos es simplemente una colección de mosaicos de conjuntos de teselas que se consideran todos a la misma profundidad dentro de la sala, y al apilar capas puede hacer que algunas cosas se dibujen sobre otras. También puede controlar ciertos aspectos de las capas desde el código, agregar o eliminar cosas, o cambiar ciertas propiedades de la capa o lo que contiene en tiempo de ejecución. Tenga en cuenta que en el editor de sala en la que está limitado a una tilemap por capa (la capa de mosaico), pero cuando se trabaja con ellos en el código, puede tener múltiples tilemaps asignadas a una sola capa, con cada mapa tiene su propia identificación única y propiedades, asi que:

  • El conjunto de mosaicos es el recurso que usas para crear capas de mosaicos y mapas de mosaicos
  • La capa de mosaico es lo que creas en el Editor de sala para agregar tilesets a
  • El mapa de azulejos es lo que usted llama la colección de mosaicos que se agregan a una capa, ya sea en el editor de la sala oa través del código, como un elemento único en esa capa

Funciones de azulejos

Además de las funciones enumeradas anteriormente para establecer mapas de mosaico, también tiene una serie de funciones que se pueden utilizar para manipular los datos de mosaicos individuales. Un "blob" de datos de mosaico se compone del índice del mosaico más algunos bits adicionales para mostrar si el mosaico se ha rotado, volteado o duplicado. También tiene la posibilidad de agregar su propia máscara de bits a los datos del mosaico, aunque esto es solo para usuarios avanzados, ya que enmascarará los bits reservados para el índice del mosaico. En esencia, hay 19 bits reservados para los índices del mosaico, pero si solo usas 8 de ellos, entonces tienes once bits sobrantes que se pueden usar para crear una máscara personalizada. Esta máscara se puede utilizar y verificar en el código para crear colisiones de fichas personalizadas o lo que sea. Sin embargo, normalmente no necesitarías crear tus propios blobs de datos de mosaico, y en cambio usarías la función tilemap_get para obtener los datos de mosaico, entonces lo manipularías usando las funciones a continuación, y luego lo establecerías nuevamente usando tilemap_set.
