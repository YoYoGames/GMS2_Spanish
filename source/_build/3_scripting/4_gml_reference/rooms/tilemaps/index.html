Funciones de la capa Tilemap

El editor de salas de GameMaker Studio 2 permite agregar mosaicos a cualquier sala usando capas. Como su nombre lo indica, una capa tilemap es simplemente una colección de mosaicos de conjuntos de mosaicos que se consideran todos a la misma profundidad dentro de la sala, y al apilar capas puede hacer que algunas cosas se dibujen por encima o por debajo de otras. También puede controlar ciertos aspectos de las capas desde el código, agregar o eliminar cosas, o cambiar ciertas propiedades de la capa o lo que contiene en tiempo de ejecución. Tenga en cuenta que en el editor de sala en la que está limitado a una tilemap por capa (la capa de mosaico), pero cuando se trabaja con ellos en el código, puede tener múltiples tilemaps asignadas a una sola capa, con cada mapa tiene su propia identificación única y propiedades, asi que:

  • El conjunto de mosaicos es el recurso que se utiliza para crear capas de mosaicos y mapas de inclinación.
  • La capa de mosaico es lo que creas en el Editor de salas para agregar conjuntos de mosaicos a
  • El tilemap es lo que se llama la colección de mosaicos que se agregan a una capa, ya sea en el editor de sala o mediante código, como un elemento único en esa capa.

Funciones de mosaico

Además de las funciones enumeradas anteriormente para configurar los mapas de mapas, también tiene una serie de funciones que se pueden usar para manipular los datos de mosaicos individuales. Un "blob" de datos de mosaico se compone del índice de mosaico más unos pocos bits adicionales para mostrar si el mosaico se ha girado, volteado o reflejado. También tiene la posibilidad de agregar su propia máscara de bits a los datos de mosaico, aunque esto es solo para usuarios avanzados, ya que estará ocultando los bits que están reservados para el índice de mosaico; esencialmente, hay 19 bits reservados para sus índices de mosaico, pero solo usas 8 de ellos, luego tienes once bits restantes que pueden usarse para crear una máscara personalizada. Esta máscara se puede usar y verificar en el código para crear colisiones de mosaicos personalizados o lo que sea. Sin embargo, normalmente no necesitaría crear sus propios blobs de tiledata, y en su lugar usaría la función tilemap_get para obtener el tiledata, entonces lo manipularías usando las funciones a continuación, y luego lo configurarías de nuevo usando tilemap_set.
