tilemap_set_mask

Descripción

Esta función se puede usar para establecer la máscara de bits de mosaico para un único mapa de mosaico. Le das el ID del elemento de mosaico (que obtienes cuando creas un elemento de mosaico usando layer_tilemap_create() o cuando usas la función layer_tilemap_get_id() ), y luego el valor de la máscara. La idea es que si tienes, por ejemplo, un conjunto de mosaicos bastante pequeño de 16 x 16 mosaicos, entonces tienes un total de 256 mosaicos diferentes en tu conjunto de mosaicos, que solo usa hasta 8 bits de tu índice en la "burbuja" de datos del mosaico. "(a partir de los 19 bits de datos disponibles). Estos bits se pueden "enmascarar" y luego el resto de los datos del mosaico se pueden usar para almacenar valores adicionales.

Una manera fácil de obtener el valor de máscara de índice inicial es tomar el número de teselas del conjunto de teselas y restar 1, por lo que un conjunto de teselas de 64x64 tiene 256 teselas y un valor de máscara de 255 (o $ff en hexadecimal), y si el juego de fichas tiene un número de fichas que no tiene una potencia de dos, luego redondea a la potencia más cercana a dos para obtener la máscara, por ejemplo, un conjunto de fichas de 20x20 tiene 400 fichas, que redondearías hasta 512, y así que obtenga un valor de índice de máscara de 511 ( $1ff en hexadecimal). Además de la máscara de índice, normalmente querrá preservar los valores invertidos, rotar y reflejar que están asignados a un mosaico también, por lo que para ayudar con eso hay algunas constantes que se pueden usar. Estos pueden ser "o" -ed junto con el valor de máscara de índice para preservar esos bits de datos. Se pueden usar las siguientes constantes (vea el siguiente código de ejemplo para el uso):
Constante Descripción
tile_rotate Se usa para establecer / obtener el bit de rotación de un blob de datos de mosaico.
tile_mirror Se usa para establecer / obtener el bit de espejo de un blob de datos de mosaico.
tile_flip Se usa para establecer / obtener el bit de una ficha de datos de mosaico.
tile_index_mask Una constante especial que es para "y" -ing con el blob de datos de mosaico para extraer el índice del mosaico.


El enmascaramiento de bits Tilemap es una función avanzada que le permite utilizar bits de la "blob" de datos de mosaico para su propio uso. Por ejemplo, el usuario puede reservar un bit para que signifique "sólido" en la estructura de datos de mosaico y luego se puede verificar si hay colisiones rápidas. Básicamente, la máscara de bits se "y" -ed contra los datos del mosaico cuando se dibuja el mapa de azulejos (por lo que no afecta a ninguna de las otras funciones), lo que significa que los bits que no están en la máscara se ignoran, pero aún pueden ser leído y escrito por el usuario para sus propios fines. Tenga en cuenta que hay dos máscaras diferentes, una global y otra por mosaico, que GameMaker Studio 2 "junta" internamente antes de utilizar para el dibujo.


Sintaxis:

tilemap_set_mask(tilemap_element_id, mask)


Argumentos:

Argumento Descripción
tilemap_element_id El valor ID único del elemento tilemap para establecer la máscara de
mask El valor de la máscara


Devoluciones:

N/A


Ejemplo:

var lay_id = layer_get_id("Tiles_sky");
var map_id = layer_tilemap_get_id(lay_id);
var mask = tile_mirror | tile_flip | tile_rotate | 255;
tilemap_set_mask(map_id, mask);

El código anterior obtiene la ID de mosaico de la capa dada y luego establece la máscara de bits de mapa de mosaico en 8bits, conservando los valores de volteo, espejo y rotación para cada mosaico en el mapa.
