Funciones generales de capa

GameMaker Studio 2 Room Editor le permite agregar capas de diferentes activos a su habitación. Estos pueden ser tilemaps (utilizando baldosas), fondos, sprites o instancias, pero tenga en cuenta que a diferencia del editor de la habitación, que no se limitan a un solo tipo de activos por capa, y pueden agregar varios tipos de activos a la misma capa - por ejemplo, puede tener un fondo y un mapa de mosaicos todo en la misma capa. Esta sección describe las funciones utilizadas al trabajar con capas de instancia, así como algunas funciones generales para usar en todas las capas:

  • layer_exists
  • layer_get_id
  • layer_get_depth
  • layer_get_name
  • layer_get_all
  • layer_get_all_elements
  • layer_get_element_layer
  • layer_get_element_type
  • layer_get_x
  • layer_get_y
  • layer_get_hspeed
  • layer_get_vspeed
  • layer_create
  • layer_destroy
  • layer_x
  • layer_y
  • layer_hspeed
  • layer_vspeed
  • layer_add_instance
  • layer_has_instance
  • layer_instance_get_instance
  • layer_element_move
  • layer_set_visible
  • layer_get_visible
  • layer_depth
  • layer_force_draw_depth
  • layer_is_draw_depth_forced
  • layer_get_forced_depth
  • layer_script_begin
  • layer_script_end
  • layer_shader
  • layer_get_script_begin
  • layer_get_script_end
  • layer_get_shader

También tenemos un par de funciones adicionales relacionadas con las capas de orientación en una habitación diferente. Puede ser que desee cambiar algo en una habitación que no es la habitación actual, o tal vez desea generar habitaciones procesalmente, o cualquier cantidad de cosas. Para habilitar esto usamos las siguientes funciones:

  • layer_set_target_room
  • layer_get_target_room
  • layer_reset_target_room

Lo que haces aquí es configurar la sala objetivo utilizando la función apropiada y luego llamar a las funciones de capa son normales. Estas funciones de capa ahora se aplicarán solo a la sala objetivo y no a la actual. Cuando hayas terminado, llamarás a la función de reinicio para devolver el alcance del código nuevamente a la habitación actual.

¡ADVERTENCIA! Mientras se dirige a otra sala, puede usar todas las funciones de capa normales, excepto que no puede crear instancias usando instance_create_layer o instance_create_depth, ni la función de capa layer_add_instance estar disponible.


Finalmente, y esto es exclusivamente para usar con instancias en capas (aunque la capa misma puede tener otros elementos, no se verán afectados por las siguientes funciones), tiene un par de funciones que se pueden usar para desactivar o reactivar todas las instancias:

  • instance_deactivate_layer
  • instance_activate_layer
