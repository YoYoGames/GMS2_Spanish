external_define


Sintaxis:

external_define(dll, name, calltype, restype, argnumb, argtype[0], argtype[1], ...argtype[10]) ;


Argumento Descripción
dll El nombre del archivo dll (cadena)
name El nombre de la función (cadena)
calltype La convención de llamadas utilizada
restype El tipo de resultado a esperar
argnumb La cantidad de argumentos (0 - 10)
argtype[0... 10] Los diferentes tipos de argumentos que se utilizan


Devoluciones:

Real (Unique function ID value)


Descripción

Esta función se puede usar para definir una llamada de función externa a un dll específico (para Windows) o dylib (para Mac). Este archivo puede ser un archivo incluido o parte de una extensión. Usted proporciona el nombre (y la ruta) del archivo, luego el nombre de la función que desea definir. A continuación, debe definir la convención de llamadas que se utilizará (consulte la lista de constantes a continuación), así como el tipo de resultado que se espera (también una constante, como se detalla a continuación). Finalmente debe dar los argumentos numéricos que la función puede tomar (de 0 a 15) y para cada uno de los argumentos debe especificar también su tipo. Tenga en cuenta que para las funciones con 4 o más argumentos, todos ellos deben ser de tipo ty_real.

NOTA: Esto es solo para dll o dylib que se han agregado como "Archivos incluidos" al IDE de GameMaker Studio 2. No funcionará con los archivos agregados como extensiones, ya que estos requieren que defina las funciones en el paquete de extensión.
¡IMPORTANTE! Esta función no está disponible con la licencia de prueba del producto.
Constante Descripción
dll_cdecl Esta es la llamada predeterminada C, C ++
dll_stdcall Esta es la llamada estándar de WinAPI (Windows dll solamente)
ty_real Un argumento de número real
ty_string un argumento de cadena terminado en nulo


Ejemplo:

my_funcion = external_define("MyDLL.dll", "MyMin", dll_cdecl, ty_real, 2, ty_real, ty_real);

El código de ejemplo anterior definirá una función externa llamada "MyMin" con dos argumentos.
