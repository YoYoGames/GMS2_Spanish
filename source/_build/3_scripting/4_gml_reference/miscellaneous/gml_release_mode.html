gml_release_mode

Sintaxis:

gml_release_mode(flag);


Argumento Descripción
flag Ya sea true para establecer el modo de lanzamiento o falso para el desarrollo general.


Devoluciones: N / A


Descripción

Cuando su juego está compilado, también incluye ciertas funciones internas para la comprobación de errores durante la ejecución. Estas comprobaciones internas cubren muchos aspectos del código de un juego, incluidos rangos, parámetros y algunos aspectos generales del compilador GML interno, y son los que generan los mensajes de error finales en muchos casos. Sin embargo, esta función se puede utilizar para desactivar estas comprobaciones internas (es decir, eliminarlas por completo del juego compilado), dando un impulso de velocidad a su proyecto final.

Tenga en cuenta que esto significa que si hace algo en el código del juego que sea incorrecto, el juego compilado simplemente se bloqueará, o puede experimentar un comportamiento muy inesperado. Por ejemplo, si ds_grid se accede a una coordenada fuera del tamaño definido, en el modo de desarrollador obtendría un mensaje de error de la ventana del compilador, pero los controles en su lugar permitirían que el juego continúe ejecutándose. Sin embargo, en el modo de lanzamiento, estas comprobaciones no estarán allí y, por lo tanto, el código podría devolver un puntero NULL o un puntero malo que, a su vez, apuntará a alguna memoria aleatoria. En el mejor de los casos, podría obtener los datos incorrectos devueltos, o en el peor podría sobrescribir algo en la memoria: datos, gráficos o incluso el código mismo.


Ejemplo:

gml_release_mode(true);

El código de ejemplo anterior configurará el juego para usar el modo de lanzamiento cuando se compile.
