gml_pragma

Sintaxis:

gml_pragma(command, [optional...]);


Argumento Descripción
command Una cadena con uno de los comandos enumerados a continuación.
[optional] Algunos de los comandos disponibles requieren un argumento o argumentos opcionales. Estos se explican a continuación para cada comando.


Devoluciones: N / A


Descripción

los gml_pragma La función afecta cómo el objetivo YYC compila su código y debe invocarse con los diferentes comandos para optimizar aún más la compilación final de su proyecto. Estos comandos se procesan de manera efectiva antes de compilar el juego y, por lo tanto, la función se puede colocar en cualquier parte de su proyecto y aún se procesará antes de que el juego esté completamente compilado. Los comandos disponibles son los siguientes:

  • " forceinline ": cuando se llama a la función con el pragma "forceinline" en un recurso de script, el YYC compilará el proyecto con el script en línea, en lugar de hacer referencia a él. Esto proporcionará un impulso de procesamiento adicional, pero se debe tener cuidado al usarlo, ya que también se inflará el tamaño del archivo ejecutable final, especialmente si el código alineado es grande y / o se usa en múltiples lugares diferentes, así como también aumentará mucho el tiempo de compilación.


  • " global ", " [código gml] ": el pragma "global" le permite llamar a algún código GML antes que nada y ejecutará la segunda cadena GML en un ámbito global, antes de que se ejecute la primera sala del juego. Por ejemplo:

    gml_pragma("global", "scr_Init()");


    Esto llamará al guión "scr_Init" antes de que se ejecute la primera sala del juego.


  • " PNGCrush ": el pragma "PNGCrush" usará el programa PNGCrush en cada textura creada. Tenga en cuenta que esto puede aumentar de manera significativa el tiempo que lleva compilar el juego, por lo que no lo quiere todo el tiempo, aunque puede hacer un ahorro significativo en el tamaño del archivo final.


  • " Texgroup.Scale ", " [TextureGroupName] ", " [Scale Divisor] " - El "Texgroup.Scale" escalará el grupo de texturas dado en la compilación. Necesitas dar dos argumentos adicionales aquí: el "[TexGroupName]", que es el nombre (una cadena) del grupo de texturas a escalar, y el "[Scale Divisor]" (también una cadena), que es el divisor que desea usar para escalar, es decir:

    gml_pragma("Texgroup.Scale", "level1", "2");


    Esto equivaldrá a la mitad de todas las texturas en el grupo de texturas "level1".


  • " UnityBuild ", " [habilitar / deshabilitar] " - Si llama a la función con este pragma y configura el argumento activar / desactivar true luego, al compilar, colapsará todos los archivos.cpp del proyecto en un único archivo que luego usará para compilar todo, por ejemplo:

    gml_pragma("UnityBuild", "true");


    El beneficio de hacer una compilación unitaria es que las compilaciones son más rápidas, pero la desventaja es que hace una compilación completa cada vez, por lo que incluso si cambia una sola parte del código, volverá a generar todo sin usar ningún archivo en caché. Esto se ha agregado específicamente para la exportación de Xbox One usando el YYC, aunque se puede invocar para otras compilaciones. Para obtener más información sobre compilaciones de unidades, consulte aquí.


Ejemplo:

gml_pragma("forceinline");

El código de ejemplo anterior forzará la secuencia de comandos donde se usa para alinear en la compilación.
