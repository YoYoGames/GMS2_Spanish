os_is_paused

Comprueba si el sistema operativo del dispositivo está en pausa o no.

Sintaxis:

os_is_paused()


Devoluciones: Boolean


Descripción

Cuando desarrolle dispositivos móviles, necesita tener una forma de verificar si el dispositivo todavía está enfocado en su juego o no, especialmente si su juego está en un dispositivo iOS, Android o Windows Phone, entonces puede ser interrumpido por llamadas telefónicas y otras aplicaciones. Puede usar esta función para verificar si el dispositivo está en pausa o no.

Vale la pena señalar que esta función ejecutará cualquier código que esté dentro del bloque de código en los dispositivos, pero luego GameMaker: Studio se detendrá inmediatamente después de que el dispositivo guarde en caché el juego como un proceso en segundo plano. Entonces, si tiene algo así como el código de ejemplo que se proporciona a continuación, la instancia que se creará no aparecerá ni ejecutará ninguno de sus eventos hasta después de que el juego se haya reiniciado desde el estado en caché. Esto significa que puede preferir tener el os_is_paused función invocar una alarma y luego en el evento de alarma (que se configurará cuando el dispositivo esté en pausa pero no se ejecute hasta que no esté en pausa) cree un menú de pausa, reinicie la música, etc.

Si está utilizando esta función en plataformas Windows, Ubuntu (Linux), MacOS o HTML5, la función solo se alterna para un paso a true cuando el juego pierde el foco, y luego regresa false de nuevo hasta que el juego vuelva a enfocarse (y lo vuelva a perder). Esto se debe a que estas plataformas no almacenan en caché las aplicaciones y los juegos, por lo que seguirán ejecutándose en segundo plano.

NOTA: en los dispositivos móviles, los sonidos y la música se detendrán cuando el sistema operativo esté en pausa. No van a reiniciar de nuevo a menos que ellos comienzan específicamente con la función apropiada.


Ejemplo:

if os_is_paused()
   {
   if !instance_exists(obj_PauseMenu) instance_create_layer(0, 0, "Controllers", obj_PauseMenu);
   }

El código anterior comprueba si el juego se ha pausado y, si tiene, crea una instancia del objeto "obj_PauseMenu" si ya no existe ninguno.
