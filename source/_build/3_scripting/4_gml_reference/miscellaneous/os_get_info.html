os_get_info


Descripción

Esta función devuelve un ds_map con información detallada sobre el sistema operativo en el que se está ejecutando el juego. La información exacta devuelta dependerá del sistema operativo y el dispositivo. Tenga en cuenta que ds_map no se elimina automáticamente de la memoria y debe usar ds_map_destroy() función cuando ya no necesita los datos que contiene.

Tenga en cuenta que en Windows Desktop y Windows UWP, obtendrá información adicional de DX11 con lo siguiente (útil para las extensiones):

  • video_d3d11_device - puntero al dispositivo DX11
  • video_d3d11_context - Puntero al contexto DX11
  • video_adapter_vendorid - cadena con el ID de proveedor de adaptadores
  • video_adapter_deviceid - cadena con el ID del dispositivo adaptador
  • video_adapter_subsysid - cadena con el ID del subsistema del adaptador
  • video_adapter_revision - cadena con el número de revisión del adaptador
  • video_adapter_dedicatedvideomemory - cadena con el tamaño de la memoria de video
  • video_adapter_dedicatedsystemmemory - cadena con el tamaño de la memoria del sistema (utilizada por el adaptador DX11)
NOTA: Esta función solo funciona en Android, iOS, Mac, Ubuntu y Windows (Desktop + UWP), en todas las demás plataformas de destino devolverá -1 en lugar de una ds_map.


Sintaxis:

os_get_info()


Devoluciones:

Real (ds_map index)


Ejemplo:

os_map = os_get_info();
if os_map != -1
   {
   var size, key, i;
   size = ds_map_size(os_map);
   key = ds_map_find_first(os_map);
   for (i = 0; i < size - 1; i++;)
      {
      map_data[i] = ds_map_find_value(os_map, key);
      key = ds_map_find_next(os_map, key);
      }
   ds_map_destroy(os_map);
   }

El código anterior verificará la información del sistema operativo para ver si ds_map es regresado. Si es así, el código recorrerá el mapa y asignará todos los valores a una matriz.
