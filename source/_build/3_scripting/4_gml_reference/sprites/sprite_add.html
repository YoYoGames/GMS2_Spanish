sprite_add


Sintaxis:

sprite_add(fname, imgnumb, removeback, smooth, xorig, yorig);

Argumento Descripción
fname El nombre (una ruta de archivo de cadena) del archivo para agregar.
imgnum Use para indicar el número de subimágenes (1 para una sola imagen o para una *.gif )
removeback Indica si se deben hacer transparentes todos los píxeles con el color de fondo (píxel inferior izquierdo).
smooth Indica si suavizar los bordes si es transparente.
xorig Indique la posición x del origen en el sprite.
yorig Indique la posición y del origen en el sprite.


Devoluciones:

Real (sprite ID value)


Descripción

Con esta función puede agregar una imagen como un elemento, cargarlo desde una fuente externa donde el archivo de imagen que se va a cargar debe estar siempre en *.png, *.gif o *.jpg/jpeg formato. La función devuelve el nuevo índice de sprites que luego debe usarse en todos los códigos adicionales que se relacionan con el sprite. Si usa esta función con HTML5 o obtiene una imagen de una URL, esta función también generará un evento asincrónico. Vea el evento asincrónico para más información.

Todas las imágenes que se convertirán en sprites animados, a excepción de *.gif (ver a continuación): debe tener un formato de "tira" (ver la imagen a continuación) y se dividirá en el número de imágenes secundarias que se especifican a continuación del ancho del elemento de regla = ancho de banda / subimágenes.

Como puede ver en la imagen de arriba, el sprite se ha colocado sobre un fondo morado oscuro, y esto puede eliminarse configurando el argumento "removeback" para true. Esto funciona verificando el píxel inferior izquierdo del sprite para ver el color y luego lo usa como el color que se eliminará. Por ejemplo, en la imagen de arriba, si tuviéramos el color del píxel inferior izquierdo en verde, todas las partes verdes del sprite habrían sido eliminadas y el resto del fondo púrpura ignorado.

Si eliges la opción "eliminar", también puedes desear que GameMaker Studio 2 suavice los bordes del sprite configurando el argumento "suave" para true. Todo lo que hace es crear un borde semitransparente alrededor de los bordes del sprite después de haber eliminado su fondo.

Finalmente, también puede especificar el origen x e y para el sprite. Este es el punto donde el sprite se "fija" en la instancia que lo usa, y siempre se calcula como relativo a la esquina superior izquierda 0,0 de una subimagen de sprite. Entonces, por ejemplo, un sprite de 32 x 32 píxeles con estos valores establecidos en (16,16) tendrá su origen en el centro.

Si está agregando un *.gif formatee la imagen, entonces solo necesita especificar el nombre del archivo y el número de imagen (que debe ser 1 para el primer fotograma) y puede dejar todos los demás argumentos como 0, indicando que solo se cargará la primera imagen del GIF.

Por defecto, todos los nuevos sprites tienen sus cuadros delimitadores calculados automáticamente (el cuadro delimitador exacto dependerá del tamaño y la transparencia del sprite); sin embargo, es posible que desee personalizarlo, en cuyo caso también debe usar la función sprite_collision_mask.

NOTA: Cuando cargue un sprite en GameMaker Studio 2, debe recordar quitarlo nuevamente (con sprite_delete ) cuando ya no sea necesario, de lo contrario existe el riesgo de una pérdida de memoria que disminuirá la velocidad y, finalmente, se bloqueará el juego.
¡IMPORTANTE! Esta función no está disponible con la licencia de prueba del producto.


Ejemplo:

spr = sprite_add("player_5.png", 16, true, true, 0, 0);

El código anterior carga un elemento en el juego y almacena su índice en la variable "spr".
