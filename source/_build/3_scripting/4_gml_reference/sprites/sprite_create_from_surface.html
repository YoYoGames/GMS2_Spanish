sprite_create_from_surface


Sintaxis:

sprite_create_from_surface(index, x, y, w, h, removeback, smooth, xorig, yorig);

Argumento Descripción
index El índice de la superficie para crear desde.
x La posición x para copiar desde.
y La posición y para copiar desde.
w El ancho del área a copiar (desde la posición x).
h La altura del área que se va a copiar (desde la posición y).
removeback Indica si se deben hacer transparentes todos los píxeles con el color de fondo (píxel inferior izquierdo).
smooth Indica si suavizar los bordes.
xorig Indica la posición x del origen en el sprite.
yorig Indica la posición y del origen en el sprite.


Devoluciones

Real (unique sprite ID value)


Descripción

Con esta función, puede crear un sprite a partir de una superficie previamente inicializada (el valor de ID de índice de superficie se devuelve cuando crea la superficie usando surface_create ) Las coordenadas xey que ingresas en la función deben ser relativas a la posición (0,0) de la superficie (la esquina superior izquierda) y no a la ventana del juego (ni a la vista si tienes una activa). Los argumentos de ancho y alto están en píxeles y definen el ancho y la altura de la parte de la superficie que se usará.

Establecer el argumento "eliminar" para true eliminará un color del sprite, marcando el píxel inferior izquierdo del sprite para ver el color y luego usarlo como el color que se va a eliminar.

Si eliges la opción "eliminar", también puedes desear que GameMaker Studio 2 suavice los bordes del sprite configurando el argumento "suave" para true. Todo lo que hace es crear un borde semitransparente alrededor de los bordes del sprite después de haber eliminado su fondo.

Finalmente, también puede especificar el origen x e y para el sprite. Este es el punto donde el sprite se "fija" en la instancia que lo usa, y siempre se calcula como relativo a la esquina superior izquierda 0,0 de una subimagen de sprite. Entonces, por ejemplo, un sprite de 32 x 32 píxeles con estos valores establecidos en (16,16) tendrá su origen en el centro.

De forma predeterminada, todos los nuevos sprites tienen sus cuadros delimitadores calculados automáticamente (la bbox exacta dependerá del tamaño y la transparencia del sprite); sin embargo, es posible que desee personalizar esto, en cuyo caso también debe usar la función sprite_collision_mask.

NOTA: Cuando crees un sprite en GameMaker Studio 2 con este método, debes recordar eliminarlo de nuevo (con sprite_delete ) cuando ya no sea necesario, de lo contrario existe el riesgo de una pérdida de memoria que disminuirá la velocidad y, finalmente, se bloqueará el juego.
¡IMPORTANTE! Esta función no está disponible con la licencia de prueba del producto.


Ejemplo:

var surf;
surf = surface_create(32, 32);
surface_set_target(surf);
draw_clear_alpha(c_black, 0);
draw_sprite(spr_Body, 0, 0, 0);
draw_sprite(spr_Clothes, 0, 0, 0);
draw_sprite(spr_Hair, 0, 0, 0);
spr_custom = sprite_create_from_surface(surf, 0, 0, 32, 32, true, true, 16, 16);
surface_reset_target();
surface_free(surf);

El código anterior crea una superficie y almacena su índice en la variable local "navegar". Luego apunta a esa superficie, la despeja y dibuja varios sprites uno encima del otro. Finalmente, crea un nuevo sprite a partir de la imagen compuesta dibujada en la superficie y asigna su índice a "spr_Custom" antes de liberar la memoria utilizada por la superficie.
