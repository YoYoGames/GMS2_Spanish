skeleton_attachment_set


Sintaxis:

skeleton_attachment_set(slot, attachment);

Argumento Descripción
slot El nombre de la ranura (una cadena) para obtener el archivo adjunto.
attachment El nombre (como una cadena o un índice de sprite) de la imagen adjunta.


Devoluciones:

String


Descripción

Un sprite de animación esquelético puede tener otras imágenes agregadas como archivos adjuntos, con estas imágenes que se agregan a una ranura nombrada (el nombre se da cuando crea la ranura de adjuntos en su programa de animación) y se dibujarán junto con la animación del sprite actual. Con esta función, puede establecer un archivo adjunto en una ranura determinada, donde debe dar los nombres (como cadenas) de la ranura y el archivo adjunto. Estos nombres están definidos por el programa de animación utilizado o (en el caso del archivo adjunto) cuando llama skeleton_attachment_create.

Tenga en cuenta que también puede pasar un sprite_index como el archivo adjunto, y ese sprite se usará, o puede usar -1 para eliminar el archivo adjunto de la ranura. Cuando pasa un índice de sprites como argumento, creará una ranura de adjuntos utilizando el nombre del sprite como la cadena para nombrar la ranura (por lo tanto, use spr_sword, por ejemplo, creará una ranura de inserción "spr_sword"), y la ranura usará el primer índice de imagen (0) del sprite, así como su compensación de origen xey, con una escala de (1,1) y una rotación de 0.

¡IMPORTANTE! Esta función no está disponible con la licencia de prueba del producto.


Ejemplo:

if skeleton_attachment_get("slot_leftHand") == ""
   {
   skeleton_attachment_set("slot_leftHand", choose("sword", "spear", "knife"));
   }

El código anterior verificaría el nombre del archivo adjunto actualmente asignado para la ranura llamada "slot_leftHand" y si se devuelve una cadena vacía, se adjunta un nuevo elemento sprite.
