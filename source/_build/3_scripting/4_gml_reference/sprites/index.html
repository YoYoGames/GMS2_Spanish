Sprites


Los Sprites son generalmente las representaciones visuales de los objetos dentro del juego que estás creando, aunque se pueden usar para muchos otros propósitos. Un sprite es una imagen individual o un conjunto de imágenes que, cuando se reproducen una tras otra, se ven como una animación en bucle. Los Sprites se pueden crear en muchos formatos, con el sprite "tradicional" formado por marcos de imágenes de píxeles, importados en GameMaker Studio 2 como *.png archivos. Sin embargo, también puede importar sprites vector que se hayan guardado en *.swf formato, o sprites de animación esquelética que se han guardado como *.json archivos con acompañamiento *.atlas y *.png archivos (para obtener más información sobre estos tipos de archivos de imagen y cómo importarlos en GameMaker Studio 2, consulte la sección Importar Sprites que no son de mapa de bits ).

Dado que los sprites son una parte integral de cualquier juego, GameMaker Studio 2 tiene una gran cantidad de funciones que se pueden usar para obtener información sobre sprites específicos, o para cambiar los que están en tu juego, o incluso para cargar tus propios sprites en una aplicación. desde un archivo de imagen de origen.

Las siguientes funciones se pueden usar para obtener información sobre un activo de sprite. Tenga en cuenta que estas funciones obtienen la información del sprite base (es decir, el activo del sprite) y no del sprite, ya que se está utilizando en una instancia. Por ejemplo, puede usar la variable de instancia sprite_width para obtener el ancho del sprite que usa una instancia y este ancho tendrá en cuenta cualquier escala que se haya aplicado. Sin embargo, si necesita saber el ancho de un sprite como lo haría normalmente, sin aplicar transformadas, necesitaría usar el sprite_get_width() función a continuación. Para obtener más información sobre las variables de instancia de sprite, consulte la sección sobre Variables de instancia.

Esto significa que la mayoría de estas funciones afectarán a todas las instancias que hagan referencia al sprite que se está cambiando, también significa que debe tener mucho cuidado para asegurarse de que exista un sprite antes de hacer referencia a él, o para asegurarse de que no se está haciendo referencia antes de cambiarlo.
