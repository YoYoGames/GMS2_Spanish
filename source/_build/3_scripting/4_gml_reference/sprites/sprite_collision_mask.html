sprite_collision_mask


Sintaxis:

sprite_collision_mask(ind, sepmasks, bboxmode, bbleft, bbtop, bbright, bbbottom, kind, tolerance);

Argumento Descripción
ind El índice del sprite para establecer el cuadro delimitador de.
sepmasks Si crear máscaras de colisión para cada subimagen del sprite ( true ), o una máscara para todos ( false )
bboxmode Qué tipo de cuadro delimitador usar 0 = automático, 1 = imagen completa, 2 = definido por el usuario.
bbleft La posición máxima a la izquierda del cuadro delimitador.
bbtop La posición máxima máxima del cuadro delimitador.
bbright La posición máxima correcta del cuadro delimitador.
bbbottom La posición inferior máxima del cuadro delimitador.
kind El tipo de máscara (0 = preciso, 1 = cuadro delimitador, 2 = disco, 3 = diamante).
tolerance Indica la tolerancia en el valor de transparencia (0 = sin tolerancia, 255 = tolerancia total).


Devoluciones

N/A


Descripción

Con esta función puede establecer el tipo de máscara de colisión que debería tener un sprite. Si selecciona ya sea automática (0) o imagen completa (1) como el modo de cuadro delimitador, entonces los valores del cuadro delimitador individual pueden establecerse en 0. Sin embargo, para una máscara definida por el usuario (2) deberá establecer estos valores. Los diferentes valores del cuadro delimitador son siempre relativos a la esquina superior izquierda del elemento sprite (independientemente del origen xey) que se considera la posición (0, 0).

Establecer el tipo de máscara establece la forma general de la máscara, pero tenga en cuenta que cualquier cosa que no sea una máscara rectangular requerirá más potencia de procesamiento al resolver las colisiones, con una caída posterior en el rendimiento. En general, solo debe usar tipos de máscara que no sean rectangulares cuando sea absolutamente necesario.

Finalmente, la tolerancia se usa para definir qué tan precisa es la máscara (cuando se utiliza con una máscara de "imagen completa", esto no tendrá ningún efecto), con una tolerancia de 0, lo que significa que la máscara seguirá cada píxel que tenga una transparencia más de 0, mientras que otros valores cambiarán el perímetro de la máscara de colisión dependiendo de la transparencia de los píxeles.

NOTA: Esta función solo es útil para sprites de mapa de bits y no funcionará con sprites SWF o JSON (Spine).
NOTA: Esta función solo funcionará en sprites duplicados y no directamente en recursos prefabricados. puedes duplicar sprites usando la función sprite_duplicate.


Ejemplo:

spr = sprite_add("player_5.png", 16, true, true, 0, 0);
sprite_collision_mask(spr, true, 1, 0, 0, 0, 0, 0, 0);

El código anterior carga un sprite de una fuente externa y almacena el nuevo índice en la variable "spr". Luego, el código establece el nuevo sprite para que tenga una máscara de colisión precisa para cada una de sus subimágenes.
