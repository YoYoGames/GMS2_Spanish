sprite_collision_mask


Sintaxis:

sprite_collision_mask(ind, sepmasks, bboxmode, bbleft, bbtop, bbright, bbbottom, kind, tolerance);

Argumento Descripción
ind El índice del sprite para establecer el cuadro delimitador de.
sepmasks Ya sea para crear máscaras de colisión para cada sub-imagen del sprite ( true ), o una máscara para todos ( false ).
bboxmode Qué tipo de caja delimitadora usar. 0 = automático, 1 = imagen completa, 2 = definido por el usuario.
bbleft La posición máxima a la izquierda del cuadro delimitador.
bbtop La posición superior máxima del cuadro delimitador.
bbright La posición máxima a la derecha del cuadro delimitador.
bbbottom La posición inferior máxima del cuadro delimitador.
kind El tipo de máscara, una constante (ver la tabla en la descripción).
tolerance Indica la tolerancia en el valor de transparencia (0 = sin tolerancia, 255 = tolerancia completa).


Devoluciones

N/A


Descripción

Con esta función puede establecer las propiedades de la máscara de colisión que debe tener un sprite. Si selecciona automático (0) o imagen completa (1) como modo de cuadro delimitador, los valores individuales del cuadro delimitador se pueden establecer en 0. Sin embargo, para una máscara definida por el usuario (2) tendrá que configurar estos valores. Los diferentes valores del cuadro delimitador son siempre relativos a la esquina superior izquierda del sprite (independientemente del origen x e y) que se considera posición (0, 0).

Al establecer el tipo de máscara, se establece la forma general de la máscara en sí, pero tenga en cuenta que cualquier otra cosa que no sea una máscara rectangular requerirá más poder de procesamiento al resolver colisiones, con una caída posterior en el rendimiento. En general, solo debe usar tipos de máscara que no sean rectangulares cuando sea absolutamente necesario.

NOTA: Esta función no permite el tipo de máscara de colisión de rectángulo girado.


El tipo de máscara que se puede establecer será una de las siguientes constantes:

Constante Descripción
bboxkind_rectangular Una forma de máscara de colisión rectangular (no giratoria) rectangular
bboxkind_ellipse Una forma de máscara de colisión elíptica
bboxkind_diamond Una forma de máscara de colisión de diamantes.
bboxkind_precise Una máscara de colisión precisa, donde la máscara se ajustará a los píxeles no transparentes del sprite, en función del valor de tolerancia dado (ver más abajo)

Finalmente, la tolerancia se usa para definir qué tan precisa es la máscara (cuando se usa con una máscara de "imagen completa", esto no tendrá efecto), con una tolerancia de 0, lo que significa que la máscara seguirá cada píxel que tenga una transparencia. sobre 0, mientras que otros valores cambiarán el perímetro de la máscara de colisión dependiendo de la transparencia de los píxeles.

NOTA: Esta función solo es útil para sprites de mapa de bits y no funcionará con sprites SWF o JSON (Spine).
NOTA: Esta función solo funcionará en sprites duplicados y no directamente en recursos prefabricados. puedes duplicar sprites usando la función sprite_duplicate.


Ejemplo:

spr = sprite_add("player_5.png", 16, true, true, 0, 0);
sprite_collision_mask(spr, true, 1, 0, 0, 0, 0, 0, 0);

El código anterior carga un sprite desde una fuente externa y almacena el nuevo índice en la variable "spr". El código luego establece que el nuevo sprite tenga una máscara de colisión precisa para cada una de sus subimágenes.
