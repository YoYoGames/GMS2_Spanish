camera_create


Descripción

Esta función creará una cámara "vacía", es decir, una cámara sin vista o sin configuraciones de proyección definidas. Una vez que haya creado la cámara, puede usar las diferentes funciones de matriz para establecer las matrices de visualización y proyección. También puede usar esta función para crear la cámara y luego usar las diferentes camera_set_* funciones para configurar la vista, sin embargo tenga en cuenta que debe proporcionar al menos una posición y tamaño de vista, de lo contrario la cámara resultante no será válida y dará resultados impares si se aplica a un puerto de visualización (si no establece el objeto siguiente, bordes o velocidad de movimiento, estos se establecerán en los valores predeterminados para una cámara de visualización en el Editor de sala).

Tenga en cuenta que las cámaras son como todos los recursos dinámicos y ocuparán la memoria cuando se creen. Esto significa que debe tener cuidado de no crear cámaras innecesariamente y destruir aquellas que no necesita llamando a la función camera_destroy(). Esto también significa que las cámaras persistirán de una habitación a otra si se asignan a un global variable (persistirán cuando se asignen a variables de instancia o locales también, pero no se puede acceder a ellas después de cambiar de sala, de ahí la pérdida de memoria), para que pueda, por ejemplo, crear una nueva cámara, asignarla a una variable global y luego, al inicio de cada sala, establezca las matrices de visualización y proyección requeridas y el puerto de visualización que necesita usar.


Sintaxis:

camera_create();


Devoluciones:

Real (unique camera ID value)


Ejemplo:

view_camera[0] = camera_create();
var viewmat = matrix_build_lookat(640, 240, -10, 640, 240, 0, 0, 1, 0);
var projmat = matrix_build_projection_ortho(640, 480, 1.0, 32000.0);
camera_set_view_mat(view_camera[0], viewmat);
camera_set_proj_mat(view_camera[0], projmat);

El código anterior crea una cámara nueva y asigna su ID para ver [0]. Las matrices de visualización y proyección se crean y almacenan en variables locales antes de usarse para configurar la cámara.
