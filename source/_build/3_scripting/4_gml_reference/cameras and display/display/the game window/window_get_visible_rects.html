window_get_visible_rects

Encuentre la región de superposición del rectángulo definido por (x1, y1) a (x2, y2) en cada una de las pantallas adjuntas.

Sintaxis:

window_get_visible_rects(x1, y1, x2, y2);


Argumento Descripción
x1 El borde izquierdo del rectángulo para verificar
y1 El borde superior del rectángulo para comprobar.
x2 El borde derecho del rectángulo para verificar
y2 El borde inferior del rectángulo para verificar.


Devoluciones: matriz


Descripción

Con esta función puede encontrar la región de superposición del rectángulo definido por (x1, y1) a (x2, y2) en cada una de las pantallas adjuntas. La función devolverá una matriz con 8 valores por pantalla (es decir: si tiene dos pantallas, la matriz tendrá una longitud de 16 índices), donde los valores [0... 3] corresponden a la superposiciónx1, overlapy1, overlapx2, overlapy2 - definiendo la región de superposición en esta pantalla y se establecerá en 0,0,0,0 si no hay solapamiento - y los valores [4... 7] corresponden a monitorx1, monitory1, monitorx2, monitory2 - las coordenadas de la pantalla en el espacio de visualización virtual. Esto puede usarse para probar si una posición de ventana guardada será visible o no (por ejemplo, el usuario puede haber desconectado un monitor externo o movido la ventana de la pantalla que dejó la posición de la ventana guardada como no válida).


Ejemplo:

var wx = window_get_x();
var wy = window_get_y();
var ww = window_get_width();
var wh = window_get_height();
display_data = window_get_visible_rects(wx, wy, wx + ww, wy + wh);
display_num = array_length_1d(display_data) / 8;

El código anterior generará una matriz 1D contenida en la variable display_data que contiene la información sobre las pantallas, así como también crea la variable display_num para mantener la cantidad de pantallas activas encontradas.
