evento_heredado

Realiza el evento heredado.

Sintaxis:

event_inherited();


Devoluciones: N / A.


Descripción

Esta función llamará al evento actual del objeto principal de la instancia. Normalmente, cuando una instancia tiene un objeto principal, hereda automáticamente todos los mismos eventos que el principal, pero si (por ejemplo) su objeto principal tiene un evento de creación y usted agrega uno a su objeto secundario, todas las instancias del objeto secundario ejecute el nuevo evento de creación que ha agregado y no el que está en el objeto principal. En caso de que necesite utilizar el evento de objeto principal y el evento de objeto secundario del mismo tipo, debe usar esta función, ya que ejecutará el evento de objeto principal antes de continuar con el resto del código o las acciones que contiene el evento secundario.


Ejemplo:

event_inherited();
switch (sprite_index)
   {
   case spr_Enemy_1: dmg += 2; break;
   case spr_Enemy_4: dmg -= 1; break;
   case spr_Enemy_10: dmg +=10; break;
   }

El código anterior llama al evento principal heredado (en el que configuramos la variable "dmg" y el índice de sprite) y luego modificamos la variable "dmg". Si no hay un padre especificado para la instancia que ejecuta este código, obtendríamos un error de "variable desconocida" ya que no se ha definido dmg.
