Generando eventos

GameMaker Studio 2 es completamente impulsado por eventos, y todas las acciones ocurren como resultado de ciertos eventos que se desencadenan en un orden específico o por una ocurrencia específica dentro del juego. Hay una serie de eventos diferentes, cada uno con una tarea específica o tiempo y algunos de ellos incluso se dividen en eventos "secundarios" separados. Normalmente GameMaker Studio 2 ejecuta estos eventos cuando detecta un activador en el juego, o cada marca de un juego (como en el caso de los eventos de paso y sorteo), pero también puede aplicar un evento a una instancia en la habitación actual desde dentro de un pedazo de código.

Para obtener más información sobre eventos, consulte la sección Eventos.

Existen las siguientes funciones que se ocupan de generar eventos:

  • event_herencia
  • event_perform
  • event_perform_object
  • event_user

También hay una serie de variables de solo lectura que se pueden utilizar con las funciones enumeradas anteriormente que son particularmente útiles para la depuración:

  • event_action
  • numero de evento
  • event_object
  • tipo de evento
