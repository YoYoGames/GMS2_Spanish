gpu_set_blendmode_ext_sepalpha


Descripción

Cuando GameMaker Studio 2 va a dibujar un píxel, hay un color de fuente (el color del píxel que vamos a dibujar) y un color de destino (el color que ya está en el píxel al que estamos dibujando), así que al determinar el nuevo color del píxel, los colores de origen y destino se calculan según el modo de fusión elegido. Cada componente de los colores se almacena como un valor de punto flotante entre 0 y 1, y el nuevo color se calcula multiplicando cada componente del color de origen por algún factor y multiplicando cada componente del color de destino por algún otro factor y luego agregando el resultados juntos componente por componente.

Esta función le permite configurar los diferentes componentes que se deben combinar para crear un modo de combinación personalizado, seleccionando diferentes partes de diferentes factores de modo de mezcla para los componentes RGB y el componente Alpha. Entonces el argumento "src" y "dest" usará los primeros 3 valores del modo de mezcla (RGB) y los argumentos "alphasrc" y "alphadest" tomarán la última parte del factor de modo de mezcla. Por ejemplo, si tiene la fuente RGB como bm_zero, entonces (0, 0, 0) se usará. Si luego configura la fuente Alpha como bm_one, se usará (1), dando un modo de mezcla fuente final de (0, 0, 0, 1).

El siguiente cuadro muestra los valores de RGBA para cada modo de fusión, donde (Rs, Gs, Bs, As) se consideran valores fuente y (Rd, Gd, Bd, Ad) son los valores de destino. Las once constantes que están disponibles para su uso pueden ser de origen o destino (o ambas) cuando se utilizan en esta función.

Constante Factor de mezcla (rojo, verde, azul, alfa)
bm_zero (0, 0, 0, 0)
bm_one (1, 1, 1, 1)
bm_src_colour (Rs, Gs, Bs, As)
bm_inv_src_colour (1-Rs, 1-Gs, 1-Bs, 1-As)
bm_src_alpha (As, As, As, As)
bm_inv_src_alpha (1-As, 1-As, 1-As, 1-As)
bm_dest_alpha (Anuncio, Anuncio, Anuncio, Anuncio)
bm_inv_dest_alpha (1-Ad, 1-Ad, 1-Ad, 1-Ad)
bm_dest_colour (Rd, Gd, Bd, Ad)
bm_inv_dest_colour (1-Rd, 1-Gd, 1-Bd, 1-Ad)
bm_src_alpha_sat (f, f, f, 1) donde f = min (As, 1-Ad)

Tenga en cuenta que puede suministrar cuatro argumentos individuales a esta función o puede suministrar una matriz de argumentos (tal como lo devuelve la función). gpu_get_blendmode_ext_sepalpha por ejemplo). Si se suministra una matriz, debe tener los siguientes cuatro elementos:

  • [0] = modo de fusión de fuente menos el componente alfa (el valor predeterminado es bm_src_alpha (As, As, As))
  • [1] = Modo de mezcla de destino menos el componente alfa (el valor predeterminado es bm_inv_src_alpha (1-As, 1-As, 1-As))
  • [2] = componente alfa fuente (por defecto es bm_src_alpha (Como))
  • [3] = componente alfa de destino (el valor predeterminado es bm_inv_src_alpha (1-As))
¡ADVERTENCIA! Esta función actualmente solo está disponible en la plataforma de destino de Windows.


Sintaxis:

gpu_set_blendmode(src, dest, alphasrc, alphadest);


Argumentos:

Argumento Descripción
src Factor de modo de mezcla fuente para RGB (ver constantes arriba).
dest Factor de modo de mezcla de destino para RGB (ver constantes arriba)
alphasrc Factor de modo de mezcla de fuente Alfa (ver constantes arriba).
alphadest Factor de modo de mezcla de destino Alfa (ver constantes arriba)


Devoluciones:

N/A


Ejemplo:

var bm = gpu_get_blendmode_ext_sepalpha();
bm[0] = bm_src_alpha;
bm[1] = bm_inv_src_alpha;
gpu_set_blendmode_ext_sepalpha(bm);

El código anterior crea una matriz local y obtiene el modo de mezcla ampliada actual con valores alfa separados. Este conjunto se manipula y devuelve para establecer el modo de mezcla.
