gpu_set_zwriteenable


Descripción

Esta función se puede utilizar para activar ( true ) y fuera ( false ) Ordenamiento en profundidad utilizando el z-buffer. Cuando una instancia se representa en GameMaker Studio 2, la profundidad de un píxel generado (su coordenada z) se almacena en el búfer z, luego, si otra instancia de la escena debe representarse en el mismo píxel, se comparan las dos profundidades. (prueba de z - ver gpu_set_ztestenable ) y el "más cercano" al observador se guarda en el búfer z, reemplazando al anterior. Sin embargo, esto no siempre es lo que desea que suceda y hay momentos en los que puede querer que se dibuje algo sobre todo lo demás, sin importar cuál sea su valor z. En esos casos, puede usar esta función para desactivar la escritura en el búfer z y así forzar que todo lo que se está dibujando se dibuje sobre todo lo demás. Tenga en cuenta que la descripción que se proporciona aquí es para el modo de comparación de prueba de z predeterminado, pero se puede cambiar

El valor predeterminado es que z-buffering está habilitado ( true ).


Sintaxis:

gpu_set_zwriteenable(enable);


Argumentos:

Argumento Descripción
enable El modo de comparación a usar (ver lista arriba)


Devoluciones:

N/A


Ejemplo:

gpu_set_zwriteenable(false);
with (obj_Menu_Containers)
   {
   draw_self();
   }
gpu_set_zwriteenable(true);

El código anterior desactiva la escritura de z-buffer, dibuja una cantidad de objetos y luego lo vuelve a encender para continuar dibujando.
