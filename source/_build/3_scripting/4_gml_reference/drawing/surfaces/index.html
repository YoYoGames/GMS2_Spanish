Superficies


En los eventos de sorteo normales, GameMaker Studio 2 realidad no dibuja directamente en la pantalla, sino que se dibuja en una superficie llamada superficie de la aplicación. Esta superficie es básicamente un "lienzo" en blanco que luego puede ser manipulado antes de ser dibujado a la pantalla cuando sea necesario, y en la mayoría de los casos GameMaker Studio 2 maneja esto por usted (aunque también puede manipularlo en código para sombreadores, escalado y muchas otras cosas - más detalles se dan a continuación).

Sin embargo, aparte de esta superficie de aplicación, también puedes crear tus propias superficies y usarlas para crear efectos especiales sorprendentes o sutiles en tu juego. Por ejemplo, puede usar superficies para "atrapar" instancias, que luego pueden destruirse, y de esta manera puede crear un efecto de calcomanía donde el elemento de la instancia se muestra en la superficie como si aún existiera, permitiendo efectos gráficos como escombros, sangre, etc. sin ningún gasto de procesamiento real. Otra cosa que puede hacer con las superficies es usarlas como texturas para ser manipuladas, o para crear sprites "sobre la marcha", o para crear superposiciones complejas. de hecho, ¡los usos de las superficies son infinitos!

Las superficies normales son bastante fáciles de usar, pero hay algunas reglas básicas que deben seguirse cuando las usa:

  • En primer lugar, debe tener en cuenta que las superficies ( excepto la superficie de aplicación) son "volátiles". Esto significa que si el dispositivo o la ventana pierde el foco o se minimiza (buenos ejemplos son cuando aparece un protector de pantalla en Windows, o en un dispositivo Android cuando la aplicación pierde el foco debido a una llamada), entonces la superficie puede ser destruida. Esto se debe a que está almacenado en la memoria de textura y puede sobrescribirse cuando la plataforma objetivo necesita esa memoria para otra cosa, lo que significa que siempre debe tener algún tipo de código de seguridad en su lugar, generalmente con el surface_exists función.

  • En segundo lugar, debe tener en cuenta que las superficies requieren grandes cantidades de memoria de textura para ser utilizadas, por lo que debe intentar mantenerlas lo más pequeñas posible. Normalmente, intentará mantenerlos no más grandes que el tamaño de la vista o la ventana de visualización.

  • Tercero, solo deberías crear superficies en el evento de sorteo. Si crea una superficie en el evento Create de una instancia, podría obtener el mismo índice que el application_surface. Esto puede causar muchos problemas y confusión al pensar que estás utilizando tu propia superficie, pero en realidad estás usando el objetivo de renderización actual. También debe intentar limitar el dibujo a una superficie en el evento de dibujo también, ya que debido a la forma optimizada en que GameMaker Studio 2 dibuja en la pantalla, se recomienda que mantenga todas las funciones de dibujo dentro del evento de dibujo, esto incluye borrar una superficie cuando se crea por primera vez, etc. Es posible dibujar en una superficie fuera del evento de atracción e incluso puede ser necesario para algunos efectos, pero no es así como se debe hacer.

  • En cuarto lugar, al dibujar a una superficie manualmente, la superficie siempre está en la posición de (0,0). Esto significa que puede necesitar convertir coordenadas absolutas en coordenadas locales para la superficie. Por ejemplo, si tiene una superficie del tamaño de una vista y desea dibujar algo que esté actualmente a la vista de esa superficie, debe restar las coordenadas xey de la vista de la cámara de las coordenadas xey reales para obtener una posición relativa de la superficie (0,0) posición. Entonces, el código se vería así:

    if view_current = 0
       {
       surface_set_target(surf);
       with (obj_Effect)
          {
          var _vx = camera_get_view_x(view_camera[1]);
          var _vy = camera_get_view_y(view_camera[1]);
          draw_sprite(sprite_index, image_index, x - _vx, y - _vy);
          }
       surface_reset_target();
       }
    else
       {
       draw_surface(surf, 0, 0);
       }

El uso básico de una superficie sería el siguiente: primero crea una superficie y asigna su índice a una variable. Luego, establecería el objetivo del dibujo en la superficie en lugar de la pantalla y dibujaría las cosas que desea, además de realizar otras manipulaciones. Una vez que haya terminado, restablecerá el objetivo del dibujo para que todos los dibujos posteriores vuelvan a aparecer en la pantalla. Una cosa a tener en cuenta es que si necesita dibujar toda la pantalla en una superficie (incluidos mosaicos, fondos, etc.) puede acceder simplemente a la superficie de la aplicación (ver más detalles a continuación) o asignar una superficie a una vista puerto usando la variable view_surface_id[0..7] como con todo esto, todo lo visible en esa vista se dibujará en la superficie correspondiente.

Existen las siguientes funciones para tratar superficies (estas funciones son específicas para crear y manipular superficies, pero para dibujarlas en la pantalla debe usar las funciones de dibujo específicas que se pueden encontrar en la sección sobre Dibujo de superficie, a continuación):

  • surface_copy
  • surface_copy_part
  • surface_get_height
  • surface_get_width
  • surface_get_texture
  • surface_getpixel
  • surface_getpixel_ext
  • surface_free
  • surface_save
  • surface_save_part

Como se mencionó anteriormente, todo el dibujo normal en su juego se realiza en la superficie de la aplicación, y esta superficie se puede cambiar y manipular como una superficie normal. Sin embargo, no es una superficie normal y usuarios avanzados encontrarán que permite un control completo sobre cómo y cuando las cosas se dibujan en GameMaker Estudio 2. Para obtener más detalles, así como funciones específicas para esta función, consulte la siguiente sección:

  1. La superficie de aplicación

Las siguientes funciones y variables existen para hacer referencia a la superficie de la aplicación:

  • application_surface

Y también tiene algunas funciones especiales que están diseñadas solo para su uso con la superficie de aplicación:

  • application_surface_enable
  • application_surface_is_enabled
  • application_get_position
  • application_surface_draw_enable

Las superficies son una herramienta gráfica increíblemente poderosa que GameMaker Studio 2 te ofrece para crear efectos especiales y una gran cantidad de otras cosas. Son relativamente simples de utilizar, ya que son básicamente un lienzo que dibuja en lugar de la pantalla, y este lienzo puede manipularse y modificarse antes de dibujarse con las funciones de esta sección.

NOTA: Al trabajar con superficies existe la posibilidad de que dejen de existir en cualquier momento debido a que se almacenan en la memoria de textura. SIEMPRE debe verificar que exista una superficie usando surface_exists antes de hacer referencia a ellos directamente.

Las siguientes funciones existen para dibujar superficies:

  1. draw_surface
  2. draw_surface_ext
  3. draw_surface_part
  4. draw_surface_part_ext
  5. draw_surface_stretched
  6. draw_surface_stretched_ext
  7. draw_surface_tiled
  8. draw_surface_tiled_ext
  9. draw_surface_general
