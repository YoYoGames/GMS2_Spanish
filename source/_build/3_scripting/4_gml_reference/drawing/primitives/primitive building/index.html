Edificio primitivo


GameMaker Studio 2 permite definir sus propias primitivas personalizadas siguiendo las especificaciones del formato de vértice que ha elegido (vea Formatos de vértice ). Estos pueden ser utilizados en cualquier sombreador que haya creado.

Las primitivas que construyas se mantienen en un búfer de vértices. Esto se debe crear de antemano y luego se debe hacer referencia a las funciones que se utilizan para construir su primitiva. El búfer de vértices se puede reutilizar tantas veces como sea necesario para crear diferentes primitivas, o se puede "congelar" para mantener un tipo primitivo específico durante el juego o nivel (que es el enfoque más rápido, por lo que si sabes que un primitivo que construyas no cambiará, entonces siempre deberías usar esta opción).

Las siguientes funciones están disponibles para el buffer de vértices:

  • vertex_create_buffer
  • vertex_create_buffer_ext
  • vertex_create_buffer_from_buffer
  • vertex_create_buffer_from_buffer_ext
  • vertex_get_buffer_size
  • vertex_get_number
  • vertex_delete_buffer

Las primitivas que construya deben seguir el formato que haya establecido utilizando las funciones de Formatos vértice, por lo que si ha definido un formato de vértice con solo datos posicionales, no tiene sentido crear su primitiva con datos de color. Debe tener en cuenta que el orden en que agrega propiedades a la primitiva que está construyendo se define por el orden en el que agregó estas propiedades al crear el formato de vértice, por lo que si ha definido el formato vectorial con la posición, color y textura de la orden coordinar, debe agregar estas propiedades a la primitiva que está siendo construida en el mismo orden; de lo contrario, obtendrá errores.

Un ejemplo de una primitiva de triángulo simple que se está construyendo se muestra en el siguiente código:

v_buff = vertex_create_buffer();
vertex_begin(v_buff, global.my_format);
vertex_position(v_buff, 10, 10);
vertex_colour(v_buff, c_white, 1);
vertex_texcoord(v_buff, 0, 0);
vertex_position(v_buff, 110, 10);
vertex_colour(v_buff, c_white, 1);
vertex_texcoord(v_buff, 1, 0);
vertex_position(v_buff, 110, 110);
vertex_colour(v_buff, c_white, 1);
vertex_texcoord(v_buff, 1, 1);
vertex_end(v_buff);
var tex = background_get_texture(background1);
shader_set(shader_prim);
vertex_submit(v_buff, pr_trianglelist, tex);
shader_reset();

Las primitivas pueden ser puntos, líneas o tiras listas o listas de triángulo o tiras, pero no se le permite ventiladores triángulo ya que la mayoría de hardware móvil no aceptará ese tipo primitivo. Las constantes utilizadas para definir estos diferentes tipos primitivos se pueden encontrar aquí.

NOTA: Puede construir la primitiva y almacenarla en el búfer de vértices en cualquier paso de su juego; sin embargo, para dibujarla, debe enviarla durante el Evento de sorteo.

Las siguientes funciones están disponibles para construir primitivas:

  • vertex_begin
  • vertex_colour
  • vertex_normal
  • vertex_position
  • vertex_position_3d
  • vertex_argb
  • vertex_texcoord
  • vertex_float1
  • vertex_float2
  • vertex_float3
  • vertex_float4
  • vertex_ubyte4
  • vertex_end
  • vertex_freeze
  • vertex_submit
