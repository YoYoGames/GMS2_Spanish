Sistemas de partículas

Antes de que puedas crear partículas, necesitas crear un sistema de partículas. Este es un "contenedor" donde coloca sus partículas y emisores (si los necesita) listos para usar, y puede colocar tantas o tan pocas partículas en cualquier sistema como crea necesario, y puede tener tantos sistemas como piensas que es necesario también Sin embargo, siempre es mejor mantener este número lo más pequeño posible debido al hecho de que cada sistema, emisor y partícula consume memoria y tener demasiados puede ralentizar su juego o causar problemas. Por ejemplo, si necesita que aparezcan algunos efectos encima de las instancias y que aparezcan otros efectos debajo de las instancias, debe crear dos sistemas y establecer sus profundidades para obtener el efecto deseado, con todas las partículas que se agregan a cada sistema que se dibuja en la profundidad tu específicas.

Como un sistema de partículas es un recurso creado dinámicamente, debe crearlo y almacenar el índice devuelto en una variable para hacer referencia al sistema en todas las demás llamadas a funciones, y es muy importante que también destruya el sistema cuando ya no lo necesite o no lo necesite. de lo contrario, tendrás una pérdida de memoria que se ralentizará y, finalmente, bloqueará tu juego. También vale la pena señalar que los sistemas de partículas vivirán para siempre después de que se crean, incluso si el índice ya no se almacena. Por lo tanto, incluso si cambia de habitación o reinicia el juego, los sistemas y las partículas permanecerán visibles en todas las habitaciones, por lo que es mejor que los destruya una vez que ya no los necesite.

Las funciones básicas para configurar un sistema de partículas se enumeran a continuación:

  • part_system_exists
  • part_system_create
  • part_system_create_layer
  • part_system_get_layer
  • part_system_layer
  • part_system_clear
  • part_system_depth
  • part_system_position
  • part_system_destroy
  • part_particles_clear
  • part_particles_count

Una vez que las partículas se agregan a un sistema de partículas y luego se rompen o se transmiten a la habitación, normalmente se actualizan automáticamente en cada paso y se dibujan según los parámetros que haya utilizado para definirlas. Sin embargo, a veces puede ser necesario controlar cuándo y cómo se actualiza el sistema, así como cómo se debe dibujar el sistema, y ​​para eso GameMaker Studio 2 ofrece las siguientes funciones:

  • part_system_automatic_update
  • part_system_automatic_draw
  • part_system_update
  • part_system_drawit
  • part_system_draw_order

Normalmente, utilizaría emisores de partículas para reventar o transmitir partículas desde un área, pero en muchos casos no son necesarias y, en realidad, es mejor crear las partículas directamente con las siguientes funciones:

  • part_particles_create
  • part_particles_create_colour
