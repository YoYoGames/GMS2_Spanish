part_type_sprite

Establece la forma de las partículas creadas en un sprite.

Sintaxis:

part_type_sprite(ind, sprite, animate, stretch, random);


Argumento Descripción
ind El índice del tipo de partícula para cambiar.
sprite El índice del sprite para establecer el tipo de partícula.
animate Si se debe seguir la animación del sprite normalmente (verdadero) o no (falso).
stretch Ya sea para estirar la animación del sprite para que coincida con la vida útil de la partícula (verdadera) o no (falsa).
random Si se elige una subimagen aleatoria (verdadera) o no (falsa).


Devoluciones: N / A


Descripción

Esta función se puede usar para establecer un tipo de partícula para usar un sprite personalizado de los recursos del juego. Puede seleccionar el sprite y luego decirle a GameMaker Studio 2 que use el sprite y animarlo o no, y si elige animado, también puede optar por cambiar las subimágenes en un intervalo tal que la animación comenzará y finalizará al principio y al final de la vida útil de cada partícula creada (de modo que si tiene un tiempo de vida fijado en un mínimo de 15 pasos y un máximo de 30 pasos, no importa qué valor tenga la vida útil de la animación, la ejecución será más rápida) para las vidas más cortas y más lento para el más largo). También puede seleccionar elegir una subimagen aleatoria para que, si el sprite no está animado, se elija una imagen diferente al azar para crear el sprite, o si la partícula está animada, comenzará la animación en un punto aleatorio.


Ejemplo:

global.p1 = part_type_create();
part_type_sprite(global.p1 , spr_Coins, 1, 0, 0);
part_type_size(global.p1, 1, 3, 0, 0);
part_type_scale(global.p1, 1, 1);
part_type_colour1(global.p1, c_white);
part_type_alpha2(global.p1, 1, 0);
part_type_speed(global.p1, 2, 4, 0, 0);
part_type_direction(global.p1, 0, 180, 0, 0);
part_type_gravity(global.p1, 0.20, 270);
part_type_orientation(global.p1, 0, 0, 0, 0, 1);
part_type_blend(global.p1, 1);
part_type_life(global.p1, 15, 60);

El código anterior establece el tipo de partícula indexado en la variable global "p1" a la forma del sprite indexado en "spr_Coins", y luego establece varias otras propiedades de partícula.
