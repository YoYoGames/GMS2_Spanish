part_type_orientation

Establece el ángulo del tipo de partícula.


Sintaxis:

part_type_orientation( ind, ang_min, ang_max, ang_incr, ang_wiggle, ang_relative );


Argumento Descripción
ind El índice del tipo de partícula para cambiar.
ang_min El ángulo inicial mínimo de las partículas.
ang_max El ángulo de inicio máximo de las partículas.
ang_incr El aumento (o disminución si se establece negativo) del ángulo en grados por paso.
ang_wiggle La cantidad que el ángulo se mueve en sentido horario / antihorario por paso.
ang_relative Si establecer su ángulo relativo a la dirección del movimiento de la partícula (verdadero) o no (falso).


Devoluciones: N / A


Descripción

Esta función se usa para determinar la orientación del sprite de la partícula cuando se crea y también se puede usar para hacer que la orientación de la partícula aumente o disminuya a lo largo de su vida útil. Los valores de orientación mínimo y máximo están predeterminados en 0, pero estos valores pueden cambiarse para aleatorizar la orientación siguiendo las instrucciones estándar de GameMaker Studio 2 de 0 grados, 90 grados para arriba, 180 grados para la izquierda y 270 grados para abajo. Si los configura con el mismo valor, todas las partículas se crearán con la misma orientación.

También puede establecer un valor de incremento que agregará (si es un número positivo) o restará (si es un número negativo) una cantidad de grados a la orientación durante su vida útil. Este valor puede ser un mínimo de (+/-) 0.01.

También puede establecer el factor "meneo", al igual que otras funciones de partículas. Este es un valor que se agregará o restará aleatoriamente a la orientación en cada paso de la vida de las partículas. Obviamente, los valores más grandes son más pronunciados que los más pequeños, y este valor puede ser incluso negativo, con un rango máximo entre -20 y 20.

Finalmente, puedes elegir tener la orientación relativa o no, lo que significa que mientras la partícula tiene una dirección (y velocidad), el sprite de la partícula estará orientado alrededor de ese vector. Tenga en cuenta que si tiene la velocidad de partícula establecida para reducir y llega a 0, la falta de velocidad establece la dirección en el valor predeterminado de 0 °, por lo que una orientación relativa hará que el sprite de la partícula "salte" a un ángulo diferente.


Ejemplo:

part_type_shape(particle2, pt_shape_spark);
part_type_size(particle2, 0.10, 0.50, 0.01, 0);
part_type_scale(particle2, 0.30, 0.30);
part_type_colour1(particle2, 8454143);
part_type_alpha1(particle2, 0.50);
part_type_speed(particle2, 4, 4, -0.07, 1);
part_type_direction(particle2, 0, 359, 0, 20);
part_type_orientation(particle2, 0, 359, 0, 20, 1);
part_type_blend(particle2, 1);
part_type_life(particle2, 1, 5);

El código anterior establecerá varios valores de partículas, incluida la orientación, que será un valor aleatorio entre 0 ½ y 359 ½. También tendrá una cantidad aleatoria añadida de 0 a 20 en cada paso, y la orientación es relativa a la dirección del movimiento.
