Dibujo de Sprites y Azulejos

Aunque puede configurar fondos en el editor de sala y puede asignar sprites a instancias, a menudo esto no es lo suficientemente flexible para el juego que está haciendo. Por ejemplo, si tiene una instancia que tiene que dibujar un menú, es posible que desee mostrar varios botones sprites a la vez, o si tiene un juego con desplazamiento de fondos de paralaje tendrá que ser capaz de configurarlos a través del código en una instancia de controlador de algún tipo. En estas situaciones, deberías utilizar las siguientes funciones que pueden dibujar sprites y activos de fondo en cualquier lugar de la habitación, con algunas de las funciones que también te permiten manipular la forma, el tamaño, el color y el alfa del sprite para lograr efectos especiales.

Puede asignar un sprite a una instancia y, de forma predeterminada, GameMaker Studio 2 lo dibujará con las propiedades dadas por las diversas variables de sprite ; sin embargo, si agrega algo al evento de dibujar, está "tomando el control" y debe decirle exactamente a GameMaker Studio 2 que dibujar Esto significa que si debe usar las siguientes funciones para definir exactamente qué es lo que la instancia debe dibujar y en qué parte de la sala está dibujarlo:

  • dibujar
  • draw_sprite
  • draw_sprite_ext
  • draw_sprite_general
  • draw_sprite_part
  • draw_sprite_part_ext
  • draw_sprite_stretched
  • draw_sprite_stretched_ext
  • draw_sprite_pos
  • draw_sprite_tiled
  • draw_sprite_tiled_ext

Si está utilizando sprites de formato SWF, también hay opciones adicionales para controlar el anti-aliasing utilizado al dibujar (consulte Importar sprites que no sean de mapa de bits para obtener más información sobre estos tipos de sprites). Tenga en cuenta que estas funciones funcionan a nivel global y configurarán el AA en todos los sprites vector SWF dibujados después de haber sido llamados.

  • draw_enable_swf_aa
  • draw_set_swf_aa_level
  • draw_get_swf_aa_level

Además de las funciones enumeradas anteriormente, también hay dos funciones especiales diseñadas para usar con animaciones esqueléticas (consulte Importar Sprites que no sean Bitmap para obtener más información sobre estos tipos de sprites y la sección Animaciones esqueléticas para funciones específicas relacionadas con estos tipos de sprites):

  • draw_skeleton
  • draw_skeleton_instance
  • draw_skeleton_collision
  • draw_skeleton_time

Los mosaicos se dibujan automáticamente en función de los datos que contienen, el conjunto de mosaicos utilizado y las propiedades del mapa de mosaicos. Sin embargo, puede dibujar fichas individuales o mosaicos con las siguientes funciones:

  • draw_tile
  • draw_tile_ext
  • tilemap_draw
