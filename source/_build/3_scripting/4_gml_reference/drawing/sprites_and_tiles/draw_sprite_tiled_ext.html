draw_sprite_tiled_ext

Dibuja un sprite en una posición dada con escala, fusión y alfa, y lo azuleja para llenar toda la vista.

Sintaxis:

draw_sprite_tiled_ext(sprite, subimg, x, y, xscale, yscale, colour, alpha);


Argumento Descripción
sprite El índice del sprite para dibujar.
subimg El subimg (frame) del sprite para dibujar (image_index o -1 se correlaciona con el cuadro actual de animación en el objeto).
x La coordenada x de dónde dibujar el sprite.
y La coordenada y de dónde dibujar el sprite.
xscale La escala horizontal del sprite. Un multiplicador '1 = escalamiento normal, 0.5 es mitad, etc.
yscale La escala vertical del sprite. Un multiplicador '1 = escalamiento normal, 0.5 es mitad, etc.
colour El color con el que mezclar el sprite. c_white es para mostrarlo normalmente.
alpha El alfa del sprite (de 0 a 1 donde 0 es transparente y 1 opaco).


Devoluciones: N / A


Descripción

Esta función tomará un sprite y luego lo insertará repetidamente en toda la vista (o en la sala si no se define ninguna vista), empezando por las coordenadas que usted da en la función y con cada azulejo a escala, color mezclado y con el alfa que defina (estas propiedades son las mismas que las utilizadas en draw_sprite_ext )

NOTA: La combinación de colores solo se recomienda para el destino HTML5 cuando WebGL está habilitado, aunque aún puede establecer el color de fusión si no está habilitado y combinará el sprite de la forma habitual. Sin embargo, todas las mezclas de esta manera crean un elemento gráfico duplicado que luego se almacena en la memoria caché y se utiliza cuando es necesario. Esto está lejos de ser óptimo y si usa múltiples cambios de color, reducirá el rendimiento de su juego a menos que active WebGL. si no desea usar WebGL, puede configurar el tamaño de la memoria caché de fuentes para intentar limitar esto si fuera necesario utilizando la función sprite_set_cache_size.


Ejemplo:

draw_sprite_tiled_ext(sprite_index, image_index, x, y, 2, 2, c_red, 0.5);

Esto dibujará las instancias asignadas sprite (sprite_index) y su marco de animación actual (image_index) en las instancias propias de la posición xey, pero a escala dos veces el tamaño normal, mezclado rojo y con la mitad del alfa normal. El sprite se colocará en mosaico horizontal y verticalmente a lo largo de la vista.
