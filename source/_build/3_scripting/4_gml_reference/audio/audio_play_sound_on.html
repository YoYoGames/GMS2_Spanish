audio_play_sound_on


Descripción

Con esta función puede reproducir cualquier recurso de sonido a través de un emisor, con cualquier cambio en la ganancia, posición, tono o velocidad del emisor que afecte a la forma en que el usuario escucha el sonido final que se está reproduciendo. Proporciona el índice de emisor que se utilizará, el índice de sonido del sonido que se reproducirá y luego especifique si el sonido es en bucle o no. Finalmente, puede asignar una prioridad, que luego se usa para determinar cómo se tratan los sonidos cuando el número de sonidos que se tocan supera el límite establecido por la función. audio_channel_num. Los sonidos de menor prioridad se detendrán en favor de sonidos de mayor prioridad, y el valor de prioridad puede ser cualquier número real (el valor real es arbitrario, y puede ser de 0 a 1 o de 0 a 100, ya que GameMaker Studio 2 los priorizará de la misma manera )

Esta función también devolverá un número de índice único para el sonido que se está reproduciendo, que luego se puede almacenar en una variable para que luego pueda pausarlo o detenerlo con las funciones apropiadas. Esto significa que si tiene varias instancias del mismo sonido reproduciéndose en cualquier momento, puede enfocar solo una instancia de ese sonido para manejar las funciones de audio.


Sintaxis:

audio_play_sound_on(emitter, sound, loop, priority);


Argumento Descripción
emitter El índice del emisor a usar.
sound El índice del sonido a usar.
loop Marca el sonido como bucle o no.
priority Establezca la prioridad del canal para el sonido.


Devoluciones:

Index


Ejemplo:

if global.SFX
   {
   audio_play_sound_on(s_emit, snd_Explode, false, 1);
   }

El código anterior verifica la variable global "SFX" y si regresa true entonces el sonido indexado en la variable "snd_Explode" se reproducirá a través del emisor indexado en la variable "s_emit" con baja prioridad y sin bucle.
