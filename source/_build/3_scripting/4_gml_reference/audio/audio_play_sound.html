audio_play_sound


Descripción

Con esta función puedes jugar cualquier recurso de sonido en tu juego. Proporciona el índice de sonido y le asigna una prioridad, que luego se usa para determinar cómo se tratan los sonidos cuando el número de sonidos que se tocan supera el límite establecido por la función. audio_channel_num. Los sonidos de menor prioridad se detendrán en favor de sonidos de mayor prioridad, y el valor de prioridad puede ser cualquier número real (el valor real es arbitrario, y puede ser de 0 a 1 o de 0 a 100, ya que GameMaker Studio 2 los priorizará de la misma manera ) El argumento final es para hacer el loop de sonido y configurarlo para true hará que el bucle de sonido hasta que se detenga y lo ajuste a false reproducirá el sonido una sola vez.

Esta función también devolverá un número de índice único para el sonido que se está reproduciendo, que luego se puede almacenar en una variable para que luego pueda pausarlo o detenerlo con las funciones apropiadas. Esto significa que si tiene varias instancias del mismo sonido reproduciéndose en cualquier momento, puede enfocar solo una instancia de ese sonido para manejar las funciones de audio.


Reproduce el sonido indicado.

Sintaxis:

audio_play_sound(index, priority, loop);


Argumento Descripción
index El índice del sonido para jugar.
priority Establezca la prioridad del canal para el sonido.
loop Establece el sonido en bucle o no.


Devoluciones:

Index


Ejemplo:

if health <= 0
   {
   lives -= 1;
   audio_play_sound(snd_PlayerDead, 10, false);
   }

El código anterior verifica la variable global "health" y si es menor o igual que 0, eliminará 1 de la variable global "lives" y reproducirá un sonido.
