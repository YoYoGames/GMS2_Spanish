audio_create_play_queue


Descripción

Esta función prepara una cola de buffer para audio. Pasa el formato de datos (solo buffer_u8 o buffer_s16 actualmente son compatibles), la frecuencia de muestreo (que puede ser de entre 1000 y 48000 Hz) y los canales que el audio requiere de una de las constantes que se enumeran a continuación:

Constante Descripción
audio_mono Audio mono (canal único).
audio_stereo Audio estéreo (doble canal).
audio_3D Audio 3D (5.1).


Tenga en cuenta que debe liberar las colas de audio creadas con esta función cuando ya no sea necesario para evitar pérdidas de memoria con la función audio_free_play_queue().

Esta función activará un evento asíncrono de reproducción de audio cuando la cola haya llegado al final, y en este caso un evento especial ds_map se creará en la variable async_load con los siguientes pares clave / valor:

  • "queue_id": la cola que finalizó la reproducción de datos
  • "buffer_id": el búfer que ya no se reproduce desde
  • "queue_shutdown": se establece en 0 durante la reproducción normal y 1 cuando se recibe el evento porque audio_free_play_queue ha sido llamado. Cuando se establece en 1, no desea poner en cola ningún dato adicional.
¡IMPORTANTE! Esta función no está disponible con la licencia de prueba del producto.


Sintaxis:

audio_create_play_queue(queueFormat, queueRate, queueChannels);


Argumento Descripción
queueFormat El formato de los datos del buffer a usar ( buffer_u8 o buffer_s16 )
queueRate La frecuencia de muestreo de los datos en la cola del buffer.
queueChannels Los canales a usar de una de las constantes enumeradas a continuación.


Devoluciones:

Index


Ejemplo:

audio_queue = audio_create_play_queue(buffer_s16, 11052, audio_mono);

El código anterior crea una nueva cola de audio y asigna su índice a una variable para referencia futura.
