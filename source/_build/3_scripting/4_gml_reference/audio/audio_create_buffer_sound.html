audio_create_buffer_sound


Descripción

Con esta función, puede crear un nuevo sonido a partir del contenido de un búfer. El búfer se habrá creado previamente (consulte las funciones del búfer para obtener detalles sobre cómo hacerlo), y se han agregado o cargado datos en él. A continuación, transfiéralo a esta función con el formato de datos (solo buffer_u8 o buffer_s16 actualmente son compatibles), la frecuencia de muestreo (que puede ser de entre 1000 y 48000 Hz) y una compensación en el búfer para obtener los datos. También debe proporcionar el número de muestras en el búfer y los canales que requiere el sonido. Estos canales están definidos por una de las siguientes constantes:

Constante Descripción
audio_mono Audio mono (canal único).
audio_stereo Audio estéreo (doble canal).
audio_3D Audio 3D (5.1).


Tenga en cuenta que después de haber creado un sonido, debe liberar el índice del puntero asociado cuando ya no sea necesario utilizar la función audio_free_buffer_sound(). Si no puede hacer esto y luego reasignar la variable o cambiar de habitación, etc., la identificación del sonido se perderá y tendrá una pérdida de memoria. También tenga en cuenta que no puede eliminar el búfer si se ha creado un sonido y el sonido no se ha liberado primero. Entonces, liberarías el sonido (o los sonidos) primero y luego eliminarías el búfer. También vale la pena señalar que agregar algo al búfer, o cambiar el tamaño del búfer, después de haber tenido un sonido creado a partir de él dará resultados inesperados y no es recomendable: una vez que haya comenzado a crear sonidos desde cualquier búfer, no debe manipular de cualquier otra manera después.

¡IMPORTANTE! Esta función no está disponible con la licencia de prueba del producto.


Sintaxis:

audio_create_buffer_sound(bufferId, bufferFormat, bufferRate, bufferOffset, bufferLength, bufferChannels);


Argumento Descripción
bufferId La ID del buffer a usar.
bufferFormat El formato de los datos en el búfer ( buffer_u8 o buffer_s16 )
bufferRate La frecuencia de muestreo de los datos en el buffer.
bufferOffset El desplazamiento en el búfer para leer los datos de muestra desde (en bytes).
bufferLength La longitud del búfer (el número de muestras en el búfer).
bufferChannels Los canales a usar de una de las constantes enumeradas a continuación.


Devoluciones:

Index


Ejemplo:

rate = 44100;
hertz = irandom_range(220, 880);
samples = 44100;
bufferId = buffer_create(rate, buffer_fast, 1);
buffer_seek(bufferId, buffer_seek_start, 0);
num_to_write = rate / hertz;
val_to_write = 1;
for (var i = 0; i < (samples / num_to_write) + 1; i++;)
   {
   for (var j = 0; j < num_to_write; j++;)
      {
      buffer_write(bufferId, buffer_u8, val_to_write * 255);
      }
   val_to_write = (1 - val_to_write);
   }
soundId = audio_create_buffer_sound(bufferId, buffer_u8, rate, 0, 44100, audio_mono);

Lo anterior crea un búfer y luego llena el proceso con datos. Estos datos se utilizan para crear un nuevo sonido, que se almacena en la variable "ID de sonido".
