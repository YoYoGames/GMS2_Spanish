audio_get_recorder_info


Descripción

Esta función devolverá un ds_map con información sobre el índice fuente del grabador dado. Puede averiguar cuántas fuentes de grabadora están disponibles utilizando la función audio_get_recorder_count, y el mapa devuelto contendrá los siguientes pares clave / valor:

  • "nombre" - un nombre para describir el dispositivo
  • "índice" - el índice que se utilizará para registrar
  • "data_format" - los datos de formato serán devueltos (esto es actualmente buffer_s16 pero otros formatos pueden ser compatibles en el futuro)
  • "sample_rate": la frecuencia de muestreo (en Hz) de los datos devueltos (actualmente sujeta a 16000hz pero esto puede cambiar en el futuro)
  • "canales" - la constante audio_mono (Más constantes para estéreo y 3D pueden ser compatibles en el futuro)

Tenga en cuenta que, si bien la función crea un ds_map para usted, no lo eliminará más tarde, por lo que debería destruir el mapa usted mismo cuando ya no lo necesite para evitar fugas de memoria.

NOTA: la mayoría de las plataformas admiten la grabación de audio de alguna forma, pero eso no significa que todos los dispositivos lo permitan, incluso si la plataforma lo hace, por lo que siempre debe verificar que el audio_get_recorder_count() la función devuelve un valor mayor que 0 para verificar que los dispositivos de grabación estén disponibles antes de usar el resto de las funciones de grabación.


Sintaxis:

audio_get_recorder_info(recorder_index);


Argumento Descripción
recorder_index El índice de la fuente del grabador para obtener la información.


Devoluciones:

ds_map


Ejemplo:

len = async_load[? "data_len"];
audio_buff = buffer_create(len, buffer_fast, 1);
buffer_copy(async_load[? "buffer_id"], 0, len, buff, 0);
audio_queue_sound(audio_queue, audio_buff, 0, len);
audio_play_sound(audio_queue, 10, 0);

El código anterior se invocará en el evento de grabación de audio asíncrona y asignará algunos datos grabados a un búfer, que luego se agregará a una cola de audio. Esto luego se juega.
