audio_play_sound_at


Descripción

Con esta función, puede reproducir cualquier recurso de sonido en una posición determinada dentro del espacio de audio. Usted proporciona el índice de sonido y luego le asigna una posición dentro del espacio 3D. La posición del oyente predeterminado es (0, 0, 0), por lo tanto, si el oyente no se ha movido y desea que el sonido provenga de la izquierda (por ejemplo), debe establecer la posición x en un valor negativo (para más información sobre la configuración de la posición del oyente ver audio_listener_position )

También puede establecer una distancia de caída (0 hará que el sonido sea silencioso, el valor predeterminado es 100) que hará que el sonido se desvanezca a medida que se aleja de la posición del oyente. La forma en que se escuche el fundido dependerá de la referencia de caída (que es la distancia bajo la cual el volumen de la fuente normalmente descenderá a la mitad) y el factor de caída (que afecta el sonido más allá de la distancia de referencia de caída). El factor predeterminado es normalmente 1, y el efecto de los diferentes valores de caída dependerá del modelo elegido (para una guía completa de los diferentes modelos de caída y cómo se usan estos valores, consulte la función audio_falloff_set_model )

Los dos últimos argumentos son establecer que el sonido sea en bucle o no y, finalmente, para asignar una prioridad al sonido. Esta prioridad se usa para determinar cómo se tratan los sonidos cuando el número de sonidos que se tocan supera el límite establecido por la función. audio_channel_num. Los sonidos de menor prioridad se detendrán en favor de sonidos de mayor prioridad, y el valor de prioridad puede ser cualquier número real (el valor real es arbitrario, y puede ser de 0 a 1 o de 0 a 100, ya que GameMaker Studio 2 los priorizará de la misma manera )

Esta función devolverá un número de índice único para el sonido que se está reproduciendo, que luego se puede almacenar en una variable para que luego pueda pausarlo o detenerlo con las funciones apropiadas. Esto significa que si tiene varias instancias del mismo sonido reproduciéndose en cualquier momento, puede enfocar solo una instancia de ese sonido para manejar las funciones de audio.


Sintaxis:

audio_play_sound_at(index, x, y, z, falloff_ref, falloff_max, falloff_factor, loop, priority);


Argumento Descripción
index El índice del sonido para jugar.
x La posición x
y La posición y.
z La posición z
falloff_ref La referencia de caída relativa al oyente (abrazadera).
falloff_max La distancia máxima de caída relativa al oyente.
falloff_factor El factor de caída (por defecto 1).
loop Marca el sonido como bucle o no.
priority Establezca la prioridad del canal para el sonido.


Devoluciones:

Index


Ejemplo:

if global.SFX
   {
   audio_play_sound_at(snd_Waterfall, x, y, 0, 100, 300, 1, true, 1);
   }

El código anterior verifica la variable global "SFX" y si regresa true luego, el sonido indexado en la variable "snd_Waterfall" se colocará en bucle en su posición de habitación, con una referencia de caída de 100, una distancia de caída de 300, un factor de caída de 1 y una prioridad baja.
