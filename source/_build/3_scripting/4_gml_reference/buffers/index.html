Tampones


Un búfer binario es una región de la memoria física que se usa para retener temporalmente los datos mientras se mueve de un lugar a otro o se manipula de alguna manera, por ejemplo, puede usar un búfer para retener paquetes de datos en espera de transmisión a través de una red o para almacenar un Paquete que se ha recibido a través de una red. GameMaker Studio 2 tiene una serie de funciones específicas relacionadas con la creación, modificación y carga / guardado de búferes, pero antes de usarlos, tenga en cuenta los siguientes puntos:

  • Cuando se crea un búfer, se borra automáticamente y se llena a 0.

  • El argumento "tamaño" de las funciones del búfer siempre se refiere al tamaño en bytes del búfer.

  • El argumento "Alineación" se refiere a cómo se almacenan los datos dentro de un búfer. Si su alineación está configurada para decir, 4, y escribe una sola pieza de datos que tiene un tamaño de 1 byte, haga una buffer_tell, obtendrá un desplazamiento de 1. Sin embargo, si escribe otro dato, también de 1 byte de tamaño, realice una buffer_tell, obtendrá un desplazamiento de 5 cuando la alineación haya "rellenado" los datos a ese tamaño. La alineación solo afecta el lugar donde se escriben las cosas, así que si haces una buffer_tell después de escribir algo, devolverá la posición de escritura actual que sigue inmediatamente a los datos que ha escrito. Tenga en cuenta, sin embargo, que si luego escribe otra parte de los datos, internamente el búfer moverá la posición de escritura a lo largo del siguiente múltiplo del tamaño de alineación antes de escribir la parte de los datos.

  • Cuando se trata de "desplazamiento", este es el valor en bytes para compensar los datos dentro del búfer dado. Por lo tanto, si su búfer está alineado con 2 bytes y desea omitir los primeros 4 lugares alineados, tendrá un desplazamiento de 2 * 4 bytes.

  • Algunas de las funciones del búfer también crean un nuevo búfer (como buffer_load por ejemplo). Recuerde que estos buffers también deberán eliminarse de la memoria cuando no estén en uso utilizando el buffer_delete función.

  • La memoria utilizada para crear búferes es la memoria del sistema, por lo que incluso cuando el juego no tiene foco, los datos almacenados en un búfer deberían ser seguros, sin embargo, si la aplicación se cierra o se reinicia, se perderá.
