buffer_create_from_vertex_buffer_ext

Sintaxis:

buffer_create_from_vertex_buffer_ext(vertex_buffer, type, alignment, start_vertex, num_vertices)


Argumento Descripción
vertex_buffer El índice de búfer de vértice a usar.
type El tipo de buffer para crear (ver la lista de constantes a continuación).
alignment La alineación de bytes para el buffer
start_vertex El vértice inicial.
num_vertices El número total de vértices para copiar.


Devoluciones: Real


Descripción

Utiliza esta función para asignar una porción de memoria como un búfer en tu juego lleno con los datos de un búfer de vértices previamente creado. La función devuelve un identificador único de almacenamiento intermedio que debe almacenarse en una variable y utilizarse para todas las demás llamadas de función al almacenamiento intermedio. La función toma la identificación para el buffer de vértice para usar (como lo devuelve la función vertex_create_buffer(), por ejemplo) con las siguientes constantes que se utilizan para definir el tipo de buffer:

Constante Descripción
buffer_fixed Un buffer de tamaño fijo.
buffer_grow Un buffer que "crecerá" dinámicamente a medida que se agreguen los datos
buffer_wrap Un buffer donde los datos se "ajustarán". Cuando los datos que se agregan alcanzan el límite del tamaño del búfer, la sobreescritura se volverá a colocar al inicio del búfer, y la escritura continuará desde ese punto.
buffer_fast Buffer especial "stripped" que es extremadamente rápido para leer / escribir. Solo se puede usar con buffer_u8 tipos de datos, y deben estar alineados a 1 byte.
buffer_vbuffer



Además del tipo de búfer, también deberá establecer la alineación de bytes para el búfer. Este valor variará según los datos que desee almacenar en el búfer, y en la mayoría de los casos un valor de 1 está perfectamente bien. Sin embargo, tenga en cuenta que para algunas operaciones es esencial una alineación específica, y una alineación incorrecta puede causar errores (para obtener más detalles sobre la alineación, consulte Alineación de zonas tampón ). La siguiente es una guía general para mostrar qué valores son los más apropiados para cada tipo de datos (cuando tenga dudas, use una alineación de 1):

  • Las cadenas deben estar alineadas a 1 byte.
  • Los enteros de 8 bits con signo o sin signo se pueden alinear con cualquier valor, pero tenga en cuenta que para un búfer rápido (consulte buffer_write ) debe estar alineado a 1.
  • Los enteros de 16 bits firmados o sin firmar se deben alinear a 2 bytes.
  • Los enteros de 32 bits firmados o sin firmar se deben alinear a 4 bytes
  • Los flotadores de hasta 16 bits se deben alinear a 2 bytes. (¡Actualmente no compatible!)
  • Los flotantes de hasta 32 bits se deben alinear a 4 bytes.
  • Los flotadores de hasta 64 bits se deben alinear a 8 bytes.

NOTA: Los búferes de vértices están alineados en 1 byte, pero puede crear el búfer con cualquier alineación dependiendo de cómo quiera tratar los datos, ya que los datos de vértice son simplemente una copia en bruto de la memoria en el búfer.

Los dos argumentos finales se usan para especificar el rango de datos de vértice que desea copiar en el búfer recién creado. El vértice de inicio puede estar entre 0 y el número de vértices -1, y luego se da el número de vértices desde ese punto para copiar. Puedes usar la función vertex_get_number() en el buffer de vértice para obtener el número total de vértices almacenados.

NOTA: Es importante que elimine de la memoria todos los recursos creados dinámicamente de esta manera cuando ya no los necesite para evitar pérdidas de memoria, de modo que cuando haya terminado con el búfer que ha creado, debería liberarlo nuevamente usando buffer_delete. También tenga en cuenta que el búfer de vértices utilizado para crear el nuevo búfer tampoco se elimina de la memoria y debe utilizar la función vertex_delete_buffer() cuando ya no es necesario.


Ejemplo:

var v_num = vertex_get_number(model_buff); player_buffer = buffer_create_from_vertex_buffer(model_buffer, buffer_grow, 1, 0, v_num - 1);

El código anterior asigna memoria a un búfer y luego copia los datos de todos los vértices del búfer de vértices dado en él, devolviendo el índice del nuevo búfer que está almacenado en la variable "player_buffer", para uso futuro.
