buffer_save_async

Sintaxis:

buffer_save_async(buffer, filename, offset, size);


Argumento Descripción
buffer El índice del buffer para guardar
filename El nombre del archivo para guardar como.
offset El desplazamiento dentro del búfer para guardar desde (en bytes).
size El tamaño del área del buffer para guardar (en bytes).


Devoluciones: Real


Descripción

Con esta función puede guardar parte de los contenidos de un búfer en un archivo, listo para ser leído de nuevo en la memoria usando el buffer_load función (o cualquiera de las otras funciones para cargar memorias intermedias). El "desplazamiento" define la posición de inicio dentro del búfer para guardar (en bytes), y el "tamaño" es el tamaño del área del búfer que se guardará desde ese desplazamiento en adelante (también en bytes). Esta función funciona de forma asíncrona, por lo que el juego continuará ejecutándose mientras se guarda. Esta función desencadenará un Evento Asíncrono de Cargar / Guardar en el que puede verificar para asegurarse de que el guardado se haya realizado correctamente.

Tenga en cuenta que puede guardar varios almacenamientos intermedios en uno llamando a esta función varias veces entre llamadas a buffer_async_group_begin() y buffer_async_group_end() (ver esas funciones para más información sobre esto).

La función devuelve un valor de ID único que luego se puede usar para verificar el evento asincrónico async_load Valor de ID, como se muestra en el ejemplo extendido a continuación. los async_load el mapa en el evento tendrá los siguientes dos pares clave / valor:

  • "id": la identificación de la función asíncrona devuelta por la función de guardar.
  • "estado": volverá true si los datos se guardaron / cargaron correctamente, y false de otra manera.




Ejemplo extendido:

los buffer_save_async() se puede llamar a la función desde cualquier evento, y dado que es asíncrona, la devolución de llamada puede ser casi instantánea o puede tomar varios segundos. Llamar a la función es simple y se vería así:

saveid = buffer_save_async(buff, "Player_Save.sav", 0, 16384);

El código anterior guarda el contenido del buffer " buff " en el archivo guardado, almacenando el ID de la llamada a la función en la variable " saveid ". Cuando finalice el guardado, se desencadenará el evento Guardar / Cargar asíncrono y podrá analizar el async_load mapa para la ID correcta de la función, como esta:

if ds_map_find_value(async_load, "id") == saveid
   {
   if ds_map_find_value(async_load, "status") == false
      {
      show_debug_message("Save failed!");
      }
   }

El código anterior primero verificará la identificación del ds_map que se ha creado, luego verificará el estado de la devolución de llamada, publicando un mensaje de depuración si ha habido algún problema.
