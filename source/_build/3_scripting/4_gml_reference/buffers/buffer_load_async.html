buffer_load_async

Sintaxis:

buffer_load_async(buffer, filename, offset, size);


Argumento Descripción
buffer El índice del buffer para cargar.
filename El nombre del archivo para cargar.
offset El desplazamiento dentro del búfer para cargar (en bytes).
size El tamaño del área del buffer para cargar (en bytes).


Devoluciones: Real


Descripción

Con esta función, puede cargar un archivo que haya creado previamente utilizando el buffer_save función (o cualquiera de las otras funciones para guardar almacenamientos intermedios) en un búfer. El "desplazamiento" define la posición de inicio dentro del búfer para cargar (en bytes), y el "tamaño" es el tamaño del área del búfer que se cargará desde ese desplazamiento en adelante (también en bytes).

La función devuelve un valor de ID único que luego se puede usar para verificar el evento asincrónico async_load Valor de ID, como se muestra en el ejemplo extendido a continuación. los async_load el mapa en el evento tendrá los siguientes dos pares clave / valor:

  1. "id": la identificación de la función asíncrona devuelta por la función de guardar.
  2. "estado": volverá true si los datos se guardaron / cargaron correctamente, y false de otra manera.




Ejemplo extendido:

los buffer_load_async() se puede llamar a la función desde cualquier evento, y dado que es asíncrona, la devolución de llamada puede ser casi instantánea o puede tomar varios segundos. Llamar a la función es simple y se vería así:

loadid = buffer_load_async(buff, "Player_Save.sav", 0, 16384);

El código anterior carga los contenidos del archivo " Player_Save.sav " al buffer dado, almacenando el ID de la llamada a la función en la variable " loadid ". Cuando se complete la carga, se desencadenará el evento Guardar / Cargar asíncrono y podrá analizar el async_load mapa para la ID correcta de la función, como esta:

if ds_map_find_value(async_load, "id") == loadid
   {
   if ds_map_find_value(async_load, "status") == false
      {
      show_debug_message("Load failed!");
      }
   }

El código anterior primero verificará la identificación del ds_map que se ha creado, luego verificará el estado de la devolución de llamada, publicando un mensaje de depuración si ha habido algún problema.
