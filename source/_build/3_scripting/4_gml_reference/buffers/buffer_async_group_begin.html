buffer_async_group_begin

Sintaxis:

buffer_async_group_begin(groupname);


Argumento Descripción
groupname El nombre del grupo (como una cadena).


Devoluciones: N / A


Descripción

Se llama a esta función cuando desea comenzar a guardar múltiples búferes en varios archivos. El "nombre de grupo" es una cadena y se usará como el nombre del directorio donde se guardarán los archivos, y se debe usar como parte de la ruta del archivo cuando se carguen los archivos en el IDE más tarde (utilizando cualquiera de los buffer_load funciones). Esta función es solo para usar con buffer_save_async() función y también debe terminar la definición de guardar llamando buffer_async_group_end() función de lo contrario, los archivos no se guardarán.

Tenga en cuenta que para las plataformas de consola (como PS4 por ejemplo), el "nombre de grupo" se utilizará como la descripción de la ranura de guardado, y el uso de esta función puede ayudarlo a evitar que se muestre la interfaz de usuario para cada archivo que se está guardando.


Ejemplo:

buffer_async_group_begin("SaveGame");
save1 = buffer_save_async(buff1, "Player_Save1.sav", 0, 16384);
save2 = buffer_save_async(buff2, "Player_Save2.sav", 0, 16384);
save3 = buffer_save_async(buff3, "Player_Save3.sav", 0, 16384);
save4 = buffer_save_async(buff4, "Player_Save4.sav", 0, 16384);
buffer_async_group_end();

El código anterior inicia un grupo de búfer y luego lo configura para guardar 4 archivos de forma asíncrona. La definición del grupo se termina (en ese momento comenzará el ahorro).
