buffer_create

Sintaxis:

buffer_create(size, type, alignment)


Argumento Descripción
size El tamaño (en bytes) del buffer.
type El tipo de buffer para crear (ver la lista de constantes a continuación).
alignment La alineación de bytes para el buffer


Devoluciones: Real


Descripción

Usas esta función para asignar una porción de la memoria como un buffer en tu juego, con la función devolviendo el identificador único del buffer que se debe almacenar en una variable y usar para todas las llamadas de funciones adicionales al buffer. El búfer se puede usar para almacenar diferentes tipos de datos (especificados cuando escribe en el búfer usando el buffer_write función, con las siguientes constantes que se utilizan para definir el tipo de búfer:

Este tipo de búfer debe usarse solo como un búfer de vértices.
Constante Descripción
buffer_fixed Un buffer de tamaño fijo.
buffer_grow Un buffer que "crecerá" dinámicamente a medida que se agreguen los datos
buffer_wrap Un buffer donde los datos se "ajustarán". Cuando los datos que se agregan alcanzan el límite del tamaño del búfer, la sobreescritura se volverá a colocar al inicio del búfer, y la escritura continuará desde ese punto.
buffer_fast Buffer especial "stripped" que es extremadamente rápido para leer / escribir. Solo se puede usar con buffer_u8 tipos de datos, y deben estar alineados a 1 byte.
buffer_vbuffer



Además del tipo de búfer, también deberá establecer la alineación de bytes para el búfer. Este valor variará según los datos que desee almacenar en el búfer, y en la mayoría de los casos un valor de 1 está perfectamente bien. Sin embargo, tenga en cuenta que para algunas operaciones es esencial una alineación específica, y una alineación incorrecta puede causar errores (para obtener más detalles sobre la alineación, consulte Buffers ). La siguiente es una guía general para mostrar qué valores son los más apropiados para cada tipo de datos (cuando tenga dudas, use una alineación de 1):

  • Las cadenas deben estar alineadas a 1 byte.
  • Los enteros de 8 bits con signo o sin signo se pueden alinear con cualquier valor, pero tenga en cuenta que para un búfer rápido (consulte buffer_write ) debe estar alineado a 1.
  • Los enteros de 16 bits firmados o sin firmar se deben alinear a 2 bytes.
  • Los enteros de 32 bits firmados o sin firmar se deben alinear a 4 bytes
  • Los flotadores de hasta 16 bits se deben alinear a 2 bytes. (¡Actualmente no compatible!)
  • Los flotantes de hasta 32 bits se deben alinear a 4 bytes.
  • Los flotadores de hasta 64 bits se deben alinear a 8 bytes.

NOTA: la alineación de bytes puede ser muy importante ya que la elección incorrecta puede afectar negativamente el rendimiento.

NOTA: Es importante que elimine de la memoria todos los recursos creados dinámicamente de esta manera cuando ya no los necesite para evitar pérdidas de memoria, de modo que cuando haya terminado con el búfer que ha creado, debería liberarlo nuevamente usando buffer_delete.


Ejemplo:

player_buffer = buffer_create(16384, buffer_fixed, 2);

El código anterior asigna 16384 bytes de memoria a un búfer y devuelve el índice de ese búfer, que se almacena en la variable "player_buffer", para su uso futuro. El búfer está alineado con un límite de dos bytes.
