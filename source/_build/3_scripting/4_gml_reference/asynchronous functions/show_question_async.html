show_question_async


Descripción

Esta función abre una ventana y muestra al usuario la pregunta que define en la función. Esta es una función asíncrona, y como tal, GameMaker Studio 2 no bloquea el dispositivo que se está ejecutando mientras se espera la respuesta, sino que más bien continúa ejecutando eventos de forma normal. La función tiene dos botones que muestran "Sí" y "No", y una vez que el usuario ha presionado uno, se desencadena un evento Dialog asíncrono que, durante la duración de ese evento solamente, tendrá un ds_map almacenado en la variable async_load.

Este mapa contendrá las dos claves, "id" y "estado". "id" es el valor que devolvió la función cuando se llamó, mientras que el "estado" será cualquiera true o false para "Sí" y "No", respectivamente.


Sintaxis:

show_question_async(string);

Argumento Descripción
String La pregunta para hacerle al usuario.


Devoluciones:

Real


Ejemplo extendido:

El evento de prensa del mouse izquierdo (por ejemplo) del objeto que muestra el mensaje tendrá el siguiente código:

msg = show_question_async("Do you want to buy some armour for " + string(armour[0, 5]) + "coins?");

Lo anterior mostrará una pregunta con la cadena dada, solicitando que el usuario presione "sí" o "No". La id de la función se almacena en la variable "msg" y se usará en el evento Dialogs asíncrono como se muestra a continuación:

var i_d, stat;
i_d = ds_map_find_value(async_load, "id");
if i_d == msg
   {
   if ds_map_find_value(async_load, "status")
      {
      coins -= armour[0,5];
      global.protection += armour[0,0];
      }
   }

El código anterior verifica la clave "id" de la respuesta ds_map contra el valor almacenado en la variable "msg". Si son iguales, comprueba si se presionó uno de los dos botones y si vuelve a aparecer true luego deducirá un valor de una variable y agregará un valor a una variable global también.
