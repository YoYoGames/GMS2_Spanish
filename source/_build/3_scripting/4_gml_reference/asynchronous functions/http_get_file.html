http_get_file


Descripción

Con esta función, puede conectarse a la URL especificada para recuperar información en forma de archivo. Como se trata de una función asincrónica, GameMaker Studio 2 no se bloqueará mientras espera una respuesta, pero seguirá funcionando a menos que reciba información de devolución de llamada. Esta información tendrá la forma de una cadena y activará un Evento Asíncrono en cualquier instancia que tenga uno definido en sus propiedades de objeto.

Este evento generará una "devolución de llamada" que será captada por cualquier evento HTTP, en cuyo caso generará un ds_map (más comúnmente conocido como un "diccionario") que es exclusivo de este evento y se almacena en la variable especial async_load. Este ds_map contendrá diferentes valores según los datos que se devuelvan, es decir: el evento se activará varias veces a medida que se reciba cada paquete de datos para que pueda mostrar una barra de progreso, por ejemplo. La estructura general para el ds_map será la siguiente:

  1. id: la ID que se devolvió desde el comando. Si disparas una serie de http_ Luego, debe saber a cuál le está respondiendo y, por lo tanto, debe usar este valor para compararlo con el valor que guardó cuando envió originalmente la solicitud para encontrar el correcto.

  3. estado: Devuelve un valor de menos de 0 para un error, 0 para completar y 1 para recibir paquetes (ver más abajo para más detalles).

  5. resultado: los datos recibidos (solo cadena).

  7. URL: la URL completa que solicitó.

  9. http_status: el código de estado HTTP sin procesar (si está disponible). Esto devuelve el código de estado web estándar para la mayoría de los navegadores, por ejemplo: 304 para "No modificado" o 204 para "Sin contenido", etc.

Si se devuelven varios paquetes a su juego, la clave de "estado" de devolución de llamada devolverá 1, en cuyo caso el ds_map tendrá las siguientes claves adicionales:

  1. "contentLength": este es el tamaño del archivo que el servidor web ha dicho que debe esperar recibir (puede ser -1 si el servidor no devuelve estos datos).

  3. "sizeDownloaded": el tamaño de los datos que ya se han descargado.
NOTA: Debe tener en cuenta que debido a la protección XSS en los navegadores, las solicitudes e intentos de cargar recursos de varios dominios están bloqueadas y puede parecer que devuelven resultados en blanco. Consulte la sección sobre Problemas de dominio cruzado para obtener más detalles.


Sintaxis:

http_get_file(url, local_target);

Argumento Descripción
url La dirección web del servidor del que desea obtener el archivo
local_target El directorio de destino del área de almacenamiento local para almacenar el archivo en


Devoluciones:

Real


Ejemplo extendido:

los http_get_file se puede llamar a la función desde cualquier evento, y dado que es asíncrona, la devolución de llamada puede ser casi instantánea o puede tomar varios segundos. Llamar a la función es simple y se vería así:

file = http_get_file("http://www.macsweeneygames.com/downloads/upgrade.zip", "\Downloads\Upgrade.zip");

El código anterior solicitará un archivo de la URL especificada y lo configurará para descargarlo al área de almacenamiento local, en el directorio "Descargas" con el nombre de archivo dado (no tiene que ser el mismo que el nombre del archivo de origen). los async_load El índice del mapa se almacenará en la variable "archivo" para que pueda verificar la devolución de llamada correcta en el evento asíncrono, y si el directorio de guardar no existe, se creará. El evento asincrónico activado sería el sub-evento HTTP, y en ese caso tendría algo como lo siguiente:

if ds_map_find_value(async_load, "id") == file
   {
   var status = ds_map_find_value(async_load, "status");
   if status == 0
      {
      var path = ds_map_find_value(async_load, "result");
      var files = zip_unzip(path, "/NewContent/");
      if files > 0
         {
         global.ExtraContent = true;
         }
      }
   }

El código anterior primero verificará la identificación del ds_map que se ha creado, luego verificará el estado de la devolución de llamada. Si el valor es igual a 0 (éxito de señalización), el resultado de la devolución de llamada se usará junto con el zip_unzip() función para descomprimir el archivo descargado en el directorio dado. Si el descomprimir tiene éxito, una variable global se establece en verdadero.
