facebook_graph_request


Sintaxis:

facebook_graph_request(graph_path, httpMethod, ds_map_parameters, ds_map_return)

Argumento Descripción
graph_path La parte del gráfico social para interactuar con
httpMethod El método http para usar (POST, GET, DELETE).
ds_map_parameters El ds_map con la información para enviar.
ds_map_return El ds_map para recibir (-1 para ninguno).


Devoluciones:

N/A


Descripción

Con esta función puede hacer que el usuario interactúe con el Gráfico social de Facebook. El argumento "graph_path" es donde defines la parte del gráfico al que deseas acceder, como la lista actual de amigos del usuario, u otros comentarios de aplicaciones o incluso un evento. La ruta exacta se puede definir utilizando los términos detallados en la sección Publicación de las páginas API Api de Facebook. A continuación, define el método http para usar, que generalmente es POST o GET, pero Facebook también acepta el método DELETE.

El siguiente argumento es un poco más complejo ya que requiere que hayas creado y rellenado un ds_map con la información correcta que GameMaker Studio 2 luego convertirá en json automáticamente cuando se envía a la API de Facebook. La información que se pone en este mapa dependerá en gran medida de qué camino elige usar y una lista completa de todos los valores posibles se pueden encontrar aquí. Un ejemplo de cómo se estructura este mapa se puede ver en el ejemplo de código que se proporciona a continuación.

Finalmente, tenemos un argumento para almacenar cualquier información que Facebook haya enviado desde la solicitud del gráfico. Esta información viene en forma de json que GameMaker Studio 2 convierte en un ds_map. Para que esto funcione correctamente, debe haber creado un ds_map previamente, y si ese mapa se ha utilizado en otro lugar y ya contiene algunos pares clave / valor, esta función lo borrará antes de que se agreguen los datos de Facebook. Es muy importante que lea la documentación de Facebook sobre los posibles valores de devolución para que sepa exactamente qué esperar. Afortunadamente Facebook tiene una página de prueba Graph Api donde puedes verificar diferentes rutas de gráficos y ver exactamente qué información devolverán.

NOTA: El usuario debe iniciar sesión para que la solicitud de gráficos funcione.


Ejemplo:

var wallPostMap;
wallPostMap = ds_map_create();
ds_map_add(wallPostMap, "message", "I just got a hi-score playing Catch The Clown!");
ds_map_add(wallPostMap, "picture", "http://MacSweeneygames.com/Clown.jpg");
ds_map_add(wallPostMap, "link", "http://MacSweeneygames.com/");
ds_map_add(wallPostMap, "name", "Catch The Clown");
ds_map_add(wallPostMap, "caption", "MacSweeneygames.com");
ds_map_add(wallPostMap, "description", "Play Catch the clown now on MacSweeney Games!");
facebook_graph_request("me/feed", "POST", wallPostMap, -1);
ds_map_destroy(wallPostMap);

El código anterior creará un ds_map y almacenará su índice en la variable local "wallPostMap". A continuación, rellena el ds_map con los pares de clave / valor relevantes que se necesitan para publicar una imagen y texto en el muro de usuarios actualmente conectados. Finalmente, envía la solicitud de gráfico y elimina el ds_map de la memoria.
