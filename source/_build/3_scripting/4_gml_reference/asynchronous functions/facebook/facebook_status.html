Estado de Facebook


Sintaxis:

facebook_status()


Devoluciones:

String


Descripción

Con esta función, puede sondear a Facebook para conocer el estado actual del inicio de sesión del usuario. Se devuelve una de las siguientes cadenas (tenga en cuenta que siempre se devuelven en mayúsculas):

  • AUTHORISED: Esto significa que el usuario ha iniciado sesión correctamente, pero no todos los permisos pueden haber sido otorgados.
  • PROCESSING: Esto se devuelve cuando el proceso de inicio de sesión no está resuelto actualmente.
  • IDLE: Esto se devuelve cuando la conexión a la API de Facebook se ha inicializado pero no se ha realizado ninguna otra acción (es decir, no se ha iniciado sesión ningún usuario).
  • FAILED: Esto significa que el usuario no ha iniciado sesión correctamente, generalmente debido a un error de conexión.

Tenga en cuenta que esta función no verifica continuamente la API de Facebook y solo se actualizará después de que se haya resuelto una acción de Facebook (como iniciar sesión, cerrar sesión o solicitar permisos).


Ejemplo:

switch (facebook_status())
   {
   case "AUTHORISED": global.Auth = true; instance_destroy(); break;
   case "FAILED": facebook_login(permissions); alarm[0] = 30; break;
   default: alarm[0] = 30;
   }

El código anterior se ejecutaría en una alarma y verifica el estado de Facebook para ver si el usuario ha iniciado sesión correctamente. Si la cadena de retorno es "AUTHORISED", una variable global se establece en verdadero y el juego continúa, si la cadena de retorno es "FAILED" el juego solicitará que el usuario inicie sesión nuevamente y luego repita la alarma para verificar el estado una vez más.
