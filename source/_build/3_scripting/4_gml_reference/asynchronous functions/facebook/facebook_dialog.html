facebook_dialog


Sintaxis:

facebook_dialog(graph_path, ds_map_parameters, ds_map_return)

Argumento Descripción
graph_path La parte del gráfico social para interactuar con
ds_map_parameters El ds_map con la información para enviar.
ds_map_return El ds_map para recibir (-1 para ninguno).


Devoluciones:

N/A


Descripción

Un diálogo de Facebook es la forma en que Facebook le pide al usuario que brinde comentarios e información, y no necesitan ningún permiso adicional porque requieren la interacción del usuario. Por ejemplo, se puede usar para enviar solicitudes de amistad o para publicar un comentario en un muro. El argumento "graph_path" es donde defines la parte del gráfico al que deseas acceder, y la ruta exacta se puede definir usando los términos delineados en la sección de Diálogos de Facebook.

El siguiente argumento es un poco más complejo ya que requiere que hayas creado y rellenado un ds_map con la información correcta que GameMaker Studio 2 luego convertirá en json automáticamente cuando se envía a la API de Facebook. La información que se pone en este mapa dependerá en gran medida de qué camino elige usar y una lista completa de todos los valores posibles se pueden encontrar aquí. Un ejemplo de cómo se estructura este mapa se puede ver en el ejemplo de código que se proporciona a continuación.

Finalmente, tenemos un argumento para almacenar cualquier información que Facebook haya enviado desde la solicitud del gráfico. Esta información viene en forma de json que GameMaker Studio 2 convierte en un ds_map. Para que esto funcione correctamente, debe haber creado un ds_map previamente, y si ese mapa se ha utilizado en otro lugar y ya contiene algunos pares clave / valor, esta función lo borrará antes de que se agreguen los datos de Facebook. Es muy importante que lea la documentación de Facebook sobre los posibles valores de devolución para que sepa exactamente qué esperar. Afortunadamente Facebook tiene una página de prueba Graph Api donde puedes verificar diferentes rutas de gráficos y ver exactamente qué información devolverán.

NOTA: El usuario no necesita iniciar sesión para usar esta función, ya que el diálogo de inicio de sesión se mostrará como parte de la solicitud de diálogo.


Ejemplo:

var interactiveMap;
interactiveMap = ds_map_create();
ds_map_add(interactiveMap, "picture", "http://MacSweeneygames.com/Clown.jpg");
ds_map_add(interactiveMap, "link", "http://MacSweeneygames.com/");
ds_map_add(interactiveMap, "name", "Catch The Clown");
ds_map_add(interactiveMap, "caption", "MacSweeneygames.com");
ds_map_add(interactiveMap, "description", "Play Catch the clown now on MacSweeney Games!");
facebook_dialog("feed", interactiveMap, -1);
ds_map_destroy(interactiveMap);

El código anterior creará un ds_map y almacenará su índice en la variable local "interactiveMap". Este mapa se llena con los pares clave / valor necesarios para generar una publicación mural interactiva. Luego el facebook_dialog abrirá una ventana en la que se le pedirá al usuario que agregue un mensaje que se enviará junto con el resto de la información, y esto se publicará en el muro de los usuarios. Finalmente, el ds_map se borra en la memoria libre.
