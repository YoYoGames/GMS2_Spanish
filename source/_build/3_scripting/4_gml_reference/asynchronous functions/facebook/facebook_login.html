facebook_login


Sintaxis:

facebook_login(permissions, ios_login_type)

Argumento Descripción
permissions Los permisos que desea que el usuario inicie sesión
ios_login_type El tipo de inicio de sesión solo de iOS (consulte la lista de constantes a continuación).


Devoluciones:

N/A


Descripción

Con esta función, GameMaker Studio 2 se conectará a Facebook y le pedirá al usuario que inicie sesión en su cuenta. También puede proporcionar solicitudes de permiso adicionales en forma de una lista ds que esta función luego convierta en una json matriz para comunicarlos a Facebook. Existe una amplia variedad de permisos que puede solicitar y están documentados en las páginas de desarrolladores de Facebook que se encuentran aquí. Si no desea solicitar permisos especiales o extendidos, deberá enviar una ds_list, pero esta vez estará vacía.

Esta función también se considera asíncrona ya que mientras la ventana de inicio de sesión de Facebook está abierta en la pantalla, el juego continuará ejecutándose en segundo plano e incluso una vez que el usuario haya completado el proceso de inicio de sesión, ninguna de las otras funciones de Facebook funcionará hasta que el facebook_status() devuelve "AUTORIZADO". Debido a la necesidad de una conexión a Internet y las diferentes velocidades de transferencia de datos, esto significa que pueden transcurrir varios segundos antes de que se reciba esta autorización, y todo el tiempo el juego seguirá en funcionamiento.

NOTA: ahora solo puede leer permisos para esta función. Si necesita permisos de escritura, necesita llamar facebook_request_publish_permissions() después de iniciar sesión. También tenga en cuenta que si su juego solicita más de los de public_profile, email y user_friends requerirá la revisión de Facebook antes de que pueda ser utilizado por personas que no sean los desarrolladores del juego.


Hay un parámetro adicional requerido para iniciar sesión en iOS (otras plataformas pueden simplemente usar fb_login_default para el argumento). Esta debería ser una de las constantes que se enumeran a continuación:

Constante Descripción
fb_login_default Utilice el comportamiento de inicio de sesión de iOS predeterminado (consulte aquí para obtener más información).
fb_login_fallback_to_webview Intente iniciar sesión en Facebook, solicitando credenciales al usuario si es necesario.
fb_login_no_fallback_to_webview Intente iniciar sesión en Facebook, pero no se realizará ninguna solicitud directa de credenciales.
fb_login_forcing_webview Intente solo el inicio de sesión de WebView, solicitando credenciales al usuario
fb_login_use_system_account Intente iniciar sesión en Facebook, prefiera la cuenta del sistema y vuelva a un cambio rápido de aplicación si es necesario.
fb_login_forcing_safari Intenta solo iniciar sesión con Safari.

Las constantes mencionadas anteriormente cambian el comportamiento de inicio de sesión en iOS, lo que le permite seleccionar qué métodos alternativos usar (si corresponde) en caso de que el inicio de sesión falle a través del proceso normal. Para obtener más información, visite esta página de Desarrollador de Facebook.


Ejemplo:

facebook_init();
var permissions = ds_list_create();
ds_list_add(permissions,"public_profile", "user_friends");
facebook_login(permissions, fb_login_default);
ds_list_destroy(permissions);

El código anterior inicializará Facebook y luego registrará al usuario actual, solicitando los permisos de lectura extendidos.
