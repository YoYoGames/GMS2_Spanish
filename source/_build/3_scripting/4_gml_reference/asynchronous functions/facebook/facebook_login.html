facebook_login


Sintaxis:

facebook_login(permissions, ios_login_type)

Argumento Descripción
permissions Los permisos que desea tener del usuario que inicia sesión.
ios_login_type El tipo de inicio de sesión de iOS solo (ver la lista de constantes a continuación).


Devoluciones:

N/A


Descripción

Con esta función, GameMaker Studio 2 se conectará a Facebook y le pedirá al usuario que inicie sesión en su cuenta. También puede proporcionar solicitudes de permisos adicionales en forma de ds_list que luego esta función convierte en una json Matriz para comunicarlos a Facebook. Hay una gran variedad de permisos que puede solicitar y están documentados en las páginas de desarrolladores de Facebook que se encuentran aquí. Si no desea solicitar ningún permiso especial o extendido, deberá enviar un ds_list, pero esta vez estará vacío.

Esta función también se considera asíncrona, mientras que la ventana de inicio de sesión de Facebook está abierta en la pantalla, el juego continuará ejecutándose en segundo plano e incluso una vez que el usuario haya completado el proceso de inicio de sesión, ninguna de las otras funciones de Facebook funcionará hasta que facebook_status() devuelve "AUTORIZADO". Debido a la necesidad de una conexión a Internet y las diferentes tasas de transferencia de datos, esto significa que pueden pasar varios segundos antes de que se reciba esta autorización, y todo el tiempo el juego seguirá funcionando.

NOTA: Ahora solo puede usar los permisos de lectura predeterminados para esta función. Si necesita permisos de escritura o permisos de lectura extendidos, debe llamar al facebook_request_publish_permissions() después de iniciar sesión. También tenga en cuenta que si su juego solicita más que el perfil público, el correo electrónico y los amigos del usuario, Facebook deberá revisarlo antes de que lo puedan usar otras personas que no sean los desarrolladores del juego.


Se requiere un parámetro adicional para iniciar sesión en iOS (otras plataformas pueden simplemente usar fb_login_default para el argumento). Esta debe ser una de las constantes enumeradas a continuación:

Constante Descripción
fb_login_default Utilice el comportamiento de inicio de sesión de iOS predeterminado (consulte aquí para obtener más detalles).
fb_login_fallback_to_webview Intente iniciar sesión en Facebook y solicite al usuario las credenciales si es necesario.
fb_login_no_fallback_to_webview Intente iniciar sesión en Facebook, pero no se realizará ninguna solicitud directa de credenciales.
fb_login_forcing_webview Solo intente iniciar sesión en WebView, solicitando credenciales al usuario
fb_login_use_system_account Intente iniciar sesión en Facebook, prefiera la cuenta del sistema y recurra a un cambio rápido de aplicación si es necesario.
fb_login_forcing_safari Intente iniciar sesión solo con Safari.

Las constantes enumeradas anteriormente cambian el comportamiento de inicio de sesión en iOS, lo que le permite seleccionar qué métodos de recuperación usar (si corresponde) en caso de que el inicio de sesión falle durante el proceso normal. Para más información por favor visite esta página de desarrolladores de Facebook.


Ejemplo:

facebook_init();
var permissions = ds_list_create();
ds_list_add(permissions,"public_profile", "user_friends");
facebook_login(permissions, fb_login_default);
ds_list_destroy(permissions);

El código anterior inicializará Facebook y luego iniciará sesión en el usuario actual, solicitando los permisos de lectura dados.
