Funciones asíncronas


Antes de tratar con las funciones asíncronas (a las que se hace referencia a partir de ahora como funciones de sincronización ) en GameMaker Studio 2 es posible que se esté preguntando qué significa realmente la palabra "asíncrono"! Bueno, la definición real del diccionario asincrónico es:

"perteneciente a una técnica de transmisión que no requiere un reloj común entre los dispositivos de comunicación; las señales de temporización se derivan de caracteres especiales en el flujo de datos mismo".

En el contexto de GameMaker Studio 2, una función asíncrona es aquella que envía a un servidor web, o solicita la entrada del usuario, o incluso transmite datos, mientras permite que GameMaker Studio 2 continúe ejecutándose sin bloqueos. Cuando se recibe la devolución de llamada de información, se activa un Evento Asíncrono especial que, a su vez, permite que cualquier instancia con un Evento Asíncrono asignado le ejecute funciones adicionales (aunque no tiene por qué ser así). Entonces, básicamente, es una forma de comunicarse con algún recurso externo (servidor, usuario o dispositivo) sin bloquear el juego o el proyecto mientras espera una respuesta.

¿Porque es esto importante? Bueno, ya que permite que GameMaker Studio 2 continúe funcionando mientras envía o recibe datos, y esto significa que puede hacer muchas cosas sin bloquear el bucle del juego como...

  • transmitir datos desde el dispositivo a su juego sin que el jugador espere que se carguen las cosas
  • tener eventos de devolución de llamada para hacer cosas solo cuando se recibe la información correcta
  • comunicar e intercambiar datos con un servidor web

Todo esto hace que todo funcione de una manera mucho más fluida y discreta. Cabe señalar que los eventos asíncronos se activarán para todas las instancias que los tengan, por lo que puede usar una función asíncrona en una instancia y tener el evento asíncrono que trata la respuesta en otra, o incluso varias otras. Puede acceder aquí para obtener más información sobre cómo funciona el evento Asíncrono: eventos asincrónicos.

Tenga en cuenta que las funciones HTTP anteriores pueden no funcionar como se esperaba debido a problemas de seguridad entre dominios. Esto significa que las solicitudes a su servidor o los intentos de cargar recursos de varios dominios están bloqueadas y puede parecer que devuelve resultados en blanco o errores 404. Una de las formas en que puede evitar esto es tener un PHP del lado del servidor que permita que ciertos dominios accedan a su servidor (esta es también una manera de proteger sus recursos y bloquear servidores que no están incluidos en la lista de permitidos de PHP).

El siguiente es un ejemplo del código que puede usar para esto:

$http_origin = $_SERVER['HTTP_ORIGIN'];
if ($http_origin == "http://127.0.0.1:51268")
   {
   header('Access-Control-Allow-Origin: *');
   }


Las versiones anteriores de GameMaker tenían diálogos emergentes que bloqueaban al corredor mientras esperaba una respuesta del usuario. Sin embargo, a la mayoría de los dispositivos modernos no les gusta esto y interpretarán que el corredor se detuvo como un error y cerrarán la aplicación. Para evitar esto, GameMaker Studio 2 tiene equivalentes asíncronos de estas funciones que cumplen el mismo propósito que las versiones anteriores, pero no bloquean al corredor del juego en ningún momento, lo que significa que cuando se llame, el juego se ejecutará normalmente en el fondo mientras el usuario responde el diálogo mostrado. Una vez que el usuario ha respondido, se llamará a un evento asincrónico de diálogo para que pueda manejar los resultados.

A continuación hay una lista de todas las funciones de diálogo asíncronas disponibles:

  • get_login_async
  • show_message_async
  • show_question_async
  • get_string_async
  • get_integer_async

Al hacer juegos, a menudo es necesario almacenar información sobre el estado del juego en un archivo de algún tipo, pero almacenar esta información en el dispositivo no siempre es la mejor opción ya que, si el jugador borró el juego y luego lo reinstaló, este la información puede perderse Para evitar esto, puede usar varios servicios en la nube diferentes, que ofrecen almacenamiento de datos a través de Internet para su recuperación y modificación en cualquier momento. Debe tener en cuenta que esta función está limitada a un único "blob" de datos por juego, por lo que cada vez que envíe un nuevo guardado al servicio en la nube, ya sea una cadena o un archivo, sobrescribirá cualquier dato previamente almacenado.

GameMaker Studio 2 compatible con esto con algunas funciones simples que funcionan en conjunto con el Evento Asincrónico en la Nube. Estas funciones se explican en las siguientes páginas:

  • cloud_synchronise
  • cloud_string_save
  • cloud_file_save
NOTA: Actualmente, solo la extensión de Servicios de Google en Android usa estas funciones, y esta extensión debe estar presente antes de poder llamarlas.

En dispositivos móviles, normalmente solo una aplicación puede estar activa en primer plano en cualquier momento, pero muchos juegos y aplicaciones operan en un entorno basado en el tiempo o interconectado donde pueden ocurrir eventos de interés para los usuarios cuando la aplicación no está en primer plano. En estos casos, las notificaciones automáticas locales y remotas pueden permitir que estos juegos notifiquen a sus usuarios cuando ocurren eventos.

Cuando el sistema operativo entrega una notificación de inserción local o remota y la aplicación de destino no se ejecuta en primer plano, presenta la notificación al usuario en forma de un banner. Si hay una alerta de notificación y el usuario hace tapping en ella, se inicia el juego y se invoca un Evento de inserción asincrónica donde puede verificar la cadena de devolución de llamada que usted especifica y resolver el impulso en función de su valor. Si la aplicación se ejecuta en primer plano cuando se entrega la notificación, la aplicación seguirá recibiendo un evento push asíncrono, pero el usuario no recibirá ningún mensaje a menos que usted mismo lo cree.

Hay algunas diferencias para tener en cuenta entre las plataformas compatibles:

  • En Android, todas las notificaciones que se reciban cuando la aplicación no se esté ejecutando / en segundo plano se almacenarán y los datos serán entregados por el evento asíncrono la próxima vez que se ejecute la aplicación y deberá habilitar notificaciones push dentro de Android Opciones de juego de Android para que trabajo.

  • En iOS, los datos de una notificación particular solo se enviarán cuando se seleccione la notificación (o si la aplicación se ejecuta en primer plano cuando se recibe la notificación) y deberá habilitar las notificaciones push dentro de las Opciones de juego de iOS para que funcionen.

Una notificación de inserción local es local para el dispositivo en el que está instalado el juego y no requiere servidor backend. Simplemente configura la fecha y la hora de la notificación y se mostrará al usuario si el juego no se está ejecutando actualmente. Este tipo de notificación es útil para establecer "recordatorios" para que el usuario vuelva a jugar su juego, o para ofrecer una recompensa diaria por jugar, etc., y cuando el usuario toque la notificación lanzará el juego y pasará datos. cadena que luego puede ser analizada por el juego para dar una recompensa o lo que sea.

Las siguientes funciones están disponibles para notificaciones locales:

  • push_local_notification
  • push_get_first_local_notification
  • push_get_next_local_notification
  • push_cancel_local_notification
NOTA: Esta funcionalidad está limitada a los módulos de destino de iOS y Android. Para Android, deberá haber instalado la extensión de servicios de Google Play y haber habilitado las notificaciones push en las opciones de juegos sociales de Android.

Los mensajes de notificación remotos son enviados por un servidor a un servicio proporcionado por la App Store de la plataforma del dispositivo, y luego reenvía esos mensajes a todos los dispositivos en los que está instalada su aplicación. Esto es compatible con GameMaker Studio 2 en los módulos de destino de iOS y Android.

No hay funciones en GameMaker Studio 2 para tratar con notificaciones remotas, ya que todas ellas deben ser generadas por su servidor y manejadas por los respectivos App Stores. Sin embargo, una vez configurado correctamente, los juegos de GameMaker Studio 2 recibirán estas notificaciones, que luego se podrán tratar en el Evento de inserción asíncrono, como lo haría con una notificación local.

Para todas las plataformas disponibles, una vez que haya realizado la configuración necesaria, cuando el juego se ejecute en un dispositivo, registrará ese dispositivo con el servicio de notificaciones push de plataformas. Esto desencadenará un evento de notificación push asíncrono y el ds_map key " type " tendrá el valor " register " así como una nueva clave, " reg_id ", que contiene la identificación de registro (o un mensaje de error en la clave " error ", si " estado " es 0). A continuación, debe enviar esta identificación de registro a su servidor, y cada dispositivo en el que esté instalado su juego tendrá una identificación de registro diferente. Su servidor debe mantener una lista de identificadores para los dispositivos registrados, como cuando envía un mensaje de notificación de inserción desde su servidor, usa los ID de registro para enviar el mensaje a los dispositivos registrados.

Tenga en cuenta que no hay garantía de que se enviarán notificaciones push remotas, y que la carga de datos permitida es bastante pequeña. Esto varía entre plataformas, pero iOS es particularmente limitado: el servicio Apple solo envía la notificación más reciente, que debe ser seleccionada por el destinatario para que los datos de carga se entreguen a su evento asíncrono, y estas notificaciones tienen un tamaño máximo de carga de 256 bytes. Por lo general, una notificación de inserción remota simplemente indicaría que hay nuevos datos disponibles en su servidor, por ejemplo.

NOTA: Android requiere que agregue la ID del remitente de GCM en Android Opciones de juego de Android. Este es el número de proyecto que se asigna al crear su proyecto API de Google Play.
