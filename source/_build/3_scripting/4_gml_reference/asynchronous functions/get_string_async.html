get_string_async


Descripción

Esta función abre una ventana y muestra un mensaje, así como un espacio para que el usuario ingrese una cadena (que contendrá la cadena predeterminada provista para comenzar). Esta es una función asíncrona y, como tal, GameMaker Studio 2 no bloquea el dispositivo en el que se está ejecutando mientras espera respuesta, sino que continúa ejecutando los eventos de manera normal. Una vez que el usuario ha escrito un vistazo a su cuerda y se pulsa el botón "Aceptar", un evento de diálogo asíncrono está activado el cual, durante la duración de dicho evento solamente, tendrá un ds_map almacenado en la variable async_load.

Este mapa contendrá las tres claves, " id ", " status " y " result ". " id " es el valor que devolvió la función cuando se invocó, el " status " será true para presionar el botón "Aceptar", o false si el mensaje fue cancelado (donde corresponda, ya que no todas las plataformas de destino permiten que se cancele el mensaje). Finalmente, " result " devolverá la cadena que el usuario ingresó (o una cadena vacía "" si no se proporcionó ninguna).


Sintaxis:

get_string_async(string, default);

Argumento Descripción
String El mensaje a mostrar en el diálogo.
default La cadena por defecto.


Devoluciones:

Real


Ejemplo extendido:

El evento de presionar con el botón izquierdo del ratón (por ejemplo) del objeto que muestra el mensaje tendría el siguiente código:

msg = get_string_async("What's your name?","Anon");

Lo anterior mostrará un mensaje solicitando que el usuario ingrese una cadena y presione "Aceptar". La identificación de la función se almacena en la variable "msg" y se usará en el evento de Diálogos asíncronos como se muestra a continuación:

var i_d = ds_map_find_value(async_load, "id");
if i_d == msg
   {
   if ds_map_find_value(async_load, "status")
      {
      if ds_map_find_value(async_load, "result") != ""
         {
         global.Name = ds_map_find_value(async_load, "result");
         }
      }
   }

El código anterior verifica la clave " id " de la devuelta ds_map contra el valor almacenado en la variable " msg ". Si son iguales, luego verifica si se presionó "Aceptar" (en lugar de cerrar / cancelar la ventana) y si vuelve a aparecer. true luego verifica el "resultado" de una cadena para asegurarse de que no se devolvieron cadenas vacías antes de establecer una variable global.
