http_get


Descripción

Con esta función, se conecta a la URL especificada para recuperar información. Como se trata de una función asincrónica, GameMaker Studio 2 no se bloqueará mientras espera una respuesta, pero seguirá funcionando a menos que reciba información de devolución de llamada. Esta información tendrá la forma de una cadena y activará un Evento Asíncrono en cualquier instancia que tenga uno definido en sus propiedades de objeto. También debe tener en cuenta que los parámetros de solicitud HTTP (los bits a veces "añadidos" al final de una URL cuando envía un formulario en una página web) son perfectamente aceptables cuando se utiliza esta función, por ejemplo:

http_get("http://www.YoYoGames.com/logon?username="+name);

pasará la cadena contenida en la variable "nombre" al servidor y recuperará los datos de esa URL. Así que, esencialmente, cada vez que se necesita pasar un pequeño y simple dato desde el cliente al servidor, esta sería una elección razonable como la función a usar.

Esta función generará múltiples "rellamadas" que serán recogidas por cualquier evento HTTP. Estos generarán un ds_map (más comúnmente conocido como un "diccionario") que es exclusivo de este evento y se almacena en la variable especial async_load. Este ds_map contendrá diferentes valores dependiendo de si hay datos devueltos o no. Por ejemplo, si ha solicitado un archivo, el evento se activará varias veces a medida que se reciba cada paquete de datos para que pueda mostrar una barra de progreso, por ejemplo. La estructura general para el ds_map será la siguiente:

  1. id: la ID que se devolvió desde el comando. Si disparas una serie de http_ Luego, debe saber a cuál le está respondiendo y, por lo tanto, debe usar este valor para compararlo con el valor que guardó cuando envió originalmente la solicitud para encontrar el correcto.

  3. status: Devuelve un valor de menos de 0 para un error, 0 para completar y 1 para recibir paquetes (ver más abajo para más detalles).

  5. result: los datos recibidos (solo cadena).

  7. url: la URL completa que solicitó.

  9. http_status: el código de estado HTTP sin procesar (si está disponible). Esto devuelve el código de estado web estándar para la mayoría de los navegadores, por ejemplo: 304 para "No modificado" o 204 para "Sin contenido", etc.

Si hay varios paquetes que se devuelven a su juego, la clave de "estado" de devolución de llamada devolverá 1, en cuyo caso el ds_map tendrá las siguientes claves adicionales:

  1. " contentLength ": este es el tamaño del archivo que el servidor web ha dicho que debe esperar recibir (puede ser -1 si el servidor no devuelve estos datos).

  3. " sizeDownloaded ": el tamaño de los datos que ya se han descargado.
NOTA: Debe tener en cuenta que debido a la protección XSS en los navegadores, las solicitudes e intentos de cargar recursos de varios dominios están bloqueadas y puede parecer que devuelven resultados en blanco. Consulte la sección sobre Problemas de dominio cruzado para obtener más detalles.


Sintaxis:

http_get(url);

Argumento Descripción
url La dirección web del servidor del que desea obtener información


Devoluciones:

Real


Ejemplo extendido:

los http_get se puede llamar a la función desde cualquier evento, y dado que es asíncrona, la devolución de llamada puede ser casi instantánea o puede tomar varios segundos. Llamar a la función es simple y se vería así:

get = http_get("http://www.MacSweeneyGames.com/logon?username=" + name);

El código anterior pasará la cadena contenida en la variable " name " al servidor dado así como también recuperará los datos de esa URL, activando un Evento Asíncrono que contendrá async_load ds_map (el índice del mapa async_load se almacenará en la variable " get "para que pueda verificar la devolución de llamada correcta"). El evento Async activado sería el sub-evento HTTP, y en ese caso tendría lo siguiente:

if ds_map_find_value(async_load, "id") == get
   {
   if ds_map_find_value(async_load, "status") == 0
      {
      r_str = ds_map_find_value(async_load, "result");
      }
   else
      {
      r_str = "null";
      }
   }

El código anterior primero verificará la identificación del ds_map que se ha creado, luego verificará el " status " de la devolución de llamada. Si el valor es igual a 0 (éxito de la señalización), el resultado de la devolución de llamada se almacenará en una variable para uso futuro; de lo contrario, la variable se establecerá en un valor predeterminado (en este caso, "nulo").
