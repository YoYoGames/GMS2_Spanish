achievement_load_leaderboard


Sintaxis:

achievement_load_leaderboard(ident, minindex, maxindex, filter)

Argumento Descripción
ident La ID única de la tabla de clasificación como se muestra en el panel del desarrollador.
minindex El valor del índice inicial para obtener los datos de la tabla de clasificación.
maxindex El valor de índice máximo para obtener los datos de la tabla de clasificación.
filter Si filtrar los resultados a aquellos que están en su lista de amigos o a todos los jugadores (ver la Descripción a continuación)


Devoluciones:

N/A


Descripción

Esta función enviará una solicitud al servidor para obtener información sobre la tabla de clasificación dada. La tabla de clasificación debe haber sido creada previamente en el tablero del desarrollador para su juego y la ID que se proporciona es la que debe proporcionar como primer argumento de la función. Se desencadenará un evento asincrónico social de devolución de llamada que contiene el async_load mapa poblado con los pares clave / valor relevantes. La clave de identificación de esto ds_map se usa para identificar la devolución de llamada correcta (puede haber más de una función de activación para cualquier evento asíncrono dado), y se emparejará con la constante achievement_leaderboard_info así como una cantidad de otros pares clave / valor para cada jugador. Los contenidos exactos del mapa se muestran a continuación:

  • " id " - Para esta función, debe ser achievement_leaderboard_info

  • " leaderboardid ": el ID único para la tabla de clasificación tal como se define en el tablero del proveedor.

  • " numentries ": la cantidad de entradas en la tabla de líderes que ha recibido.

  • " PlayerN ": el nombre del jugador, donde "N" es un valor entero que corresponde a su posición dentro de la lista de entradas de la tabla de clasificación.

  • " PlayeridN ": la identificación de usuario única del jugador, "N".
  • " RankN " - El rango del jugador "N" dentro de la tabla de clasificación.

  • " ScoreN " - El puntaje del jugador "N".

Puede especificar la porción exacta de la tabla de líderes para mostrar dando un valor de índice mínimo y máximo, así como solicitar que los resultados se filtren para recuperar solo los puntajes de sus amigos o los publicados por todos los jugadores utilizando las siguientes constantes de filtro:

  • achievement_filter_friends_only - Obtenga solo las puntuaciones de los jugadores amigos.

  • achievement_filter_all_players - Obtenga todos los puntajes para todos los jugadores.

NOTA: esta función no funciona en la plataforma Android.


Ejemplo extendido:

El siguiente código probablemente se invocará después de que el jugador haya iniciado sesión en su cuenta de juego usando achievement_login para obtener una lista de todas las posiciones de los líderes de la lista de amigos para el juego, ya sea en Android o iOS:

if os_type == os_android
   {
   achievement_load_leaderboard("CgkIs9_51u0PEAIQBw", 1, 100, achievement_filter_friends_only);
   }
else
   {
   achievement_load_leaderboard("leaderboard1id", 1, 100, achievement_filter_friends_only);
   }

Esto enviará una solicitud de información en la tabla de clasificación dada y generará una devolución de llamada asincrónica con el async_load ds_map que contiene los siguientes datos:

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_leaderboard_info
   {
   var lbid = ds_map_find_value(async_load,"leaderboardid");
   if lbid == "CgkIs9_51u0PEAIQBw" || lbid == "leaderboard1id"
      {
      global.numentries = ds_map_find_value(async_load,"numentries");
      for(var i = 0; i < numentries; i++;)
         {
         global.playername[i] = ds_map_find_value(async_load, "Player" + string(i));
         global.playerid[i] = ds_map_find_value(async_load, "Playerid" + string(i));
         global.playerrank[i] = ds_map_find_value(async_load, "Rank" + string(i));
         global.playerscore[i] = ds_map_find_value(async_load, "Score" + string(i));
         achievement_get_pic(global.playerid[i]);
         }
      }
   }

El código anterior verifica el resultado ds_map en el evento asincrónico social y si su "id" coincide con la constante que se comprueba, comprueba para asegurarse de que la tabla de clasificación que se envía coincide con la llamada de función original antes de recorrer el mapa y almacenar todos los valores diferentes en varias matrices. También solicita una imagen para cada una de las entradas en el mapa (ver achievement_get_pic )
