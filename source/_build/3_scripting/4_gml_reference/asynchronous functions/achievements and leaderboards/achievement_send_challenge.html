enos_enviar_el desafío


Sintaxis:

achievement_send_challenge(playerid, challengeid, score, type, message)

Argumento Descripción
playerid La ID única del jugador a desafiar.
challengeid El ID de desafío único (como se encuentra en el panel del proveedor).
score La puntuación a batir.
type el tipo de desafío que se emitirá: una de las dos constantes que se indican a continuación.
message El mensaje a adjuntar al reto.


Devoluciones:

N/A


Descripción

Esta función enviará un desafío a través de la red al jugador elegido. Puedes obtener el ID de jugador usando el achievement_load_friends o la achievement_load_leaderboard funciones y también debe proporcionar el desafío, que es el valor único dado el desafío cuando lo creó en su panel de control de iTunes Connect o Google Play. También debe proporcionar una puntuación y un breve mensaje de texto, así como establecer el tipo de desafío. Esta puede ser una de las siguientes constantes:

  • achievement_type_score_challenge - Un desafío basado en el valor de puntuación.
  • achievement_type_achievement_challenge - Un reto basado en un logro.

Esta función activará un evento social asíncrono de devolución de llamada para el jugador que recibirá el desafío, y en este caso tendrá un ds_map referenciado en la variable async_load. La clave de identificación de este ds_map se utiliza para identificar la devolución de llamada correcta (puede haber más de una función de activación para cualquier evento asíncrono dado), y se emparejará con la constante achievement_challenge_received así como un número de otros pares clave / valor para cada jugador. El contenido exacto del mapa se muestra a continuación:

NOTA: el usuario también puede recibir notificaciones de brindis por los desafíos recibidos, pero esto dependerá de los valores que haya elegido utilizando la función. achievement_show_challenge_notifications.
  1. " id " - Para esta función debería ser achievement_leaderboard_info

  2. " playerid " - La identificación del jugador para el desafío.

  3. " issuerid ": el ID del emisor para el desafío.

  4. " state ": el estado del desafío, que tendrá un valor de 0 - 3 (como una cadena) para inválido, pendiente, completado o rechazado.

  5. " message " - El mensaje de texto para desafío.

  6. " completeddate ": la fecha de finalización del desafío.

  7. " issueddate ": la fecha de emisión para impugnación.

  8. " type " - El tipo de desafío dado. Puede ser una de las dos constantes:
    • achievement_type_score_challenge - Un desafío basado en el valor de puntuación.
    • achievement_type_achievement_challenge - Un reto basado en un logro.
  9. " identificador ": la cadena de identificación para el desafío, según lo establecido en el panel del proveedor

  10. " puntaje ": el puntaje vinculado con el desafío (si corresponde).


Ejemplo extendido:

Para enviar un desafío a través de la red tendría este código:

achievement_send_challenge(global.playerid[0], global.challengeid[0], score, achievement_type_score_challenge, "Beat that sucker!");

Esta solicitud activará el evento social en tu juego para el jugador al que se dirigió el desafío, y esto se puede abordar de la siguiente manera:

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_challenge_received
   {
   player_id = ds_map_find_value(async_load, "playerid");
   issuer_id = ds_map_find_value(async_load, "issuerid");
   state = ds_map_find_value(async_load, "state");
   message = ds_map_find_value(async_load, "message");
   date_completed = ds_map_find_value(async_load, "completeddate");
   date_issued = ds_map_find_value(async_load, "issueddate");
   ach_type = ds_map_find_value(async_load, "type");
   ach_ident = ds_map_find_value(async_load, "identifier");
   ach_score = ds_map_find_value(async_load, "score");
   }

El código anterior comprueba la devolución. ds_map en el evento asíncrono social y si su "id" coincide con la constante que se está verificando, luego extrae los valores relevantes para cada una de las claves en el mapa y las almacena en variables para uso futuro.
