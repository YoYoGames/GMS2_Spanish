logro_send_challenge


Sintaxis:

achievement_send_challenge(playerid, challengeid, score, type, message)

Argumento Descripción
playerid La identificación única del jugador para desafiar.
challengeid El ID de desafío único (como se encuentra en el tablero del proveedor).
score El puntaje a batir.
type el tipo de desafío que se emitirá: una de las dos constantes que se detallan a continuación.
message El mensaje para unirme al desafío.


Devoluciones:

N/A


Descripción

Esta función enviará un desafío a través de la red al jugador elegido. Puedes obtener el playerid usando el achievement_load_friends o el achievement_load_leaderboard funciones y también debe proporcionar el challengeid, que es el valor único dado el desafío cuando lo creó en su tablero iTunes Connect o Google Play. También debe proporcionar una puntuación y un breve mensaje de texto, así como establecer el tipo de desafío. Esta puede ser una de las siguientes constantes:

  • achievement_type_score_challenge - Un desafío basado en el valor del puntaje.
  • achievement_type_achievement_challenge - Un desafío basado en un logro.

Esta función desencadenará un evento social asincrónico de devolución de llamada para el jugador que recibirá el desafío, y en este caso tendrá un ds_map referenciado en la variable async_load. La clave de identificación de esto ds_map se usa para identificar la devolución de llamada correcta (puede haber más de una función de activación para cualquier evento asíncrono dado), y se emparejará con la constante achievement_challenge_received así como una cantidad de otros pares clave / valor para cada jugador. Los contenidos exactos del mapa se muestran a continuación:

NOTA: que el usuario también puede recibir notificaciones de tostado por desafíos recibidos, pero esto dependerá de los valores que haya elegido utilizando la función achievement_show_challenge_notifications.
  1. " id " - Para esta función, debe ser achievement_leaderboard_info

  2. " playerid ": la identificación del jugador para el desafío.

  3. " issuerid ": la identificación del emisor para el desafío.

  4. " estado ": el estado del desafío, que tendrá un valor de 0 - 3 (como una cadena) para inválido, pendiente, completado o rechazado.

  5. " mensaje " - El mensaje de texto para desafío.

  6. " completeddate ": la fecha de finalización del desafío.

  7. " issueddate " - La fecha de emisión del desafío.

  8. " tipo " - El tipo de desafío dado. Puede ser una de dos constantes:
    • achievement_type_score_challenge - Un desafío basado en el valor del puntaje.
    • achievement_type_achievement_challenge - Un desafío basado en un logro.
  9. " identificador ": la cadena de identificación para el desafío, tal como se establece en el tablero del proveedor

  10. " puntaje " - El puntaje empatado con el desafío (si corresponde).


Ejemplo extendido:

Para enviar un desafío a través de la red, tendrías este código:

achievement_send_challenge(global.playerid[0], global.challengeid[0], score, achievement_type_score_challenge, "Beat that sucker!");

Esta solicitud activará el Evento social en tu juego para el jugador al que se dirigió el desafío, y esto se puede tratar de la siguiente manera:

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_challenge_received
   {
   player_id = ds_map_find_value(async_load, "playerid");
   issuer_id = ds_map_find_value(async_load, "issuerid");
   state = ds_map_find_value(async_load, "state");
   message = ds_map_find_value(async_load, "message");
   date_completed = ds_map_find_value(async_load, "completeddate");
   date_issued = ds_map_find_value(async_load, "issueddate");
   ach_type = ds_map_find_value(async_load, "type");
   ach_ident = ds_map_find_value(async_load, "identifier");
   ach_score = ds_map_find_value(async_load, "score");
   }

El código anterior verifica el resultado ds_map en el evento asincrónico social y si su "id" coincide con la constante que se está verificando, luego extrae los valores relevantes para cada una de las claves en el mapa y las almacena en variables para uso futuro.
