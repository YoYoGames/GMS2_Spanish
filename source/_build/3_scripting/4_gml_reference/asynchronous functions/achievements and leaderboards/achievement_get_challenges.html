aprovechamiento


Sintaxis:

achievement_get_challenges()


Devoluciones:

N/A


Descripción

Esta función enviará una solicitud al servidor para obtener información sobre todos los desafíos actuales y activará un evento social asíncrono de devolución de llamada que contiene el async_load Mapa rellenado con los pares clave / valor relevantes. La clave de identificación de este ds_map se utiliza para identificar la devolución de llamada correcta (puede haber más de una función de activación para cualquier evento asíncrono dado), y se emparejará con la constante achievement_challenge_list_received así como una serie de otros pares clave / valor para cada desafío. El contenido exacto del mapa se muestra a continuación:

  1. " id " - Para esta función debería ser achievement_challenge_list_received

  2. " numchallenges ": la cantidad de desafíos (locales y remotos) disponibles actualmente.

  3. " ChallengeNplayerid ": la identificación del jugador para el desafío, donde "N" es un número entero, EG: "Challenge5playerid" es la identificación del jugador para el quinto desafío de la lista.

  4. " ChallengeNissuerid ": la identificación de la persona que emitió el desafío, donde "N" es un número entero, EG: "Challenge2issuerid" es la identificación del emisor para el segundo desafío en la lista.

  5. " ChallengeNstate ": el estado del desafío "N", que tendrá un valor de 0 - 3 (como una cadena) para inválido, pendiente, completado o rechazado.

  6. " ChallengeNmessage ": el mensaje de texto para el desafío "N".

  7. " ChallengeNissueddate " - La fecha de emisión para el desafío "N".

  8. " ChallengeNcompleteddate " - La fecha de finalización del desafío "N".

  9. " ChallengeNtype " - El tipo de desafío dado. Puede ser una de las dos constantes:
    • achievement_type_score_challenge - Un desafío basado en el valor de puntuación.
    • achievement_type_achievement_challenge - Un reto basado en un logro.
  10. " ChallengeNidentifier ": la cadena de identificación para el desafío.

  11. " ChallengeNscore " - La puntuación empatada con el desafío.
NOTA: Esta función es solo para iOS.


Ejemplo extendido:

El siguiente código probablemente se llamará después de que el jugador haya iniciado sesión en su cuenta de juego usando achievement_login:

achievement_get_challenges();

Esto enviará una solicitud de la información de desafío actual y generará una devolución de llamada asíncrona con el especial async_load ds_map que contiene los siguientes datos:

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_challenge_list_received
   {
   var numentries = ds_map_find_value(async_load,"numchallenges");
   for(var i = 0; i < numentries; i++;)
      {
      player_id[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"playerid");
      issuer_id[i] = ds_map_find_value(async_load, "Challenge" + string(i) +"issuerid");
      state[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"state");
      message[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"message");
      date_completed[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"completeddate");
      date_issued[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"issueddate");
      ach_type[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"type");
      ach_ident[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"identifier");
      ach_score[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"score");
   }

El código anterior comprueba la devolución. ds_map en el Evento asíncrono social y si su "id" coincide con la constante que se está verificando, entonces recorre el mapa almacenando todos los valores diferentes en una serie de arreglos. Esta información se puede usar, por ejemplo, para crear tus propias páginas de desafío personalizadas en el juego.
