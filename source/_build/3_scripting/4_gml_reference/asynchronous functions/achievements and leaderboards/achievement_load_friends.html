achievement_load_friends


Sintaxis:

achievement_load_friends()


Devoluciones:

N/A


Descripción

Esta función enviará una solicitud al servidor para obtener información sobre todos los usuarios que hayan iniciado sesión y activará un evento social asincrónico de devolución de llamada que contiene el async_load mapa poblado con los pares clave / valor relevantes. La clave de identificación de esto ds_map se usa para identificar la devolución de llamada correcta (puede haber más de una función de activación para cualquier evento asíncrono dado), y se emparejará con la constante achievement_friends_info así como una cantidad de otros pares clave / valor para cada amigo. Los contenidos exactos del mapa se muestran a continuación:

  1. " id " - Para esta función, debe ser achievement_friends_info

  2. " FriendN ": el nombre del amigo, donde "N" es un valor entero que corresponde a su posición dentro de la lista de amigos.

  3. " FriendidN ": la identificación de usuario única del amigo, "N".
NOTA: esta función es solo para iOS.


Ejemplo extendido:

El siguiente código probablemente se invocará después de que el jugador haya iniciado sesión en su cuenta de juego usando achievement_login para obtener una lista de todos los amigos de los usuarios:

achievement_friends_info();

Esto enviará una solicitud de información sobre los amigos de los usuarios y generará una devolución de llamada asincrónica con el async_load ds_map que contiene los siguientes datos:

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_friends_info
   {
   var numfriends = ds_map_find_value(async_load, "numfriends");
   global.numfriends = numfriends;
   for(var i=0; i < numfriends; i++;)
      {
      global.friendname[i] = ds_map_find_value(async_load, "Friend" + string(i));
      global.friendid[i] = ds_map_find_value(async_load, "Friendid" + string(i));
      achievement_get_pic(global.friendid[i]);
      }
   }

El código anterior verifica el resultado ds_map en el evento asincrónico social y si su "id" coincide con la constante que se está verificando, recorre el mapa almacenando todos los valores diferentes en un número de matrices y solicita una imagen para cada una de las entradas en el mapa (ver achievement_get_pic )
