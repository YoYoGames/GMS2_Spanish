show_message_async


Descripción

Esta función abre una ventana y muestra al usuario el mensaje que define en la función. Esta es una función asíncrona y, como tal, GameMaker Studio 2 no bloquea el dispositivo en el que se está ejecutando mientras espera respuesta, sino que continúa ejecutando los eventos de manera normal. Una vez que el usuario ha pulsado el botón "Aceptar", un evento de diálogo asíncrono está activado el cual, durante la duración de dicho evento solamente, tendrá un ds_map almacenado en la variable async_load.

Este mapa contendrá las dos claves, " id " y " status ", siendo " id " el valor que fue devuelto por la función cuando se llamó, y el " status " será true para presionar el botón "Aceptar", o false si el mensaje fue cancelado (cuando esté disponible, ya que no todas las plataformas de destino permiten la cancelación de diálogos).


Sintaxis:

show_message_async(string);

Argumento Descripción
String El mensaje a mostrar al usuario.


Devoluciones:

Real


Ejemplo extendido:

El evento de presionar con el botón izquierdo del ratón (por ejemplo) del objeto que muestra el mensaje tendría el siguiente código:

msg = show_message_async("You will now be taken to the store");

Lo anterior mostrará un mensaje con la cadena dada. La identificación del mensaje se almacena en la variable " msg " y se usará en el evento de Diálogos asíncronos como se muestra a continuación:

var i_d, stat;
i_d = ds_map_find_value(async_load, "id");
if i_d == msg
   {
   if ds_map_find_value(async_load, "status")
      {
      url_open("https://play.google.com/store");
      }
   }

El código anterior verifica la clave " id " de la devuelta ds_map contra el valor almacenado en la variable " msg ". Si son iguales, luego verifica si se presionó el botón "Aceptar" (en lugar de cerrar o cancelar la ventana) y si vuelve a aparecer. true se abre una url.
