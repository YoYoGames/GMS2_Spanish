get_login_async


Descripción

Esta función abre una ventana que le pide al usuario que ingrese un nombre de usuario y contraseña. Estos argumentos se pueden establecer como una cadena vacía o puede almacenar valores previamente ingresados ​​para usarlos si lo desea. Esta es una función asíncrona, y como tal, GameMaker Studio 2 no bloquea el dispositivo que se está ejecutando mientras se espera la respuesta, sino que más bien continúa ejecutando eventos de forma normal. Una vez que el usuario ha ingresado los detalles y presionado el botón "Aceptar", se desencadena un evento de Interacción del usuario asíncrono que, durante la duración de ese solo evento, tendrá un ds_map almacenado en la variable async_load.

Este mapa contendrá las dos claves, "nombre de usuario" y "contraseña", con la correspondiente entrada de usuario almacenada en cada una. Como es un ds_map que ha sido creado, esto puede ser utilizado, por ejemplo, por el json_encode función lista para ser enviada a un servidor o escrita en un archivo en el dispositivo elegido.

Vale la pena señalar que esta función devolverá un número de índice para el ds_map creado que luego se puede verificar en el evento correspondiente para que pueda "orientar" un ds_map específico en caso de que espere que se devuelva más de un mapa asincrónico (quizás de alguna otra función). Fuera de los eventos asincrónicos, async_load ds_map devolverá un valor "id" de -1.


Sintaxis:

get_login_async(name, password);

Argumento Descripción
username El nombre de usuario predeterminado
password La contraseña predeterminada


Devoluciones:

Real


Ejemplo extendido:

El evento de creación (por ejemplo) del objeto que controla el inicio de sesión de nuestro usuario tendría el siguiente código:

ini_open("Profile.ini");
u = ini_read_string("User","0","");
p = ini_read_string("User","1","");
ini_close();
login = get_login_async(u,p);

El código anterior abre un archivo ini (o crea uno si no existe) y obtiene el nombre y la contraseña almacenados en ese archivo. Si no existen, se devuelve el valor predeterminado de una cadena vacía (""). Estos valores se usan luego en get_login_async() para preguntar al usuario los detalles de su nombre de usuario y contraseña, con el índice de solicitud almacenado en la variable "iniciar sesión". Tenga en cuenta que mientras espera que el usuario brinde la información necesaria, el juego y sus eventos continuarán ejecutándose normalmente.

Ahora que el código asíncrono se ha desactivado, debemos verificar el valor de retorno en el evento asíncrono para Diálogos de la siguiente manera:

if ds_map_find_value(async_load, "id") == login
   {
   u = ds_map_find_value(async_load, "username");
   p = ds_map_find_value(async_load, "password");
   }

El código anterior verifica la clave "id" de async_load ds_map y si tiene el mismo valor que el almacenado en la variable de "inicio de sesión", los detalles del mapa se leen en las variables correspondientes que luego puede almacenar o usar para verificar contra valores de base de datos, etc...
