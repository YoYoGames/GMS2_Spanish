http_post_string


Descripción

En informática, se utiliza una solicitud posterior cuando el cliente necesita enviar datos al servidor como parte de la solicitud de recuperación, como al cargar un archivo o enviar un formulario completo, y lo mismo se aplica a esta función en GameMaker Studio 2. En contraste con el método de solicitud http_get donde solo se envía una URL al servidor, http_post_string() también incluye una cadena que se envía al servidor y puede dar como resultado la creación de un nuevo recurso o la actualización de un recurso existente o ambos. Cabe señalar que los parámetros de solicitud HTTP (los bits a veces "añadidos" al final de una URL cuando se envía un formulario en una página web) son perfectamente aceptables cuando se utiliza esta función también.

Este evento generará una "devolución de llamada" que será captada por cualquier evento HTTP, en cuyo caso generará un ds_map (más comúnmente conocido como un "diccionario") que es exclusivo de este evento y se almacena en la variable especial async_load. Este ds_map contendrá diferentes valores dependiendo de si hay datos devueltos o no. Por ejemplo, si ha solicitado un archivo, el evento se activará varias veces a medida que se reciba cada paquete de datos para que pueda mostrar una barra de progreso, por ejemplo. La estructura general para el ds_map será la siguiente:

  • id: la ID que se devolvió desde el comando. Si disparas una serie de http_ Luego, debe saber a cuál le está respondiendo y, por lo tanto, debe usar este valor para compararlo con el valor que guardó cuando envió originalmente la solicitud para encontrar el correcto.

  • estado: Devuelve un valor de menos de 0 para un error, 0 para completar y 1 para recibir paquetes (ver más abajo para más detalles).

  • resultado: los datos recibidos (solo cadena).

  • URL: la URL completa que solicitó.

  • http_status: el código de estado HTTP sin procesar (si está disponible). Esto devuelve el código de estado web estándar para la mayoría de los navegadores, por ejemplo: 304 para "No modificado" o 204 para "Sin contenido", etc.

Si se devuelven varios paquetes a su juego, la clave de "estado" de devolución de llamada devolverá 1, en cuyo caso el ds_map tendrá las siguientes claves adicionales:

  1. "contentLength": este es el tamaño del archivo que el servidor web ha dicho que debe esperar recibir (puede ser -1 si el servidor no devuelve estos datos).

  3. "sizeDownloaded": el tamaño de los datos que ya se han descargado.
NOTA: Debe tener en cuenta que debido a la protección XSS en los navegadores, las solicitudes e intentos de cargar recursos de varios dominios están bloqueadas y puede parecer que devuelven resultados en blanco. Consulte la sección sobre Problemas de dominio cruzado para obtener más detalles.


Sintaxis:

http_post_string(url, string);

Argumento Descripción
url La dirección web del servidor del que desea obtener información
string La cadena que desea enviar a la URL especificada


Devoluciones:

Real


Ejemplo extendido:

los http_post_string se puede llamar a la función desde cualquier evento, y dado que es asíncrona, la devolución de llamada puede ser casi instantánea o puede tomar varios segundos. Llamar a la función es simple y se vería así:

var str = "name=" + global.player_name + "&score=" + string(global.player_score);
post = http_post_string("http://www.angusgames.com/game?game_id=" + string(global.game_id), str);

El código anterior envía una solicitud de recuperación a la URL especificada con los parámetros dados, así como el envío de los datos adicionales (nombre del jugador y puntaje en forma de codificación URL) almacenados en la variable str. Esto desencadenará todos los Http Events asíncronos definidos si se recibe una devolución de llamada, y puede verificar el "id" devuelto contra el almacenado en la variable post para asegurarse de que ejecuta el código correcto en caso de que haya utilizado varias http_post_string() funciones. El siguiente código de ejemplo muestra cómo se haría esto:

var r_str = "null";
if ds_map_find_value(async_load, "id") == post
   {
   if ds_map_find_value(async_load, "status") == 0
      {
      r_str = ds_map_find_value(async_load, "result");
      }
   }

El código anterior primero verificará la identificación del ds_map que se ha creado, luego verificará el estado de la devolución de llamada. Si el valor es igual a 0 (éxito de señalización), el resultado de la devolución de llamada se almacenará en una variable para uso futuro, de lo contrario la variable simplemente mantendrá un valor predeterminado (en este caso "nulo").
