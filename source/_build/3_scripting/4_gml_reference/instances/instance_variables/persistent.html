persistente


Descripción

Esta variable se puede leer para averiguar si la instancia está marcada como persistent o no, o puede usarse para establecer la persistencia a true (persistente) o false (no persistente) para la instancia. Una instancia persistente es aquella que será "transferida" de una sala a otra, lo que significa (por ejemplo) que solo debe crearse una vez al inicio del juego y estará presente en todas las demás salas. Se debe tener cuidado con la persistencia ya que es fácil perder la pista de las instancias persistentes que pueden conducir a problemas más adelante en el desarrollo del juego.

Un objeto persistente todavía tendrá activados los eventos Game Start, Game End, Room Start y Room End, sin embargo, si reinicia el juego (con, por ejemplo, el game_restart() función) todos los objetos persistentes serán eliminados y solo existirán cuando sean creados nuevamente por el juego. También tenga en cuenta que si desactiva un objeto persistente, ya no pasará de una habitación a otra a menos que se vuelva a activar antes de que se desencadene el evento de fin de habitación.


Sintaxis:

persistent;


Devoluciones:

Boolean


Ejemplo:

if !persistent
   {
   persistent = true;
   }

El código anterior verificará la instancia para ver si la persistencia está marcada como false y si la instancia NO es persistente, el indicador de persistencia se establece en true.
