alarma


Descripción

Esta matriz de 1 dimensión se usa para obtener el valor actual de las alarmas que pueda tener la instancia, o se puede usar para establecer esas alarmas. Hay doce alarmas integradas en cada instancia de un objeto, y cada una tiene su propio evento que se ejecutará cuando esta variable llegue a 0. Se debe tener en cuenta que la alarma no finaliza en ese punto (aunque el evento se haya activado) como el siguiente paso bajará a -1, por lo que si necesita detener una alarma por cualquier razón, debe establecer su valor de matriz en -1 no 0. Los tiempos de alarma se calculan en pasos de juego, con un valor de 30 en 30 pasos etc...

NOTA: No se ejecutará una alarma sin acciones ni código. Sin embargo, incluso con solo un comentario y sin código, la alarma contará hacia abajo.

Puede configurar el conjunto de alarmas directamente (y recuperar el valor de alarma actual directamente usando el conjunto), pero en algunas circunstancias este no es el método más apropiado, por lo que también puede usar la función alarm_set() para configurar una alarma, y ​​la función alarm_get() para obtener el valor de una alarma


Sintaxis:

alarm[val];


Devoluciones:

Real (steps in the alarm, -1 if the alarm isn't running)


Ejemplo:

if canshoot
   {
   if keyboard_check_pressed(vk_space)
      {
      canshoot = false;
      alarm[0] = room_speed;
      instance_create_layer(x, y, "Bullets", obj_Bullet);
      }
   }

El código anterior comprueba una variable y luego si es verdadera, comprueba si se presiona una tecla y, si eso es cierto, finalmente establece la variable en falso, establece una alarma (donde se establecerá en verdadero de nuevo) y crea una instancia.
