image_index


Descripción

Un sprite se compone de una o más subimágenes que pueden hacer que el sprite parezca animado cuando cambian de una a otra, o pueden intercambiarse en el código para dar diferentes "estados", al igual que un botón tiene en ventanas. Si el sprite está animado, puedes obtener el fotograma actual de la animación marcando image_index variable, o si desea cambiar el estado de un sprite estático, puede seleccionar una nueva imagen secundaria configurando esta variable en la subimagen deseada para el sprite. Tenga en cuenta que los cambios en esta variable sea visible, la instancia debe tener ya sea ningún caso sorteo (y por lo GameMaker Studio 2 por defecto dibujar el sprite) o ser dibujado usando una de las funciones de dibujo extendidos como draw_self() o draw_sprite_ext().

Tenga en cuenta que al usar sprites de animación esquelética, aún puede obtener y configurar el image_index valores - vea la función skeleton_animation_get_duration para ejemplos de cómo hacer esto.


Sintaxis:

image_index;


Devoluciones:

Real


Ejemplo:

if image_speed > 0
   {
   if image_index > image_number - 1 instance_destroy();
   }

El código anterior comprueba si el sprite está animando, y si lo está, comprueba si el actual image_index es mayor que el número de subimágenes y, si lo es, destruye la instancia.
