Variables de instancia


Cuando cree un nuevo objeto, vendrá con ciertas variables ya inicializadas con valores predeterminados. Estas variables se denominan variables de instancia, ya que se crearán para cada instancia que coloque del objeto en sus salas de juego, y una vez que se crea una instancia, los valores de estas variables son exclusivos de esa instancia y solo de esa instancia. Algunas de estas variables las usarás mucho, como x y y, mientras que otros se requieren menos dependiendo de lo que quieras hacer, pero en general son muy útiles y, cuando sea posible, deberías buscar usarlos en lugar de crear tus propias variables personalizadas.

La mayoría de las variables de instancia se pueden establecer y leer, lo que le permite cambiar las propiedades y el comportamiento de una instancia simplemente al ajustar el valor de una determinada variable. Puede evitar que una instancia dibuje, por ejemplo, simplemente configurando visible variable incorporada a false. A continuación puede encontrar las diferentes variables que se inicializan para todas las instancias de todos los objetos en su juego.

Estas variables tratan con las propiedades generales de la instancia:
  • carné de identidad
  • visible
  • sólido
  • persistente
  • profundidad
  • capa
  • alarma
  • object_index
  • salud
  • vive
  • Puntuación

Estas variables están relacionadas con el sprite asignado a la instancia y se pueden usar para cambiar lo que se dibuja y cómo:
  • sprite_index
  • sprite_width
  • sprite_height
  • sprite_xoffset
  • sprite_yoffset
  • image_alpha
  • image_angle
  • image_blend
  • image_index
  • número de imagen
  • image_speed
  • image_xscale
  • imagen_escala

Estas variables tratan con la máscara de colisión:
  • mask_index
  • bbox_bottom
  • bbox_left
  • bbox_right
  • bbox_top

Estas variables tratan con la posición de la instancia y el movimiento:
  • dirección
  • fricción
  • gravedad
  • direccion de gravedad
  • hspeed
  • vspeed
  • velocidad
  • xstart
  • ystart
  • X
  • y
  • xprevio
  • anterior

Estas variables están relacionadas con las rutas y cómo la instancia interactúa con una, si se asigna:
  • path_index
  • path_position
  • path_positionprevious
  • path_speed
  • path_scale
  • path_orientation
  • path_endaction

Estas variables son para configurar una instancia para usar una línea de tiempo:
  • timeline_index
  • timeline_running
  • timeline_speed
  • timeline_position
  • timeline_loop
