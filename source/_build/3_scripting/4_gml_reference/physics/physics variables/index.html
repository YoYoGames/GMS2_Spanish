Variables de Física

Al igual que con el sistema de colisiones y movimientos "tradicionales", el sistema de física también tiene una serie de variables de instancia asignadas a cada instancia de un objeto cuando se crea. Algunas de estas propiedades pueden ser definidas por el usuario en GML, pero muchas son de solo lectura y aún más solo son aplicables en el evento de colisión.

Las siguientes variables existen en todos los objetos habilitados por física y pueden leerse y escribirse (es decir, cambiarse) usando código de instancias y scripts:

  • phy_active
  • phy_angular_velocity
  • phy_angular_damping
  • phy_linear_velocity_x
  • phy_linear_velocity_y
  • phy_linear_damping
  • phy_speed_x
  • phy_speed_y
  • phy_position_x
  • phy_position_y
  • phy_position_xprevious
  • phy_position_yprevious
  • phy_rotation
  • phy_fixed_rotation
  • phy_bullet

Ciertas variables son variables de sólo lectura, lo que significa que la información que contienen no se puede cambiar a través de código o scripts de ejemplo (pero se pueden cambiar internamente por GameMaker Studio 2 por lo que estos son variables y no constantes). La siguiente tabla muestra las variables de solo lectura que existen en todos los objetos habilitados por física:

  • phy_speed
  • phy_com_x
  • phy_com_y
  • phy_dynamic
  • phy_kinematic
  • phy_inertia
  • phy_mass
  • phy_sleeping

Finalmente, hay una serie de variables temporales que existen en los objetos habilitados para física y se crean solo en el evento de colisión. Estas variables son como las enumeradas anteriormente y son de solo lectura:

  • phy_collision_points
  • phy_collision_x
  • phy_collision_y
  • phy_col_normal_x
  • phy_col_normal_y
