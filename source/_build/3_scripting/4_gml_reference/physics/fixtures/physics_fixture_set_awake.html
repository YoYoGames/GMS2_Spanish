physics_fixture_set_awake

Esta función le dice a GameMaker Studio 2 que el dispositivo debe tratarse como si estuviera despierto o no.

Sintaxis:

physics_fixture_set_awake(fixture, state)


Argumento Descripción
fixture el índice del accesorio
flag si un accesorio está despierto ( true ) o no ( false )


Devoluciones: N / A


Descripción

Cuando comienzas una sala con física o vincula accesorios a instancias, los dispositivos siempre se consideran "despiertos"... es decir, procesan eventos e interactúan con las instancias circundantes. Sin embargo, esto a veces puede ocasionar problemas, especialmente si tiene varias instancias con accesorios que están uno al lado del otro cuando se inicia una habitación (piense en una torre hecha de varias instancias con accesorios) ya que estando "despiertos" se moverán y posiblemente cambie de posición debido al comienzo repentino de la gravedad y las colisiones que los afectan. En estos casos, puede usar esta función para marcar el dispositivo como "dormido", en cuyo caso no se procesará nada cuando se cree por primera vez en la sala.


Ejemplo:

physics_fixture_set_awake(fix_Cloud, true);

El código anterior indica que el dispositivo está "despierto" cuando está vinculado a una instancia.
