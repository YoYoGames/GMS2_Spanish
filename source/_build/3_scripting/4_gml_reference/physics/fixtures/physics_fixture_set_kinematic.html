physics_fixture_set_kinematic

Esta función indica que el dispositivo está indicado para comportarse cinemáticamente.

Sintaxis:

physics_fixture_set_kinematic(fixture)


Argumento Descripción
fixture el índice del accesorio


Devoluciones: N / A


Descripción

Habrá ciertas instancias en un juego basado en la física que desee mover, pero no desea que actúen fuerzas como la gravedad, ni fuerzas provocadas por colisiones con objetos dinámicos (piense en mover plataformas en un juego de plataforma, por ejemplo ) Para tales objetos, simplemente establecer la densidad de un dispositivo en 0 significará que la física supondrá que el objeto está destinado a ser estático. Sin embargo, ¿debería configurar el phy_speed_x, phy_speed_y o phy_angular_velocity variables para una instancia que ha tenido un accesorio vinculado a él, se convertirá en un objeto cinemático y comenzará el movimiento. Cambiar la instancia de simulación media estática a cinemática puede ser costoso en términos de procesamiento (dependiendo del estado de las instancias en el mundo de la física) y, por lo tanto, se recomienda configurar el dispositivo para que sea cinemático utilizando esta función antes de vincularlo a un ejemplo, para que no se vea afectado por colisiones y fuerzas (como la gravedad), pero puede moverse con velocidad lineal y / o angular constante.


Ejemplo:

physics_fixture_set_kinematic(fix_Cloud);

El código anterior indicará que el dispositivo indexado en "fix_Cloud" debe marcarse como cinemático.
