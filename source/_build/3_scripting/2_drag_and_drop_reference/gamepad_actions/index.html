Biblioteca de acciones de Gamepad


GameMaker Studio 2 tiene una serie de acciones dedicadas que se pueden utilizar para detectar controles analógicos y digitales desde múltiples juegos conectados. Estas acciones requieren que especifiques un valor de índice de gamepad, que es un número (que cuenta desde 0) que representa el gamepad que se selecciona. Tenga en cuenta que cuando un gamepad está conectado a su dispositivo (o se elimina), se desencadena un evento del sistema asíncrono (sin embargo, para tratar con esto, es posible que necesite usar código).

Las "ranuras" del gamepad están indexadas desde 0 con las ranuras 0 - 3 inclusive, que son solo para gamepads Xinput, es decir: controladores Xbox360 y compatibles. Sin embargo, también puede verificar las ranuras 4 - ii inclusive para los gamepads DirectInput, lo que significa que puede detectar muchos otros modelos de controlador cuando está conectado a través de estas ranuras.

Las acciones del gamepad disponibles se enumeran a continuación:

Gamepad Get Axis
Gamepad Get Trigger
Gamepad Get Count
Gamepad Get Connected
Gamepad Set Axis Deadzone
Si el botón Gamepad presionó
Si Gamepad abotona
Si se lanza el botón Gamepad


Compatibilidad


La siguiente lista muestra la compatibilidad actual entre las plataformas (tenga en cuenta que esto cambiará con futuras actualizaciones):

  • Windows es totalmente compatible con hasta un máximo de 11 dispositivos conectados permitidos a la vez. Tenga en cuenta que en Windows, las primeras 4 ranuras de gamepad (0 - 3) se manejan usando el dll XInput, lo que significa que solo los controladores XBox pueden ser 100% compatibles y que para otros tipos de controladores debe verificar el resto de las ranuras del gamepad (4 - 11).

  • Mac OS es compatible con hasta un máximo de 4 dispositivos conectados permitidos a la vez, y estos dispositivos SOLAMENTE pueden ser del tipo Playstation 3 o Xbox 360. Tenga en cuenta que la opción " Build for Mac AppStore " en Mac Game Options debe ser APAGADO para que el soporte de la almohadilla funcione.

  • Ubuntu también es compatible con la entrada de GamePad, pero es posible que deba instalar bibliotecas adicionales desde el repositorio de Ubuntu. Puede hacerlo fácilmente abriendo un terminal de línea de comando y escribiendo lo siguiente:

    sudo apt-get install jstest-gtk
    sudo apt-get install joystick.

    Esto instalará el soporte de GUI para el joystick así como también el joystick.

  • Los juegos HTML5 admiten gamepads en la mayoría de los principales navegadores, excepto Safari.

  • La compatibilidad con Gamepad también se extiende a iOS con el gabinete iCade. El eje izquierdo se asigna al controlador stick (aunque la entrada es digital, no analógica), los cuatro botones "cara" se asignan a los botones frontales del gabinete, y los cuatro botones laterales se asignan a los que se encuentran en la parte posterior del gabinete.

  • Android exportación de Android admite controladores NYKO y controladores Bluetooth genéricos (incluido el OUYA), pero solo cuando están habilitados, lo que significa que deberá marcar la opción iCade / Bluetooth en la sección General de Android Opciones de juegos de Android. Requieren API nivel 12 para que funcionen por completo y se debe tener en cuenta que GameMaker Studio 2 se registrará como conectado a cualquier dispositivo Bluetooth con el que esté emparejado, ya sea que esté realmente conectado o no. Por lo tanto, esto debe tenerse en cuenta al asignar y verificar "espacios".

  • En PS4, si desea utilizar el seguimiento del panel táctil, debe usar las funciones para device_mouse_* botones (no hay acciones DnD ™ para esto).
