Biblioteca de estructura de datos


En los juegos, a menudo necesita almacenar información de una manera precisa y ordenada. Por ejemplo, puede que necesite almacenar listas de artículos que una persona transporta o puede querer almacenar una grilla de lugares que aún necesitan ser visitados. Ahora puede usar múltiples variables para esto, pero si quiere hacer operaciones más complicadas, como clasificar los datos o buscar un elemento en particular, necesita escribir grandes bloques de código de bloques de acción que pueden ser lentos de ejecutar, difíciles de depuración y un verdadero dolor para escribir!

Para remediar esto, GameMaker Studio 2 tiene una serie de estructuras de datos integradas a las que se puede acceder a través de funciones especializadas. Hay cuatro tipos diferentes de estructura de datos disponibles mediante Arrastrar y soltar (hay seis si usa GML ), cada uno con sus pros y sus contras dependiendo del tipo de información que desea almacenar y cómo desea manipularla. más tarde - los cuatro tipos son:

Liza Una lista es una estructura de datos que almacena la información secuencialmente a medida que se agrega. Los índices de lista comienzan en 0 y subirán para cada elemento que se agrega al final de la lista, aunque también puede insertar datos en una posición que no sea la última. A diferencia de la estructura de datos Stack (explicada a continuación), puede acceder a los datos desde cualquier índice de la lista en cualquier momento.
Mapas La estructura de datos del mapa es excepcionalmente útil ya que te permite almacenar claves y pares de valores. Por ejemplo, un personaje en su juego puede tener una cantidad de elementos diferentes (claves) y para cada elemento individual puede tener un número de ellos (valores), como en un juego de rol donde puede tener 10 pociones de salud, 5 pociones de maná y 100 de oro Los mapas mantienen todos estos pares juntos en un solo lugar y puede agregar pares al mapa y buscar el valor correspondiente a ciertas teclas también usando algunas funciones simples. Los mapas no están ordenados de ninguna manera (reconocible), lo que significa que para encontrar una determinada clave, puede que tenga que recorrer todo (lo cual es muy lento), y tampoco hay forma de mantener dos claves iguales, ni puede asignar una clave dos valores
Rejillas Una cuadrícula es una matriz bidimensional de valores donde se define su ancho y alto. Esto establece el número total de "celdas" de cuadrícula dentro de la cuadrícula (el ancho y el alto se multiplican) y estas celdas se utilizan para mantener diferentes valores de datos, que pueden ser cualquiera de los tipos de datos permitidos. Este tipo de estructura le permite establecer y recuperar el valor de las celdas en la grilla dando la posición x (columna) y y (fila) de la misma (ambas filas y columnas comienzan en 0 y van hasta el ancho de la grilla - 1 y la altura de la grilla -1). Al acceder a las estructuras de datos de la cuadrícula, siempre debe tratar de usar valores enteros para la posición de la celda, y GameMaker Studio 2 todos los índices que no sean enteros si no lo hace. Si eso no es lo que necesita, tendrá que hacer el redondeo usted mismo antes de pasar el índice que desea verificar.
Stacks Una estructura de datos de pila es la denominada estructura de último en entrar primero en salir (LIFO). Puede insertar valores en una pila y eliminarlos de nuevo sacándolos de la pila, y el valor que se insertó en la pila más recientemente es el primero que se saca de allí (solo piense en una pila de monedas, donde cada la moneda que agrega debe ser removida primero antes de que pueda llegar al resto de las monedas debajo). Las pilas se usan a menudo cuando hay interrupciones que manejar, o cuando se tienen funciones recursivas, o incluso cuando se construye una IA rudimentaria para tus juegos.

Esencialmente, todas las estructuras de datos funcionan de la misma manera: usted crea una estructura de datos y almacena su valor de índice en una variable. A continuación, utiliza este índice para hacer referencia a la estructura de datos en todas las demás llamadas a funciones que pueden realizar operaciones en él. Finalmente, una vez que termina, destruye la estructura de datos nuevamente para eliminarla de la memoria. Puedes usar tantas estructuras al mismo tiempo como necesites, y todas las estructuras pueden almacenar tanto cadenas como valores reales.

NOTA: Al igual que con todos los recursos dinámicos, las estructuras de datos ocupan la memoria y, por lo tanto, siempre deben destruirse cuando ya no son necesarias para evitar fugas de memoria que disminuirán la velocidad y, finalmente, bloquearán su juego.
