Mouse y biblioteca de acciones de teclado


La biblioteca de acciones del mouse y el teclado es donde puede encontrar las acciones necesarias para sondear y procesar la entrada desde el teclado o los dispositivos del mouse. Las acciones disponibles se enumeran a continuación:

Si el ratón presionado
Si el mouse hacia abajo
Si el ratón lanzado
Si el teclado presionado
Si el teclado hacia abajo
Si el teclado lanzado

Las otras tres acciones aquí se refieren al teclado virtual. Si necesita entrada de texto o numérica en su juego en un dispositivo con pantalla táctil, puede usar las acciones del teclado virtual que se proporcionan para usar el teclado del SO del dispositivo. Estas acciones activarán u ocultarán el teclado del sistema operativo y aceptarán la entrada de la misma forma que un teclado físico, por lo que las acciones normales del teclado de GameMaker Studio 2 deberían funcionar con los teclados virtuales (por ejemplo, la acción "Si se liberó una tecla"). y cualquier entrada del teclado virtual se almacenará en las diferentes variables del teclado, como keyboard_string o keyboard_last_char, etc...).

Abrir o cerrar el teclado virtual generará eventos asíncronos del sistema, en los que el async_load El mapa DS se llenará con los siguientes pares clave / valor:

  • " event_type ": el tipo de evento del sistema recibido, que será el " estado del teclado virtual " para los teclados virtuales.

  • " screen_height ": la altura del teclado virtual (en píxeles). Esto será 0 si el teclado es invisible.

  • " keyboard_status ": el estado actual del teclado, devuelto como una de las siguientes cadenas:

    • "ocultación"
    • "oculto"
    • "demostración"
    • "visible"

Las acciones de teclado virtual disponibles son:

Mostrar teclado virtual
Ocultar teclado virtual
Obtener la altura del teclado virtual
Si se muestra el teclado virtual
