Establecer variable de instancia


Descripción

Con esta acción puede establecer una de las variables integradas. Todas las instancias de todos los objetos tienen una cantidad de variables ya creadas dentro de ellas para contener información esencial como la posición x / y actual, la velocidad, la dirección, la visibilidad y muchas otras cosas. Las variables de instancia que puede usar son:

  • X Coordinate - La posición a lo largo del eje x en la habitación.

  • Coordenada Y: la posición a lo largo del eje y en la sala.

  • Dirección - La dirección de movimiento para la instancia (por defecto es 0, donde la derecha es 0 °, arriba es 90 °, la izquierda es 180 ° y abajo es 270 °).

  • Velocidad: la velocidad (en píxeles por cuadro de juego) a la que se está moviendo la instancia.

  • Velocidad horizontal: la velocidad horizontal (en píxeles por cuadro de juego) a la que se mueve la instancia. Una velocidad negativa significa que se mueve hacia la izquierda, una velocidad positiva es la correcta y el valor predeterminado es 0.

  • Velocidad vertical: la velocidad vertical (en píxeles por cuadro de juego) en la que se mueve la instancia. Una velocidad negativa significa que se mueve hacia arriba, una velocidad positiva está bajada y el valor predeterminado es 0.

  • Visible: la variable de alternancia de visibilidad (el valor predeterminado es true a menos que se establezca lo contrario en el Editor de objetos, configure en false hacer invisible). Tenga en cuenta que una instancia invisible aún ejecuta todos sus eventos, excepto el evento Draw.

  • Persistente: la variable de alternancia de persistencia (el valor predeterminado es false a menos que se establezca lo contrario en el Editor de objetos, configure en true para hacer persistente). Esto hará que una instancia "persista" de una habitación a otra, es decir: la instancia no se limpiará en el extremo de la sala, sino que se trasladará a la sala contigua. Tenga en cuenta que cuando se persiste una instancia, no llamará a su evento Crear ni Destruir, sino que llamará a los eventos de Limpieza, Final de sala y Inicio de sala.

  • Layer: la capa a la que se ha asignado la instancia.

  • Profundidad: el valor de profundidad asignado a la capa.

  • Sprite: el recurso de sprite asignado a la instancia.

  • Índice de imagen - El índice de imagen del sprite asignado. Este valor establece el marco del sprite asignado a la instancia, y si el valor es mayor que el número de fotogramas en la animación, se "reproducirá" de forma tal que un sprite con 5 fotogramas si se establece en el fotograma 7 realmente mostrará el fotograma 2.

  • Ángulo de imagen: el ángulo en el que se debe establecer la instancia (por defecto es 0, donde la derecha es 0 °, arriba es 90 °, la izquierda es 180 ° y abajo es 270 °).

  • Image Alpha - El valor alfa para la instancia (por defecto es 1) de 0 a 1, donde 0 es completamente transparente y 1 es completamente opaco.

  • Mezcla de imagen: el color que se combinará con el sprite de instancia (el valor predeterminado es blanco).

  • Velocidad de imagen: la velocidad de animación para la instancia.

  • Escala horizontal: la escala horizontal a la que se debe dibujar el sprite de la instancia (esto también afectará el cuadro delimitador de la instancia y las colisiones). Una escala de 1 indica que no hay escalado (1: 1), los valores más pequeños se reducirán (0.5, por ejemplo, tendrá la mitad del ancho del sprite), los valores más grandes se escalarán y los valores negativos reflejarán el sprite y lo escalarán a menos que el valor utilizado es exactamente -1 (en cuyo caso el sprite solo se refleja a lo largo del eje y sin escalar).

  • Escala vertical: la escala vertical a la que se debe dibujar el sprite de la instancia (esto también afectará el cuadro delimitador de la instancia y las colisiones). Una escala de 1 indica que no hay escalado (1: 1), los valores más pequeños se reducirán (0.5, por ejemplo, tendrá la mitad de la altura del sprite), los valores más grandes se escalarán y los valores negativos cambiarán el sprite y lo escalarán a menos que el valor utilizado es exactamente -1 (en cuyo caso el sprite simplemente se voltea a lo largo del eje y sin escalar).

  • Máscara: el índice de máscara para la instancia. El valor predeterminado es -1 (lo que significa que la instancia usará la máscara de colisión para el índice de sprite actualmente asignado), pero si se establece en cualquier otro recurso de sprite, se usará la máscara de colisión para esa instancia en lugar del sprite asignado al sprite índice.

  • Fricción: la fricción para usar cada cuadro del juego para ralentizar la instancia. Este valor simplemente se deducirá del valor de velocidad actual de cada cuadro de juego (o se agregará si el valor de velocidad es negativo).

  • Gravedad: la gravedad aplicará una fuerza constante en la dirección de gravedad (ver a continuación) de la instancia, lo que influye en su velocidad (el valor predeterminado es 0). Tenga en cuenta que la gravedad es una fuerza acumulativa y acelerará el objeto si elige no limitar la velocidad final.

  • Dirección de gravedad: se puede usar para establecer la dirección de movimiento cuando la gravedad de las instancias es mayor que 0 (el valor predeterminado es 270). Tenga en cuenta que las instrucciones en GameMaker Studio 2 se calculan como 0 °, derecha, 90 °, arriba, 180 ° hacia la izquierda y 270 ° hacia abajo.

  • Puntuación - Establezca el score variable.

  • Vidas - Establecer el lives variable.

  • Salud - Establecer el health variable.

Una vez que selecciona la variable incorporada para usar, entonces establece el valor para esa variable. El valor que use dependerá de la variable seleccionada (consulte la lista anterior para conocer los valores predeterminados y los rangos) y puede indicar que el valor es relativo al valor actual de la variable. También puede agregar variables adicionales para establecer en la misma acción haciendo clic en el ícono más al lado de la acción, y seleccionando otra variable de instancia y dándole un valor.
