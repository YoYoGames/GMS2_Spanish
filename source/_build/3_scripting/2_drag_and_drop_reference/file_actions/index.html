Biblioteca de archivos


Las Acciones de archivo se ocupan de dos tipos de archivos diferentes: archivos de búfer y archivos de Ini, y también tienen ciertas acciones genéricas de archivo para cambiar el nombre o copiar archivos existentes. Los archivos de búfer se crean guardando los datos de un búfer que ha creado previamente y estos datos pueden cargarse en un búfer nuevamente en cualquier momento en el futuro. Puede encontrar más información sobre búferes aquí: Acciones de búfer.

Los archivos Ini son archivos pequeños y livianos que son compatibles con la mayoría de las plataformas. Son ideales para almacenar pequeñas cantidades de información, como preferencias de interfaz, puntuaciones locales altas, datos de nivel, etc., y son muy fáciles de usar. Los archivos Ini no tienen que haberse creado previamente para usar estas acciones, y puede leer desde un archivo Ini inexistente y simplemente obtendrá un default valor de retorno (que usted especifica), sin embargo, le recomendamos que cree al menos un archivo ini "base" para abrir y modificar antes de usar las acciones. Este archivo base ini se puede crear simplemente llamando a la acción Abrir archivo.i, seguido por el archivo Cerrar.i, ya que al cerrar el archivo se escribirá en el disco, o puede incluir uno en los archivos incluidos del árbol de recursos. Si está utilizando un archivo incluido en el árbol de recursos como su base Ini, también debe leer la sección del manual sobre cómo funciona el sistema de archivos.

NOTA: Para los juegos con localizaciones es muy importante que las letras acentuadas se puedan leer desde archivos externos. Esto significa que primero debe crear el archivo ini en formato UTF8 y luego agregarlo a GameMaker Studio 2 como un archivo incluido para que se exporte al ejecutar el juego y se use en lugar del archivo en formato ANSI predeterminado creado por el ini. funciones cuando ninguna ha sido suministrada previamente. De esta forma, puede leer y escribir correctamente con todos los acentos y manteniendo las letras no latinas.
