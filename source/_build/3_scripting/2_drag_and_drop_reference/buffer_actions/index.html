Biblioteca de almacenamiento intermedio


Un búfer es una región de memoria física que se utiliza para guardar temporalmente datos mientras se mueven de un lugar a otro o se manipulan de alguna manera; por ejemplo, puede usar un búfer para guardar los datos del estado del juego. GameMaker Studio 2 tiene una serie de acciones específicas relacionadas con la creación, modificación y carga / guardado de búferes, pero antes de utilizarlos tenga en cuenta los siguientes puntos:

  • Cuando se crea un búfer, se borra automáticamente y se llena a 0.
  • El argumento de "tamaño" de las funciones del búfer siempre se refiere al tamaño en bytes del búfer.

  • Los buffers DnD™ siempre están alineados a 1 byte. La "alineación" se refiere a cómo se almacenan los datos dentro de un búfer y se usa cuando se utilizan las funciones de búfer GML, pero para arrastrar y colocar, no es relevante.

  • Los datos en los buffers se leen y escriben de forma secuencial (es decir, usted llama a la acción Write Buffer tres veces, luego escribe los primeros datos, avanza, los segundos datos, avanza y luego escribe los últimos datos), con algo llamado el valor "buscar" para indicarle en qué lugar del buffer se realizará la próxima lectura / escritura.

  • Todos los datos se escriben en el búfer y se devuelven del búfer como una cadena, por lo que si ha almacenado números reales, deberá usar una de las acciones de tipo de datos para convertirlo de la cadena al valor.

La memoria utilizada para crear memorias intermedias es la memoria del sistema, por lo que incluso cuando el juego no tiene foco, cualquier información almacenada en un búfer debe ser segura; sin embargo, si la aplicación se cierra o se reinicia, se perderá.
