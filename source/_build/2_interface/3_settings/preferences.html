Preferencias específicas de la plataforma

Esta sección del manual cubre las diferentes Preferencias específicas de la plataforma que deben configurarse antes de que pueda compilar sus juegos para ciertos objetivos. En general, estos solo necesitan configurarse una vez para comenzar (aunque las actualizaciones futuras de los tiempos de ejecución pueden requerir que cambie ciertas cosas, pero se mencionarán en las Notas de la versión ) y se usan para apuntar a cualquier SDK que sea necesario y para configurar las opciones básicas. cómo desea que GameMaker Studio 2 compile sus proyectos de juegos para cualquier plataforma determinada. Tenga en cuenta que las plataformas disponibles dependerán de la licencia que tenga y que no todas estarán disponibles.

A continuación se muestra una lista de todas las plataformas de destino disponibles y sus preferencias:

Las preferencias de Amazon Fire se dividen en varias secciones enumeradas a continuación. Algunos de ellos se refieren a Android herramientas de Android, ya que el objetivo de AMazon Fire se basa en Android y comparte los mismos requisitos de SDK.

En la parte superior de las preferencias, tiene la opción de sincronizar la configuración de la exportación de Android. Esto es para aquellas personas que tienen la licencia móvil y que ya han configurado y usado la exportación de Android, ya que Amazon Fire usa casi exactamente las mismas herramientas de construcción que Android. Simplemente haga clic en esto para importar todas las configuraciones apropiadas desde Android, luego revíselas para asegurarse de que sean correctas.

Bajo el botón Sincronizar, puede establecer el tamaño del montón de memoria para el JDK de Android. A continuación, debe configurar las distintas rutas para que GameMaker Studio 2 sepa dónde buscar las distintas herramientas necesarias para crear el paquete ejecutable final y probar su juego. Se divide en tres secciones para Android SDK, Android NDK y Java JDK. Estas rutas deberían haberse completado automáticamente para usted, pero si tiene algún problema, puede hacer clic en el botón a la derecha para abrir el explorador de archivos y seleccionar manualmente las ubicaciones para cada elemento. Puede encontrar detalles de las versiones necesarias para cada uno de los elementos del SDK aquí. Las rutas deben validarse a medida que se aplican y un mensaje mostrará "Encontrado" o le informará de cualquier error (en cuyo caso, debe revisar toda la información para asegurarse de que sea correcta).

Debajo tienes las diferentes opciones de Empaque que son:

  • Realice siempre la instalación completa de APK: marcar esto hará que GameMaker Studio 2 elimine de su dispositivo a los corredores anteriores de su juego, así como a todos los archivos asociados (por ejemplo, archivos *.ini) de su dispositivo de prueba antes de instalar una nueva versión de el juego.

  • Instalar en paquete: cuando esta opción está marcada, compilar un ejecutable final *.apk El paquete también lo instalará automáticamente en el dispositivo de destino seleccionado actualmente.

Finalmente llegamos al KeyStore. Este es un archivo que se utilizará para "firmar" todas sus aplicaciones de Amazon Fire así que complete todos los detalles correctamente y tenga en cuenta que este archivo será necesario para crear y actualizar todas sus aplicaciones de Amazon Fire en el futuro, lo que significa que después de configurarlo, se recomienda tomar una captura de pantalla de la configuración utilizada y hacer una copia de seguridad del archivo final. Si pierde este archivo, no podrá actualizar ningún juego existente que se haya cargado en la tienda de Google Play.

NOTA: El mismo almacén de claves se puede utilizar para los objetivos de Android y Amazon Fire.


Para crear un nuevo archivo de almacén de claves se requieren los siguientes detalles:

  • Nombre de archivo: este es el nombre del archivo KeyStore. NOTA: ¡ Este no es su nombre o el nombre de la compañía, sino el nombre del archivo que se generará! Si ya tiene un archivo de almacén de claves de proyectos anteriores, puede hacer clic en el botón del explorador de archivos aquí y dirigir GameMaker Studio 2 al archivo de almacén de claves anterior (o hacer clic en el botón Importar ). En este caso, aún debe completar el resto de la información requerida (tal como estaba cuando generó el archivo por primera vez ), ¡pero no debe hacer clic en el botón Generar Key Hash!. Eso generará un nuevo archivo de almacén de claves, sobrescribiendo el anterior.

  • Nombre común: este sería normalmente tu nombre.

  • Contraseña: su contraseña de seguridad para el archivo KeyStore que debe tener al menos seis caracteres.

  • Alias: este es otro nombre que se puede utilizar para el archivo KeyStore y debe ser diferente al nombre ingresado anteriormente.

  • Contraseña de alias: su contraseña de seguridad de alias para el archivo KeyStore, que debe tener al menos seis caracteres.

  • Unidad organizativa: el departamento dentro de la empresa de la que forma parte.

  • Organización: El nombre de su empresa.

  • Ubicación: El nombre del pueblo o ciudad donde se encuentra.

  • País: el código estándar de dos letras para el país donde reside.

Una vez que haya completado esta información y esté satisfecho con ella, presione el botón marcado Generar Key Hash para GameMaker Studio 2 para crear el archivo de KeyStore necesario. Esto puede demorar un momento, pero una vez finalizado, normalmente no será necesario cambiarlo nuevamente. Tenga en cuenta que puede hacer clic en el botón Importar para importar un archivo de almacén de claves creado anteriormente (consulte la sección de Nombre de archivo, arriba) y también puede recuperar el hash SHA1 del almacén de claves haciendo clic en el botón Mostrar Key Hash. Al presionar esto se generarán dos claves hash de seguridad especiales directamente vinculadas a su archivo de almacén de claves que algunos sitios web requieren para la funcionalidad correcta (por ejemplo, Facebook o Google Play para el hash Sha1). Si necesita esto, simplemente presione el botón, luego copie y pegue las claves hash que se generan.

Para obtener más detalles sobre cómo configurar los SDK de Amazon Fire requeridos, consulte este artículo en la Base de conocimientos de YoYo Games.

Las preferencias de HTML5 tienen las siguientes opciones:

  • Tipo de paquete predeterminado: al crear un paquete ejecutable final de HTML5, puede elegir entre Paquete como archivos sueltos, que generará simplemente todos los archivos requeridos en una carpeta, o Paquete como Zip, que creará un archivo ZIP comprimido que tendrá el juego y cualquier archivo adicional dentro de. Por defecto, se le dará una opción de cuál de estas dos opciones desea compilar en la compilación, pero puede configurar esta preferencia para usar siempre una o la otra. Tenga en cuenta que cuando esta opción está configurada en Mostrar diálogo, en la ventana de diálogo real que aparece en la compilación, al elegir "Recordar opción de empaquetado" se establecerá esta preferencia en la opción que seleccione.

Las preferencias de iOS tienen las siguientes opciones:

  • Identificador de equipo predeterminado: Aquí puede agregar su Identificador de equipo predeterminado, según lo asignado por Apple. Esta identificación del equipo se utilizará cuando los archivos del juego se envíen a Xcode para crear la aplicación, y permitirá que Xcode genere los certificados de firma necesarios. Tenga en cuenta que esta configuración se aplicará de forma predeterminada a todos los juegos creados para iOS, pero se puede anular en función de cada proyecto desde las Opciones de juego generales de iOS.

  • Suprimir compilación y ejecución: de forma predeterminada, cuando creas un proyecto de iOS y luego creas un paquete ejecutable final, GameMaker Studio 2 compilará el paquete y luego intentará ejecutarlo en cualquier dispositivo conectado. Marcar esta opción suprimirá la creación de la aplicación y su ejecución en el dispositivo y simplemente creará un proyecto XCode en el Mac host.

  • Ruta de instalación de macOS: La ruta de instalación para los archivos de proyecto en la compilación Mac.


Las preferencias de macOS tienen las siguientes opciones:

  • Identificador de equipo predeterminado: Aquí puede agregar su Identificador de equipo predeterminado, según lo asignado por Apple. Esta identificación del equipo se utilizará cuando los archivos del juego se envíen a Xcode para crear la aplicación, y permitirá que Xcode genere los certificados de firma necesarios. Tenga en cuenta que esta configuración se aplicará de forma predeterminada a todos los juegos creados para macOS, pero se puede anular por proyecto desde General MacOS Game Options.


El Steam SDK se usa para las Windows, Mac OS and Ubuntu (Linux) de Windows, Mac OS and Ubuntu (Linux) que se distribuirán a través de la plataforma Steam. Esta sección contiene las opciones de configuración necesarias para que funcione en sus proyectos:

  • Path To Steam SDK: la ruta del sistema donde se ha instalado Steam SDK. Una vez que haya configurado esta ruta al SDK, deberá activar Steam en las Opciones del juego y luego, para cada plataforma, agregar la ID de la aplicación.


Las preferencias de Windows tienen las siguientes opciones:

  • Ruta de Visual Studio: esta opción es para señalar GameMaker Studio 2 a la instalación de MS Visual Studio en la PC de compilación.

  • Tipo de paquete predeterminado: al crear un paquete ejecutable final de Windows, puede elegir entre crear un instalador (utilizando NSIS ) o un archivo ZIP comprimido que tendrá el juego y cualquier otro archivo adicional. Por defecto, se le dará una opción de cuál de estas dos opciones desea compilar en la compilación, pero puede configurar esta preferencia para usar siempre una o la otra. Tenga en cuenta que cuando esta opción está configurada en Mostrar diálogo, en la ventana de diálogo real que aparece en la compilación, al elegir "Recordar opción de empaquetado" se establecerá esta preferencia en la opción que seleccione.


Las preferencias de Windows UWP tienen las siguientes opciones:

  • Ubicación de Visual Studio: esta opción es para señalar GameMaker Studio 2 a la instalación de MS Visual Studio en la PC de compilación.

  • Arquitectura del paquete predeterminado: al crear un paquete ejecutable final de Windows UWP, puede elegir entre crear un paquete x86, x64 o ARM que tendrá el juego y cualquier otro archivo adicional. Aquí puede establecer un valor predeterminado para el paquete a utilizar o seleccionar Mostrar diálogo para que aparezca una ventana emergente cuando presiona el botón de compilación preguntando cuál elegir. En general, desea apuntar a x86 para computadoras con Windows 10, x64 para Xbox One y ARM para dispositivos con Windows 10.

  • Ubicación de Win UWP Tools: esta opción es para señalar GameMaker Studio 2 a la ruta de instalación de las herramientas de UWP en la PC de compilación.

  • Respuesta automática a la creación de un paquete de tienda: al crear un paquete ejecutable final de Windows UWP, se le preguntará si desea crear un paquete de tienda o no. Puede configurar esta opción para omitir automáticamente esta ventana emergente usando Sí o No de manera predeterminada, o puede configurar Mostrar diálogo para que la ventana siempre aparezca cuando presione el botón de compilación.

Tenga en cuenta que, además de configurar estas preferencias, también deberá configurar los dispositivos a los que se dirigen algunas de estas plataformas, de lo contrario, es posible que no pueda compilar ni probar sus proyectos. Consulte la sección sobre Compilación y la sección sobre el Administrador de dispositivos para obtener más información.

El manual no muestra las distintas preferencias de la Consola debido a las diferentes restricciones legales mantenidas por las diferentes compañías. Si tiene una licencia de consola, puede obtener esta información del servicio YoYo Games asistencia de YoYo Games, pero primero debe comunicarse con YoYo Games con el correo electrónico de identificación de la consola y solicitar permiso para ver las diferentes secciones.
