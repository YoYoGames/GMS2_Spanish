Opciones de juego


Las Opciones del juego se dividen en dos secciones, Opciones principales y Configuración de la plataforma. Los detalles de las Opciones principales se dan en las siguientes secciones:

La ventana de Opciones generales es donde puede establecer ciertas cosas que afectarán el juego, independientemente de la plataforma en la que esté diseñado para ejecutarse. La ventana tiene las siguientes secciones:

  • Utilice DnD ™ como tipo de script predeterminado: cuando esta opción está marcada, todos los nuevos scripts y eventos de objeto se abrirán usando el editor DnD ™ en lugar del editor de código.

  • Marcos de juego por segundo: este valor establece el límite en el número de cuadros por segundo que GameMaker Studio 2 debe tratar de mantener sus juegos cuando se ejecuta. Este valor representa el número de veces que el bucle del juego se ejecutará en un segundo (un bucle de un solo juego se considera un único "marco") y se establece en un valor predeterminado de 30, aunque 60 también se utiliza a menudo. Existen muchos argumentos acerca de qué velocidad de habitación es la ideal, pero la respuesta real depende del tipo de juego que está haciendo, el tamaño de la sala, el número de instancias y qué tan intenso es el código. Por ejemplo, si tiene una habitación grande y una velocidad de sala de 30, el juego puede aparecer "entrecortado" ya que cuanto mayor sea el área que está mirando, más fácil será para el ojo percibir velocidades de habitación bajas, pero si el juego es intensivo, luego establecer la velocidad de la habitación en 100 puede causar que el juego se "retrase" cuando los fps disminuyen debido a la cantidad de procesamiento que se está realizando. Así que tenga cuidado con este valor y trate de ajustarlo siempre a lo que necesita su juego.

  • Color de dibujo predeterminado: el color predeterminado para dibujar texto o foros (y otras cosas) cuando no se ha establecido un color con el código.

  • Generar imágenes de proyectos: Al hacer clic en esto, se abrirá la herramienta Generador de imágenes del proyecto.

  • Fecha de inicio del proyecto: la fecha en que se inició el proyecto.

  • Tiempo de uso del proyecto: el tiempo que ha pasado desde que inició el proyecto.

  • GUID del proyecto: este es el identificador único que se crea para cada proyecto dentro de GameMaker Studio 2. Si desea cambiar esto, puede hacer clic en el botón Crear nuevo GUID para generar uno nuevo.

  • Genere mipmaps para páginas de texturas separadas: esta opción hará que GameMaker Studio 2 cree mipmaps para aquellos sprites que hayan marcado la opción de Página de textura separada en la configuración de Textura del Editor de Sprite. Habilitar mipmapping significa que la página de textura original del sprite debe escalarse y filtrarse en varias resoluciones dentro del archivo de textura. Esto normalmente está desactivado, pero si está utilizando vistas para acercarse o alejarse en áreas grandes de la sala de juegos, o si está trabajando con un juego 3D, entonces puede habilitar esto y luego usar las Funciones de Mipmapping para ajustar cómo juego trata con los diferentes niveles de mipmap. Tenga en cuenta que si no está marcando sprites como el uso de páginas de texturas individuales, aún puede habilitar mipmapping en base a grupos de texturas desde el Administrador de grupos de texturas. Esta opción está apagada por defecto.

  • Identificación de Steam: si está creando un juego que se va a distribuir a través de Steam, debe proporcionar aquí la ID de la aplicación Steam. Asegúrese de haber agregado la ruta al SDK de Steam Works en las Preferencias de plataforma al usar esta opción, y asegúrese de tener todo configurado correctamente en el tablero de Steam Works.

  • Permitir estadísticas de juego anónimas: habilite o deshabilite el envío de datos de análisis de juegos de GameMaker Studio 2 a YoYo Games, tanto desde el IDE como desde los juegos que cree. Cuando esté habilitado, enviará datos anónimos sobre la PC que ejecuta el programa / juego (cosas como el tamaño del monitor, CPU, GPU, memoria, etc.) y los ejecutables finales también enviarán datos sobre cómo se llama el juego. Esta información se está acumulando para que YoYo Games pueda orientar mejor sus servicios a las necesidades del usuario, y en el futuro YoYo Games espera poner esta información a disposición de los desarrolladores (al igual que Steam hace con Steam Software Survey ). Para detalles completos, vea la parte 6 de la sección Información que recopilamos de la Política de privacidad de YoYo Games. Esta opción está disponible solo para aquellos usuarios que tengan una Licencia Pagada (Escritorio, Móvil, HTML5, UWP o Consola) y los usuarios con la Licencia de Prueba siempre enviarán datos estadísticos anónimos.

  • Habilite el control de fuente: si desea que el proyecto esté bajo SCM (gestión de control de fuente), debe verificarlo. Está desactivado por defecto (consulte aquí para obtener más información sobre el control de fuente).

  • Autor: El nombre del autor del juego (o editor o estudio).

  • Última modificación del proyecto: esta sección simplemente muestra la marca de fecha / hora de la última vez que se modificó el proyecto (esto no es editable).

GameMaker Studio 2 requiere que se instalen ciertos complementos como extensiones para su proyecto si desea usar funciones extendidas como tablas de clasificación o compras desde la aplicación en plataformas específicas. Al hacer clic en el botón " Descargar " se descargará la extensión y la instalará en el proyecto actual, listo para usar.

El árbol de recursos también tendrá otras opciones de juego disponibles dependiendo de las plataformas de destino disponibles para la licencia que tenga y no todas ellas estén disponibles o visibles. Las siguientes opciones de plataforma existen:

  1. Windows
  2. Mac OS
  3. Ubuntu
  4. Windows UWP
  5. HTML5
  6. Amazon Fire
  7. Android
  8. iOS
