Los eventos gestuales


Los eventos en la categoría Evento Gesto se activarán siempre que GameMaker Studio 2 detecte un "gesto" del mouse o un evento de pantalla táctil (mientras que estos eventos gestuales están diseñados específicamente para uso móvil, también se pueden usar en otros objetivos para detectar el mouse, aunque no detectarán toques múltiples en este caso). El objetivo del sistema de gestos es tratar de reconocer las entradas a un nivel más alto que las funciones de lectura directa y táctil del mouse, y está diseñado para simplificar la implementación de las entradas utilizadas comúnmente en los dispositivos táctiles.

NOTA: estos eventos no se activarán en la plataforma HTML5. Si está buscando gestos en ese objetivo, entonces debería usar las funciones del dispositivo.


Puede optar por detectar gestos de instancia o gestos globales, donde los eventos de gestos de instancia solo se activarán cuando el toque / clic inicial se realice en una instancia dentro de la sala. Tenga en cuenta que la instancia debe tener una máscara de colisión válida (consulte las secciones Editor de Sprite - Máscara de colisión y Editor de objetos - Máscara de colisión para obtener más información) para que se active este evento. Sin embargo, los eventos globales se activarán al tocar / hacer clic en cualquier lugar dentro de la sala de juegos y para todas las instancias que tengan el evento.

Cuando se reconoce un evento de gesto, desencadenará uno o más de los sub-eventos disponibles, y el sub-evento desencadenado dependerá del tipo de gesto que se haya detectado. En todos los casos, sin embargo, se generará un mapa de DS para usted y se almacenará en la variable incorporada event_data. Las claves disponibles dependerán del tipo de evento en el que haya sido creado y se muestren en cada una de las siguientes subsecciones.

NOTA: la variable event_data solo es válido en estos eventos, ya que ds_map que apunta se crea automáticamente al comienzo del evento, luego se destruye nuevamente al final, y esta variable se restablece a un valor de -1 en todos los demás momentos.


Vale la pena señalar que si tiene múltiples instancias debajo de la posición que se está tocando y todas tienen un evento de gesto, todas se dispararán, no solo la instancia "más alta". También tenga en cuenta que al usar vistas múltiples y arrastrar una instancia, los valores devueltos se basarán en la vista en la que se encontraba cuando se recibió el toque / clic inicial; esto es cierto para todos los eventos posteriores en ese gesto para esa instancia. Así que al tocar y arrastrar una instancia en una vista y luego soltarla en otra vista, se devolverán los valores relativos a la vista inicial donde se detectó el gesto por primera vez.


Los eventos "Tap", "Drag" y "Flick" se basan en un solo toque o clic del mouse en la pantalla y el event_data DS Map contendrá las siguientes claves:

Llave Descripción del valor
"gesture"
Este es un valor de ID que es exclusivo del gesto que está en juego. Esto le permite vincular las diferentes partes de los gestos de varias partes (como inicio de arrastrar, arrastrar y arrastrar) juntos.

"touch"
Este es el índice del toque que se está utilizando para el gesto. En general, esto comenzará en 0 y aumentará para cada dedo que se mantenga presionado, luego se reiniciará a 0 cuando se eliminen todos los dedos, pero si el usuario está tocando la pantalla en otro lugar cuando este evento se desencadena con otro toque, entonces el valor será mayor que 0.

"posX"
Esta es la posición X del espacio de la habitación del toque.

"posY"
Esta es la posición Y del espacio de la habitación del toque.

"rawposX"
Esta es la posición X ventana-espacio prima de tacto (equivalente a obtener la posición del ratón usando device_mouse_raw_x() )

"rawposY"
Esta es la posición Y sin formato del espacio de la ventana táctil (equivalente a obtener la posición del mouse usando device_mouse_raw_y() )

"guiposX"
Esta es la posición X del espacio gui del tacto (equivalente a obtener la posición del mouse usando device_mouse_x_to_gui() )

"guiposY"
Esta es la posición Y del espacio gui del tacto (equivalente a obtener la posición del mouse usando device_mouse_y_to_gui() )

"diffX"
Esta es la diferencia de espacio de la sala X entre la posición del toque actual y la posición del último toque en este gesto.

"diffY"
Esta es la diferencia Y del espacio de la habitación entre la posición del toque actual y la posición del último toque en este gesto.

"rawdiffX"
Esta es la diferencia X en bruto entre la posición del toque actual y la posición del último toque en este gesto.

"rawdiffY"
Esta es la diferencia en Y sin procesar entre la posición del toque actual y la posición del último toque en este gesto.

"guidiffX"
Esta es la diferencia X de gui-espacio entre la posición del toque actual y la posición del último toque en este gesto.

"guidiffY"
Esta es la diferencia Y del espacio-gui entre la posición del toque actual y la posición del último toque en este gesto.

"isflick"
Solo disponible en el evento Drag End. Esto se establece en 1 si el final del arrastre se detecta como una película, lo que significa que no necesita un evento Flick por separado si está manipulando el arrastre de todos modos.

El evento Tap se activará cuando se toque o haga clic en una instancia o, si se trata de un evento global, cuando el juego registre un toque o haga clic en cualquier parte de la sala. Un toque se considera un toque y liberación rápidos, y si el toque dura demasiado, se considerará un Arrastre (y activará los eventos Gesto de arrastre en lugar del evento Tap). Este evento generará un event_data Mapa DS que luego puede usar para obtener información sobre el evento. Por ejemplo:

Crear evento

x_goto = x;
y_goto = y;

Tap Evento

x_goto = event_data[? "posX"];
y_goto = event_data[? "posY"];

Paso Evento

var _pd = point_distance(x, y, x_goto, y_goto);
move_towards_point(x_goto, y_goto, clamp(_pd, 0, 5);


El código anterior detectará un toque en la pantalla y luego obtendrá la posición del tap para mover la instancia a esa posición. Tenga en cuenta que si desea tener un tiempo de detección de tap más largo o más corto, puede configurarlo con la función gesture_drag_time. Esto establece el tiempo entre la detección inicial y la derivación convirtiéndose en un arrastre, así que ajústelo a un valor más alto para que la detección de toma sea más larga o menor para acortarla (el valor está en segundos y el valor predeterminado es 0.16).


El evento Double Tap se activará cuando se toque o haga clic en una instancia dos veces seguidas (o, si se trata de un evento global) cuando el juego registre dos toques o clics rápidos en cualquier lugar de la habitación. Un doble toque se considera dos toques rápidos y lanzamientos, pero si alguno de los toques dura demasiado, se considerará un Arrastre (y activará los eventos de gesto de Arrastre en lugar del evento Doble toque). Este evento generará un event_data Mapa DS que luego puede usar para obtener información sobre el evento. Por ejemplo:

Crear evento

x_goto = x;
y_goto = y;

Evento Doble Tap

instance_destroy();


El código anterior simplemente detecta un doble toque y luego destruye la instancia. Tenga en cuenta que puede establecer el tiempo entre toques para activar un doble toque usando la función gesture_double_tap_time (que tiene un valor predeterminado, en segundos, de 0.16) y también puede establecer la distancia para la detección entre pulsaciones con la función gesture_double_tap_distance (si se detecta un segundo toque fuera de esta distancia, se considerará un evento de toque regular).


El evento Drag Start se activará cuando el usuario mantenga un toque o un clic sin soltarlo. Se activará una vez cuando haya transcurrido un tiempo establecido después del toque inicial, que es 0.16 segundos por defecto (aunque puede establecerlo en cualquier otro valor en segundos usando la función gesture_drag_time ) Después de que se haya activado este evento, y mientras el usuario mantenga presionado un toque / clic, el evento Arrastrar se disparará en cada paso hasta que se haya liberado el toque / clic. Este evento generará un event_data Mapa DS que luego puede usar para obtener información sobre el evento. Por ejemplo:

Crear evento

drag_offsetX = 0;
drag_offsetY = 0;

Arrastrar evento de inicio

drag_offsetX = x - event_data[?"posX"];
drag_offsetY = y - event_data[?"posY"];


El código anterior usa el evento Arrastrar inicio para obtener la posición del toque / clic y usarlo para establecer un valor de compensación para los ejes xey. Esto se puede usar al arrastrar la instancia para asegurarse de que no "salte" a la posición en la que se detectó el toque / clic (consulte el evento Arrastrando a continuación para continuar con este ejemplo).


El evento Dragging se activa después del evento Drag Start y se activará para cada paso que el usuario mantenga al tocar / hacer clic en la instancia (o la pantalla, si se trata de un evento global) y mover más que el umbral de arrastre definido. Esta distancia es de 0.1 pulgadas por defecto, pero se puede configurar usando la función gesture_drag_distance. Si no hay movimiento o el movimiento está por debajo del umbral definido, el evento no se disparará. Este evento generará un event_data Mapa DS que luego puede usar para obtener información sobre el evento. Por ejemplo:

Crear evento

drag_offsetX = 0;
drag_offsetY = 0;

Arrastrar evento de inicio

drag_offsetX = x - event_data[?"posX"];
drag_offsetY = y - event_data[?"posY"];

Evento de arrastre

x = event_data[?"posX"] + drag_offsetX;
y = event_data[?"posY"] + drag_offsetY;


El código de ejemplo anterior utiliza las variables de desplazamiento establecidas en el evento Arrastrar inicio para mover la instancia cuando se desencadena el evento de arrastre.


El evento Drag End se activa cuando el usuario suelta el toque / clic en la instancia (o la pantalla si el evento es global). Este evento generará un event_data Mapa de DS que luego puede usar para obtener información sobre el evento, pero en este caso el mapa tendrá una clave adicional: " isflick ". Flick se calcula como la distancia por segundo que se ha producido el arrastre, y el valor para el" isflick "la clave será true si es mayor que la distancia definida por segundo valor, o false de otra manera. Tenga en cuenta que el valor predeterminado es de 2 pulgadas por segundo, pero puede establecerlo en otro valor utilizando la función gesture_flick_speed. También tenga en cuenta que hay un evento Flick dedicado que también se activará si el " isflick "variable es true. Un ejemplo de uso sería:

Crear evento

flickVelX = 0.0;
flickVelY = 0.0;

Evento de arrastrar final

isFlick = event_data[?"isflick"];
if (isFlick)
    {
    flickVelX = event_data[?"diffX"];
    flickVelY = event_data[?"diffY"];
    }
else
    {
    flickVelX = 0;
    flickVelY = 0;
    }

Paso Evento

x += flickVelX;
y += flickVelY;
flickVelX *= 0.7;
flickVelY *= 0.7;


El código anterior simplemente obtiene la diferencia en la posición xey del último evento Dragging y el evento actual Drag End, y si el movimiento ha sido mayor que el umbral flick, establece algunas variables que se usan para mover la instancia en el paso evento


El evento Flick solo se activa cuando se mantiene presionada, se arrastra y luego se suelta, y la distancia entre la última posición de arrastre y la posición de liberación es mayor a 2 pulgadas por segundo (esta es la configuración predeterminada, aunque puede cambiarse) usando la función gesture_flick_speed ) Este evento generará un event_data Mapa DS que luego puede usar para obtener información sobre el evento. Por ejemplo:

Crear evento

flickVelX = 0.0;
flickVelY = 0.0;

Flick Event

flickVelX = event_data[?"diffX"];
flickVelY = event_data[?"diffY"];

Paso Evento

x += flickVelX;
y += flickVelY;
flickVelX *= 0.7;
flickVelY *= 0.7;


El código anterior simplemente obtiene la diferencia en la posición xey del último evento de arrastre y el evento Flick actual, y si el movimiento ha sido mayor que el límite de flick, establece algunas variables que se usan para mover la instancia en el paso evento.


Los eventos "Pellizcar" se basan en dos toques a la pantalla de los dispositivos que se reconocen a la vez, donde uno (o ambos) se han movido más de una cierta distancia. El ángulo de movimiento de los toques junto con el movimiento de cada toque es lo que determinará la detección de un evento Pinch o Rotate, donde (en el caso del tipo de evento Pinch ):

  • Si uno de los toques no se mueve, el otro debe moverse hacia él o alejarse de él dentro de un ángulo de umbral (que se puede establecer usando las funciones gesture_pinch_angle_towards() y - gesture_pinch_angle_away() )
  • Si ambos toques se están moviendo, sus velocidades deben estar en direcciones aproximadamente opuestas y la misma verificación de umbral angular también se realiza para garantizar que los toques se muevan en una alineación aproximada.

Cuando se detectan dos toques y un movimiento con los criterios anteriores, se activará un evento de pellizco, y en cada uno de los eventos event_data DS Map se completará con las siguientes claves:

Llave Descripción del valor
"gesture"
Este es un valor de ID que es exclusivo del gesto que está en juego. Esto le permite vincular las diferentes partes de los gestos de varias partes (como inicio de arrastrar, arrastrar y arrastrar) juntos.

"touch1"
Este es el índice del primer toque que se está utilizando como parte del gesto de pellizco. En general, esto será 0, pero si el usuario toca la pantalla en cualquier otro lugar cuando este evento se desencadena con otro toque, entonces el valor será mayor que 0.

"touch2"
Este es el índice del segundo toque que se está utilizando como parte del gesto de pellizco. En general, será 1 más que el valor de touch1, pero puede tener algún otro valor dependiendo del número de toques que se detecten en otro lugar.

"posX1"
Esta es la posición X del espacio de la habitación del primer toque.

"posY1"
Esta es la posición Y del espacio de la habitación del primer toque.

"rawposX1"
Esta es la posición X ventana-espacio en bruto de la primera touch (equivalente a obtener la posición del ratón usando device_mouse_raw_x() )

"rawposY1"
Esta es la posición Y sin formato del espacio de la ventana del primer toque (equivalente a obtener la posición del mouse usando device_mouse_raw_y() )

"guiposX1"
Esta es la posición X del espacio gui del primer toque (equivalente a obtener la posición del mouse usando device_mouse_x_to_gui() )

"guiposY1"
Esta es la posición Y del espacio-gui del segundo toque (equivalente a obtener la posición del mouse usando device_mouse_y_to_gui() )

"posX2"
Esta es la posición X del espacio de la habitación del segundo toque.

"posY2"
Esta es la posición Y del espacio de la habitación del segundo toque.

"rawposX2"
Esta es la posición X ventana-espacio en bruto de la primera tacto.

"rawposY2"
Esta es la posición Y ventana-espacio en bruto de la segunda táctil.

"guiposX2"
Esta es la posición X del espacio gui del segundo toque.

"guiposY2"
Esta es la posición Y del espacio-gui del segundo toque.

"midpointX"
La posición X del punto medio entre los dos toques en el espacio de la habitación.

"midpointY"
La posición Y del punto medio entre los dos toques en el espacio de la habitación.

"rawmidpointX"
Esta es la posición X del espacio de ventana sin formato del punto medio.

"rawmidpointY"
Esta es la posición Y sin formato del espacio de la ventana del punto medio.

"guimidpointX"
Esta es la posición X del espacio gui del punto medio.

"guimidpointY"
Esta es la posición Y del espacio gui del punto medio.

"relativescale"
Esta es la diferencia de escala en comparación con el último evento en este gesto (por lo que para los eventos Pinch In siempre será menor que 1.0, mientras que para los eventos Pinch Out siempre será mayor que 1.0)

"absolutescale"
Esta es la escala en comparación con donde estaban los dedos cuando comenzó el gesto (por lo que si la distancia entre los dedos se ha reducido a la mitad, entonces será 0,5, mientras que si la distancia se ha duplicado será 2,0).

El evento Pinch Start se activará cuando una instancia (o la pantalla si el evento es global) ha sido tocada por dos "dedos" (y se mantiene el toque) y luego se mueve uno o ambos "dedos". Si los toques se alejan el uno del otro o hacia el otro más de la distancia de verificación mínima (que es de 0,1 pulgadas por defecto, pero se puede establecer mediante la función gesture_pinch_distance ), y el ángulo entre ellos está dentro del valor definido (esto predeterminado es 45 ° pero se puede configurar usando gesture_pinch_angle_towards() y - gesture_pinch_angle_away() ), luego se activará un evento Pinch Start. En este caso, puede establecer variables o almacenar la fecha de posición para uso futuro. Por ejemplo:

Evento de inicio de pinch

pinching = true;
pinch_x = event_data[? "midpointX"]; pinch_y = event_data[? "midpointY"];


El código anterior detectará un pellizco y almacenará la posición del punto medio para ese pellizco.


Los eventos Pinch In y Pinch Out se activarán en cada paso que la distancia entre los dos toques que componen el pellizco cambie sobre el umbral mínimo (establecido en +/- 0.1 pulgadas por defecto, pero puede cambiarlo usando la función gesture_pinch_distance ) Si no hay movimiento de los pinchazos, estos eventos no se dispararán. Estos eventos generarán un event_data Mapa DS que luego puede usar para obtener información sobre el evento. Por ejemplo:

Global Pinch In / Pinch Out Event

var _scale = event_data[? "relativescale"];
var _w = camera_get_view_width(view_camera[0]);
var _h = camera_get_view_height(view_camera[0]);
var _x = camera_get_view_x(view_camera[0]) + (_w / 2);
var _y = camera_get_view_y(view_camera[0]) + (_h / 2);

_w *= _scale;
_h = _w * (room_height / room_width);
_x -= _w / 2;
_y -= _h / 2;

camera_set_view_pos(view_camera[0], _x, _y);
camera_set_view_size(view_camera[0], _w, _h);


El código anterior escalará la vista en función de la escala relativa de los toques de pellizco.


El evento Pinch End se activará cuando el usuario libere uno (o ambos) de los toques del dispositivo. Este evento generará event_data Mapa DS que luego puede usar para obtener información sobre el evento. Por ejemplo:

Evento Pinch End

var _pinchx = event_data[? "midpointX"]; var _pinchy = event_data[? "midpointY"]; var _w = camera_get_view_width(view_camera[0]);
var _h = camera_get_view_height(view_camera[0]);
var _x = _pinchx - (_w / 2);
var _y = _pinchy - (_h / 2);

camera_set_view_pos(view_camera[0], _x, _y);


El código anterior establecerá que la posición de la vista se centre en el punto medio de los dos toques que componen el pellizco cuando se sueltan los toques.



Los eventos "Girar" se basan en dos toques en la pantalla de dispositivos que se reconocen a la vez, y donde ha habido una rotación angular constante entre los dos dentro de un tiempo específico. El ángulo de movimiento de los toques junto con el movimiento de cada toque es lo que determinará la detección de un evento Pinch o Rotate, donde (en el caso del tipo de Evento de Giro ):

  • Debes mantener presionados dos toques durante un tiempo mínimo especificado (el tiempo predeterminado es 0.16 segundos, pero puedes cambiarlo usando la función gesture_rotate_time() )
  • Dentro de este período mínimo de tiempo, deben rotar en una dirección constante (si la dirección de rotación cambia dentro de ese tiempo, entonces no se inicia la rotación).
  • La cantidad de rotación debe exceder el ángulo de umbral mínimo (que se establece en 5 ° por defecto, pero esto se puede cambiar con la función gesture_rotate_angle() )

Este es un valor de ID que es exclusivo del gesto que está en juego. Esto le permite vincular las diferentes partes de los gestos de varias partes (como inicio de arrastrar, arrastrar y arrastrar) juntos.

El evento Rotate Start se activará cuando una instancia (o la pantalla si el evento es global) ha sido tocada por dos "dedos" (y el toque se mantiene) y luego uno o ambos "dedos" se rotan desde su inicio posición. La rotación de los toques debe haber comenzado dentro de un corto período de tiempo (0.16 segundos por defecto, pero se puede configurar usando la función gesture_rotate_time() ) y ser mayor que el umbral angular mínimo (por defecto 5 °, pero esto se puede cambiar usando la función gesture_rotate_angle() ) Si estas comprobaciones son verdaderas, se activará un evento Girar inicio y puede usarlo para almacenar valores o establecer variables para usar con el resto de los eventos de rotación. Por ejemplo:

Crear evento

rotating = false;
view_a = camera_get_view_angle(view_camera[0]);

Rotar evento de inicio

rotating = true;


El código anterior simplemente configura algunas variables para girar la cámara de visualización, y luego en el evento Girar inicio, establece uno de ellos para true.


El evento giratorio se activará cada vez que los toques en la pantalla giren uno alrededor del otro, siempre que el movimiento sea mayor que el umbral angular mínimo (por defecto 5 °, pero esto se puede cambiar utilizando la función gesture_rotate_angle() ) Este evento se puede usar para establecer variables y manipular instancias, por ejemplo:

Evento giratorio

var _relangle = event_data[?"relativeangle"];
var _a = camera_get_view_angle(view_camera[0]);
_a += _relangle;
camera_set_view_angle(view_camera[0], _a);


El código anterior gira la vista de la cámara dependiendo del movimiento de rotación de los toques en el evento.


El evento Rotar fin se activará cuando uno (o ambos) toques que componen el gesto se liberen desde la pantalla del dispositivo. Este evento se puede usar para establecer variables y manipular instancias, por ejemplo:

Rotar evento final

rotating = false;

Paso Evento

if !rotating
    {
    var _a = camera_get_view_angle(view_camera[0]);
    var _adif = angle_difference(view_a, _a);
    _a += median(-5, _adif, 5);
    camera_set_view_angle(view_camera[0], _a);
    }


El código anterior utiliza el evento Rotar fin para detectar cuándo el usuario detiene el gesto y luego establece una variable. Esta variable se usa luego en el evento de paso para rotar la cámara de visualización a su posición original.
