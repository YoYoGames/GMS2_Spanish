El editor de guiones


Puede usar el código en una serie de lugares dentro de GameMaker Studio 2 desde objetos, a salas, a instancias, y hay una serie de funciones integradas que le facilitan la vida al programar con GML o Arrastrar y soltar. Sin embargo, a menudo es necesario que usted cree sus propias funciones para realizar una tarea específica, o para condensar una gran sección de código en una parte más manejable, o incluso para poder reutilizar un bloque de código particular en muchos lugares diferentes, Es por eso que GameMaker Studio 2 te da la posibilidad de crear scripts.

NOTA: Si bien esta sección trata sobre los scripts, la ventana de código en la que se escriben los scripts es la misma que para los eventos, el código de creación de sala o instancia o Shaders, y la funcionalidad que se describe aquí se aplica a todos ellos.

NOTA: Esta página hace referencia a GML para escribir guiones, pero también puede crear guiones usando Arrastrar y soltar (consulte aquí ). No importa lo que use para codificar su proyecto, la información en esta página sigue siendo válida para el editor en sí.

Los scripts se escriben utilizando el lenguaje de scripts GML (consulte la sección de información general de GML para obtener más detalles), que es el lenguaje de programación incorporado que utiliza GameMaker Studio 2. Una vez que esté más familiarizado con GameMaker Studio 2 y desee utilizarlo en toda su extensión, es recomendable que comience a aprender a usar este lenguaje, ya que aumenta enormemente sus posibilidades al crear juegos. También puede escribir scripts usando Drag and Drop, pero eso requiere un editor diferente y se describe en la sección del manual de Action Scripts.

Un script (como cualquier función incorporada) puede tomar diferentes variables de entrada, con valores reales, de cadena, booleanos y especiales como all o noone siendo aceptadas, y estas variables de entrada generalmente se llaman argumentos, aunque a menudo también se les conoce como parámetros. Para ejecutar un script desde un objeto o una línea de tiempo, use el nombre del script como si fuera una función o utilizando la función GML script_execute. Puedes nombrar el script haciendo clic derecho en él en el árbol de recursos y seleccionando Cambiar nombre, pero tenga en cuenta que el nombre del script debe cumplir con las reglas de scripting para las funciones, por lo que deben comenzar con una letra y solo deben contener letras, números o el símbolo debajo de la barra "_".

Como se mencionó anteriormente, puede pasar varios argumentos a un script (16 en total), y los utilizará para realizar la tarea para la que lo ha escrito. Para comprender mejor esto, realice, por ejemplo, una acción (o un código en GML) y piense cómo funciona... colóquelo en un evento, especifique una serie de parámetros y hará que su instancia haga algo. Los scripts son exactamente lo mismo, con la única diferencia de que las escribe. No solo puede pasar argumentos a una secuencia de comandos, sino que también puede pedirle a una secuencia de comandos que devuelva un valor (usando la return palabra clave), por lo que puede usarlos para crear métodos de cálculo (métodos matemáticos) o devolver una identificación de instancia después de una compleja detección de colisiones o cualquier otra cosa. Sin embargo, tenga en cuenta que el uso de return finaliza la secuencia de comandos, por lo que no se ejecutará ningún código después de que se haya utilizado, y esto significa que una cierta cantidad de pensamiento debe centrarse en cómo se estructuran las secuencias de comandos.

Cuando cree un recurso por primera vez, la ventana del editor de secuencias de comandos se abrirá con las siguientes opciones:

El editor de secuencias de comandos se abre en una ventana con pestañas en la parte superior para permitirle tener varias secuencias de comandos en una ventana (aunque este comportamiento se puede cambiar desde las Preferencias para dar una nueva ventana a cada secuencia de comandos). Puede hacer clic en un script y luego arrástrelo para reordenar las pestañas, o si lo prefiere, puede retirarlo de la ventana actual y colocarlo en el área de trabajo para crear una nueva ventana para ese script (o agregarlo a una ventana diferente) ), y también puede maximizar el editor de scripts para crear un nuevo espacio de trabajo. Si arrastra una pestaña de script fuera de la ventana IDE, entonces se generará un nuevo IDE para contener este recurso de script, y se puede usar como lo haría con la ventana principal.

Tenga en cuenta que si está editando código de un evento de objeto en el editor de secuencias de comandos y ha maximizado el editor de secuencias de comandos o lo tiene en una ventana o espacio de trabajo independiente, haga clic con el botón derecho El menú del editor tendrá algunas opciones extra:

  • Restaurar en el área de trabajo: esto sacará al editor de secuencias de comandos de un estado de área de trabajo / maximizado y lo encadenará al objeto en el área de trabajo.

  • Ir a Objeto: Esto lo llevará al área de trabajo en la que se encuentra el objeto con el código y se enfocará en el objeto.

  • Agregar / Abrir evento: Esto le permite agregar un nuevo evento al objeto al que pertenece la secuencia de comandos actual, y abrirá una nueva pestaña de código en el Editor de secuencias de comandos para el evento agregado. Si el evento seleccionado ya tiene código, se abrirá en una nueva pestaña.

Para información sobre el resto del clic derecho. opciones de menú, consulte la sección sobre Edición a continuación.

El editor de secuencias de comandos principal es donde escribirá todo el código para crear la secuencia de comandos o completar un evento de objeto. Los scripts y los eventos de código se escriben siguiendo las reglas generales para la sintaxis de GML (consulte la sección Información general de GML para obtener más detalles), y puede obtener ayuda al escribir sus scripts desde la ventana de autocompletar:

Esta ventana se abrirá mientras escribe su código (normalmente después de una breve pausa, pero esto puede configurarse en las Preferencias ) y le ofrece una selección de respuestas "mejor ajustadas" en orden alfabético. Mientras usas el editor de código puedes hacer clic y arrastre el mouse para seleccionar una fila de código o mantenga presionada luego haga clic y arrastre para seleccionar columnas de código. Cualquier cosa seleccionada de esta manera se puede copiar, cortar y pegar usando los atajos de ventana estándar de / + C, / + X y / + V.

Vale la pena señalar que puede comentar su código de varias maneras. El comentar su código es muy importante para mantener la productividad, ya que mantiene las cosas claras y separa las secciones del código de tal manera que es obvio lo que hace cada parte y por qué está allí. También significa que si está trabajando en un equipo, puede dejar notas para otros miembros y ellos sabrán exactamente lo que ha hecho o para qué es una sección específica del código. Incluso si eres un desarrollador en solitario, es importante dejar comentarios para tu código, especialmente si tienes que dejar el proyecto y volver a él en una fecha posterior.

Comentar su código es simple, ya que solo tiene que usar " // "antes de cualquier línea de texto o puede comentar párrafos enteros usando" /* text here */ ". Hay incluso dos métodos abreviados de teclado para comentar y descomentar las líneas de código seleccionadas: / + K comentar, y / + + K descomentar La siguiente imagen ilustra una secuencia de comandos utilizando estos dos métodos:

Los scripts también pueden tener ciertos comentarios de estilo JSDoc agregados, de modo que cuando los use en su código, se muestren en forma automática junto con sus argumentos y otros detalles. Puede obtener más información sobre esto en la sección Comentarios de secuencias de comandos de JSDoc para obtener más información. Es importante tener en cuenta que la lista de tareas puede formatear y analizar los comentarios de las secuencias de comandos para que los elementos se puedan agregar a la lista automáticamente (para obtener más información, consulte aquí ).

Además de los comentarios también puedes utilizar especiales. #region Etiquetas para marcar una sección de código para el plegado de código. ¿Cómo funciona esto es que agregue un #region etiqueta (junto con cualquier texto de comentario) en la parte superior de una sección de código y luego agregue una #endregion etiqueta en la parte inferior de la sección y luego puedes "doblar" el código, es decir: esconderlo para poder reducir el número de líneas de código visibles en el editor en cualquier momento:

El código real se ve algo como esto:

#region This is an example region

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Further comment here if required

Como habrás notado, las partes del texto del script están coloreadas. El editor de scripts "sabe" acerca de los objetos existentes, las variables y funciones incorporadas, etc. y los codifica por colores en consecuencia. La codificación por colores ayuda mucho a evitar errores como puede ver de inmediato si ha escrito mal algún nombre o si usa una palabra clave como variable, o si tiene un error de sintaxis. Sin embargo, si no le gusta el código de color, en las Preferencias puede activarlo y desactivarlo (también puede usar F10), así como personalizar los colores que se utilizan.

Puede hacer clic derecho También en cualquier parte del editor para abrir el siguiente menú:

Además de las funciones normales que espera cortar, copiar y pegar líneas de código, también tiene la opción de comentar o descomentar cualquier texto seleccionado, agregar o eliminar sangrías, así como dividir la ventana de código en 1, 2 o 3 paneles separados (tenga en cuenta que las opciones relativas a #regions no estará disponible si no ha definido ninguno en el código que se está editando). Estas últimas opciones dividirán la ventana en diferentes partes donde luego puede arrastrar un script y mostrarlo junto con los otros para comparar y editar.

Esto es excepcionalmente útil para comparar dos scripts o para ver scripts conectados al mismo tiempo, sin necesidad de otra ventana o tener que cambiar constantemente entre pestañas. Las otras opciones en este menú son alternar un punto de interrupción para usar con el Depurador y convertir el código GML a DnD™ o viceversa. Tenga en cuenta que si convierte el código a DnD™, entonces no creará acciones individuales para cada línea de código, sino que usará una acción de código único para contener todo lo que estaba anteriormente en el evento o script.

Otra herramienta muy útil que tiene a su disposición cuando edite sus scripts es el uso de Fragmentos de código. Presionando F4 abrirá la ventana emergente del fragmento de código, que le permite seleccionar uno de los métodos de código más utilizados. También puede definir sus propios fragmentos de código si lo desea, guardándolos en el directorio donde instaló GameMaker Studio 2 en la siguiente ubicación: \GameMaker Studio 2\TextEditor\snippets.txt. Puedes editar este archivo con cualquier editor de texto siguiendo estas reglas:

  • Cada fragmento está en una línea separada (no debe haber líneas en blanco).
  • Cada fragmento comienza con el nombre del fragmento (que es lo que se muestra en el menú) seguido de los dos puntos (:) seguidos del fragmento real del código.
  • Use $ para una nueva línea y coloque el texto que debe seleccionarse primero entre | simbolos

Vea los ejemplos que ya están en el archivo para ver cómo está configurado siguiendo las reglas anteriores, y se recomienda encarecidamente que realice una copia de seguridad antes de comenzar a cambiar cualquier cosa, y también haga una copia de seguridad de las modificaciones que realice en el archivo., como debería reinstalar el programa o actualizarlo, revertir los cambios que realice, puede usar la copia de seguridad para rehacerlos nuevamente.


La barra de información en la parte inferior del IDE muestra el número de línea actual en el que se encuentra y la posición a lo largo de la línea. También es donde puede ver el código auxiliar, que es una línea de texto que muestra la función que está editando actualmente junto con los argumentos que requiere. A medida que complete la función en el editor, los argumentos se resaltarán para mostrarle cuál está editando actualmente. Si ha utilizado los comentarios de secuencias de comandos de JSDoc dentro de una secuencia de comandos personalizada, la información que ha proporcionado también se mostrará aquí.

Tenga en cuenta que si está desarrollando para la Web (es decir, orientado a HTML5 ), entonces hay un protocolo de script adicional que puede usar, que es el prefacio de un nombre de script con gmcallback_, por ejemplo:

gmcallback_create_button

Usar el nombre de script anterior significaría que el script gmcallback_create_button no se ofuscará, por lo que se puede usar en extensiones de JavaScript y otras áreas de su juego, por ejemplo, al usar el clickable_* funciones
