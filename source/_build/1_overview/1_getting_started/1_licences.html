Licencias de GameMaker Studio 2


Hay varios tipos diferentes de licencias que puede obtener para GameMaker Studio 2, cuya elección dependerá en gran medida de la audiencia a la que desee dirigirse, así como de si solo desea divertirse haciendo juegos o si desea venderlos. Distribúyelos también. A continuación puede encontrar detalles de cada tipo de licencia.

GameMaker Studio 2 viene con una licencia de prueba para comenzar, y luego puedes comprar diferentes actualizaciones según tus necesidades. La licencia de prueba tiene ciertas limitaciones, pero le permite probar las funciones más importantes y los proyectos prototipo. Lo más importante a tener en cuenta con la licencia de prueba del producto es que, si bien puede probar juegos en sus máquinas, no puede crear paquetes ejecutables finales para ninguna plataforma de destino. Aparte de eso, también está limitado en cuanto a los recursos disponibles:

  • Objetos - 15
  • Sprites - 20 (no se permite la carga dinámica, no se permiten SWF o spine Spine)
  • Sonidos - 10 (no se permiten buffers de audio)
  • Conjuntos de azulejos - 2
  • Scripts - 10
  • Rutas - 5 (no se permite la creación dinámica de rutas)
  • Líneas de tiempo - 2 (no se permite la creación dinámica de líneas de tiempo)
  • Fuentes - 5 (no se permite la creación de fuentes dinámicas)
  • Habitaciones - 5 (no se permite la creación dinámica de habitaciones)
  • Shaders - 0
  • Archivos incluidos - 0
  • Extensiones - 0
  • Configuraciones - 0


Aparte de esos límites de recursos, hay algunos otros elementos que faltan en el IDE:

  • No puedes crear páginas de texturas
  • No puedes crear grupos de texturas
  • No puedes crear grupos de audio
  • No puede agregar recursos existentes a través del menú del botón derecho
  • No puedes usar el Control de Fuente incorporado
  • No puede descargar activos del Mercado
  • No puedes importar proyectos antiguos de versiones anteriores de GameMaker Studio
  • No puedes importar paquetes YYZ
  • No puedes cambiar la pantalla de YoYo Games
  • No puede deshabilitar analíticas IDE anónimas

Con la versión del creador de GameMaker Studio 2 se eliminan la mayoría de las limitaciones de la licencia de prueba (ver más arriba), excepto la capacidad de cambiar la pantalla de bienvenida, la capacidad de deshabilitar las estadísticas IDE anónimas y la capacidad de usar el objetivo YYC. Una vez comprado, podrás compilar proyectos en uno de los siguientes objetivos:

  • Windows
  • Mac OS

La plataforma de destino disponible dependerá de la versión del IDE que compró ( Creator para macOS o Creator para Windows ) y usará la VM (un corredor + código interpretado) para ejecutar y construir sus juegos. Para obtener información sobre la configuración, consulte la sección de asistencia técnica sobre los SDK requeridos o la sección sobre Objetivos de escritorio.


Con la versión de escritorio de GameMaker Studio 2 se eliminan todas las limitaciones de la licencia de prueba (ver más arriba) y puede compilar proyectos para los siguientes objetivos de escritorio:

  • Windows
  • Mac OS
  • Ubuntu

Todos los objetivos vienen con dos opciones de compilación, una es la máquina virtual que usa un corredor y un código interpretado y la otra es el YYC ( YoYo Compiler) que crea un código nativo y un ejecutable compilado. Tenga en cuenta que la opción YYC requerirá una configuración adicional. Para obtener más información, consulte la sección sobre SDK requeridos o la sección de asistencia técnica en los objetivos de escritorio.


Con la versión web de GameMaker Studio 2 se eliminan todas las limitaciones de la licencia de prueba (ver más arriba) y puede compilar proyectos para el destino HTML5. Los juegos se compilan utilizando JavaScript. Puede encontrar más información en el servicio de asistencia de YoYo Games aquí: Configuración de la Web (HTML5).


Con la versión de Amazon Fire de GameMaker Studio 2 se eliminan todas las limitaciones de la licencia de prueba (ver más arriba) y puede compilar proyectos para el objetivo de Amazon Fire. Los juegos se compilan con el SDK de Android y se pueden compilar con la VM (un corredor con código interpretado) o el YYC (compilador de YoYo que crea el código nativo). Para obtener más información sobre la configuración, consulte la sección sobre SDK requeridos o la sección de asistencia técnica en la licencia de Amazon Fire.


Con la versión móvil de GameMaker Studio 2 se eliminan todas las limitaciones de la licencia de prueba (ver más arriba) y puede compilar proyectos con los siguientes objetivos:

  • Android
  • Amazon Fire
  • iOS

Todos los objetivos vienen con dos opciones de compilación, una es la máquina virtual que usa un corredor y un código interpretado y la otra es el YYC ( YoYo Compiler) que crea un código nativo y un ejecutable compilado. Tenga en cuenta que la opción YYC requerirá una configuración adicional. Para obtener más información sobre la configuración, consulte la sección sobre SDK requeridos o la sección de asistencia técnica sobre objetivos móviles.


Con la versión UWP de GameMaker Studio 2 se eliminan todas las limitaciones de la Licencia de prueba (ver más arriba) y puede compilar proyectos como ejecutables compatibles con UWP. Estos ejecutables pueden configurarse para ejecutarse en una de las tres plataformas UWP:

  • Escritorio Windows 10
  • Windows 10 Mobile
  • XBox One

El objetivo de UWP viene con dos opciones de compilación, una es la VM que usa un código de corredor e interpretado y la otra es el YYC ( YoYo Compiler) que crea un código nativo y un ejecutable compilado. Tenga en cuenta que la opción YYC requerirá una configuración adicional. Para obtener más información sobre la configuración, consulte la sección sobre SDK necesarios o la sección de asistencia técnica sobre los objetivos de UWP.


Con la versión de Consolas de GameMaker Studio 2 se eliminan todas las limitaciones de la Licencia de prueba (ver más arriba) y puede compilar proyectos para las siguientes plataformas de destino

  • XBox One
  • Playstation 4
  • Interruptor de nintendo

Cada uno de los destinos de la consola viene con dos opciones de compilación, una es la VM que usa un código de corredor e interpretado y la otra es el YYC ( YoYo Compiler) que crea un código nativo y un ejecutable compilado. Tenga en cuenta que la opción YYC requerirá una configuración adicional. Para obtener más información sobre la configuración, consulte la sección sobre los SDK necesarios.

¡IMPORTANTE! Debido a las restricciones de los distintos NDA asociados con las exportaciones de la consola, no hay información sobre la configuración o programación de estos objetivos en el manual de GameMaker Studio 2. Una vez que sea un desarrollador registrado para la plataforma elegida, póngase en contacto con el servicio de asistencia a través de su cuenta de YoYo y solicite acceso a la documentación correspondiente.

GameMaker Studio 2 también se puede utilizar en un entorno universitario. Para obtener más información sobre esto, comuníquese con YoYo Games a través del Portal de Educación de YoYo Games.


¡IMPORTANTE! Si ha estado usando una licencia de prueba para GameMaker Studio 2 y luego compra una de las otras licencias (Creador, Escritorio, Amazon Fire, Mobile, UWP, Web o Consola), deberá cerrar la sesión del IDE y reiniciar GameMaker Studio 2. Cuando reinicie el programa y vuelva a iniciar sesión, la nueva licencia debe aplicarse correctamente. Puede cerrar sesión en el menú Archivo.
