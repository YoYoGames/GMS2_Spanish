Licencias de GameMaker Studio 2


Hay varios tipos diferentes de licencias que puede obtener para GameMaker Studio 2, cuya elección dependerá en gran medida de la audiencia a la que desee dirigirse y de si solo desea divertirse haciendo juegos o si desea venderlos. distribuirlos también. A continuación puede encontrar detalles de cada tipo de licencia.

GameMaker Studio 2 viene con una licencia de prueba para comenzar, y luego puede comprar diferentes actualizaciones según sus necesidades. La licencia de prueba tiene ciertas limitaciones, pero le permite probar las características y prototipos más importantes. Lo más importante a tener en cuenta con la licencia de prueba del producto es que aunque puede probar juegos en sus máquinas, no puede crear paquetes ejecutables finales para ninguna plataforma objetivo. Aparte de eso, también está limitado en cuanto a los recursos disponibles:

  • Objetos - 15
  • Sprites - 20 (no se permite carga dinámica, no se permiten sprites SWF o Spine)
  • Sonidos - 10 (no se permiten búferes de audio)
  • Juegos de azulejos - 2
  • Guiones - 10
  • Rutas - 5 (no se permite la creación dinámica de rutas)
  • Líneas de tiempo: 2 (no se permite la creación dinámica de la línea de tiempo)
  • Fuentes - 5 (no se permite la creación dinámica de fuentes)
  • Habitaciones - 5 (no se permite la creación de salas dinámicas)
  • Shaders - 0
  • Archivos incluidos - 0
  • Extensiones - 0
  • Configuraciones - 0


Aparte de esos límites de recursos, faltan algunos otros elementos del IDE:

  • No puedes crear páginas de textura
  • No puedes crear grupos de texturas
  • No puedes crear grupos de audio
  • No puede agregar recursos existentes a través del menú contextual
  • No puede usar el control de fuente incorporado
  • No puede descargar activos desde Marketplace
  • No puede importar proyectos anteriores de versiones anteriores de GameMaker Studio
  • No puede importar paquetes YYZ
  • No puedes cambiar la pantalla de presentación de YoYo Games
  • No puede deshabilitar análisis IDE anónimos

Con la versión Creator de GameMaker Studio 2 se eliminan la mayoría de las limitaciones de la licencia de prueba (ver arriba), excepto la capacidad de cambiar la pantalla de bienvenida, la capacidad de deshabilitar estadísticas IDE anónimas y la posibilidad de utilizar el objetivo YYC. Una vez comprado, podrá compilar proyectos para uno de los siguientes objetivos:

  • Windows
  • Mac OS

La plataforma objetivo disponible dependerá de la versión del IDE que haya comprado ( Creador para macOS o Creator para Windows ) y usará la máquina virtual (un corredor + código interpretado) para ejecutar y construir sus juegos. Para obtener información de configuración, consulte la sección del servicio de ayuda sobre SDK necesarios o la sección sobre Objetivos de escritorio.


Con la versión de escritorio de GameMaker Studio 2 se eliminan todas las limitaciones de la licencia de prueba (ver arriba) y puede compilar proyectos para los siguientes objetivos de escritorio:

  • Windows
  • Mac OS
  • Ubuntu

Todos los objetivos vienen con dos opciones de compilación, una es la máquina virtual que utiliza un código interpretado y el corredor y la otra que es el YYC (compilador YoYo ) que crea código nativo y un ejecutable compilado. Tenga en cuenta que la opción YYC requerirá una configuración adicional. Para obtener más información, consulte la sección sobre SDK requeridos o la sección de Helpdesk en Objetivos de escritorio.


Con la versión web de GameMaker Studio 2 se eliminan todas las limitaciones de la licencia de prueba (ver arriba) y puede compilar proyectos para el destino HTML5. Los juegos se compilan usando JavaScript. Puede encontrar más información en el servicio de ayuda de YoYo Games aquí: Configuración para Web (HTML5).


Con la versión Amazon Fire de GameMaker Studio 2 se eliminan todas las limitaciones de la licencia de prueba (ver arriba) y puede compilar proyectos para el objetivo Amazon Fire. Los juegos se compilan con el SDK de Android y se pueden compilar utilizando la máquina virtual (un corredor con código interpretado) o el YYC (compilador YoYo que crea el código nativo). Para obtener más información sobre la configuración, consulte la sección sobre SDK requeridos o la sección del servicio de ayuda en la licencia de Amazon Fire.


Con la versión móvil de GameMaker Studio 2 se eliminan todas las limitaciones de la licencia de prueba (ver arriba) y puede compilar proyectos para los siguientes objetivos:

  • Android
  • Amazon Fire
  • iOS

Todos los objetivos vienen con dos opciones de compilación, una es la máquina virtual que utiliza un código interpretado y el corredor y la otra que es el YYC (compilador YoYo ) que crea código nativo y un ejecutable compilado. Tenga en cuenta que la opción YYC requerirá una configuración adicional. Para obtener más información sobre la configuración, consulte la sección sobre SDK requeridos o la sección del servicio de asistencia en Destinos móviles.


Con la versión UWP de GameMaker Studio 2 se eliminan todas las limitaciones de la licencia de prueba (ver arriba) y puede compilar los proyectos como ejecutables compatibles con UWP. Estos ejecutables se pueden configurar para ejecutarse en una de las tres plataformas UWP:

  • Escritorio Windows 10
  • Windows 10 Mobile
  • XBox One

El objetivo UWP viene con dos opciones de compilación, una es la máquina virtual que utiliza un corredor y el código interpretado y la otra que es el YYC (compilador YoYo ) que crea el código nativo y un ejecutable compilado. Tenga en cuenta que la opción YYC requerirá una configuración adicional. Para obtener más información sobre la configuración, consulte la sección sobre SDK requeridos o la sección del servicio de asistencia sobre objetivos UWP.


Con la versión de consolas de GameMaker Studio 2 todas las limitaciones de la licencia de prueba (ver más arriba) se retiran y se puede compilar proyectos para las siguientes plataformas de destino

  • XBox One
  • Playstation 4

Cada uno de los objetivos de la consola incluye dos opciones de compilación, una es la máquina virtual que utiliza un corredor y el código interpretado, y la otra es el YYC (compilador YoYo ) que crea código nativo y un ejecutable compilado. Tenga en cuenta que la opción YYC requerirá una configuración adicional. Para obtener más información sobre la configuración, consulte la sección sobre SDK requeridos o la sección del servicio de ayuda sobre los Objetivos de la consola.


GameMaker Studio 2 también se puede usar en un entorno universitario. Para obtener más información al respecto, contáctese con YoYo Games a través del portal educativo YoYo Games.


¡IMPORTANTE! Si ha estado utilizando una licencia de prueba para GameMaker Studio 2 y luego compra una de las otras licencias (Creator, Desktop, Amazon Fire, Mobile, UWP, Web o Console), entonces deberá cerrar sesión en el IDE y reiniciar GameMaker Studio 2. Cuando reinicia el programa y vuelve a iniciar sesión, la nueva licencia debe aplicarse correctamente. Puede cerrar sesión en el menú Archivo.
