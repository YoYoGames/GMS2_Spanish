Preferencias - Editores de texto


Las Preferencias de los editores de texto se usan para definir cómo se ven y se sienten los diferentes editores de código / script. La página principal tiene las siguientes opciones:


  • Retardo de comprobación de la sintaxis (ms) - El tiempo de retardo (en milisegundos) antes de que se ejecute el verificador de sintaxis. Este valor predeterminado es 2000 ms.

  • Retardo de finalización de código (ms) - El tiempo de retardo (en milisegundos)

  • Abrir scripts en el editor de pantalla completa: al marcar esta opción, los recursos de script se abrirán automáticamente en un nuevo espacio de trabajo. Si no está marcado, se abrirán en una ventana dentro del espacio de trabajo actual. Esta opción está desmarcada de forma predeterminada.

  • Abrir sombreadores en el editor de pantalla completa: al marcar esta opción, los recursos de sombreado se abrirán automáticamente en un nuevo espacio de trabajo. Si no está marcado, se abrirán en una ventana dentro del espacio de trabajo actual. Esta opción está desmarcada de forma predeterminada.

  • Abrir eventos de objeto en el editor de pantalla completa: al marcar esta opción, los editores de código de evento de objeto se abrirán automáticamente en un nuevo espacio de trabajo. Si no está marcado, se abrirán en una ventana encadenada al editor de eventos del objeto. Esta opción está desmarcada de forma predeterminada.

  • Abrir scripts de eventos como pestañas en la misma ventana: al agregar código o arrastrar y soltar a un evento, la ventana de código está encadenada a la ventana de eventos, y cuando esta opción está marcada, cualquier evento adicional que se abra para edición se agregará al mismo ventana como pestañas en la parte superior. Al desmarcar esta opción, cada evento se abrirá como su propia ventana encadenada única. Esta opción está seleccionada de forma predeterminada.

  • Abra scripts enlazados como pestañas en la misma ventana: cuando tiene varios scripts vinculados y los abre, de forma predeterminada se agrupan en la misma ventana que las pestañas. Al desmarcar esta opción, los obligará a abrir cada uno en su propia ventana / área de trabajo. Esta opción está seleccionada de forma predeterminada.

  • Editor de texto panorámico a la vista en el enfoque: cuando abre un editor de texto desde un evento de objeto, al seleccionar esta opción se desplazará automáticamente el área de trabajo para llevar el editor dentro de los límites. Esta opción está apagada por defecto.

  • Ancho de ventana de código predeterminado: establezca el ancho predeterminado (en píxeles) para la ventana de código cuando se abra. El valor predeterminado es 900.

  • Altura de la ventana de código predeterminado: establezca la altura predeterminada (en píxeles) para que se abra la ventana de código. El valor predeterminado es 600.

  • Ancho mínimo de ventana de código: establezca el ancho mínimo (en píxeles) permitido por la ventana de código. El valor predeterminado es 300.

  • Altura mínima de la ventana de código: establezca la altura mínima (en píxeles) permitida por la ventana de código. El valor predeterminado es 200.

Además de las opciones anteriores, también hay dos subcategorías para el código y las preferencias del editor de diálogo (que se muestran a continuación ya que las preferencias son exactamente las mismas para ambos):

Las preferencias de Color se usan para cambiar la forma en que el editor de códigos y scripts muestra diferentes textos. Cada entrada tiene una serie de sub-entradas que le permiten definir exactamente cómo se verá cada fragmento de texto, como se muestra en la imagen de ejemplo anterior. En general, puede establecer el color, la fuente, el tamaño y el estilo para cada una de las diferentes opciones. Más abajo también puede establecer un estilo más general para cosas como selección de línea de corchetes de código, etc.

En las preferencias de color tiene las diferentes opciones de estilo. Estas preferencias se utilizan para establecer el comportamiento del editor de código / script, con las siguientes opciones disponibles:

  • Mantener pestañas: si está tildado, esto mantendrá pulsadas las teclas de tabulación como caracteres de tabulación - '\ t' - en el editor, mientras que si no se marca, las pestañas se convierten en espacios. El valor predeterminado es desactivado.

  • Tamaño de la pestaña: la cantidad de espacios de caracteres que presionar "Tab" debe sangrar el código. El valor predeterminado es 4.

  • Mostrar espacios en blanco: mostrar "." para mostrar espacios en blanco (ver imagen a continuación). El valor predeterminado es desactivado.

  • Mostrar números de línea: si mostrar los números de línea a la izquierda o no. El valor predeterminado está activado.

  • Mostrar líneas de pestañas: muestra "...." para mostrar el espacio con pestañas. El valor predeterminado es desactivado.

  • Habilitar codificación de color: habilite o deshabilite la codificación de color para el editor. Si está desactivado, el código no se diseñará, mientras que cuando se lo utilizará se diseñará utilizando el formato indicado anteriormente en la sección Color. El valor predeterminado está activado.

  • Límite de línea de documento: establezca el número máximo de líneas para cualquier documento de editor determinado. El valor predeterminado es 0 (0 o menos es esencialmente infinito) y cualquier otro valor limitará las líneas permitidas.

  • Hacer coincidir sangría en líneas nuevas: habilite o deshabilite la sangría automática para su código. El valor predeterminado está activado, lo que mantendrá las pestañas establecidas en una línea cuando se agregue una nueva línea.

  • Sangría automática después de la llave: habilite o deshabilite la sangría automática para su código al agregar llaves {}. El valor predeterminado está activado, que tabulará la siguiente línea de código cuando se agregue una nueva llave {y una nueva línea.

  • Habilite la finalización del código: habilite o deshabilite la ventana de finalización del código. Al escribir el código y esto está habilitado, obtendrá una ventana emergente que muestra posibles funciones que puede usar en función del texto de entrada actual. El valor predeterminado está activado.

  • Corchetes de función de agregar automáticamente: habilite o deshabilite la adición automática de corchetes () cuando usa el autocompletar. Al escribir código y esto está habilitado, el IDE agregará automáticamente los dos corchetes () que requiere una función, colocando el cursor dentro de ellos si la función toma argumentos, o colocando el cursor detrás de ellos si no lo hace. Esta función solo funciona cuando la función ha sido seleccionada desde la función autocompletar. El valor predeterminado está activado.

  • Deshacer / Rehacer el límite de la pila: aquí puede establecer el límite de la pila Deshacer / Rehacer. Cualquier cosa debajo de 0 es esencialmente infinita (limitada por la memoria disponible) mientras que los valores positivos limitarán la pila a ese número, eliminando cualquier acción en la cola de la pila si se alcanza el límite. El valor predeterminado es -1.

  • Resalte las coincidencias de selección: Esto resaltará ligeramente cualquier sección de código que esté duplicada en otro lugar. Por lo tanto, si selecciona una función, por ejemplo, todas las instancias de esta función en la ventana de código actual también se resaltarán. El valor predeterminado está activado.

  • Codificación de color / finalización de código para funciones obsoletas: Esto resaltará cualquier sección de código que esté obsoleta. El valor predeterminado está activado.
