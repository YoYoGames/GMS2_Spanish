Preferencias - General


Las Preferencias generales se usan para definir el comportamiento general del IDE de GameMaker Studio 2. Los más generales se enumeran en la página principal, mientras que muchos otros se dividen en múltiples subcategorías, que se explican más adelante en esta página.

  • Idioma IDE: esta opción le permite seleccionar el idioma para el IDE de la lista de opciones suministradas. El predeterminado es inglés.

  • IDE Skin: aquí puede seleccionar entre las diferentes máscaras que el IDE puede usar. Por defecto GameMaker Studio 2 viene con dos máscaras, una luz y una oscuridad, y la configuración inicial usa la piel oscura.

  • Dispositivo de audio predeterminado: aquí puede elegir el dispositivo de audio predeterminado para el IDE de GameMaker Studio 2 a usar. Puede actualizar la lista haciendo clic en el botón Actualizar

  • Respuesta automática a la confirmación de salida: esta opción le permite establecer si se debe mostrar un mensaje de confirmación cuando salga de GameMaker Studio 2. De forma predeterminada, está configurado como "OK", pero se puede configurar para omitir el mensaje como si hubiera hecho clic en Aceptar.

  • Habilitar notificaciones: al marcar esto, el IDE mostrará notificaciones sobre nuevos tiempos de ejecución, etc. Esto está activado por defecto.

  • Tiempo de espera de notificación (ms): Establece el período de tiempo (en milisegundos) durante el cual se muestra una notificación. El valor predeterminado es 6000.

  • Volver a cargar automáticamente los archivos modificados: si está marcado, el IDE escaneará y verificará automáticamente los archivos para ver si alguno de ellos se ha modificado fuera de GameMaker Studio 2 y los volverá a cargar si lo hubiera hecho.

  • Abrir nuevos recursos para editar en la creación: si está marcado, cada vez que cree un nuevo recurso en el Árbol de recursos, el editor de recursos correspondiente se abrirá automáticamente. Esto se comprueba de manera predeterminada y al desmarcarlo, significa que debe abrir manualmente todos los recursos recién creados.

  • Abre nuevos recursos para editar al arrastrar archivos a IDE: Ciertos recursos (como sprites o sonidos) se pueden arrastrar al Área de trabajo principal en GameMaker Studio 2 IDE y se detectarán y agregarán como un nuevo recurso en el Árbol de recursos. De forma predeterminada, el editor de recursos correspondiente también se abrirá para su edición cuando esto ocurra, pero al desmarcar esta opción evitará que se abran los editores y se le solicitará que los abra de forma manual.

  • Permitir que las pestañas vacías se cierren mediante acceso directo: puede cerrar cualquier pestaña abierta usando el atajo de teclado / + W, pero si deshabilita esta opción, ese acceso directo ya no funcionará. Esto está habilitado por defecto.

  • Deshabilitar animaciones de transición IDE: GameMaker Studio 2 IDE tiene una serie de transiciones animadas para cosas como abrir ventanas, etc... sin embargo, no son del gusto de todos y puede experimentar un aumento de rendimiento si están deshabilitadas en máquinas más antiguas o con menor potencia si se desactivan. Puede deshabilitarlos aquí, y están activados por defecto.

  • Mostrar pantalla de bienvenida al inicio: esto está activado de manera predeterminada y simplemente le dice a GameMaker Studio 2 que muestre una breve pantalla de bienvenida al iniciar el programa. Al desmarcar esto, evitará que se muestre la pantalla de bienvenida.

  • Habilitar invalidación DPI: esta opción le permite anular la configuración PPP predeterminada con la suya. Primero debe habilitar la opción y luego establecer el valor de Anulación de DPi sin formato (el valor predeterminado es 96) o establecer el Porcentaje de DPi nativo, y una vez modificado, se le solicitará que reinicie el IDE, lo que puede requerir que guarde cualquier trabajo que se está realizando en el proyecto actualmente abierto.

  • Suscríbase a las encuestas de opinión: Cuando se selecciona esta opción, le está diciendo GameMaker Studio 2 que desea participar en encuestas de opinión, lo que significa que occaisionally mientras se trabaja un mensaje se mostrará sobre la zona que se está trabajando y solicitar una calificación y realimentación. Esta opción está activada por defecto.

Las preferencias de fondo se refieren al fondo del espacio de trabajo y a lo que se mostrará, así como a la forma en que se mostrará. Las opciones son:

  • Imagen de fondo: este es el archivo de imagen que se usará como fondo para los diferentes espacios de trabajo. Se proporciona una imagen predeterminada y está habilitada de manera predeterminada, pero puede seleccionar su propia imagen aquí. Los tipos de imagen que puede seleccionar son *.png, *.gif, *.jpg, *.jpeg, *.tif, *.tiff, *.ico, *.bmp, *.2bp o *.mdi, pero tenga en cuenta que, independientemente del formato de imagen inicial, todos los archivos se guardarán para su uso como *.png archivos, tan animados *.gif los archivos, por ejemplo, solo se representarán como un marco inicial único.

  • Escalado de imagen: controla cómo se escalará la imagen de fondo dentro de los diferentes espacios de trabajo. Las opciones son:

    • Estirar - La imagen se alargará para adaptarse al espacio de trabajo, cambiando la escala en función de los muelles visibles
    • Mosaico: la imagen se embaldosará para cubrir el espacio de trabajo
    • Mejor ajuste: la imagen se ajustará para adaptarse al espacio de trabajo proporcionalmente, con barras añadidas alrededor del borde si no es un ajuste de 1: 1
    • Estiramiento de escritorio: el fondo cubrirá todo el espacio de trabajo, independientemente de los muelles visibles
  • Mostrar imágenes de fondo en espacios de trabajo: esta opción controla si se debe mostrar o no la imagen de fondo seleccionada. Por defecto, esto está marcado como activado.

Las preferencias de compilación tratan ciertas opciones relacionadas con el resultado al compilar su juego para probar o crear un paquete, así como ciertas áreas del proceso de compilación. Las opciones son:

  • Abrir automáticamente la ventana de error de sintaxis en la detección: habilite o deshabilite la notificación de error de sintaxis al compilar. Esto está desactivada por defecto.

  • Mostrar la ventana de compilación durante la compilación: de forma predeterminada, la ventana de compilación se mostrará cuando compile sus proyectos, incluso si el documento de compilación está cerrado (se abrirá para la compilación). Desmarcar esto desactivará este comportamiento y la ventana de compilación estará visible si está abierta.

  • Mostrar los comandos del compilador en la salida de compilación: esto se comprueba de manera predeterminada y obliga al compilador a mostrar todos los comandos relevantes en la ventana de compilación. Al desmarcar esto, la salida será más pequeña y tendrá menos información.

  • Mostrar salida del compilador detallado: al comprobar esto, se forzará al compilador a mostrar comandos adicionales relacionados con el proceso de compilación. En general, esto solo es necesario cuando se realiza una depuración y es posible que el soporte lo solicite en caso de que tenga un problema. Está desactivado por defecto (y esta es la configuración recomendada).

  • Mostrar el paquete en el explorador en la finalización de la compilación: al verificar esto, GameMaker Studio 2 abrirá la ventana del explorador del sistema operativo en la ubicación donde se creó el paquete ejecutable final. Está activado por defecto.

  • Ventana de salida clara en el inicio de compilación: de forma predeterminada, la ventana del compilador borrará todos los mensajes anteriores de la ventana de salida de compilación al inicio de cada compilación, sin embargo, al desmarcar esto se mantendrán los registros de compilación previos. Esto está activado por defecto y es la configuración recomendada ya que los registros de compilación grandes pueden ralentizar los tiempos de compilación.

  • Puerto del depurador: Establece el número de puerto para el Módulo de depuración que se utilizará. El valor predeterminado es 6509.

  • Máximo de trabajos de compilación en paralelo: este valor establece la cantidad máxima de núcleos de CPU para dedicar a la compilación de un proyecto. El valor predeterminado es 8 y, en general, no debería tener que cambiar esto, pero si está en una computadora portátil o una máquina de gama más baja, es posible que desee reducir este valor. El valor predeterminado está establecido en 8.


Las preferencias de Confirmar diálogo tratan algunas de las ventanas emergentes de confirmación general que aparecerán al realizar ciertas acciones en GameMaker Studio 2 IDE. Las opciones son:

  • Respuesta automática al cierre de un área de trabajo mediante un atajo de teclado: el atajo de teclado mencionado anteriormente se puede usar para cerrar cualquier pestaña abierta, pero a veces puede cerrar accidentalmente una pestaña vacía que necesite. Para evitar esto, puede configurar esta opción como "Mostrar mensaje" (la opción predeterminada) para preguntar si realmente desea cerrar esa pestaña vacía. Las otras configuraciones son "No", simplemente no hacer nada si se usa el atajo, o "Bien" para cerrar la pestaña sin ningún mensaje.

  • Respuesta automática al reinicio de una compilación en ejecución: cuando reinicia una compilación y otra anterior aún se está ejecutando, se le preguntará si desea detener la compilación actual primero. Al desmarcar este mensaje, se suprimirá ese mensaje y se forzará a GameMaker Studio 2 a asumir que ha hecho clic en "Aceptar" como respuesta. Esta opción está configurada en "Mostrar mensaje" de forma predeterminada.

  • Respuesta automática a la advertencia cuando se intenta depurar YYC: el objetivo de YYC se compila en código nativo y, como tal, no se puede depurar utilizando el depurador incorporado. Cuando intente utilizar el depurador en el objetivo de YYC, recibirá una advertencia que le informará sobre esto, y el hecho de que el proyecto se compilará utilizando la VM en su lugar. Esta opción está configurada en "Mostrar mensaje" de forma predeterminada, pero establecerla en "Aceptar" suprimirá esta advertencia.

  • Respuesta automática para restablecer el diseño del proyecto: cuando selecciona restablecer el diseño del proyecto (desde el menú de diseños ), de manera predeterminada, se mostrará un mensaje de advertencia sobre cómo cerrará cualquier ventana de recursos abiertos. Sin embargo, puede suprimir este mensaje y restablecer automáticamente el diseño seleccionando "Aceptar" o hacer que no haga nada seleccionando "No". El valor predeterminado es tener "Mostrar mensaje".

  • Respuesta automática a la desactivación de las estadísticas anónimas: cuando deshabilita las estadísticas anónimas del juego en las opciones generales del juego, se le mostrará un mensaje de advertencia pidiéndole que considere cuidadosamente la opción. Puede suprimir este mensaje seleccionando "Aceptar" aquí o hacer que no haga nada seleccionando "No". El valor predeterminado es tener "Mostrar mensaje".

Las preferencias aquí están relacionadas con el depurador GameMaker Studio 2 y usted tiene las siguientes opciones:

  • Respuesta automática al cierre del depurador: esta opción le permite configurar la respuesta automática al cerrar el depurador. Si se establece en "Mostrar mensaje", se le mostrará el diálogo cuando desee cerrar el depurador, mientras que al establecerlo en "Sí" o "No" no se mostrará este diálogo y se realizará la tarea como se indica.

  • Habilite la depuración en tiempo real: al marcar esta opción, podrá depurar los proyectos de su juego en tiempo real a medida que los ejecuta desde el IDE. Esto está marcado por defecto.

Las preferencias aquí son para la ventana Ir a, que puede abrir usando el atajo de teclado / + T.

  • Número máximo de resultados de búsqueda para mostrar: esta opción controla el número máximo de resultados devueltos cuando usa la ventana Ir a búsqueda.

El archivo de ayuda está diseñado para funcionar como una página web y, como tal, requiere el uso del servidor web micro de GameMaker Studio 2 o el uso de un navegador externo. Estas configuraciones se pueden cambiar aquí:

  • Puerto manual de ayuda: Configura el número de puerto para el micro servidor web manual. El valor predeterminado es 51291.

  • Use un navegador externo para obtener ayuda: si está marcado, al abrir el archivo de ayuda se abrirá usando el navegador predeterminado para su sistema. Esto está activado por defecto, y apagarlo obligará al archivo de ayuda a abrirse en el espacio de trabajo dentro del IDE.

Las preferencias de entrada controlan ciertos aspectos de la entrada del mouse y del teclado dentro del IDE. Las opciones disponibles son:

  • Tiempo lento de doble clic (ms): algunas áreas del IDE requieren un doble clic más lento para funcionar correctamente, por ejemplo al cambiar el nombre de un recurso (un doble clic rápido abrirá el editor de recursos, mientras que un doble clic lento le permitirá cambiarle el nombre ) Esta opción le permite establecer la velocidad de detección, con un valor predeterminado de 500 ms.

  • Tiempo de bloqueo de desplazamiento (ms): esta opción se refiere al tiempo que GameMaker Studio 2 debe esperar entre las ventanas de desplazamiento superpuestas. Esencialmente, cuando se desplaza por la ventana "A", el cursor puede desplazarse sobre otro área desplazable en la ventana "B", por lo que esta preferencia define cuánto tiempo tiene que esperar después de dejar de desplazarse "A" antes de poder desplazarse " B ", por lo que no accidentalmente comienzas a desplazarte por diferentes paneles. El valor predeterminado es 500ms.

  • Velocidad de desplazamiento: Controla la velocidad de desplazamiento dentro de los diferentes espacios de trabajo. El valor predeterminado es 1.
  • Distancia de arrastre de tabulación para des-maximizar el escritorio (px): cuando arrastra una pestaña del área de trabajo fuera de su posición predeterminada dentro del IDE, si solo hay un área de trabajo activa en ese momento, la ventana de GameMaker Studio 2 no se maximizará y se moverá con el mouse (si tiene más de un espacio de trabajo activo, se creará una nueva instancia del IDE para contener el área de trabajo). Esta configuración controla qué tan lejos tiene que arrastrar la pestaña antes de que el área de trabajo la reconozca, y el valor predeterminado es 50px.

  • Nivel de zoom mínimo para interactuar con el contenido de la ventana (%): esta opción controla el punto de corte del zoom en el que el mouse ya no podrá interactuar con las ventanas del espacio de trabajo (es decir: agregar nodos, arrastrar ventanas, etc.). El valor predeterminado es 40.

  • El zoom de la rueda del mouse requiere la tecla Ctrl: cuando se marca esta opción, todos los editores (y el área de trabajo principal) tendrán habilitado el zoom con / + la rueda del mouse , mientras lo desmarca, habilitará el zoom solo con la rueda del mouse .


Las preferencias de la ventana de salida controlan las diferentes opciones de color y formato para las diferentes ventanas de salida diferentes en el IDE. Cada una de las diferentes ventanas tiene el mismo conjunto de opciones, que se enumeran a continuación:

  • Color: el color predeterminado para el texto que se escribe en la ventana.

  • Nombre de la fuente: el nombre de la fuente a usar (las fuentes disponibles dependerán de las instaladas en su sistema).

  • Estilo: el estilo de fuente a usar, como negrita o cursiva (tenga en cuenta que no todas las ventanas permitirán los mismos estilos y también dependerá de la fuente utilizada).

  • Tamaño: El tamaño para establecer la fuente para la ventana de salida dada.

  • Anti-Aliasing: aquí puede establecer el modo anti-aliasing para el renderizado de fuentes de Windows de salida. Puede optar por dejar que el IDE decida al seleccionar Usar predeterminado, o puede forzarlo activando o desactivando seleccionando cualquiera de esas opciones.

Las preferencias de ruta controlan las diferentes rutas de archivos requeridas por los elementos en GameMaker Studio 2 IDE, así como también por los editores externos que requiera. Las siguientes opciones existen para el IDE:

  • Directorio temporal: la ubicación para guardar todas las carpetas temporales. Por defecto esto es \Users\<Username>\AppData\Local\Temp\GameMakerStudio2.

  • Directorio de caché IDE: la ubicación para guardar el caché IDE. Por defecto esto es \Users\<Username>\AppData\Roaming\GameMakerStudio2-Dev\Cache.

  • Directorio de caché de activos: la ubicación para guardar el caché de activos para cada proyecto. Por defecto esto es \Users\<Username>\AppData\Roaming\GameMakerStudio2-Dev\Cache.

  • Ubicación de "Mis proyectos": la ubicación en la que GameMaker Studio 2 creará inicialmente nuevos proyectos. Por defecto esto es \Users\<Username>\Documents\GameMakerStudio2.

  • Eliminar automáticamente el directorio temporal al cerrar: Al comprobar esto, forzará a GameMaker Studio 2 a eliminar automáticamente la carpeta Temp que crea por proyecto para compilar, etc. Esta configuración está activada por defecto, y al desmarcarla la desactivará (lo que significa que tendrá que eliminar manualmente cualquier archivo temporal más adelante).

  • Eliminar automáticamente la memoria caché de activos al cerrar: si está marcada, la carpeta de memoria caché del compilador de activos se eliminará cuando salga de GameMaker Studio 2. Esto está desactivado por defecto, y habilitarlo significará que cada vez que cargue y ejecute cualquier proyecto, la memoria caché tendrá que ser reconstruida (lo que puede tomar tiempo según el tamaño del juego).

  • Eliminar automáticamente la memoria caché IDE al cerrar: si está marcada, la carpeta de la memoria caché del compilador IDE se eliminará cuando salga de GameMaker Studio 2. Esto está desactivado por defecto, y habilitarlo significará que cada vez que inicie GameMaker Studio 2, la memoria caché tendrá que ser reconstruida.

  • Eliminar carpeta temporal: al hacer clic en este botón se eliminará la carpeta temporal del proyecto.

  • Eliminar caché de activos: al hacer clic en este botón se eliminará el caché de activos del compilador para el proyecto.

  • Eliminar el caché IDE: al hacer clic en este botón, se eliminará el caché IDE.

Las siguientes opciones existen para establecer rutas a editores externos:

  • Ruta al editor / visor externo para archivos SWF: si está trabajando con sprites de formato SWF, puede establecer esto en la ruta de su visor / editor preferido y al hacer clic en el botón Editar imagen en el Editor de Sprite, se abrirá el programa. El valor predeterminado aquí es no tener ninguna ruta.

  • Ruta al editor / visor externo para los archivos de Spine: si está trabajando con Sprites de formato de Spine, puede establecer esto en la ruta de su visor / editor preferido y cuando haga clic en el botón Editar Imagen en el Editor de Sprite, se abrirá el programa. El valor predeterminado aquí es no tener ninguna ruta.


Las preferencias de ahorro de energía aquí afectarán la cantidad de potencia de la CPU y de la GPU que se utilizará para generar GameMaker Studio 2 IDE (esencialmente, esto reducirá la tasa de fotogramas base en la que se ejecuta el programa y, por lo tanto, menor consumo de energía). Tenga en cuenta que algunos de estos son específicos del proceso de fondo de File Watcher. File Watcher escanea las carpetas del proyecto en busca de cambios externos; por ejemplo, si cambia una secuencia de comandos gml en Notepad ++, el observador de archivos detectará este cambio y le preguntará si desea volver a cargarlo o descartarlo, y como tal la frecuencia con la que se escanea sucede puede afectar gravemente el rendimiento y el consumo de energía.

Las opciones aquí son:

  • Velocidad de fotogramas de potencia de CA (%): modificador de velocidad de fotogramas IDE base cuando se conecta a una entrada de alimentación. El valor predeterminado es 100%.

  • Velocidad de fotogramas por potencia de la batería (%): el modificador de velocidad de fotogramas IDE base cuando se utiliza la batería para alimentar la máquina. El valor predeterminado es 35%.

  • Velocidad de fotogramas de fondo (%): el modificador de velocidad de fotogramas IDE base cuando GameMaker Studio 2 no está enfocado (independientemente de la fuente de alimentación). El valor predeterminado es 15%.

  • Frecuencia de búsqueda de archivos (ms): Esta es la frecuencia con la que se ejecutará el vigilante de archivos cuando se encuentre en el escritorio (o en una computadora portátil acoplada). El valor predeterminado es 2000.

  • Frecuencia del monitor de archivos portátiles (ms): esta es la frecuencia con la que se ejecutará el vigilante de archivos cuando se encuentre en modo portátil. El valor predeterminado es 5000.

Las preferencias aquí contienen opciones de cómo la lista de Windows reciente mostrará elementos:

  • Abrir automáticamente en la carga del proyecto: cuando se habilita, la lista de Windows Reciente se mostrará cuando cargue un nuevo proyecto. El valor predeterminado está deshabilitado.

  • Mueva las ventanas enfocadas hacia arriba: cuando esto esté habilitado y haga clic en cualquier elemento en la lista Ventana reciente, el elemento seleccionado se moverá a la parte superior de la lista. Por defecto, esto está desactivado.

  • Mostrar solo ventanas actualmente abiertas: al habilitar esto, la lista de Windows Reciente eliminará aquellos elementos que ya no están abiertos en un área de trabajo (es decir, al cerrar cualquier ventana, también se eliminará de la lista). Por defecto, esto está desactivado.

  • Máximo de ventanas para mostrar: Esto establece el número máximo de ventanas abiertas previamente que se mostrarán dentro de la lista de Windows Recientes. El valor predeterminado es 10.


Las preferencias aquí contienen opciones sobre cómo se extraerán ciertos recursos en sus proyectos:

  • Color de dibujo predeterminado inicial para nuevos proyectos: cuando utiliza una fuente o dibuja una forma usando las funciones de GameMaker Studio 2 o acciones de DnD™ sin especificar un color, el color que se establece aquí es el que se utilizará. El color predeterminado es negro ( $00000000 ).

Las preferencias aquí afectarán la forma en que se mueve dentro de los diferentes espacios GameMaker Studio 2 trabajo de GameMaker Studio 2. Las opciones son:

  • Rango de grados de navegación del teclado del área de trabajo: dentro de un espacio de trabajo puede tener múltiples ventanas abiertas para diferentes objetos y recursos, y para moverse rápidamente entre ellos puede usar el atajo de teclado / + + <Arrow Keys> para moverse entre ellos. Dado que las ventanas dentro de un espacio de trabajo no están alineadas con la cuadrícula, el IDE debe elegir a qué ventana se refiere cuando presiona cualquiera de las 8 direcciones posibles, y esta opción le permite establecer el "ángulo de aspecto" que se utiliza. El valor predeterminado es 40 °.

  • Relleno de columna de cadena de área de trabajo (px): los elementos del área de trabajo a menudo están "encadenados" entre sí (por ejemplo, el Editor de código está encadenado al Editor de eventos está encadenado al Editor de objetos) y esta opción le permite establecer la distancia horizontal entre dos ventanas encadenadas. El valor predeterminado es 30px.

  • Relleno de fila de cadena de espacio de trabajo (px): como se mencionó anteriormente, los elementos de espacio de trabajo a menudo están "encadenados" entre sí y esta opción le permite establecer la distancia vertical entre dos ventanas encadenadas. El valor predeterminado es 20px.

  • Conteo de segmento de renderizado de cadena de área de trabajo: aquí puede establecer el número máximo de segmentos para cadenas de renderizado. Bajar este valor puede ayudar al rendimiento en las máquinas de gama baja. El valor predeterminado es 20.

  • Color de enlace de cadena de espacio de trabajo: este es el color que se usará para mostrar las cadenas entre los elementos del espacio de trabajo. El color predeterminado es $99DD00 con un alfa de 153 ( $99 ).

  • La navegación por el teclado en el área de trabajo resetea el zoom: como se mencionó anteriormente, puede saltar de una ventana a otra dentro del área de trabajo usando los atajos de teclado. También puede acercar y alejar el área de trabajo utilizando / + la rueda del mouse . Establecer esta opción significará que si tiene el espacio de trabajo ampliado y utiliza los accesos directos para saltar a otra ventana, el área de trabajo se devolverá a una vista 1: 1 y se centrará en la ventana seleccionada. Esta opción está apagada por defecto.

  • Las cadenas de espacios de trabajo se pueden superponer: cuando está habilitada, esta opción permitirá que las cadenas de espacios de trabajo se crucen y se superpongan. Esto está desactivada por defecto.
