Preferencias - Runtime Feeds


Las Preferencias de Runtime Feeds se utilizan para mostrar y cambiar los tiempos de ejecución actualmente instalados, así como el tiempo de ejecución en uso actual. GameMaker Studio 2 puede tener varias herramientas de tiempo de ejecución diferentes instaladas a la vez y puede establecer cuál usar de esta preferencia. Lo ideal es utilizar el tiempo de ejecución más actualizado, pero a veces puede tener un problema, o es posible que deba usar uno más antiguo para poder comparar, etc... y así poder cambiar las herramientas de tiempo de ejecución que GameMaker Studio 2 utiliza para construir tu juego aquí.

La sección inicial de esta preferencia está relacionada con el lugar desde el que extrae las herramientas de tiempo de ejecución y requiere que ingrese una URL a un canal RSS. Luego se escaneará esta fuente en busca de actualizaciones de las herramientas de tiempo de ejecución que luego se pueden descargar e instalar. Esta sección mostrará la URL de la fuente maestra de tiempo de ejecución de forma predeterminada, pero puede hacer clic en el botón Agregar para crear una nueva fuente de tiempo de ejecución en la que ingrese el nombre y la URL de la fuente RSS, y luego hacer clic en Aplicar para decirle a GameMaker Studio 2 que acceda a ella.. Deberá cerrar y volver a abrir la ventana de Preferencias para que se muestre la nueva fuente.

NOTA: si opta por el canal Beta, los tiempos de ejecución del Canal Beta se mostrarán en una fuente "Beta" junto con "Maestro" y puede cambiar entre los dos en cualquier momento.


Una vez que se haya creado un feed en tiempo de ejecución, se incluirá en esta sección como una subcategoría, y siempre debe tener al menos un feed en la lista (de forma predeterminada, este sería el feed maestro). Las preferencias de Runtime Feed se utilizan para seleccionar, instalar y desinstalar las diferentes herramientas de Runtime. En la parte superior, se utilizan las herramientas de tiempo de ejecución actuales, la fecha en que se creó y la ubicación en la que está almacenado en el disco duro. Debajo de estos detalles, puede encontrar una lista de todos los tiempos de ejecución disponibles, y puede elegir cualquiera de esta lista para cambiar a ser el tiempo de ejecución "en ejecución" utilizado para compilar sus proyectos.

Para seleccionar un tiempo de ejecución diferente para usar, simplemente encuéntrelo en la lista y luego haga doble clic en eso. Si el tiempo de ejecución no se ha descargado e instalado, se le pedirá que lo haga, y una vez que se haya instalado, se le pedirá que cierre y reinicie el IDE para que se pueda inicializar el nuevo tiempo de ejecución.

Esta lista de tiempo de ejecución detalla todos los tiempos de ejecución disponibles, su número de versión, la fecha en que se publicaron y, si están instalados, la ruta a donde se han instalado. El lado derecho también muestra los diferentes iconos de estado, que son los siguientes:

Esto marca el tiempo de ejecución actual que se ha seleccionado para usar al construir sus proyectos.
Al hacer clic en esto, se iniciará la descarga e instalación del tiempo de ejecución seleccionado (se mostrará una barra de estado en la parte superior derecha del IDE).
Al hacer clic en esto se eliminará el tiempo de ejecución seleccionado de su disco duro.

Puede actualizar la lista de tiempo de ejecución en cualquier momento desde la fuente RSS dada haciendo clic en el botón de actualización .
