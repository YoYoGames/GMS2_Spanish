Micro Servidor Web


Esta es una de las herramientas principales. La función principal es conectar y entregar archivos a algunas de las plataformas de destino disponibles para GameMaker Studio 2 (como el destino de HTML5, por ejemplo). Cuando ejecuta un juego en un destino que requiere el Micro Web Server, se abrirá una nueva ventana a medida que el juego se sirva en el dispositivo elegido:

NOTA: La primera vez que ejecute un juego en un destino que requiera el Micro Web Server, es posible que reciba una alerta de firewall. Debe permitir que el servidor web se comunique a través del firewall si esto sucede.


El servidor web mostrará una cierta cantidad de información de depuración relacionada con los archivos que se han servido, y tiene un botón para eliminar esto en la parte inferior. Junto al botón "Borrar" está la ruta al archivo JS raíz para el proyecto, mientras que debajo está la opción Velocidad de red. Esto se establece en "Sin límite" de forma predeterminada, pero al hacer clic en él se abrirá una lista de diferentes opciones de red:

Cambiar esta configuración también cambiará la opción Tamaño del paquete a la derecha. También puede editar esto manualmente, en cuyo caso la velocidad de la red se configurará en "Personalizado". Si lo configura en un valor diferente al máximo, la velocidad de transferencia de la red se reducirá de manera efectiva y le permitirá probar juegos en un entorno de bajo ancho de banda.

También puede ver la lista de direcciones IP permitidas junto con el número de puerto que se está utilizando, pero no se pueden editar en el Micro Web Server y, en su lugar, deben configurarse en la configuración de dispositivos del administrador de destino para el objetivo del dispositivo dado.
